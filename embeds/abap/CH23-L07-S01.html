Validation · Determination · Action (분류 + 실행 시점)

RAP의 비즈니스 로직 셋은 목적과 실행 시점이 다릅니다 — 검사(Validation) · 자동 채움(Determination) · 사용자 실행(Action). 상황을 분류해 보고, 아래 타임라인에서 각 로직이 언제 실행되는지 확인하세요.

① 이 상황은 무엇일까? (V / D / A)

② 실행 타임라인 — 언제 실행되나

모든 로직을 action으로 만들지 마세요. 사용자가 눌러야 하는 동작만 action이고, 저장 전에 반드시 지켜야 하는 규칙은 validation이 자연스럽습니다. 그리고 validation에서 값을 고치면 안 돼요 — 값 자동 채움은 determination의 일입니다.
