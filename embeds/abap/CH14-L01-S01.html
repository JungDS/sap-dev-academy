코드 JOIN vs Database View (같은 결과 · 다른 정의 위치)

같은 결합을 코드 JOIN으로 쓰든 Database View로 등록하든 결과는 같습니다. 상태를 바꿔 inner join이 마스터 없는 행을 어떻게 떨어뜨리는지, View Field 축소가 결과에 어떻게 영향을 주는지 보세요.

① 코드 JOIN (CH13 스타일)
SELECT c~concert_id p~perf_name
       c~date c~hall_id
  FROM zconcert AS c
  INNER JOIN zperf AS p
    ON c~perf_id = p~perf_id
  INTO TABLE lt_out.
② Database View ZV_CONCERT (SE11)
Basis: zconcert, zperf
Join: zconcert~perf_id = zperf~perf_id
View Fields

↓ 두 정의 모두 같은 결과를 읽는다
View = 저장소가 아니다. 데이터는 basis table에 있고 View는 그 관점일 뿐입니다. 여러 테이블 View는 inner join이라 짝 없는 행은 빠집니다. View Field는 외부 프로그램의 구조 계약이니 필요한 필드만 노출하세요.
