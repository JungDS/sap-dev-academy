SAP LUW 타임라인 — IN UPDATE TASK는 언제 실행될까?

단계를 진행하며 Update Task 대기 큐와 DB가 어떻게 변하는지 보세요. 함수를 "호출"한 순간 DB에 들어갈까요?

① 화면 입력→ ② UPDATE TASK 등록→ ③ 추가 처리→ ④ COMMIT WORK
📋 Update Task 대기 큐 0건
💾 DB (확정) 0건
💡 SAP LUW(업무 묶음, 여러 화면) 안에서 IN UPDATE TASK로 변경을 모아 두었다가, COMMIT WORK 한 순간 Update Work Process가 실제 DB 쓰기(DB LUW)를 수행합니다.
