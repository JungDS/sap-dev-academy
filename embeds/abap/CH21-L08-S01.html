Dynamic Type Inspector (CAST vs CASE TYPE OF)

부모 타입 변수 lo_mgr에 어떤 실제 객체가 담겼는지 고르고, CAST(다운캐스트)와 CASE TYPE OF 중 무엇으로 다룰지 바꿔 보세요. 변수의 정적 타입은 같아도, 담긴 객체의 동적 타입에 따라 결과가 달라집니다.

기본은 다형성, CAST는 최소화. 가능하면 공통 메서드(book( ) 재정의)로 풀고, 정말 구체 타입 기능이 필요할 때만 CAST/CASE TYPE OF를 씁니다. 타입별 분기가 자꾸 늘어나면 설계(다형성)를 재검토하세요. CASE TYPE OF의 순서는 구체 타입부터.
