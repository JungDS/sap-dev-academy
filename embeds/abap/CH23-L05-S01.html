본문(DDL) ↔ 메타데이터(DDLX) 분리 실험실 (Metadata Extension)

UI 주석을 본문 ZC_Concert에 둘지, 별도 Metadata Extension(DDLX)으로 뺄지 토글해 보세요. 대상 entity가 @Metadata.allowExtensions를 허용해야 붙고, DDLX 블록은 요소 뒤에 세미콜론(;)을 씁니다. 두 파일이 어떻게 나뉘는지 보세요.

Metadata Extension은 필드를 추가하는 도구가 아닙니다. 기존 CDS entity에 annotation만 별도 source로 붙여요. 그래서 annotate entity … with이고, select source나 필드를 바꾸지 않습니다. @Metadata.layer는 여러 extension이 있을 때 평가 우선순위에 쓰입니다(고객별 덮어쓰기 전략은 이후 단계).
