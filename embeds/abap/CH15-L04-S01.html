검증 게이트와 메시지 콘솔 (AT SELECTION-SCREEN → MESSAGE)

시나리오를 고르고 검증 실행을 누르면, AT SELECTION-SCREEN이 어떤 MESSAGE로 막는지(&1·&2 치환), sy-msg*가 어떻게 채워지는지, E가 START-OF-SELECTION을 어떻게 잠그는지 봅니다.

선택화면 입력

p_from
p_to
p_conc

MESSAGE

조회 전에 막는다. 입력 검증은 AT SELECTION-SCREEN에서 하고, MESSAGE e###(class) WITH ...로 사용자가 고칠 이유를 알립니다. E는 화면으로 되돌려 START를 잠그고, S/I는 흐름을 막지 않습니다. message class를 쓰면 다국어·일관 관리에 유리합니다.
