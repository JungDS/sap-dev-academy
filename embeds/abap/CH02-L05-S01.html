pi를 직접 적으면 vs 상수 하나로 (같은 계산, 두 버전)

반지름 3과 5인 원의 둘레·넓이를 구하는 같은 코드입니다. pi를 줄마다 직접 적은 쪽(왼쪽)과 상수 gc_pi 하나로 묶은 쪽(오른쪽)을 비교해 보세요.

✗ pi를 줄마다 직접
" 반지름 3
lv_c = 2 * '3.1415926' * 3.
lv_a = '3.1415927' * 3 * 3.   " …27 오타!

" 반지름 5
lv_c = 2 * '3.14' * 5.
lv_a = '3.14159' * 5 * 5.
  • 줄마다 pi가 다르다 — '3.1415926'·'3.1415927'·'3.14'·'3.14159'
  • 끝자리 오타(…27)·정밀도 불일치로 결과가 미묘하게 틀어진다
직접 적은 pi: 4곳 (다 다름)
✓ 상수 gc_pi 하나
CONSTANTS gc_pi TYPE p LENGTH 8 DECIMALS 7 VALUE '3.1415926'.

" 반지름 3
lv_c = 2 * gc_pi * 3.
lv_a = gc_pi * 3 * 3.

" 반지름 5
lv_c = 2 * gc_pi * 5.
lv_a = gc_pi * 5 * 5.
  • gc_pi 하나만 읽어 모든 계산이 같은 값
  • 사람이 잘못 적을 일이 없다 — 정밀도를 바꿔도 선언 한 줄만
직접 적은 pi: 0곳
변수로도 같은 계산을 할 수 있다. 하지만 변수는 실수로 덮어써져도 ABAP이 막지 않아 이후 계산이 조용히 틀어진다. gc_pi처럼 절대 바뀌면 안 되는 값은 상수로 못 박으면 재대입이 문법 오류라 그런 사고가 원천 차단된다.
