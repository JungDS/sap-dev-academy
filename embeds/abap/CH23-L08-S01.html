Cloud 준비도 판정 카드 (이게 ABAP Cloud에서 될까?)

ABAP Cloud의 핵심은 서버 위치가 아니라 released API만 사용 · 제한된 언어 범위 · RAP 중심 · Clean Core입니다. 각 시나리오가 cloud-ready인지 판정해 보고, 제한되는 경우의 대안도 확인하세요.

① 이 개발은 ABAP Cloud에서 가능한가?

② classic도 배운 게 낭비가 아니다 — 둘 다 쓸 곳이 다르다

classic ABAP (온프레미스·유지보수)

  • 기존 시스템 읽기·유지보수
  • Dynpro·SALV·classic 도구
  • SAP GUI 흐름

ABAP Cloud (신규·cloud-ready)

  • RAP · CDS · released API
  • ADT 중심 개발
  • OData / Fiori 소비
제한은 불편이 아니라 안전장치예요. 표준 코어를 직접 바꾸지 않고 공개된 확장 지점과 released API로만 확장하면, SAP가 표준을 업그레이드해도 내 개발이 덜 깨집니다(Clean Core). released 여부는 CL_*·BAPI_* 이름이 익숙하다고 추측하지 말고 시스템의 release 정보를 확인하세요.
