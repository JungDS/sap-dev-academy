Singleton — 몇 번을 불러도 같은 하나

양쪽을 여러 번 눌러 보세요. get_instance()는 늘 같은 주소(인스턴스 1개)를, NEW는 매번 새 주소(인스턴스 N개)를 만듭니다.

✓ get_instance() 인스턴스 0개
⚠ NEW 매번 인스턴스 0개
💡 CREATE PRIVATE로 외부 NEW를 막고 get_instance()만 열어 두면, 최초 1회만 생성되고 이후엔 같은 인스턴스를 돌려줍니다 → 설정·캐시처럼 하나여야 하는 상태를 한 곳에. (단, 전역 상태 남용은 테스트·동시성을 어렵게 하니 꼭 필요할 때만.)
