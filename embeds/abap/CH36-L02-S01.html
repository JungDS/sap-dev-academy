ZI_Booking 해부 — Interface View(CDS)

밑줄 부분을 클릭해 기반(ZI_) 뷰의 구성을 살펴보세요. 예매를 root로, 공연·회차를 association으로 잇습니다.

ZI_Booking (CDS View)
define root view entity ZI_Booking
  as select from zbooking
  association [1..1] to ZI_Concert as _Concert
    on $projection.concert_id = _Concert.concert_id
{
  key booking_id,
      concert_id, perf_no, customer, seats, status,
      @Semantics.user.createdBy: true
      created_by,
      _Concert, _Perf
}
밑줄 친 구문을 클릭하면 설명이 나옵니다.
💡 기반(ZI_)은 재사용·관계 중심, 화면용 주석은 다음 레슨의 Projection(ZC_)에. association은 { }에 넣어야 소비 측에서 따라갈 수 있습니다.
