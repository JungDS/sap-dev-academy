3단계 — 범위만큼만 반복 (예: 3~5단)

1~9를 다 돌며 거르지 말고, p_from부터 필요한 횟수만큼만 돌고 회차 끝에 단을 1 올립니다. 정답을 보기 전에 빈칸을 직접 채워 보세요. (아래 💡는 루프 전 입력 점검 권장 로직)

ABAP
PARAMETERS: p_from TYPE i, p_to TYPE i.
DATA: gv_dan TYPE i, gv_mul TYPE i, gv_cnt TYPE i, gv_res TYPE i.
💡 시작·종료 범위를 점검하는 로직은 여기(루프 전)에 추가하는 게 좋습니다 — 펼쳐 보기

루프 전에 두 번만 확인하면 됩니다 — ① 양 끝(시작 ≥ 2 · 종료 ≤ 9) → ② 순서(시작 ≤ 종료). 둘 다 통과해야 실행하고, 어긋나면 안내 후 멈춥니다. 신기하게도 이 둘이면 시작·종료가 모두 2~9 안에 들어옵니다(까닭은 맨 아래 🪄).

두 관문을 IF / ELSEIF로 적으면:

IF p_from < 2 OR p_to > 9.
  WRITE: / '단은 2~9 사이로 입력하세요.'.
  EXIT.
ELSEIF p_from > p_to.
  WRITE: / '시작 ≤ 종료로 입력하세요.'.
  EXIT.
ENDIF.

🪄 그런데 왜 이 두 줄로 2~9 전체가 검사될까?

①은 사실 시작의 왼쪽(≥2)과 종료의 오른쪽(≤9) — 두 경계만 직접 봅니다. 나머지 두 경계는 ②시작 ≤ 종료가 옮겨 줘요. ①을 지나왔으니 이미 시작 ≥ 2·종료 ≤ 9인데:

  • 종료 ≤ 9 이고 시작 ≤ 종료 ⇒ 시작도 ≤ 9 종료의 오른쪽 경계를 상속
  • 시작 ≥ 2 이고 종료 ≥ 시작 ⇒ 종료도 ≥ 2 시작의 왼쪽 경계를 상속

그래서 시작·종료 둘 다 결국 2~9 안에 갇힙니다. 정수축으로 보면 — 초록 실선=직접 검사한 경계, 파랑 점선=②가 옮겨 준(상속) 경계:

gv_cnt = p_to - p_from + 1.   " 출력할 단 개수
gv_dan = .            " 시작 단부터

DO  TIMES.
  DO 9 TIMES.
    gv_mul = sy-index.
    gv_res = gv_dan * gv_mul.
    WRITE: / gv_dan, 'x', gv_mul, '=', gv_res.
  ENDDO.
  gv_dan = gv_dan + .        " 다음 단으로
ENDDO.
"몇 번 돌고, 단을 어떻게 올릴까?"를 떠올리며 빈칸을 채워 보세요.
