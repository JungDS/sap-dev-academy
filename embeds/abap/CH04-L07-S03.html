3단계 — 범위만큼만 반복 (예: 3~5단)

1~9를 다 돌며 거르지 말고, pa_from부터 필요한 횟수만큼만 돌고 회차 끝에 단을 1 올립니다. 정답을 보기 전에 빈칸을 직접 채워 보세요. (아래 💡는 루프 전 입력 점검 권장 로직)

ABAP
PARAMETERS: pa_from TYPE i, pa_to TYPE i.
DATA: gv_dan TYPE i, gv_mul TYPE i, gv_cnt TYPE i, gv_res TYPE i.
💡 시작·종료 범위를 점검하는 로직은 여기(루프 전)에 추가하는 게 좋습니다 — 펼쳐 보기

루프 전에 두 번만 확인하면 됩니다 — ① 양 끝(시작 ≥ 2 · 종료 ≤ 9) → ② 순서(시작 ≤ 종료). 둘 다 통과해야 실행하고, 어긋나면 안내 후 멈춥니다. 신기하게도 이 둘이면 시작·종료가 모두 2~9 안에 들어옵니다(까닭은 맨 아래에서 그림으로).

두 관문을 IF / ELSEIF로 적으면:

IF pa_from < 2 OR pa_to > 9.
  WRITE: / '단은 2~9 사이로 입력하세요.'.
  EXIT.
ELSEIF pa_from > pa_to.
  WRITE: / '시작 ≤ 종료로 입력하세요.'.
  EXIT.
ENDIF.

구구단 점검 로직, 왜 이렇게 검사할까?

조건문이 어떻게 시작·종료를 모두 2~9로 가두는지, IF와 ELSEIF를 한 조각씩 뜯어 봅니다.

IF  pa_from < 2  OR  pa_to > 9

한 IF가 OR로 두 가지를 함께 봅니다. 둘 중 하나라도 참이면 오류 메시지를 내고 멈추죠. 그래서 아래 두 조건은 같은 IF의 왼쪽·오른쪽입니다.

IF 왼쪽 조건 pa_from < 2

  • pa_from 이 2보다 작으면 참 → WRITE문 실행 후 프로그램 종료 (OR이라 오른쪽 조건 결과와 관계없이 참)
  • pa_from 이 2 이상이면 거짓 → IF 오른쪽 조건 검사
참 → 오류 (< 2)거짓 → 다음 (≥ 2)

i 시작의 상한(9 이내)은 여기서 아직 안 봅니다

IF 오른쪽 조건 pa_to > 9

  • pa_to 가 9보다 크면 참 → WRITE문 실행 후 프로그램 종료
  • pa_to 가 9 이하면 거짓 → IF 두 조건 모두 거짓이므로 ELSEIF로 이동
거짓 → 다음 (≤ 9)참 → 오류 (> 9)

i 종료의 하한(2 이상)은 여기서 아직 안 봅니다

ELSEIF 조건 pa_from > pa_to

ELSEIF는 이전 IF가 거짓일 때만 진행

  • pa_from < 2 가 거짓 → pa_from ≥ 2 (2 이상)
  • pa_to > 9 가 거짓 → pa_to ≤ 9 (9 이하)
➜

ELSEIF pa_from > pa_to

참(시작 > 종료)이면 → WRITE문 실행 후 프로그램 종료
거짓(시작 ≤ 종료)이면 → ENDIF 다음 문장 실행

A 왜 pa_from 도 9 이하일까?

  • IF 오른쪽이 거짓 → pa_to ≤ 9, ELSEIF가 거짓(통과) → pa_from ≤ pa_to.
  • 이으면 pa_from ≤ pa_to ≤ 9 → 그래서 pa_from도 9 이하!

B 왜 pa_to 도 2 이상일까?

  • IF 왼쪽이 거짓 → pa_from ≥ 2, ELSEIF가 거짓(통과) → pa_from ≤ pa_to.
  • 이으면 2 ≤ pa_from ≤ pa_to → 그래서 pa_to도 2 이상!
🎯

두 조건을 모두 통과한 순간 pa_from·pa_to는 자동으로 둘 다 2~9 범위 안에 있게 됩니다.

gv_cnt = pa_to - pa_from + 1.   " 출력할 단 개수
gv_dan = .            " 시작 단부터

DO  TIMES.
  DO 9 TIMES.
    gv_mul = sy-index.
    gv_res = gv_dan * gv_mul.
    WRITE: / gv_dan, 'x', gv_mul, '=', gv_res.
  ENDDO.
  gv_dan = gv_dan + .        " 다음 단으로
ENDDO.
