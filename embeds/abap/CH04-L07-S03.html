3단계 — 범위만큼만 반복 (예: 3~5단)

1~9를 다 돌며 거르지 말고, p_from부터 필요한 횟수만큼만 돌고 회차 끝에 단을 1 올립니다. 정답을 보기 전에 빈칸을 직접 채워 보세요. (아래 💡는 루프 전 입력 점검 권장 로직)

ABAP
PARAMETERS: p_from TYPE i, p_to TYPE i.
DATA: gv_dan TYPE i, gv_mul TYPE i, gv_cnt TYPE i, gv_res TYPE i.
💡 시작·종료 범위를 점검하는 로직은 여기(루프 전)에 추가하는 게 좋습니다 — 펼쳐 보기 
IF p_from < 2 OR p_to > 9.
  WRITE: / '단은 2~9 사이로 입력하세요.'.
  EXIT.
ENDIF.

EXIT는 지금 실행 중인 범위를 중단합니다 — 반복문(DO/WHILE) 안이면 그 반복을 빠져나가고(CH04-L05), 반복문 밖(프로그램 본문)이면 프로그램(현재 처리 블록)을 멈춥니다. 그래서 루프 전 입력 점검에서 값이 잘못되면 EXIT로 바로 중단할 수 있어요(루프 안에서 매번 거를 필요가 없습니다).

gv_cnt = p_to - p_from + 1.   " 출력할 단 개수
gv_dan = .            " 시작 단부터

DO  TIMES.
  DO 9 TIMES.
    gv_mul = sy-index.
    gv_res = gv_dan * gv_mul.
    WRITE: / gv_dan, 'x', gv_mul, '=', gv_res.
  ENDDO.
  gv_dan = gv_dan + .        " 다음 단으로
ENDDO.
"몇 번 돌고, 단을 어떻게 올릴까?"를 떠올리며 빈칸을 채워 보세요.
