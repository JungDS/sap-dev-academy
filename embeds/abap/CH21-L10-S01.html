예약 로직을 클래스로 묶었습니다. + 공개(PUBLIC)는 외부에서 호출, − 비공개(PRIVATE)는 클래스 안에서만. 아래에서 좌석 수를 바꿔 가며 book( )을 직접 실행해 보세요.

+book( iv_seats ) RAISING zcx_fully_booked

↗EVENTS sold_out매진 시 발생 → handler가 반응

동작
캡슐화. 정원·회차 같은 상태는 PRIVATE로 숨기고, 외부는 remaining( )·book( ) 같은 PUBLIC 메서드로만 다룹니다. 실패는 RAISING zcx_fully_booked 예외로, 매진이라는 사건은 EVENTS sold_out으로 알립니다. 실제 저장은 이후 DML 챕터의 몫입니다.
