제안 vs 검증 스위치보드 (Value Table · Foreign Key)

두 스위치를 켜고 끄며 없는 값 C999가 어떻게 처리되는지 보세요. Value Table은 "후보 제안"일 뿐 검증을 강제하지 않고, 실제로 막아 주는 것은 Foreign Key입니다.

Value Table ZCONCERT
Domain 수준 — "이 도메인 값은 보통 여기서 온다"는 제안
Foreign Key ZBOOKING-CONCERT_ID
테이블 필드 수준 — "이 값은 반드시 저 Check Table에 있어야 한다"
선언 상태 → C999 입력 결과

없음있음거부실제 관계가 있어 검증

보너스 · Conversion Routine(ALPHA) — 화면 표시값 ≠ 내부 저장값

ID 비교·조회 때 형식이 안 맞으면 "분명히 있는데 없음"처럼 보일 수 있습니다.
기억법 — Value Table = "이 도메인은 보통 여기서 온다"(제안), Foreign Key = "이 필드는 반드시 저 테이블을 참조한다"(검증). 검증이 걸렸는지는 테이블 필드의 Foreign Key 화면까지 확인해야 합니다.
