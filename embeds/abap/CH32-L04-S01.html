SELECT in LOOP — 1 + N의 재앙

예매 건수 N을 늘려 보세요. LOOP 안에서 매 행 SELECT하면 DB 왕복이 1+N회로 폭발하지만, 미리 한 번에 읽으면(FAE/JOIN) 2회로 끝납니다.

❌ LOOP 안 SELECT
LOOP… SELECT SINGLE … ENDLOOP
DB 왕복
✅ 사전 조회(FAE/JOIN)
SELECT … FOR ALL ENTRIES → READ
DB 왕복
💡 FOR ALL ENTRIES(CH13)나 JOIN으로 한 번에 읽고, 메모리에서 READ TABLE … BINARY SEARCH(CH06)로 찾습니다. FAE는 드라이버 테이블이 비면 전체 조회되니 비었으면 건너뜁니다. 중첩 SELECT는 금지.
