사람이 늘면 변수가 폭발한다 (3명만 담아 봐도)

이름·나이·금액을 가진 사람 세 명을 담아 보세요. Structure 없이(왼쪽) vs Structure로(오른쪽). 네 번째 사람이 등장하는 순간을 상상하면 차이가 분명해집니다.

✗ 단일 값 변수 — 이름에 번호가 붙기 시작한다
DATA gv_name   TYPE string.
DATA gv_age    TYPE i.
DATA gv_amount TYPE p LENGTH 8 DECIMALS 2.

DATA gv_name2   TYPE string.   " 2번째 사람…
DATA gv_age2    TYPE i.
DATA gv_amount2 TYPE p LENGTH 8 DECIMALS 2.

DATA gv_name3   TYPE string.   " 3번째 사람…
DATA gv_age3    TYPE i.
DATA gv_amount3 TYPE p LENGTH 8 DECIMALS 2.
  • 선언만 9줄 — 사람마다 3줄씩 늘어난다
  • gv_name2·gv_age3… 번호 붙은 이름은 "이 셋이 한 사람"이라는 걸 사람 머릿속으로만 묶는다
  • 4번째 사람? → 또 3줄 + 대입·출력 코드 전부 복사
사람 1명 추가 = 선언 3줄 + 곳곳의 코드 수정
이 코드가 메모리에 만드는 것 — 변수 9개가 따로따로
gv_namegv_agegv_amount gv_name2gv_age2gv_amount2 gv_name3gv_age3gv_amount3
✓ Structure + 타입 — 사람 1명 = 변수 1개
TYPES: BEGIN OF ts_person,
         name   TYPE string,
         age    TYPE i,
         amount TYPE p LENGTH 8 DECIMALS 2,
       END OF ts_person.

DATA gs_p1 TYPE ts_person.   " 정훈영
DATA gs_p2 TYPE ts_person.   " 손흥민
DATA gs_p3 TYPE ts_person.   " 유재석
  • 모양은 한 번만 정의 — 사람은 변수 하나로 끝
  • "한 사람 = 한 묶음"이 코드에 그대로 보인다(gs_p1-name·gs_p1-age…)
  • 4번째 사람? → DATA gs_p4 TYPE ts_person. 한 줄
사람 1명 추가 = 선언 1줄
이 코드가 메모리에 만드는 것 — 변수 3개, 전부 같은 모양
gs_p1👤 name🔢 age💰 amount
gs_p2👤 name🔢 age💰 amount
gs_p3👤 name🔢 age💰 amount
😱 실무 스케일 — 사람의 정보가 3개일 리 없다: 성별·생일·부서·직급·입사일·연봉·연락처 +7

✓ Structure — TYPES 안에 7줄이 끝
TYPES: BEGIN OF ts_person,
         name   TYPE string,
         age    TYPE i,
         amount TYPE p LENGTH 8 DECIMALS 2,
         gender TYPE c LENGTH 1,   " + 추가는
         birth  TYPE d,            "   여기
         dept   TYPE string,       "   7줄뿐
         grade  TYPE string,
         hire   TYPE d,
         salary TYPE i,
         phone  TYPE string,
       END OF ts_person.

DATA gs_p1 TYPE ts_person.   " 그대로 — 수정 0줄
DATA gs_p2 TYPE ts_person.   " 그대로
DATA gs_p3 TYPE ts_person.   " 그대로
⬇ TYPES에 7줄 추가 → 세 변수에 자동 반영
gs_p1👤 name🔢 age💰 amount🚻 gender🎂 birth🏢 dept🎖️ grade📅 hire💵 salary📞 phone
gs_p2👤 name🔢 age💰 amount🚻 gender🎂 birth🏢 dept🎖️ grade📅 hire💵 salary📞 phone
gs_p3👤 name🔢 age💰 amount🚻 gender🎂 birth🏢 dept🎖️ grade📅 hire💵 salary📞 phone

변수 선언(gs_p1～p3)은 한 글자도 안 바뀝니다. 4번째 사람? 여전히 1줄.

단일 값 변수 = 곱셈(사람 × 정보), Structure = 덧셈(정보만큼만).
정보 20개 · 사람 10명이면? 단일 값 변수는 200개, Structure는 TYPES 한 번 + 변수 10개.
번호 붙은 변수(gv_name2…)가 보이기 시작하면 그건 "묶어야 한다"는 신호입니다. 관련된 값을 한 덩어리로 묶는 그릇, 그게 이 레슨의 Structure입니다.
