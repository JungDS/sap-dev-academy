왜 변수가 필요한가 — 같은 값을 여러 곳에 쓸 때

인사말을 세 번 출력합니다. 따옴표로 직접 적은 쪽(왼쪽)과 변수 하나에 담은 쪽(오른쪽). 인사말을 바꿔야 하면 어디를 고쳐야 할까요?

✗ 리터럴을 직접
WRITE / 'Hello, ABAP!'.
WRITE / 'Hello, ABAP!'.
WRITE / 'Hello, ABAP!'.
  • 같은 문구가 세 곳에 흩어져 있다
  • 바꾸려면 세 줄을 다 고쳐야 — 하나 빠뜨리면 버그
바꿀 곳: 3군데
✓ 변수 하나에 담아
DATA gv_msg TYPE string.
gv_msg = 'Hello, ABAP!'.

WRITE / gv_msg.
WRITE / gv_msg.
WRITE / gv_msg.
  • 값은 한 곳(gv_msg)에서 관리
  • 바꿀 땐 대입 한 줄만 — 출력 세 줄은 그대로
바꿀 곳: 1군데
계산 결과를 보관하거나, 같은 값을 여러 번 쓰거나, 나중에 바꿔야 할 때 — 값을 담아 둘 그릇인 변수가 필요합니다.
