CDS View Entity 활성화 관찰기 (테이블 → 모델 → 소비)

단계를 따라가며 zconcert 테이블 위에 ZI_Concert라는 읽기 모델을 만들어 봅니다. 필드 오타 주입을 켜면 활성화가 어떻게 실패하는지, 끄면 SELECT FROM ZI_Concert로 소비까지 어떻게 이어지는지 관찰하세요.

관점 전환이 핵심. CDS View Entity는 프로그램을 하나 더 만드는 게 아니라, 읽기 모델을 재사용 가능한 이름으로 만드는 일입니다. 활성화해도 바뀌는 건 테이블 데이터가 아니라 ZI_Concert라는 모델 정의예요.
