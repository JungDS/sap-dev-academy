사람 셋을 담아 보면 (단일 값 변수만으로)

이름·나이·금액을 가진 사람 세 명을, 지금까지 배운 단일 값 변수만으로 담아 보세요. 그런데 같은 선언을 반복해서 적다 보니 실수도 끼어들었습니다. 점선 밑줄 세 곳을 살펴보세요.

✗ 선언만 9줄 — 게다가 실수까지 숨어든다
DATA gv_name   TYPE string.
DATA gv_age    TYPE i.
DATA gv_amount TYPE p LENGTH 8 DECIMALS 2.

DATA gv_naem2   TYPE string.   " 2번째 사람…
DATA gv_age2    TYPE i.
DATA gv_amount2 TYPE p LENGTH 8.

DATA gv_name3   TYPE c LENGTH 10.   " 3번째 사람…
DATA gv_age3    TYPE i.
DATA gv_amount3 TYPE p LENGTH 8 DECIMALS 2.
⚠ 점선 밑줄 세 곳 — 뭐가 잘못됐을까? (정답 보기)
  • 이름 오타 — gv_naem2. 선언 자체는 통과합니다. 나중에 gv_name2를 쓰는 순간에야 "없는 변수" 오류가 나고, 원인 찾기는 내 몫입니다.
  • DECIMALS 누락 — p LENGTH 8만 적으면 소수 자리가 0. 오류도 안 나고, 1500.50을 넣으면 반올림되어 1501이 됩니다.
  • string 대신 c 10 — 10자가 넘는 이름은 조용히 잘립니다. 역시 아무 경고 없음.
같은 선언을 사람마다 반복하는 동안, 이런 실수를 막아 줄 장치가 없습니다.
사람 1명 추가 = 선언 3줄 + 번호 관리 + 실수 기회 3번
이 코드가 메모리에 만드는 것 — 변수 9개가 따로따로 (오타 이름 그대로!)
gv_namegv_agegv_amount gv_naem2gv_age2gv_amount2 gv_name3gv_age3gv_amount3
gv_naem2·gv_age2·gv_amount2가 "같은 사람"이라는 사실은 코드 어디에도 없습니다 — 여러분 머릿속에만 있죠. 심지어 오타(gv_naem2)조차 그대로 변수가 됐습니다.
