# 제목·설명 자연스러움 점검 (한국어 표현)

> 📅 최종수정: 2026-06-24 16:35 KST (작성: AI 점검)
> 🎯 content/abap 전 챕터/레슨의 **title·intro·direction**을 한국어 화자 기준으로 훑어 **부자연·어색·저급 의심**을 모두 수록(경중 표기). 적용 여부는 정훈영 판단 — 골라서 알려주면 반영.
> 표기: 추천안은 `코드체`, 한 항목에 여러 대안은 ` · `로 나열.

## 🔴 명백히 어색 — 수정 권장

### CH03-L01 · title `Domain(얕게)`
- **문제**: 부사 "얕게"를 괄호로 제목에 붙인 형태가 부자연(제목 접미사로 어울리지 않음). 나선 학습(뒤에서 깊게 재방문)을 뜻하려는 의도로 보임.
- **추천**: `Domain 기초` · `Domain 첫걸음` · `Domain 살펴보기` · `Domain — 기술 속성 정의`

### CH03-L02 · title `Data Element(얕게)`  ← (사용자 지적)
- **문제**: 위와 동일. "(얕게)"가 제목으로 어색.
- **추천**: `Data Element 기초` · `Data Element 첫걸음` · `Data Element — 의미·라벨 입히기` · `Data Element 살펴보기`

### CH03-L03 · title `PARAMETERS로 보상(F4·라벨)`
- **문제**: "보상"이 제목 명사로 뜻이 모호(무엇을 보상하는지 불명). direction의 "DDIC의 수고가 화면에서 보상받는다"는 비유는 괜찮으나 제목에 "보상"만 떼어 쓰면 어색.
- **추천(title)**: `PARAMETERS로 받는 결실 — 라벨·F4` · `DDIC가 화면에서 빛을 발한다 — PARAMETERS(라벨·F4)` · `PARAMETERS — 라벨·F4 자동 적용`
- **추천(direction)**: "DDIC에 들인 수고가 화면에서 빛을 발한다 — PARAMETERS의 라벨·F4." (보상받는다 → 빛을 발한다/결실을 맺는다)

### CH06-L04 · title `집단행 순회 · 컨트롤레벨`
- **문제**: "집단행"은 사전에 없는 어색한 조어(여러 행을 뜻하려는 듯). "컨트롤레벨"은 띄어쓰기 누락.
- **추천(title)**: `여러 행 순회 · 컨트롤 레벨` · `전체 행 순회 · 컨트롤 레벨`
- **연계**: CH06-L03 `단일행 접근` → `한 행 접근` (한 행 ↔ 여러 행 대비가 자연스러움)

### CH11-L05 · title `SALV 1차 범위와 심화 항목 분리`
- **문제**: "범위와 심화 항목 분리"가 행정/기획 문서 말투라 학습자 친화적이지 않음(저급 의심).
- **추천(title)**: `여기까지의 SALV, 심화는 다음에` · `SALV — 어디까지 다루나` · `지금의 SALV와 미루는 심화`
- **추천(direction)**: "지금 다루는 SALV는 어디까지이고, 어떤 심화를 뒤로 미루는지 짚는다."

### CH21-L06 · direction `색을 줄 단위를 고른다 — 행·컬럼·셀.`
- **문제**: "색을 … 단위를" 처럼 목적격 조사(을/를)가 겹쳐 비문.
- **추천(direction)**: `색을 어느 단위로 줄지 고른다 — 행·컬럼·셀.` · `행·컬럼·셀 중 어디에 색을 줄지 고른다.`

## 🟡 경미 / 취향 — 검토 후 선택

### CH10-L04 · title `Local Class로 모듈화 (Static-first)`
- **문제**: "(Static-first)" 영어 괄호가 입문 제목엔 다소 생경.
- **추천**: `Local Class로 모듈화 — 정적 메서드부터` (direction에 이미 "정적 메서드부터"가 있어 일관)

### CH14-L08 · title `Classic View와 CDS View Entity 비교 준비`
- **문제**: "비교 준비"가 다소 사무적.
- **추천**: `Classic View의 한계와 CDS 맛보기` · `클래식 뷰에서 CDS로 — 다음 장 예고`

### CH25-L04 · direction `잃어버린 갱신 문제와 낙관/비관적 잠금.`
- **문제**: "잃어버린 갱신 문제"는 직역투(원어 'lost update'는 정식 용어라 그대로 둬도 무방).
- **추천(원하면)**: `한쪽 변경이 덮여 사라지는 문제(lost update)와 낙관/비관적 잠금.`

### 모티프 `1차 / 2차` (예: CH11 `SALV 1차 (간단 ALV)`, CH13 `Open SQL 2차`)
- **문제**: "차수"가 학습자에겐 행정적으로 들릴 수 있음(이해는 됨).
- **대안(원하면)**: `기본 / 심화` 또는 `(1) / (2)` 표기. — 전 챕터 일관성에 영향이 크니 *보류·일괄 결정* 권장.

## 참고 — 점검했으나 문제없다고 본 것(오해 방지)
- `(캡스톤)`(CH05-L05·CH06-L06 등): 의도된 학습 모티프 + 대상 세대에 통용되는 외래어 → 유지 OK.

- 도입부 줄임표/대시 문체("…하고 싶다", "— 그리고 멈추고 건너뛴다") : 동기부여 톤으로 일관 → 유지.
