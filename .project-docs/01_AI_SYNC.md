## 🎯 단일 집중 목표 (Single Overarching Goal)

**"개발자 시선으로 0부터 안내하는 SAP 학습 여정 — 동기부여형 + 이론·체험 병행"**

분류 순서로 외우는 학습이 아니라, **불편을 먼저 겪고 그 해결책으로 개념을 배우는** 과정을 만든다.
현재 1순위 도메인은 **ABAP 커리큘럼**(35챕터/192레슨, [02_PROGRESS](02_PROGRESS.md)에서 현황). UI5/Fiori·모듈·통합실습은 이후 범위.

## ✅ 완료 정의 (Definition of Done) — 레슨/페이지 1개 기준

- [ ] **입문자 가독성**: 완전 입문자가 따라올 수 있게 작성([04 R5](04_CONVENTIONS.md) 입문자 작성법). 압축 설명 0.
- [ ] **흐름 골격**: 왜 필요한가 → 무엇인가 → 어떻게 쓰나 → 실수/주의 → 정리.
- [ ] **용어 한 줄 풀이**: 어려운 용어는 처음 등장 시 인라인으로 풀고, glossary에 등록([04 R6](04_CONVENTIONS.md)).
- [ ] **코드 = 체험 (필수)**: 본문에 코드가 1줄이라도 나오면 그 페이지에서 직접 해보는 체험/시뮬레이션을 넣는다([06_SAMPLE_LIBRARY](06_SAMPLE_LIBRARY.md)에서 수단 선택). 정적 코드만 있으면 미완.
- [ ] **시각화 동반**: 텍스트 나열 대신 그림/표/인터랙션을 함께.
- [ ] **이름 풀 준수**: 사람 이름은 고정 풀에서만, 1번은 항상 정훈영([04 R7](04_CONVENTIONS.md)).
- [ ] **검증**: 빌드 통과 + 콘솔 오류 0 + 인터랙션 동작 확인([07_BROWSER_TESTING](07_BROWSER_TESTING.md)).

작업 단위 = **한 번에 하나의 레슨/컴포넌트**에 집중한다.

## 🚨 반드시 지킬 규칙 (R = Rule, 위반 금지)

| ID | 규칙 |
|----|------|
| **R1** | **HTML/JSON 생성물은 손으로 고치지 않는다.** 내용은 `content/abap/**.md`에서만 → `npm run build:abap`로 재생성([03](03_ARCHITECTURE.md)). |
| **R2** | **`sapui5/`(구 repo)는 읽기 전용 참고 원본.** 절대 수정하지 않는다. [[sapui5-readonly]] |
| **R3** | **입문자용으로 작성.** 아는 사람용 압축 설명 금지. 용어 첫 등장 시 한 줄 풀이([04 R5](04_CONVENTIONS.md)). |
| **R4** | **코드 → 체험 필수.** 코드가 나오면 같은 페이지에서 실행/조작 체험을 붙인다. |
| **R5** | **제목은 기술형, 동기는 `intro`/`direction`.** 제목에 서사 문구 금지. |
| **R6** | **classic-first SQL 경계 = CH18.** CH07~CH16 Open SQL은 전부 classic, CH18부터 modern. [[abap-curriculum-design]] |
| **R7** | **사람 이름은 고정 풀에서만, 1번은 항상 정훈영([04 R7](04_CONVENTIONS.md)).** 즉석 작명·풀 밖 이름 금지. |
| **R8** | **공통 자산은 수정 전 전체를 먼저 읽는다**(`shell.js`·`base.css` 등) — 중복·충돌 방지. 코드 임의 생략(`// 생략`) 금지. |
| **R9** | **검증 없이 "완료"라 하지 않는다.** 빌드/콘솔/동작을 실제로 확인한 결과만 보고한다. |
