## 1. 프로젝트 분석 기준

- 학습 대상: 개발·전공 경험 없는 입문자. 설명은 압축하지 않고 "왜 필요한가 -> 무엇인가 -> 어떻게 확인하는가 -> 실수/주의 -> 정리" 순서로 풀어쓴다.
- R15 게이팅: 이미 배운 것만으로 이해할 수 있어야 한다. 후속 개념은 L1 예고까지만 허용하고, 코드나 정의를 앞당기지 않는다.
- R2 코드=체험: 코드가 1줄이라도 있으면 같은 페이지에서 조작하거나, 최소한 어떤 시뮬레이터/다이어그램/버튼 흐름으로 체험할지 명시한다.
- 참고 우선순위: 로컬 `content/abap`과 `.project-docs`가 NotebookLM보다 최신이다. NotebookLM은 큰 방향 참고로만 사용한다.

## 2. 이 챕터의 강의 목표

메모리에서 사라지던 데이터를 DB 테이블로 영속화한다.

**반드시 강화할 지점**
- Transparent Table과 Internal Table의 목적 차이를 반복
- MANDT, Key, Data Element 사용 기준을 필수로 다룸

**대표 체험 설계**
- 메모리 표와 DB 표를 나란히 두고 프로그램 종료 후 무엇이 남는지 비교한다.

**원본 신호**

- 방향: 값을 영구히 — DB에 1:1로 대응하는 투명 테이블을 만든다.

**개선 강의안**
- 도입 불편: 학습자가 이 레슨 직전에 어떤 답답함을 느꼈는지 먼저 묻고, 그 답답함의 해결책으로 Transparent Table 생성 (SE11)를 제시한다.
