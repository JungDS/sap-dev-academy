## 느린데 어디가 문제일까 — 측정부터

성능은 **추측 말고 측정**이다. 가장 흔한 병목은 DB 접근 — [[ST05]](SQL Trace)로 어떤
SELECT가 얼마나 느리고 몇 번 도는지 본다.

## 사용 흐름

1. `ST05` → Trace On → 대상 프로그램 실행 → Trace Off.
2. Display Trace로 SQL 목록·시간·횟수 분석.

볼 것:
- **느린 단일 쿼리** — 시간이 큰 SELECT(인덱스·조건 점검).
- **Identical Selects** — 같은 쿼리 반복(버퍼링·한 번 읽기로).
- **많은 횟수** — LOOP 안 SELECT 의심(L04).

## 흔한 실수와 점검

- **추측 튜닝** → 측정 없이 고치면 엉뚱한 데 손댄다. ST05 먼저.
- **운영에서 장시간 Trace** → 부하·로그. 짧게·대상 한정.
