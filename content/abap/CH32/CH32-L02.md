## 코드 어느 부분이 느린가

DB뿐 아니라 ABAP **처리 자체**(루프·내부 테이블·메서드)가 느릴 수 있다. [[SAT]]
(구 SE30, Runtime Analysis)로 어느 코드 블록이 시간을 먹는지 본다.

## 사용 흐름

1. `SAT` → 측정 변형 지정 → 프로그램 실행 → 결과.
2. **Hit List** — 가장 시간을 많이 쓴 호출 순.
3. ABAP / Database / System 시간 비율 확인.

볼 것:
- **ABAP 시간이 큼** → 비효율 루프·정렬·내부 테이블 접근(SORTED/HASHED 고려).
- **DB 시간이 큼** → ST05로(L01).

## 흔한 실수와 점검

- **측정 오버헤드** → SAT 자체가 느리게 만든다. 상대 비교용으로.
- **대표 데이터로** → 소량 데이터로 측정하면 병목이 안 보인다.
