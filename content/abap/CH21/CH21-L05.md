## 셀마다 다른 모양·동작

색뿐 아니라 셀마다 **편집 가능 여부·버튼·체크박스** 같은 모양을 다르게 줄 수 있다.
역시 행 구조에 **스타일 테이블**(`LVC_T_STYL`)을 품는 방식이다.

```abap
TYPES: BEGIN OF ty_perf,
         "... 기존 필드 ...
         cellstyles TYPE lvc_t_styl,
       END OF ty_perf.

LOOP AT lt_perf ASSIGNING FIELD-SYMBOL(<p>).
  " 매진 회차는 좌석수 셀을 읽기 전용으로
  IF <p>-seatsocc >= <p>-capacity.
    <p>-cellstyles = VALUE #( ( fieldname = 'SEATSOCC'
                                style = cl_gui_alv_grid=>mc_style_disabled ) ).
  ENDIF.
ENDLOOP.

ls_layout-stylefname = 'CELLSTYLES'.
```

- `mc_style_disabled`(비활성)·`mc_style_enabled`(편집)·`mc_style_button` 등.
- `stylefname` — 스타일 테이블 컬럼 이름을 Layout에 지정.
- **본격 편집(입력 받기·검증)** 은 CH28(Editable Grid)에서.

## 흔한 실수와 점검

- **색과 스타일 혼동** → 색=`ctab_fname`, 모양·동작=`stylefname`.
- **편집만 켜고 처리 없음** → 편집 표시는 됐어도 값 반영·검증은 별도(CH28).
