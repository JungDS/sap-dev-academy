## DB에서 정렬해 받기

내부 테이블을 `SORT`(CH06)할 수도 있지만, DB에서 바로 정렬해 받을 수 있다 —
`ORDER BY`다.

## ORDER BY (classic)

```abap
SELECT persid name age
  FROM ztperson
  INTO CORRESPONDING FIELDS OF TABLE lt_person
  ORDER BY age DESCENDING name ASCENDING.
```

- `ORDER BY 필드 [ASCENDING|DESCENDING]`, 여러 필드 나열 가능.
- `ORDER BY PRIMARY KEY` — 기본 키 순.

## ORDER BY vs ABAP SORT

- **소량·이미 읽은 데이터** → ABAP `SORT`로도 충분.
- **DB가 정렬에 유리(인덱스)** 하거나 정렬해 바로 쓰면 → `ORDER BY`.
- 무분별한 ORDER BY는 DB 부하가 될 수 있으니 상황에 맞게.

## 흔한 실수와 점검

- **대량 ORDER BY 부하** → 인덱스 없는 정렬은 비싸다. 필요한 만큼만.
- **콤마/@** → classic이라 공백 구분, `@` 없이.
