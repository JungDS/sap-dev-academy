## 집계한 다음에 거르기

`WHERE`는 **그룹 짓기 전** 행을 거른다. "인원 5명 이상인 부서만"처럼 **집계
결과로** 거르려면 `HAVING`이 필요하다.

## WHERE vs HAVING

| | 시점 | 대상 |
|---|---|---|
| `WHERE` | GROUP BY **전** | 개별 행 |
| `HAVING` | GROUP BY **후** | 집계 결과 |

```abap
SELECT dept_id COUNT( * ) AS cnt
  FROM ztperson
  INTO CORRESPONDING FIELDS OF TABLE lt_sum
  WHERE age >= 20          " 먼저 개별 행 필터
  GROUP BY dept_id
  HAVING COUNT( * ) >= 5.  " 그 다음 그룹 필터
```

## 두 번 거른다 — 단계로 보기

이 한 문장은 **두 번** 거른다.

1. `WHERE age >= 20` — 20세 미만은 **그룹 만들기 전**에 제외(개별 **행**).
2. `HAVING COUNT( * ) >= 5` — 남은 사람을 부서별로 센 뒤, 5명 미만 부서는 제외(**그룹**).

집계 조건(`COUNT`·`SUM`)은 그룹이 생긴 **뒤**에야 의미가 있으므로 `WHERE`가 아니라
`HAVING`에 둔다. `WHERE COUNT( * ) >= 5`는 잘못된 방향이다(아직 COUNT가 없다).

아래 파이프라인을 한 단계씩 진행하며 **WHERE는 행 하나**를, **HAVING은 그룹 하나**를
어떻게 떨어뜨리는지 보자.

## 흔한 실수와 점검

- **집계 조건을 WHERE에** → `WHERE COUNT(*)…`는 오류. 집계 조건은 `HAVING`.
- **순서** → `WHERE → GROUP BY → HAVING → ORDER BY`를 지킨다.
