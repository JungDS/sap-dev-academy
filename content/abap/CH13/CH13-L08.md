## 🎫 5단계 — 공연별 현황 한눈에

지금까지 예매를 행 단위로 봤다. 이제 **공연별로 묶어** 예매 좌석합계를 집계하고,
정원과 비교해 잔여 현황을 만든다. 이 실습은 **세 질문**에 답하면 도구가 정해진다.

1. 예매가 **없는 공연도** 보여야 하나? → 예 → `LEFT OUTER JOIN`
2. **취소(C)** 예매를 합계에서 빼야 하나? → 예 → JOIN 조건에 `b~status <> 'C'`
3. 예매 행 여러 개를 **공연별 한 행**으로 줄여야 하나? → 예 → `GROUP BY` + `SUM`

```abap
TYPES: BEGIN OF ts_stat,
         concert_id TYPE zconcert-concert_id,
         artist     TYPE zconcert-artist,
         capacity   TYPE zconcert-capacity,
         booked     TYPE i,
       END OF ts_stat.

DATA: lt_stat TYPE TABLE OF ts_stat,
      lo_alv  TYPE REF TO cl_salv_table.

SELECT c~concert_id c~artist c~capacity SUM( b~seats ) AS booked
  FROM zconcert AS c
  LEFT OUTER JOIN zbooking AS b
    ON c~concert_id = b~concert_id AND b~status <> 'C'
  INTO TABLE lt_stat
  GROUP BY c~concert_id c~artist c~capacity.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = lo_alv CHANGING t_table = lt_stat ).
    lo_alv->display( ).
  CATCH cx_salv_msg.
ENDTRY.
```

## 직접 해 볼 과제

1. 공연별 **예매 좌석합계**가 맞게 나오는지 확인.
2. 결과 구조에 **잔여석(capacity − booked)** 컬럼을 더해 LOOP로 계산해 본다.
3. (도전) `HAVING`으로 "예매가 정원의 80%를 넘은 공연"만 추린다.

## 흔한 실수와 점검

- **LEFT JOIN 이유** → 예매 없는 공연도 0으로 보이게(INNER면 빠진다).
- **classic 집계** → `SUM( )`·`GROUP BY`는 classic OK. modern `@`·콤마는 [Chapter 19 · Modern Open SQL](CH19-L01.html).
- **합계 매핑** → SELECT 필드 순서와 `ts_stat` Component 순서를 맞춘다.
