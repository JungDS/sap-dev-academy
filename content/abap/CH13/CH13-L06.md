## 클래식 뷰의 한계

지금까지의 DDIC 뷰(Database/Projection/Help/Maintenance)는 **GUI로 정의**하는
클래식 방식이다. 강력하지만 한계가 있다 — 계산 컬럼·복잡한 가공·버전 관리·현대
도구(Fiori/OData) 연계가 약하다.

## CDS View Entity (미리보기)

현대 ABAP은 **CDS**(Core Data Services)로 뷰를 **코드(소스)** 로 정의한다.

| | Classic View | CDS View Entity |
|---|---|---|
| 정의 | SE11 GUI | 소스 코드(ADT) |
| 가공/계산 | 제한적 | 풍부(식·집계·연관) |
| 연계 | 약함 | OData/Fiori/RAP 친화 |
| 버전관리 | 약함 | 코드라 우수 |

- 본격 학습은 **CH21(CDS View Entity 기초)**.
- 지금은 "클래식 뷰 다음에 CDS가 온다"는 지도를 갖는다.

## 흔한 오해와 점검

- **클래식 뷰는 끝물?** → 여전히 유효(특히 유지보수 뷰). 다만 신규 조회·연계는 CDS.
- **CDS를 지금 깊이** → 나선형. CH21에서.
