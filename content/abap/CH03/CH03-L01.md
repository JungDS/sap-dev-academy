## 로컬 타입의 벽

CH02의 [[TYPES]]는 편했지만 **그 프로그램 안에서만** 산다. 회사 전체가 쓰는
"금액", "통화코드" 같은 타입을 매 프로그램이 또 정의하는 건 낭비고, 정의가
갈리면 사고다. 이제 타입을 **전역**으로 올린다 — 그 시작이 [[DDIC]]의 [[Domain]]이다.

## DDIC와 Domain

[[DDIC]](SAP Dictionary)는 시스템 전체의 데이터 정의를 한 곳에서 관리하는 중앙
사전이다. 그 안에서 [[Domain]]은 **순수한 기술 속성**을 담는다.

- 데이터 타입과 길이(예: `DEC` 8자리, 소수 2)
- (선택) 허용 값 목록 = [[고정값]](Fixed Values)

> Domain은 "어떻게 생긴 값인가"만 안다. "이 값이 무슨 의미인가"는 다음 레슨의
> Data Element가 맡는다. 역할이 갈린다.

## SE11에서 Domain 만들기 (얕게)

`SE11`(Dictionary) → Domain → 이름 `ZDO_AMOUNT` → Create.

- Data Type `DEC`, Length `8`, Decimals `2` → 활성화(Ctrl+F3).

상태값처럼 정해진 값만 받아야 한다면 **Value Range** 탭에 [[고정값]]을 넣는다
(예: `O`=Open, `C`=Closed). 이 고정값은 나중에 화면 F4·입력검증으로 되돌아온다.

## 흔한 실수와 점검

- **이름 규칙** → 전역 객체도 `Z`/`Y`로 시작. 저장 시 패키지/이송요청은 CH01-L05.
- **Domain에 라벨을 기대** → 라벨·의미는 Domain이 아니라 Data Element 소관.
