## 의미와 이름표가 빠졌다

[[Domain]]은 "8자리 소수 2"라는 **기술**만 안다. 하지만 화면이나 테이블에서 이
값이 "송장 금액"인지 "단가"인지, 라벨을 뭐라 띄울지는 모른다. 그 **의미·라벨**
계층이 [[Data Element]]다.

## Data Element = 의미 + 라벨 (+ Domain)

[[Data Element]]는 [[Domain]] 위에 의미를 입힌다.

- 어떤 [[Domain]]을 쓰는지(기술 속성 상속)
- 필드 **라벨**(Short / Medium / Long / Heading)
- (선택) 문서 / F1 도움말

테이블이나 화면의 각 필드는 보통 **Data Element를 참조**한다 → 타입·라벨·도움말을
한꺼번에 물려받는다.

## SE11에서 Data Element 만들기

`SE11` → Data Element → `ZDE_AMOUNT` → Create.

- Domain에 `ZDO_AMOUNT` 지정(앞 레슨에서 만든 것).
- **Field Label** 탭에 길이별 라벨 입력(예: "금액") → 활성화.

```abap
" 프로그램에서 전역 Data Element로 변수 선언
DATA lv_amount TYPE zde_amount.
```

이제 `lv_amount`는 회사 표준 금액 정의를 그대로 따른다.

## Domain vs Data Element 한눈에

| | Domain | Data Element |
|---|---|---|
| 담당 | 기술(타입·길이·고정값) | 의미·라벨·도움말 |
| 비유 | "8자리 소수 둘" | "이건 '금액'이다" |
| 참조 | — | Domain을 참조 |

## 한 Domain, 여러 Data Element

같은 기술 속성을 **여러 의미**가 나눠 쓸 수 있다. 예로 공항 코드는 똑같이 3자리지만(같은 Domain), "출발 공항"과 "도착 공항"은 **다른 의미·라벨**이다.

| Data Element | 라벨 | 참조 Domain |
|---|---|---|
| `ZDE_FROM_AIRP` | 출발 공항 | `ZDO_AIRPORT`(3자리) |
| `ZDE_TO_AIRP` | 도착 공항 | `ZDO_AIRPORT`(3자리) |

→ Domain은 **기술을 한 번** 정의해 재사용하고, Data Element는 그 위에 **의미·라벨을 따로** 입힌다. 이게 두 계층을 나눈 이유다.

> 💡 **참고** — Data Element에 **Parameter ID**(SET/GET Parameter)를 달면, 화면이 그 필드의 **마지막 입력값을 기억**했다가 다음에 자동으로 채워 준다(사용자 편의). 지금은 "그런 속성이 있다" 정도만 알아 두자.

## 흔한 실수와 점검

- **Domain 없이 Data Element** → 가능은 하지만(직접 타입 지정), 보통은 Domain을 둬서
  기술 속성을 재사용한다.
- **라벨 비움** → 화면·ALV에서 빈 라벨이 뜬다. Field Label을 채운다.
