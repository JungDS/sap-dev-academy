## 수고의 보상을 받을 시간

[[Domain]]·[[Data Element]]를 만드느라 손이 갔다. 그 투자가 **화면에서 한 번에
회수**되는 걸 [[PARAMETERS]]로 체감한다.

## PARAMETERS — 가장 단순한 입력 화면

[[PARAMETERS]]는 프로그램 실행 시 **입력 필드 한 칸**을 선택화면에 만든다.

```abap
REPORT zdemo_param.

PARAMETERS p_amt TYPE zde_amount.

START-OF-SELECTION.
  WRITE: / '입력값:', p_amt.
```

실행(F8)하면 입력 화면이 먼저 뜨고, 값을 넣으면 `p_amt`로 들어온다.

## 공짜로 따라오는 것들

`p_amt`의 타입을 [[Data Element]] `zde_amount`로 줬더니:

- **라벨** — Data Element의 Field Label("금액")이 입력칸 옆에 자동으로 뜬다.
- **[[F4]] 도움말** — Domain에 [[고정값]]이나 검색도움말이 있으면 입력칸에서 F4를
  눌러 **선택 목록**이 나온다.
- **입력검증** — 고정값을 벗어난 값은 자동으로 거부된다.

> 이게 "보상"이다. 타입을 `TYPE c LENGTH 8`처럼 줬다면 라벨도 F4도 없다.
> **Data Element로 줬기에** 화면이 똑똑해졌다.

## 유용한 PARAMETERS 옵션

입력칸에 몇 가지 속성을 더할 수 있다.

```abap
PARAMETERS p_amt  TYPE zde_amount OBLIGATORY.   " 필수 입력(빈칸이면 거부)
PARAMETERS p_cnt  TYPE i          DEFAULT 10.   " 화면에 미리 채울 기본값
PARAMETERS p_name TYPE c LENGTH 20 LOWER CASE.  " 소문자 보존(기본은 대문자 변환)
```

- `OBLIGATORY` — 비워 두면 실행을 막는다(필수).
- `DEFAULT 값` — 화면에 미리 채워지는 기본값.
- `LOWER CASE` — 입력을 대문자로 바꾸지 않고 그대로 둔다.

## 흔한 실수와 점검

- **라벨이 안 떠요** → Data Element의 Field Label이 비었거나, 타입을 표준 타입으로
  직접 줬다. Data Element 참조로 바꾼다.
- **F4가 없어요** → Domain에 고정값/검색도움말이 없다. 입력검증이 필요하면 추가.
- **PARAMETERS 이름 길이** → 8자 이내(예: `p_amt`).
