## 수고의 보상을 받을 시간

[[Domain]]·[[Data Element]]를 만드느라 손이 갔다. 그 투자가 **화면에서 한 번에
회수**되는 걸 [[PARAMETERS]]로 체감한다.

## PARAMETERS — 가장 단순한 입력 화면

[[PARAMETERS]]는 프로그램 실행 시 **입력 필드 한 칸**을 선택화면에 만든다.

```abap
REPORT zdemo_param.

PARAMETERS pa_amt TYPE zde_amount.

WRITE: / '입력값:', pa_amt.
```

실행(F8)하면 **입력 화면이 먼저** 뜨고, 값을 넣고 실행하면 그제야 아래 [[WRITE]]가
돌며 `pa_amt`의 값이 찍힌다.

> ⚠️ **이름은 최대 8자(문법 제한).** PARAMETERS의 이름이 8자를 넘으면 **구문
> 오류**가 난다. 접두어는 SAP 표준 교재 관례인 **`pa_`** 를 따른다. 이 과정의
> 예제도 전부 `pa_`다. 참고로 실무 코드에서는 8자 안에서 한 글자라도 아끼려고
> `p_`로 줄여 쓴 경우도 자주 만난다(같은 관례의 축약형이니 당황하지 말 것).

## 공짜로 따라오는 것들

`pa_amt`의 타입을 [[Data Element]] `zde_amount`로 줬더니:

- **라벨** — 입력칸 옆 글자는 정확히는 프로그램의 **Selection Text**(선택화면 라벨)인데,
  Selection Text 화면에서 **Dictionary 참조 체크 한 번**이면 Data Element의 Field
  Label("금액")을 그대로 이어받는다. 아무것도 안 하면 `PA_AMT` 같은 기술 이름이 보인다.
- **[[F4]] 도움말** — Domain에 [[고정값]]이 있거나 Data Element에 검색도움말이
  연결돼 있으면 입력칸에서 F4를 눌러 **선택 목록**이 나온다.

> 이게 "보상"이다. 타입을 `TYPE c LENGTH 8`처럼 줬다면 라벨도 F4도 없다.
> **Data Element로 줬기에** 화면이 똑똑해졌다.

> ⚠️ **단, F4 ≠ 검증.** F4는 *골라 주는 도움말*일 뿐이라, 목록 밖 값을 직접 쳐도
> 기본으로는 **그냥 통과**한다. Domain 고정값 기준으로 **막으려면** 선언에
> `VALUE CHECK`를 붙여야 한다: `PARAMETERS pa_stat TYPE zde_status VALUE CHECK.`
> (DDIC 타입을 참조할 때만 붙일 수 있다.)

## 유용한 PARAMETERS 옵션

입력칸에 몇 가지 속성을 더할 수 있다.

```abap
PARAMETERS pa_amt  TYPE zde_amount OBLIGATORY.   " 필수 입력(빈칸이면 거부)
PARAMETERS pa_cnt  TYPE i          DEFAULT 10.   " 화면에 미리 채울 기본값
PARAMETERS pa_name TYPE c LENGTH 20 LOWER CASE.  " 소문자 보존(기본은 대문자 변환)
PARAMETERS pa_stat TYPE zde_status VALUE CHECK.  " Domain 고정값으로 입력 검증
```

- `OBLIGATORY` — 비워 두면 실행을 막는다(필수).
- `DEFAULT 값` — 화면에 미리 채워지는 기본값.
- `LOWER CASE` — 입력을 대문자로 바꾸지 않고 그대로 둔다. (기본 동작은 **자동 대문자 변환**이다.
  소문자를 살려야 하는 값이면 꼭 붙인다.)
- `VALUE CHECK` — Domain [[고정값]]과 대조해 **목록 밖 값을 거부**한다. DDIC 타입 참조일
  때만 가능하고, **이게 없으면 F4만 나올 뿐 검증은 되지 않는다.**

## 보상을 직접 확인 — 같은 칸, 두 타입

아래에서 같은 입력칸의 타입을 **Data Element ↔ 표준 타입 직접 지정**으로 갈아 끼워 보자.
라벨·[[F4]]가 붙었다 떨어지고, **검증은 `VALUE CHECK` 스위치가 따로**라는 것까지
한눈에 보인다. F4 목록 밖 값(`X`)을 넣고 `VALUE CHECK`를 껐다 켜 보라. `OBLIGATORY`·`LOWER CASE`도 함께 실험.

## 흔한 실수와 점검

- **라벨이 안 떠요** → Data Element의 Field Label이 비었거나, 타입을 표준 타입으로
  직접 줬거나, Selection Text의 **Dictionary 참조 체크가 꺼져** 있다.
- **F4가 없어요** → Domain에 고정값이 없고, Data Element에 검색도움말도 없다.
- **F4가 뜨니 검증도 되겠지** → 아니다. F4는 *도움말*, 막는 건 **`VALUE CHECK`**.
  목록 밖 값이 그냥 통과한다면 이걸 빠뜨린 것.
- **PARAMETERS 이름 길이** → 8자 이내(예: `pa_amt`).
