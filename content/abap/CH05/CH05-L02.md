## 구조도 전역으로

CH03에서 타입을 전역(Domain·Data Element)으로 올렸듯, **구조체 모양**도 전역으로
올릴 수 있다. 여러 프로그램·화면·테이블이 같은 구조를 공유하게 하는 게
[[DDIC]] Structure다.

## SE11에서 Structure 만들기

`SE11` → Data Type → **Structure** → `ZST_PERSON` → Create.

- 컴포넌트(필드)를 추가하고, 각 필드 타입은 보통 [[Data Element]]로 준다
  (라벨·도움말까지 상속).

| Component | Typing | 예 |
|---|---|---|
| NAME | Data Element | `ZDE_NAME` |
| AGE | Data Element | `ZDE_AGE` |
| AMOUNT | Data Element | `ZDE_AMOUNT` |

활성화하면 전역 구조 타입이 된다.

## 프로그램에서 쓰기

```abap
DATA gs_person TYPE zst_person.

gs_person-name = '정훈영'.
WRITE gs_person-name.
```

로컬 구조(`BEGIN OF`)와 **쓰는 법은 똑같다** — 차이는 "정의가 어디 사느냐"뿐.
전역이라 여러 곳에서 같은 모양을 공유하고, 한 곳만 고치면 전부 반영된다.

## DDIC 타입으로 쓸 수 있는 것 — Domain만 빼고

변수나 컴포넌트의 타입으로 줄 수 있는 DDIC 객체는 여럿이다. 단 **Domain은 직접 못 준다**(CH03 — Domain은 Data Element를 통해서만).

| DDIC 객체 | 변수/컴포넌트 타입으로? | 비고 |
|---|---|---|
| **Data Element** | ✅ | 가장 흔함(라벨·도움말 상속) |
| **Structure** | ✅ | 구조 안에 구조(중첩) |
| **Transparent Table** | ✅ | 그 테이블의 **행 한 줄** 모양 |
| **View** | ✅ | 뷰의 행 모양 |
| **Table Type** | ✅ | 내부 테이블 타입(→ CH06) |
| **Domain** | ❌ | 변수 타입 직접 불가 — DE 경유 |

- 컴포넌트에 Data Element 대신 **다른 구조**를 주면 **중첩 구조**가 된다(다음 레슨에서 INCLUDE·APPEND와 함께).

> 💡 **심화(지금은 가볍게)** — 컴포넌트가 **전부 문자**(C·N 등)인 구조는, 통째로 한 문자 변수처럼 다룰 수도 있다(예: `year`·`month`·`day` 8자리 구조에 `sy-datum`을 통째로 대입). "그런 트릭이 있다" 정도만 알아 두자.

## 언제 로컬, 언제 DDIC?

- 그 프로그램에서만 쓰는 임시 묶음 → **로컬**(`BEGIN OF`).
- 여러 프로그램·테이블이 공유 → **DDIC Structure**.
- (참고) 다음 챕터의 투명 테이블도 사실 이 구조 위에 행이 쌓이는 형태다.

## 흔한 실수와 점검

- **필드를 표준 타입으로 직접** → 가능하지만 라벨·도움말을 잃는다. 가능하면 Data
  Element 참조.
- **이름 규칙/활성화** → `Z`/`Y` 시작, 활성화(Ctrl+F3) 안 하면 사용 불가.
