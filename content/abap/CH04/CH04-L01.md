## 대입과 초기화 — `=` 와 `CLEAR`

계산에 들어가기 전에 가장 기본부터 짚고 가자. **값을 변수에 넣는 일**이다. `=`는 수학의
'같다'가 아니라, **오른쪽을 왼쪽 변수에 넣어라**라는 뜻의 *대입*이다.

```abap
DATA gv_a TYPE i.
DATA gv_b TYPE i.

gv_a = 10.          " 오른쪽 값(10)을 왼쪽 변수에 넣는다
gv_b = gv_a.        " gv_a의 값을 gv_b에 복사 → gv_b = 10
gv_a = 25.          " 다시 넣으면 이전 값(10)은 덮여 사라진다 → gv_a = 25
```

변수를 **처음 상태로 비우려면** [[CLEAR]]를 쓴다(숫자는 0, 문자는 공백으로 돌아간다).

```abap
DATA gv_amount TYPE i VALUE 500.

CLEAR gv_amount.     " gv_amount = 0
```

> 옛 코드에는 `MOVE a TO b`(= `b = a`)도 보이는데, 지금은 **`=`가 표준**이다.
> `MOVE`는 굳이 새로 쓰지 않는다(legacy).

## 값을 받았지만, 계산을 못 한다

[[PARAMETERS]]로 숫자를 입력받는 데까지 왔다([Chapter 3](CH03-L03.html)). 그런데 막상 그 값으로
**무언가를 계산**하려니 방법이 없다. 입력받은 두 수를 더하고 싶고, 금액에 부가세를
곱하고 싶다. 변수에 값을 **담는 것**과, 그 값으로 **계산하는 것**은 다른 일이다.

이번 레슨에서 ABAP의 **산술 연산자**부터 익힌다.

## 사칙연산 — `+ - * /` 와 제곱 `**`

연산 결과도 `=`로 변수에 담는다(왼쪽 변수 = 오른쪽 계산 결과).

```abap
DATA gv_result TYPE i.

gv_result = 7 + 3.      " 10
gv_result = 7 - 3.      " 4
gv_result = 7 * 3.      " 21
gv_result = 2 ** 10.    " 1024 (2의 10제곱)

WRITE: / gv_result.
```

> ⚠️ **연산자 앞뒤에는 공백이 필수**다. `7+3`(X) → `7 + 3`(O). ABAP은 토큰을
> 공백으로 끊어 읽기 때문에 붙여 쓰면 오류가 난다.

## 계산 순서 — 우선순위와 괄호

여러 연산이 한 줄에 섞이면 **정해진 순서**로 계산된다. 제곱 `**`이 가장 먼저,
그다음 `*`·`/`, 마지막으로 `+`·`-` 순이다. 같은 등급끼리는 **왼쪽부터** 차례로 한다
(단 `**`끼리 이어질 때만 예외로 **오른쪽부터** 계산한다).

```abap
DATA gv_n TYPE i.

gv_n = 2 + 3 * 4.        " 14  — 곱하기가 먼저: 3 * 4 = 12, 거기에 + 2
gv_n = ( 2 + 3 ) * 4.    " 20  — 괄호가 먼저: 5 * 4
```

순서를 바꾸고 싶으면 괄호 `( )`로 묶는다. 묶은 부분이 **먼저** 계산된다. 이때도
괄호 안팎에 **공백**을 둔다: `(2+3)`(X) → `( 2 + 3 )`(O).

## 나눗셈은 소수까지 — 타입에 주의

정수([[I]]) 변수에 나눗셈 결과를 담으면 **반올림**되어 소수가 사라진다. 소수가
필요하면 [[P]](Packed) 타입을 쓴다.

```abap
DATA gv_i   TYPE i.
DATA gv_dec TYPE p LENGTH 8 DECIMALS 2.

gv_i   = 10 / 4.    " 3  (2.5가 반올림됨!)
gv_dec = 10 / 4.    " 2.50
```

> ⚠️ **0으로 나누지 말 것** — 나누는 수가 `0`이면 나눗셈 자체가 정의되지 않아
> 프로그램이 **런타임 오류**(실행 도중 멈추는 오류)로 중단된다. 나누는 값이 `0`이
> 될 수 있는지 늘 확인한다.

그래서 보통은 **나누기 전에 `0`인지 검사**한다. 검사는 `IF`문으로 하는데, `IF`는
아직 안 배웠으니 지금은 **"0이 아닐 때만 나눈다"는 흐름만** 읽고 넘어가자(정식 학습은
[Chapter 4 · IF와 조건식](CH04-L03.html)).

```abap
DATA gv_x TYPE i.
DATA gv_y TYPE i VALUE 0.

" [선행 사용] IF는 뒤 레슨(IF와 조건식)에서 배운다. 지금은 분석·암기 말고 흐름만
IF gv_y <> 0.            " gv_y가 0이 아니면 ( <> 는 '같지 않다' )
  gv_x = 100 / gv_y.     " 그때만 나눈다 → 런타임 오류 없음
ENDIF.                   " gv_y가 0이면 나누기를 통째로 건너뛴다
```

## 정수 몫과 나머지 — `DIV` · `MOD`

`/`는 소수까지 계산하지만, 때로는 **나눈 몫**(정수)과 **나머지**만 따로 알고 싶다.
그때 `DIV`(몫)와 `MOD`(나머지)를 쓴다. 둘 다 `+ - * /`와 같은 갈래의 산술 연산자다.

```abap
DATA gv_q TYPE i.
DATA gv_r TYPE i.

gv_q = 17 DIV 5.     " 몫 → 3
gv_r = 17 MOD 5.     " 나머지 → 2
```

> 💡 `MOD`는 **짝수·홀수 판별**에 자주 쓴다. `n MOD 2`가 `0`이면 짝수, `1`이면 홀수다.

## 문장형 연산 — `ADD` · `SUBTRACT` · `MULTIPLY` · `DIVIDE`

`=` 말고, **문장처럼 읽히는** classic 연산문도 있다. 현업 코드에 자주 보이니
눈에 익혀 둔다.

```abap
DATA gv_n TYPE i VALUE 100.

ADD 10 TO gv_n.          " gv_n = gv_n + 10  → 110
SUBTRACT 30 FROM gv_n.   " gv_n = gv_n - 30  → 80
MULTIPLY gv_n BY 2.      " gv_n = gv_n * 2   → 160
DIVIDE gv_n BY 4.        " gv_n = gv_n / 4   → 40
```

> 💡 `ADD 10 TO gv_n`은 `gv_n = gv_n + 10`과 똑같다. 둘 중 편한 쪽을 쓰면 된다.

## 날짜도 계산된다 — `D` 타입의 특수 산술

[[D]](날짜) 타입은 겉보기엔 `YYYYMMDD` 형식의 **8자리 문자**다(앞 레슨에서 봤다).
그런데 **예외적으로 더하기·빼기가 된다**. 단, 보통의 산술과 **다른 특수한 계산**(달력
규칙)이 들어간다.

```abap
DATA gv_date TYPE d.

gv_date = '20260131'.       " 1월 31일
gv_date = gv_date + 1.      " 보통 산술이면 '20260132'를 기대하지만,
                            " 날짜는 특수 처리라 '20260201'(2월 1일)이 된다.
```

`'20260132'` 같은 **없는 날짜**로 가지 않고, 달이 바뀌며 **다음 날**로 정확히 넘어간다.
덕분에 날짜로 자연스럽게 계산할 수 있다.

```abap
DATA gv_start TYPE d VALUE '20260101'.
DATA gv_due   TYPE d.
DATA gv_days  TYPE i.

gv_due  = gv_start + 30.        " 30일 뒤 → 20260131
gv_days = gv_due - gv_start.    " 두 날짜 차이(일수) → 30
```

- **날짜 + 숫자** → 그만큼 **뒤의 날짜**
- **날짜 - 날짜** → 그 사이의 **일수**(정수)

`'20260101'`처럼 **따옴표 안 8자리**로 날짜 리터럴을 적는다. 시간([[T]])도 똑같이
**특수 산술**이 적용된다. `HHMMSS` 6자리에 초를 더하면 분·시가 알아서 넘어간다.

## 흔한 실수와 점검

- **연산자 공백 누락** → `a+b`가 아니라 `a + b`. 괄호도 `( a + b )`처럼 띄운다.
- **계산 순서 착각** → `2 + 3 * 4`는 `20`이 아니라 `14`(곱하기 먼저). 원하는 순서는 괄호로.
- **나눗셈 소수가 사라짐** → 결과 변수를 [[I]] 대신 [[P]](`DECIMALS`)로.
- **0으로 나누기** → 런타임 오류로 멈춘다. 나누는 수가 `0`이 아닌지 확인.
- **날짜 리터럴 형식** → `'20260101'`(YYYYMMDD), 8자리·따옴표.
