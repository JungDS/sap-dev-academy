## 낱개 변수의 난립

사람 하나를 다루는데 `lv_name`, `lv_age`, `lv_amount`가 따로 논다. 함수에 넘기거나
초기화할 때 일일이 챙겨야 하고, "이 셋이 한 묶음"이라는 의미도 코드에 안 보인다.
관련된 값을 **하나로 묶고 싶다** — 그게 [[Structure]](구조체)다.

## BEGIN OF ~ END OF

[[DATA]]로 구조체를 선언한다. 묶을 필드를 `BEGIN OF`와 `END OF` 사이에 둔다.

```abap
DATA: BEGIN OF ls_person,
        name   TYPE string,
        age    TYPE i,
        amount TYPE p LENGTH 8 DECIMALS 2,
      END OF ls_person.
```

이렇게 만든 변수 하나를 [[Work Area]](작업 영역)라 부른다 — 한 건(레코드)을 담는
그릇이다. `ls_` 접두어가 관례(local structure).

## 컴포넌트 접근 — 하이픈(-)

구조체 안의 필드(컴포넌트)는 **하이픈 `-`** 로 접근한다.

```abap
ls_person-name   = 'Kim'.
ls_person-age    = 30.
ls_person-amount = '1500.50'.

WRITE: / ls_person-name, ls_person-age, ls_person-amount.
```

## 타입으로 빼두기 (재사용)

구조체 **모양**을 [[TYPES]]로 정의하면 여러 변수가 같은 구조를 공유한다.

```abap
TYPES: BEGIN OF ty_person,
         name TYPE string,
         age  TYPE i,
       END OF ty_person.

DATA: ls_a TYPE ty_person,
      ls_b TYPE ty_person.
```

## 흔한 실수와 점검

- **점(.)으로 접근** → ABAP의 컴포넌트 접근은 `-`다. `ls_person-name`(O), `.name`(X).
- **END OF 이름** → `END OF ls_person.` 처럼 같은 이름으로 닫는다.
