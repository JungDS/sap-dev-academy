## 언제나 똑같이 행동하는 프로그램

지금까지 만든 코드는 **입력이 무엇이든 늘 같은 일**을 한다. 하지만 진짜 프로그램은
"잔액이 부족하면 거절, 충분하면 승인"처럼 **상황을 보고 다르게** 행동해야 한다.
그 첫걸음이 [[IF]]다.

## 비교연산자 — 참/거짓을 묻는다

두 값을 비교하면 **참(true) 또는 거짓(false)** 이 나온다. 기호와 영문 둘 다 쓸 수 있다.

| 뜻 | 기호 | 영문 |
|---|---|---|
| 같다 / 다르다 | `=` / `<>` | `EQ` / `NE` |
| 크다 / 작다 | `>` / `<` | `GT` / `LT` |
| 크거나같다 / 작거나같다 | `>=` / `<=` | `GE` / `LE` |

## IF / ELSEIF / ELSE

조건이 참이면 그 블록을, 아니면 다음을 본다. 모든 [[IF]]는 `ENDIF`로 닫는다.

```abap
PARAMETERS p_amt TYPE i.

IF p_amt > 1000.
  WRITE: / '큰 금액'.
ELSEIF p_amt > 0.
  WRITE: / '소액'.
ELSE.
  WRITE: / '0 또는 음수'.
ENDIF.
```

## 여러 조건 묶기 — `AND` · `OR` · `NOT` · 괄호

```abap
IF p_amt > 0 AND p_amt <= 1000.
  WRITE: / '소액 결제 가능'.
ENDIF.

IF ( p_amt > 5000 OR p_vip = 'X' ) AND NOT p_blocked = 'X'.
  WRITE: / 'VIP 혜택'.
ENDIF.
```

> ⚠️ `AND`와 `OR`가 섞이면 **괄호 `( )`로 묶어** 의도를 분명히 한다. 괄호 안쪽도
> 연산자처럼 **앞뒤 공백**이 필요하다: `( p_amt > 5000 )`.

## 비었는지 묻기 — `IS INITIAL`

값이 **초기값(숫자 0·문자 공백)** 인지 확인할 땐 `IS INITIAL`이 깔끔하다.

```abap
IF p_amt IS INITIAL.        " p_amt = 0 과 같은 의미
  WRITE: / '입력값이 없습니다'.
ENDIF.
```

## ABAP에는 boolean 타입이 없다 (중요)

많은 언어에 있는 "참/거짓 전용 타입"이 ABAP에는 **없다.** 대신 **한 글자(C 1)** 에
관례로 `'X'`(참)·공백(거짓)을 담는다. 가독성을 위해 표준 [[CONSTANTS]] `abap_true`(='X')·
`abap_false`(=공백)를 쓴다.

```abap
DATA gv_flag TYPE c LENGTH 1.

gv_flag = abap_true.            " 'X'
IF gv_flag = abap_true.
  WRITE: / '플래그 켜짐'.
ENDIF.
```

비교 결과를 바로 플래그에 담고 싶으면 `XSDBOOL( )`(또는 `BOOLC( )`)를 쓴다 —
조건이 참이면 `'X'`, 거짓이면 공백을 돌려준다.

```abap
gv_flag = xsdbool( p_amt > 1000 ).   " 큰 금액이면 'X'
```

> 💡 이 `abap_true`/`abap_false` 약속은 뒤(CH10 모듈화)에서 다시 만난다. **여기서
> 한 번 제대로 익혀 두면** 이후엔 자연스럽게 읽힌다.

## 흔한 실수와 점검

- **`ENDIF` 누락** → 모든 `IF`는 `ENDIF`로 닫는다.
- **`=` 의 두 얼굴** → IF 안의 `=`는 "같냐?"(비교), 문장에서의 `=`는 "넣어라"(대입).
- **boolean을 숫자처럼** → `IF gv_flag.`만 쓰지 말고 `IF gv_flag = abap_true.`로 분명히.

## 정리

프로그램에 **판단**이 생겼다. 그런데 한 값을 여러 경우로 나눌 땐 `IF/ELSEIF`가
길어진다. 그럴 때 더 읽기 좋은 도구가 있다 → **CH04-L04(CASE 분기)**.
