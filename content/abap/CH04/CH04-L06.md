## WRITE로 찍어 보는 건 한계가 있다

변수가 어떻게 변하는지 보려고 `WRITE`를 여기저기 넣어 본 적 있을 것이다. 확인이
끝나면 다시 지워야 하고, 반복문 안이면 출력이 폭발한다. 더 좋은 방법이 있다 —
실행을 **잠깐 멈추고** 변수를 직접 들여다보는 **디버거**(Debugger)다.

## 디버거를 켜는 두 가지 방법

**① 코드에 [[BREAK-POINT]] 심기** — 멈추고 싶은 줄에 한 문장 넣는다. 실행이 그 줄에
닿으면 디버거가 뜬다.

```abap
PARAMETERS p_amt TYPE i.

BREAK-POINT.             " 여기서 멈춘다
WRITE: / '입력값:', p_amt.
```

**② 실행 중 `/h`** — 프로그램을 실행한 상태에서 명령창(좌상단)에 `/h`를 입력하고
Enter → 그다음 단계부터 디버거가 켜진다. 코드를 안 고쳐도 된다.

> 💡 첫 실습: CH03에서 만든 `PARAMETERS p_amt` 프로그램을 실행해, 내가 입력한 값이
> 정말 `p_amt`에 들어왔는지 디버거에서 확인해 보자.

## 한 줄씩 실행하기 — F5 · F6 · F7 · F8

디버거가 뜨면 노란 화살표가 **곧 실행할 줄**을 가리킨다. 단축키로 진행한다.

| 키 | 이름 | 동작 |
|---|---|---|
| `F5` | Step Into | **한 줄** 실행 |
| `F6` | Execute | 한 줄 실행하되, 다른 코드 호출은 **통째로** 한 번에 |
| `F7` | Return | 지금 들어와 있는 호출 단위를 **빠져나간다** |
| `F8` | Continue | 다음 멈춤 지점까지(또는 끝까지) **쭉** 실행 |

> 지금은 `F5`(한 줄씩)와 `F8`(끝까지)만으로 충분하다. `F6`·`F7`이 빛을 보는
> "다른 코드 호출(모듈)"은 CH10에서 배운다 — 지금 외울 필요 없다.

## 변수 값 보기 (그리고 바꾸기)

디버거 아래쪽 변수 영역에 보고 싶은 변수 이름(`p_amt`, `sy-index` 등)을 적으면
**현재 값**이 보인다. 값을 직접 고쳐서 "이 값이면 어떻게 흘러갈까"를 테스트할 수도
있다(실제 데이터가 아니라 메모리상 임시값만 바뀐다).

CH04-L05의 반복문을 디버거로 열고 `F5`를 누를 때마다 `sy-index`가 1→2→3으로
변하는 걸 보면, "반복이 돈다"는 말이 눈에 들어온다.

## WATCH POINT — 값이 바뀌는 순간 잡기

어디서 값이 바뀌는지 모를 때 [[WATCH POINT]]를 건다. "`gv_n`이 바뀌면 멈춰라"라고
걸어 두면, 그 변수가 변하는 **바로 그 줄**에서 자동으로 멈춘다.

::embed code-learning/step-debugger.html | F5로 한 줄씩 실행하며 변수 값을 관찰 — 디버거 체험::

## 흔한 실수와 점검

- **BREAK-POINT를 지우지 않고 이송** → 운영에 멈춤이 남는다. 확인 끝나면 제거.
- **`/h`를 너무 늦게** → 보고 싶은 줄을 이미 지난 뒤. 그 줄 앞에서 `/h`.
- **F5만 계속 눌러 지침** → 관심 구간 전까지는 `F8`로 건너뛰고, 거기서 `F5`.

## 정리

이제 코드를 한 줄씩 멈춰 변수를 들여다볼 수 있다 — 앞으로 모든 챕터에서 "디버거로
들여다보기"를 반복한다. 연산·분기·반복·디버깅을 다 모아, 드디어 **구구단**을
완성할 차례다 → **CH04-L07**.
