## 🎫 필요한 예매만

[Chapter 11 · SALV](CH11-L01.html)에서 예매 목록 **전체**를 표로 봤다. 하지만 실제
업무에서는 "특정 공연·특정 상태"만 필요할 때가 더 많다. 정훈영 담당자가 공연 C001의
예약 건만 보고 싶다면, 전체 6건을 띄워 눈으로 찾는 방식은 곧 한계가 온다.

이번 실습은 이 챕터에서 배운 [[Range Table]] 지식을 콘서트 예매 앱에 연결한다.
사용자가 공연ID·상태를 입력하면, 프로그램은 그 조건을 `WHERE ... IN`에 연결해
**필요한 예매만** 가져온 뒤 SALV로 보여 준다.

```abap
TABLES zbooking.

SELECT-OPTIONS: s_conc FOR zbooking-concert_id,
                s_stat FOR zbooking-status.

DATA: lt_book TYPE TABLE OF zbooking,
      lo_alv  TYPE REF TO cl_salv_table.

START-OF-SELECTION.
  SELECT * FROM zbooking INTO TABLE lt_book
    WHERE concert_id IN s_conc
      AND status     IN s_stat.

  TRY.
      cl_salv_table=>factory(
        IMPORTING r_salv_table = lo_alv
        CHANGING  t_table      = lt_book ).
      lo_alv->get_functions( )->set_all( abap_true ).
      lo_alv->display( ).
    CATCH cx_salv_msg.
      MESSAGE '예매 목록 ALV 생성 실패' TYPE 'I'.
  ENDTRY.
```

여기서 핵심은 SALV가 아니라 위에서 **이 두 줄**이다.

```abap
SELECT-OPTIONS: s_conc FOR zbooking-concert_id,
                s_stat FOR zbooking-status.
```

그리고 조건이다. 두 줄 사이는 `AND`라 **둘 다** 만족하는 예매만 남는다.

```abap
WHERE concert_id IN s_conc
  AND status     IN s_stat.
```

> 상태값은 업무 의미가 있다 — **`N` = 예약, `C` = 취소**. 이런 고정값의 관리(Domain·
> 고정값)는 [Chapter 3 · Domain](CH03-L01.html)에서 배운 DDIC 개념으로 돌아가 확인한다.

아래에서 조건을 바꿔 가며 결과가 어떻게 줄어드는지, `sy-subrc`·`sy-dbcnt`까지 확인해 보자.

## 직접 해 볼 과제

1. **`전체 (빈 범위)`** → 6건 전부 나오는지 확인(빈 range는 true).
2. **`공연 C001만`** → C001 예매만 남는지 확인.
3. **`예약만 (status=N)`** → N 상태만 남는지 확인.
4. **`취소 제외 (E)`** → C 상태가 빠지는지 확인.
5. (도전) `+ 범위 행 추가`로 Include/Exclude 행을 직접 만들어, 화면 조작 하나가
   Range Table 한 행이 되는 흐름을 설명해 보기.

## 흔한 실수와 점검

- **빈 SELECT-OPTIONS** → 비우면 전체 조회(빈 range는 모두 통과). 막고 싶으면
  [Chapter 15 · 이벤트와 선택화면](CH15-L01.html)의 입력 필수 검증을 쓴다.
- **두 조건을 OR로 오해** → 코드에 `AND`가 있으니 공연·상태 **둘 다** 만족해야 한다.
- **0건과 SALV 오류 혼동** → 조회가 0건이어도 SALV는 **빈 표**를 띄운다. SALV 생성
  실패(`cx_salv_msg`)와는 다른 문제다.
- **classic 유지** → `IN s_conc`는 classic range다. modern `@`는
  [Chapter 19 · Modern Open SQL](CH19-L01.html).

## 정리

- [[SELECT-OPTIONS]]가 공연·상태 조건을 selection table로 만든다.
- `WHERE concert_id IN s_conc AND status IN s_stat`는 **두 조건을 모두** 만족하는 예매만 조회한다.
- 빈 range는 전체 통과가 될 수 있으니 주의한다.
- [Chapter 11 · SALV](CH11-L01.html) 표시와 결합하면 **조건 입력형 예매 목록 리포트**가 된다.
