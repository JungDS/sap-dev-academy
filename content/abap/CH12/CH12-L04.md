## 여러 값, 그리고 제외

단일 From/To 입력만으로는 부족할 때가 있다. 사용자는 "C001**과** C003만",
"C001부터 C005까지지만 C004는 **빼고**", "예약(N)은 넣고 취소(C)는 **빼고**"처럼
조건을 여러 줄로 만들고 싶어 한다.

[[SELECT-OPTIONS]] 오른쪽 **화살표(다중 선택) 버튼**을 누르면 이 요구를 처리하는
팝업이 열린다. 거기서 넣은 여러 값·범위·제외는 결국 [[Range Table]]의 **여러 행**이 된다.

## 녹색은 포함(I) · 빨강은 제외(E)

다중 선택 팝업은 화면 의미를 Range Table 행으로 바꿔 준다. 탭 이름보다 **행의 의미**를
먼저 잡으면 된다.

| 화면 의미 | Range Table 행 |
|---|---|
| 포함할 단일 값 | `[[SIGN]] = I`, `OPTION = EQ` |
| 포함할 범위 | `SIGN = I`, `OPTION = BT` |
| 제외할 단일 값 | `SIGN = E`, `OPTION = EQ` |
| 제외할 범위 | `SIGN = E`, `OPTION = BT` |

> 🟢🔴 선택화면에서 `SIGN = I`(Include) 조건은 **녹색**, `SIGN = E`(Exclude) 조건은
> **빨강** 아이콘으로 구분된다. 색은 장식이 아니라 실제 `SIGN` 값과 연결된다.

## 포함은 넓히고, 제외는 뺀다

여러 행이 있을 때 결과는 이렇게 정해진다.

1. **포함(I) 행 중 하나라도** 맞으면 후보가 된다(OR처럼 후보를 넓힌다).
2. 그 후보가 **제외(E) 행에 걸리면** 최종 결과에서 빠진다.

예를 들어 다음 두 행을 보자.

| SIGN | OPTION | LOW | HIGH | 뜻 |
|---|---|---|---|---|
| I | BT | C001 | C003 | C001부터 C003까지 포함 |
| E | EQ | C002 |  | C002는 제외 |

공연 후보를 하나씩 판정하면 — C001·C003은 통과, **C002는 포함 범위에 있어도 제외에
걸려 탈락**, C004는 포함에 안 들어와 탈락이다. 즉 **제외가 포함보다 더 센 필터**처럼
작동한다.

## 흔한 실수와 점검

- **포함 여러 줄을 AND로 생각한다** → 같은 필드의 포함(I) 여러 줄은 후보를 **넓힌다**.
  "C001도 되고 C003도 된다"에 가깝다(OR).
- **녹색/빨강을 색 장식으로 본다** → 녹색=포함(I), 빨강=제외(E). 실제 `SIGN`이 다르다.
- **제외만 넣고 둔다** → 평가 규칙상 동작은 하지만, 실무에서는 "어느 범위에서 무엇을
  빼는지" 포함 범위를 함께 명확히 두는 편이 안전하다.

## 정리

- 다중 선택 팝업은 여러 [[Range Table]] 행을 화면에서 만드는 도구다.
- 포함은 `SIGN = I`, 제외는 `SIGN = E`.
- 포함은 후보를 넓히고(OR), 제외는 후보에서 뺀다.
- 녹색/빨강 아이콘은 실제 `SIGN` 값과 연결된다.

---

포함·제외는 잡았다. 그럼 각 행의 **비교 방식(`OPTION`)** — `EQ`·`BT`·`CP`는 각각 무엇을 어떻게 비교할까 → 다음 레슨.
