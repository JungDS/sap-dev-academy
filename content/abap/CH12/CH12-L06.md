## 이미 가진 목록으로 조회

화면이나 앞 단계에서 만든 **내부 테이블의 값들**을 조건으로 DB를 조회하고 싶다.
[[FOR ALL ENTRIES]](FAE)가 그 도구다 — 단, **함정이 많아 규칙을 지켜야** 한다.

## 기본형 (classic)

```abap
" 전제: lt_dept (조회 기준이 될 내부 테이블)에 dept_id들이 있음
IF lt_dept IS NOT INITIAL.          " ← 필수!
  SELECT persid name dept_id
    FROM ztperson
    INTO CORRESPONDING FIELDS OF TABLE lt_person
    FOR ALL ENTRIES IN lt_dept
    WHERE dept_id = lt_dept-dept_id.
ENDIF.
```

## 반드시 지킬 규칙

- **빈 테이블 금지** → `lt_dept`가 비어 있으면 FAE는 **WHERE를 무시하고 전체
  조회**한다(대형 사고). 항상 `IS NOT INITIAL` 확인.
- **중복 자동 제거** → FAE 결과는 **중복 행이 제거**된다. 중복이 의미 있으면 주의.
- **WHERE에 기준 테이블 필드** → `lt_dept-dept_id`처럼 기준 테이블 필드를 조건에.

## 흔한 실수와 점검

- **IS NOT INITIAL 누락** → 빈 테이블 → 전체 조회 폭탄. 1순위 점검.
- **중복 제거 간과** → 결과 건수가 기대와 다를 수 있다.
