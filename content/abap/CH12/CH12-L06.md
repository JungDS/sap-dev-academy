## 화면 없이 조건 만들기

배치/내부 호출처럼 화면이 없을 때, [[Range Table]]을 **코드로 직접** 채워 조회
조건으로 쓴다. 구조(SIGN/OPTION/LOW/HIGH)를 알면 어렵지 않다.

## 코드로 행 추가 (classic)

```abap
DATA: lr_age TYPE RANGE OF ztperson-age,
      ls_age LIKE LINE OF lr_age,
      lt_p   TYPE TABLE OF ztperson.

ls_age-sign   = 'I'.
ls_age-option = 'BT'.
ls_age-low    = 20.
ls_age-high   = 40.
APPEND ls_age TO lr_age.

SELECT * FROM ztperson INTO TABLE lt_p
  WHERE age IN lr_age.       " classic (@ 없이)
```

- `TYPE RANGE OF <필드>`로 Range Table 타입을 간단히 만든다.
- 한 행 = 한 조건. 여러 조건은 여러 번 `APPEND`.

## 흔한 실수와 점검

- **SIGN/OPTION 빈 값** → 채우지 않으면 조건이 안 먹는다. I/E·EQ/BT 등 필수.
- **modern 혼입** → classic 구간이라 SELECT에 현대식 escape 자제(→ CH19).
