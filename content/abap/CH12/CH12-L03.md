## 범위를 조회에 꽂기

[[SELECT-OPTIONS]]가 채운 [[Range Table]]을 `WHERE ... IN`으로 SELECT에 바로
꽂는다. 한 줄로 범위·다중 조건이 적용된다.

> 이 챕터는 **classic**이다. `WHERE age IN s_age`처럼 **`@` 없이** 쓴다.
> 현대식 Open SQL 구문은 CH19부터.

## WHERE … IN (classic)

```abap
DATA lt_person TYPE TABLE OF ztperson.

SELECT * FROM ztperson
  INTO TABLE lt_person
  WHERE age IN s_age.        " classic: @ 없이 Range Table 직접
```

- `IN s_age` 하나로 사용자가 입력한 모든 범위·목록 조건이 적용된다.
- 비어 있으면(아무 입력 없음) **조건 없음**으로 동작 → 전체 조회. 의도와 다르면
  검증을 추가한다.

## 여러 SELECT-OPTIONS 결합

```abap
SELECT * FROM ztperson INTO TABLE lt_person
  WHERE age  IN s_age
    AND name IN s_name.
```

## 흔한 실수와 점검

- **modern escape 습관** → 이 챕터는 classic이라 host 변수 그대로(현대식은 CH19).
- **빈 SELECT-OPTIONS = 전체** → 무조건 조회가 의도가 아니면 입력 필수 처리.
