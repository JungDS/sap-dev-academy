## 화면을 상황에 맞게

"기본 조회만 하는 사람에겐 고급 입력칸을 숨기고, 운영 담당자에겐 보여 준다." 같은
report를 두 개 만들기보다, 화면이 그려지기 **직전(PBO)** 마다 특정 필드를 숨기거나
비활성화할 수 있다. [[AT SELECTION-SCREEN OUTPUT]]이 그 자리다.

## LOOP AT SCREEN INTO gs_screen

```abap
PARAMETERS: pa_mode   TYPE c LENGTH 1,
            pa_secret TYPE c LENGTH 10.

DATA gs_screen TYPE screen.

AT SELECTION-SCREEN OUTPUT.
  LOOP AT SCREEN INTO gs_screen.
    IF gs_screen-name = 'PA_SECRET' AND pa_mode <> 'A'.
      gs_screen-active = '0'.            " 화면에서 비활성(숨김)
      MODIFY SCREEN FROM gs_screen.      " work area를 화면에 반영
    ENDIF.
  ENDLOOP.
```

- 화면 요소를 `LOOP AT SCREEN INTO gs_screen`으로 돌며 `active`·`input`·`invisible`
  같은 속성을 바꾼다.
- `gs_screen`(work area)을 고친 뒤 **반드시** `MODIFY SCREEN FROM gs_screen`으로 화면에 반영한다.
- 이 이벤트는 화면을 그릴 때마다(매 PBO) 실행된다.

> 📌 **명시 work area를 쓴다.** 짧은 `LOOP AT SCREEN.`/`MODIFY SCREEN.`(내장 `screen`)
> 형태는 공식문서상 피하는 표기다(기존 코드에서 만날 수는 있다). 또 `screen-name`은
> **대문자 필드명**(`PA_SECRET`)으로 비교한다.

아래에서 모드를 바꾸고 화면을 그려, work area 수정이 `MODIFY SCREEN` 후에야 화면에
반영되는 걸 확인하자.

## 흔한 실수와 점검

- **`MODIFY SCREEN` 누락** → work area만 바꾸고 안 반영하면 화면은 그대로다.
- **loop 밖에서 MODIFY SCREEN** → `LOOP AT SCREEN` **안**에서만 의미가 있다.
- **`screen-name` 소문자 비교** → 화면 필드명은 대문자다(`PA_SECRET`).
- **OUTPUT에서 기본값 재대입** → 매 화면마다 사용자 입력을 덮어쓴다. 기본값은
  [[INITIALIZATION]](이전 레슨), 여기선 **화면 속성만**.

## 정리

- [[AT SELECTION-SCREEN OUTPUT]]은 화면을 보내기 **직전(PBO)** 의 조정 지점이다.
- 대표 작업은 `LOOP AT SCREEN INTO gs_screen` → 속성 변경 → `MODIFY SCREEN FROM gs_screen`.
- 기본값(INITIALIZATION)·검증(AT SELECTION-SCREEN)·조회(START-OF-SELECTION)와 역할을 섞지 않는다.

---

화면 모양은 제어했다. 이제 사용자가 실행을 누른 순간 **입력을 검증**하자 → 다음 레슨(AT SELECTION-SCREEN).
