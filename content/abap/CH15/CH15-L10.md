## 🎫 7단계 — 이벤트로 엮은 리포트

CH13의 집계 리포트에, CH15의 **Report Event**를 입혀 실무형 리포트로 만든다 —
기본값·입력검증·조회·출력을 이벤트마다 제자리에.

```abap
TABLES zconcert.
PARAMETERS p_conc TYPE zconcert-concert_id.

DATA: lt_book TYPE TABLE OF zbooking,
      lv_chk  TYPE zconcert-concert_id,
      lo_alv  TYPE REF TO cl_salv_table.

INITIALIZATION.
  SELECT SINGLE concert_id FROM zconcert INTO p_conc.   " 첫 공연을 기본값으로

AT SELECTION-SCREEN.
  SELECT SINGLE concert_id FROM zconcert INTO lv_chk    " 존재 확인(classic)
    WHERE concert_id = p_conc.
  IF sy-subrc <> 0.
    MESSAGE '없는 공연 번호입니다' TYPE 'E'.       " 화면으로 복귀(CH15-L04)
  ENDIF.

START-OF-SELECTION.
  SELECT * FROM zbooking INTO TABLE lt_book WHERE concert_id = p_conc.

END-OF-SELECTION.
  TRY.
      cl_salv_table=>factory(
        IMPORTING r_salv_table = lo_alv CHANGING t_table = lt_book ).
      lo_alv->display( ).
    CATCH cx_salv_msg.
  ENDTRY.
```

## 직접 해 볼 과제

1. 기본 공연이 화면에 채워지는지(INITIALIZATION) 확인.
2. 없는 공연 번호를 넣어 `MESSAGE E`로 막히는지 확인.
3. (도전) 회차(`p_perf`)도 입력받아 회차별로 거른다.

## 흔한 실수와 점검

- **검증을 START-OF-SELECTION에** → 너무 늦다. 입력검증은 `AT SELECTION-SCREEN`(CH15-L04).
- **이벤트 순서** → INITIALIZATION → AT SELECTION-SCREEN → START → END 순.
