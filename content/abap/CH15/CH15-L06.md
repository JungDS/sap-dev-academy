## 처리 후 마무리

`START-OF-SELECTION`의 조회·가공이 끝난 뒤 실행되는 게 `END-OF-SELECTION`이다.
**최종 출력·요약**을 여기서 정리하면 흐름이 깔끔해진다.

```abap
START-OF-SELECTION.
  SELECT * FROM ztperson INTO TABLE lt_person WHERE dept_id IN s_dept.

END-OF-SELECTION.
  " 조회 결과를 ALV로 표시 (CH11)
  TRY.
      cl_salv_table=>factory(
        IMPORTING r_salv_table = lo_alv
        CHANGING  t_table      = lt_person ).
      lo_alv->display( ).
    CATCH cx_salv_msg.
  ENDTRY.
```

## 언제 나누나

- 단순 리포트는 `START-OF-SELECTION` 한 곳으로 충분.
- "조회"와 "출력"을 **의도적으로 분리**하면 가독성↑ → 출력을 `END-OF-SELECTION`에.

## 흔한 실수와 점검

- **END-OF-SELECTION에서 조회** → 본 조회는 START. END는 마무리.
- **과한 분리** → 작은 프로그램까지 억지로 나눌 필요는 없다.
