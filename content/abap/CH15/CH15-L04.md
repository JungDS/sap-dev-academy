## 잘못된 입력은 막는다

사용자가 실행(또는 Enter)을 누른 순간(PAI), 입력값을 **검증**하고 잘못됐으면
화면으로 되돌려 다시 입력하게 한다. [[AT SELECTION-SCREEN]]이 그 자리다.

## 전체 검증 — AT SELECTION-SCREEN

필드를 지정하지 않은 `AT SELECTION-SCREEN`은 **모든 입력이 들어온 뒤 마지막에 한 번**
실행된다. 화면 전체를 놓고, 여러 필드의 **조합 규칙**을 검증하기 좋은 자리다.

```abap
AT SELECTION-SCREEN.
  IF p_from > p_to.
    MESSAGE '시작일이 종료일보다 늦습니다' TYPE 'E'.  " E → 화면으로 복귀
  ENDIF.
```

- `MESSAGE ... TYPE 'E'`(Error)면 처리를 멈추고 **선택화면으로 되돌아간다**.
- 오류 시 **모든 입력 필드**가 다시 입력 대기 상태가 된다.

## 필드 하나만 콕 집어 — AT SELECTION-SCREEN ON <필드>

특정 필드 하나만 검증하려면 [[AT SELECTION-SCREEN ON]] `<필드>`를 쓴다.
전체 `AT SELECTION-SCREEN`보다 **먼저**, 그 필드를 처리하는 차례에 실행된다.

```abap
AT SELECTION-SCREEN ON p_date.
  IF p_date > sy-datum.
    MESSAGE '미래 날짜는 안 됩니다' TYPE 'E'.
  ENDIF.
```

핵심 차이는 **오류가 났을 때 화면이 어떻게 되는가**다.

> 🔎 `ON p_date`에서 에러가 나면 — **`p_date`만 다시 입력 대기**가 되고, 나머지 필드는
> **읽기 전용으로 잠긴다.** 사용자의 시선을 "고쳐야 할 그 칸"으로 모아 준다.
> (필드 지정 없는 전체 검증은 오류 시 모든 필드가 다시 열린다.)

## 전체 vs 필드 — 한눈에

| | `AT SELECTION-SCREEN` | `AT SELECTION-SCREEN ON p_x` |
|---|---|---|
| 언제 | 모든 입력 후 **마지막 1회** | `p_x` 차례에 (전체보다 **먼저**) |
| 검증 범위 | 화면 전체·**필드 간 관계** | **그 필드 하나** |
| 오류 시 | **모든 필드** 다시 입력 대기 | **그 필드만** 입력 대기, 나머지 읽기 전용 |
| 어울리는 곳 | 여러 필드 조합 규칙(기간 from↔to 등) | 개별 필드 단순 검증 |

<details>
<summary>ON 으로 더 다룰 수 있는 것들 (고급 — 맛보기)</summary>

- `AT SELECTION-SCREEN ON BLOCK <블록>` — 화면 **블록 단위** 검증
- `AT SELECTION-SCREEN ON RADIOBUTTON GROUP <그룹>` — **라디오 그룹** 검증
- `AT SELECTION-SCREEN ON HELP-REQUEST FOR <필드>` — 그 필드에서 **F1(도움말)** 을 누른 순간
- `AT SELECTION-SCREEN ON VALUE-REQUEST FOR <필드>` — 그 필드에서 **F4(입력 도움)** 을 누른 순간

검증(`ON`·`ON BLOCK`)과 도움말(F1·F4 = `HELP/VALUE-REQUEST`)은 목적이 다르다 — 본격 활용은 **CH15-L08(Selection Screen 고급)** 에서 다룬다.
</details>

## MESSAGE 제대로 알기 — 타입과 메시지 클래스

CH08에서 `MESSAGE`를 맛봤다. 이제 정식으로. 메시지는 **타입**에 따라 동작이 다르다.

| 타입 | 뜻 | 동작 |
|---|---|---|
| `I` | Information | 팝업으로 알리고 **계속 진행** |
| `S` | Status | 다음 화면 **상태바**에 표시(가벼운 안내) |
| `W` | Warning | 경고 — 고치거나 그냥 진행 가능 |
| `E` | Error | **막고** 입력 화면으로 복귀 |
| `A` | Abort | 처리를 **중단** |
| `X` | eXit | 런타임 **덤프**(프로그램 오류 진단용) |

### 메시지 클래스 (SE91)

매번 문자열을 직접 적는 대신, **메시지 클래스**(`SE91`)에 메시지를 번호로 등록해
재사용한다(다국어·일관성). `MESSAGE e001(zmsg)`처럼 부른다.

```abap
MESSAGE e001(zmsg).   " zmsg 클래스의 001번을 Error로
```

### 메시지에 값 끼우기 — &1~&4

메시지 텍스트에 `&1`~`&4` 자리표시자를 두고 `WITH`로 값을 채운다.

```abap
" 텍스트: "공연 &1 은(는) 정원(&2)을 초과했습니다"
MESSAGE e002(zmsg) WITH ls_concert-concert_id ls_concert-capacity.
```

## CH09과의 관계

- **존재/형식**은 DDIC(Foreign Key·F4)가 선언적으로(CH09).
- **업무 규칙**(미래 날짜 금지 등)은 여기서 코드로. CH09-L06의 역할 분리 그대로.

## 흔한 실수와 점검

- **TYPE 'E' 아닌 'I'** → 정보 메시지는 막지 못한다. 막으려면 `E`.
- **검증을 START-OF-SELECTION에** → 너무 늦다. 입력 단계에서 막는다.
- **필드 하나 문제인데 전체 `AT SELECTION-SCREEN`** → 오류 시 멀쩡한 필드까지 다 열려 사용자가 헷갈린다. 그 칸만 막으려면 `ON <필드>`.

---

입력까지 검증했다. 이제 메인 로직 — 조회를 실행하자 → START-OF-SELECTION(다음 레슨).
