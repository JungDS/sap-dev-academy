## "길이를 안 적어도 되는" 타입

타입 중에는 **그 자체로 완전해서 길이를 추가로 지정할 필요가 없는** 것들이
있다. 이를 [[Complete 타입]]이라 한다. 입문 단계에서 가장 자주 쓴다.

## 대표 Complete 타입

| 타입 | 뜻 | 크기 · 범위 | 예 |
|---|---|---|---|
| [[STRING]] | 가변 길이 문자열 | 길이 **가변** — 내용만큼 자람 | `'안녕'`, 긴 텍스트 |
| `I` | 정수(integer) | **4[[바이트]]** · -2,147,483,648 ～ +2,147,483,647 (약 ±21억) | `42`, `-7` |
| `F` | [[부동소수점]](float) | **8바이트** · 약 ±1.8 × 10³⁰⁸까지 · **근삿값**(유효숫자 15～16자리) | `3.14` |
| `D` | 날짜 | **8자리 문자** · `YYYYMMDD` (0001～9999년) | `'20260618'` |
| `T` | 시간 | **6자리 문자** · `HHMMSS` (000000～235959) | `'143000'` |

> 💡 **왜 Complete인가** — `I·F·D·T`는 **길이가 고정**돼 있고 `STRING`은 **가변**(내용만큼 자람)이라,
> 어느 쪽도 길이를 따로 적을 필요가 없다. 그래서 *Complete*다. (±21억을 넘는 정수가 필요하면
> `INT8`(8바이트)도 있다.)

```abap
DATA: gv_text  TYPE string,
      gv_qty   TYPE i,
      gv_today TYPE d.

gv_qty   = 10 + 5.
gv_today = '20260623'.   " 날짜는 따옴표 안 8자리(YYYYMMDD)
WRITE: / gv_qty, / gv_today.
```

> 💡 "오늘 날짜"를 자동으로 채워 주는 시스템 값(`sy-datum`)도 있는데, 그건 흐름
> 제어를 배우는 **CH04**에서 다른 시스템 값들과 함께 정리한다. 지금은 날짜를 직접
> 적어 넣는다.

## 다섯 타입의 "모양"을 눈으로

선언문은 실행해 볼 로직이 없다 — 대신 **어떤 모양의 박스가 생기는지**를 보면 된다.
[[STRING]]은 내용만큼 늘어나는 가변 박스, `D`·`T`는 자릿수가 정해진 문자 박스.
선언 직후엔 뭐가 들어 있는지(타입 기본값), 값을 넣으면 어떻게 바뀌는지 토글해 보자.

## 숫자를 적는 법

숫자 [[리터럴]]은 따옴표 없이 그대로 적는다 — 정수 `42`, 음수 `-7`. 그런데 **소수**는
따옴표로 감싼다: `'123.45'`. ABAP에서 마침표 `.`는 **문장의 끝**이라, 따옴표 없이
`123.45`라고 쓰면 `123` 다음에서 문장이 끊겨 버리기 때문이다.

```abap
DATA gv_price TYPE p LENGTH 8 DECIMALS 2.

gv_price = '123.45'.    " O — 소수는 따옴표 안에
" gv_price = 123.45.    " X — '.'이 문장 끝으로 해석됨
```

> 타입이 다른 값을 대입하면 ABAP이 **자동으로 변환**한다(예: 문자 `'10'` → 정수
> `10`). 다만 받는 변수가 짧으면 **잘릴** 수 있으니 길이에 주의한다.

## 정수 연산의 함정

`I`는 정수라 나눗셈 결과가 **반올림**된다.

```abap
DATA gv_r TYPE i.
gv_r = 7 / 2.     " 3.5 → 4 (반올림)
WRITE gv_r.
```

소수를 정확히 다루려면 다음 레슨의 `P`(packed) 타입이 필요하다.

## 흔한 실수와 점검

- **날짜를 순수 숫자로 착각** → `D`는 내부적으로 8자리 문자다. `'20260618'`처럼 다룬다.
- **정수 나눗셈 반올림** → `7 / 2`가 4가 된다. 의도와 다르면 `P` 타입을 쓴다.

---

여기까진 길이를 안 적었다. 그런데 "이름은 최대 10글자", "금액은 소수 2자리"
처럼 **길이·자릿수를 내가 정해야** 하는 타입이 있다 → 다음 레슨(Incomplete).
