## 리터럴의 한계

[Chapter 1](CH01-L04.html)에서 `WRITE 'Hello'.`처럼 값을 **따옴표에 직접** 적었다. 그런데 같은 값을
여러 번 쓰거나, 계산 결과를 보관하려면? 따옴표 리터럴로는 안 된다.
**값을 담아 둘 그릇**, 곧 [[변수]]가 필요하다.

## DATA로 변수 선언

[[DATA]] 문으로 변수를 선언한다. `이름 TYPE 타입` 형태다.

```abap
DATA gv_name TYPE string.
DATA gv_count TYPE i.
```

- `gv_` 접두어는 관례(**g**lobal **v**ariable). 아직 Subroutine(프로그램을 잘게 나누는 단위)을 안 배웠으니 **선언하는 변수는 모두 전역**이다. 그래서 `g`로 시작한다(로컬 변수 `lv_`는 뒤에서). 강제는 아니지만 팀 표준을 따른다.
- `TYPE` 뒤에 어떤 타입인지 적는다(다음 레슨들의 주제).

::embed CH02-L01-S02 | 변수 박스 — 타입 라벨 박스에 값 쪽지 담기::

## 값 넣고 출력하기

선언한 변수에 값을 대입(`=`)하고 출력해 본다.

```abap
DATA gv_msg TYPE string.
gv_msg = 'Hello, ABAP!'.
WRITE gv_msg.
```

이제 메시지를 한 곳(`gv_msg`)에서 관리한다. 바꿀 때 한 줄만 고치면 된다.

## 선언과 동시에 초기값 — `VALUE`

`=`로 나중에 넣는 대신, 선언하면서 **바로 초기값**을 주려면 `VALUE`를 붙인다.

```abap
DATA gv_qty TYPE i VALUE 10.            " 선언과 동시에 10으로 시작
DATA gv_tag TYPE c LENGTH 1 VALUE 'X'.
```

`VALUE`를 안 적으면 변수는 **타입의 기본값**으로 시작한다. 숫자는 `0`, 문자는 공백이다.

## 여러 변수 한 번에 (체인)

콜론(`:`)으로 [[DATA]]를 체인해 여러 변수를 한 번에 선언한다.

```abap
DATA: gv_first TYPE string,
      gv_last  TYPE string,
      gv_age   TYPE i.
```

## TYPE 말고 LIKE — 기존 변수를 본떠

`TYPE` 뒤에는 **타입 이름**을 적지만, `LIKE` 뒤에는 **이미 선언한 변수**를 적는다.
"저 변수와 똑같은 타입으로 하나 더"라는 뜻이다.

```abap
DATA gv_price TYPE p LENGTH 8 DECIMALS 2.
DATA gv_total LIKE gv_price.     " gv_price와 같은 타입
```

원본(`gv_price`)의 타입이 바뀌면 `gv_total`도 자동으로 따라 바뀐다. "이 변수와
짝이 되는 변수"를 만들 때 편하다.

## 흔한 실수와 점검

- **선언 안 한 변수 사용** → `gv_xxx is unknown` 오류. 먼저 [[DATA]]로 선언.
- **타입 누락** → `DATA gv_x.`도 문법상 통과는 된다. 하지만 **길이 1짜리 문자**(C)로
  선언돼 버린다(타입 생략 시의 암묵 기본값). 의도한 게 아니라면 `TYPE`을 명시한다.
- **LIKE 대상이 아래에 있음** → `LIKE`는 **이미(위에) 선언된** 변수만 본뜬다.
  참조할 원본을 먼저 선언한다.
