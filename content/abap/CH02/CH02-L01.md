## 리터럴의 한계

CH01에서 `WRITE 'Hello'.`처럼 값을 **따옴표에 직접** 적었다. 그런데 같은 값을
여러 번 쓰거나, 계산 결과를 보관하려면? 따옴표 리터럴로는 안 된다 —
**값을 담아 둘 그릇**, 곧 [[변수]]가 필요하다.

## DATA로 변수 선언

[[DATA]] 문으로 변수를 선언한다. `이름 TYPE 타입` 형태다.

```abap
DATA lv_name TYPE string.
DATA lv_count TYPE i.
```

- `lv_` 접두어는 관례(local variable). 강제는 아니지만 팀 표준을 따른다.
- `TYPE` 뒤에 어떤 타입인지 적는다(다음 레슨들의 주제).

## 값 넣고 출력하기

선언한 변수에 값을 대입(`=`)하고 출력해 본다.

```abap
DATA lv_msg TYPE string.
lv_msg = 'Hello, ABAP!'.
WRITE lv_msg.
```

이제 메시지를 한 곳(`lv_msg`)에서 관리한다 — 바꿀 때 한 줄만 고치면 된다.

## 여러 변수 한 번에 (체인)

콜론(`:`)으로 [[DATA]]를 체인해 여러 변수를 한 번에 선언한다.

```abap
DATA: lv_first TYPE string,
      lv_last  TYPE string,
      lv_age   TYPE i.
```

## TYPE 말고 LIKE — 기존 변수를 본떠

`TYPE` 뒤에는 **타입 이름**을 적지만, `LIKE` 뒤에는 **이미 선언한 변수**를 적는다.
"저 변수와 똑같은 타입으로 하나 더"라는 뜻이다.

```abap
DATA lv_price TYPE p LENGTH 8 DECIMALS 2.
DATA lv_total LIKE lv_price.     " lv_price와 같은 타입
```

원본(`lv_price`)의 타입이 바뀌면 `lv_total`도 자동으로 따라 바뀐다. "이 변수와
짝이 되는 변수"를 만들 때 편하다.

## 흔한 실수와 점검

- **선언 안 한 변수 사용** → `lv_xxx is unknown` 오류. 먼저 [[DATA]]로 선언.
- **타입 누락** → `DATA lv_x.` 만으론 부족. `TYPE`을 붙인다.
- **대입과 비교** → 둘 다 `=`를 쓴다(ABAP). 문맥으로 구분된다.

---

값은 담았다. 그런데 `TYPE` 뒤에 뭘 적나? 타입에는 **길이를 안 적어도 되는 것**과
**적어야 하는 것**이 있다. 먼저 전자(Complete) → 다음 레슨.
