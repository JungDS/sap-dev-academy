## 코드에 박힌 정체불명의 값들

코드 곳곳에 `1000`, `'X'`, `0.1` 같은 값이 흩어져 있으면 두 가지가 괴롭다.
**첫째**, 이 `1000`이 한도인지 수량인지 뜻을 알 수 없다(이런 값을 *매직 넘버*라
부른다). **둘째**, 한도가 2000으로 바뀌면 여기저기 박힌 `1000`을 **빠짐없이**
찾아 고쳐야 한다 — 하나라도 놓치면 버그다.

값에 **이름**을 붙이면 둘 다 해결된다. 그게 [[CONSTANTS]](상수)다.

## CONSTANTS로 선언하기

[[DATA]]와 거의 같지만 `VALUE`로 **값을 못 박는다**. 한번 정한 값은 실행 중에
바꿀 수 없다.

```abap
CONSTANTS gc_max_amount TYPE i VALUE 1000.
CONSTANTS gc_flag_on    TYPE c LENGTH 1 VALUE 'X'.

DATA lv_amt TYPE i VALUE 1500.

IF lv_amt > gc_max_amount.       " 1000 대신 '이름'으로 읽힌다
  WRITE: / '한도 초과'.
ENDIF.
```

- `gc_` 접두어가 관례(global constant).
- 한도가 바뀌면 **선언 한 줄(`VALUE 1000`)만** 고치면 끝 — 쓰는 곳은 그대로다.
- 상수에 값을 다시 대입(`gc_max_amount = 2000.`)하려 하면 **문법 오류**가 난다.

## SAP가 미리 만들어 둔 상수도 있다

매번 `'X'`/공백을 직접 쓰는 대신, SAP가 제공하는 표준 상수 `abap_true`(='X')·
`abap_false`(=공백)를 쓰면 뜻이 더 분명해진다(참/거짓 플래그용). 이 약속은
**CH04(흐름 제어)** 에서 조건문과 함께 다시 만난다.

> 💡 **옛 코드에서** — 클래스가 흔해지기 전에는 전역 상수·타입을 **Type Group**(`TYPE-POOLS`로 불러 씀)에 모아 뒀다. 지금은 **클래스의 `CONSTANTS`**가 그 자리를 대신한다(Type Group은 구식). 옛 코드에서 `TYPE-POOLS abap.` 같은 줄을 보면 "전역 상수 묶음이구나" 정도로 읽으면 된다.

## 흔한 실수와 점검

- **VALUE 누락** → `CONSTANTS gc_x TYPE i.`는 값이 없어 의미가 없다. `VALUE`를 붙인다.
- **상수에 재대입** → 상수는 못 바꾼다. 바뀔 값이면 [[DATA]](변수)로.
- **여전히 매직 넘버** → "이 숫자가 뭐지?" 싶으면 상수로 이름 붙일 후보다.
