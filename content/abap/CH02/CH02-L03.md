## "길이를 정해야" 완성되는 타입

어떤 타입은 **길이를 함께 지정해야** 비로소 완전해진다. 이를
[[Incomplete 타입]]이라 한다. 대표는 `C`·`N`·`P`다.

## C · N · P

| 타입 | 뜻 | 지정할 것 |
|---|---|---|
| `C` | 고정 길이 문자(character) | 길이 |
| `N` | 숫자형 문자(numeric text) | 길이 |
| `P` | 압축 소수(packed) | 길이 + 소수 자릿수 |

```abap
DATA: gv_code TYPE c LENGTH 4,            " 4글자 코드
      gv_zip  TYPE n LENGTH 5,            " 우편번호(숫자 문자)
      gv_amt  TYPE p LENGTH 8 DECIMALS 2. " 금액, 소수 2자리

gv_code = 'ABCD'.
gv_amt  = 7 / 2.    " 3.50 (정확)
WRITE: / gv_code, / gv_amt.
```

- `C LENGTH n` — 딱 n글자. 모자라면 공백으로 채워진다.
- `N LENGTH n` — 숫자처럼 보이지만 **문자**다(앞의 0이 보존됨, 예: 우편번호).
- `P LENGTH n DECIMALS d` — 금액·수량 등 **정확한 소수** 계산용.

> **왜 N을 쓰나?** 우편번호 `01234`를 정수 `I`에 넣으면 앞의 0이 사라져 `1234`가
> 된다. "숫자처럼 보이지만 계산하지 않는 값"은 `N`(또는 `C`)으로 둔다.

> 💡 **`P`의 LENGTH는 "글자 수"가 아니라 칸(바이트) 수** — `LENGTH 1`이 곧 한 칸
> (= 1[[바이트]])이다. 문자는 한 칸에 1개 들어가지만, `P`는 한 칸을 둘로 쪼개 숫자
> **두 자리**를 담는다. 단 **마지막 칸의 한 자리는 숫자가 아니라 부호(+/−) 자리**다.
> 그래서 `LENGTH 8`이면 담을 수 있는 숫자는 최대 **15자리**(= 8×2−1, 소수 자릿수
> 포함)다. `C`·`N`의 LENGTH(글자 수)와 의미가 다르니 헷갈리지 말 것.

## 일부만 떼어 보기 — offset/length

문자형(`C`·`N`) 변수는 **일부 구간만** 읽거나 쓸 수 있다. `변수+시작(길이)` 형태이고,
시작 위치는 **0부터** 센다.

```abap
DATA gv_date TYPE c LENGTH 8 VALUE '20260623'.

WRITE gv_date+0(4).    " 2026  (앞 4글자 = 연)
WRITE gv_date+4(2).    " 06    (5번째부터 2글자 = 월)
gv_date+6(2) = '01'.   " 끝 2글자(일)만 '01'로 교체 → '20260601'
```

> 날짜·코드처럼 **자리마다 의미가 정해진** 문자열에서 특정 부분만 다룰 때 유용하다.

## 칸으로 그려 보기

말보다 그림이 빠르다. `LENGTH`가 만드는 **칸**을 직접 보자. C의 빈칸 채움과 잘림, N의 앞 0,
P의 바이트 칸(그리고 부호 반 칸)이 한눈에 보이고, offset이 **어느 칸을 집는지**도
눌러서 확인할 수 있다.

## 흔한 실수와 점검

- **길이 누락** → `DATA gv_x TYPE c.`는 길이 1로 선언된다. `LENGTH`를 명시.
- **금액에 F 사용** → 부동소수점은 오차가 난다. 돈은 반드시 `P … DECIMALS`.
- **N에 알파벳 대입** → `N`에는 숫자 문자만. 알파벳은 `C`.
- **offset 범위 초과** → `+시작(길이)`가 변수 끝을 넘으면 오류다
  (**시작+길이 ≤ 전체 길이**). 8글자 변수에 `+7(2)`는 한 글자 벗어난다.
