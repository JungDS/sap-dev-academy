## 같은 타입을 또 적기 싫다

금액을 다루는 변수가 여럿이면 `TYPE p LENGTH 8 DECIMALS 2`를 매번 반복하게
된다. 정의가 바뀌면(예: 소수 3자리로) 전부 고쳐야 한다. **한 번 정의해 이름을
붙이고 재사용**하고 싶다. 그게 [[TYPES]]다.

## TYPES로 내 타입 만들기

```abap
TYPES ty_amount TYPE p LENGTH 8 DECIMALS 2.

DATA: gv_price TYPE ty_amount,
      gv_total TYPE ty_amount.
```

이제 금액 타입은 `ty_amount` 한 곳에서 관리된다. 정의를 바꾸면 그걸 쓰는 모든
변수가 함께 바뀐다.

- `ty_` 접두어는 타입 관례.
- [[TYPES]]는 **타입을 정의**할 뿐 [[메모리]]를 잡지 않는다(변수는 [[DATA]]가 만든다).

## 여러 타입 한 번에

```abap
TYPES: ty_amount TYPE p LENGTH 8 DECIMALS 2,
       ty_code   TYPE c LENGTH 4.
```

## 도면과 실물 — 눈으로 확인

[[TYPES]]는 **도면**이고, 변수는 그 도면으로 찍어낸 **실물**이다. 아래에서 도면의
`DECIMALS`를 바꿔 보자. 실물 변수 3개가 **한꺼번에** 바뀐다("한 곳에서 관리"가
바로 이것). 도면에 값을 직접 넣으려 하면 어떻게 되는지도 눌러 보자.

::embed CH02-L04-S01 | TYPES 도면 → DATA 실물 — 한 곳 고치면 전부 바뀐다 | 560::

## 아직은 "로컬"이다

여기서 만든 [[TYPES]]는 **이 프로그램 안에서만** 쓰는 로컬 타입이다. 금액·코드
타입을 **다른 프로그램과도 공유**하려면 전역(DDIC)으로 올려야 하는데, 그 이야기는
[다음 챕터](CH03-L01.html)에서 본격적으로 다룬다.

## 흔한 실수와 점검

- **TYPES로 값 대입 시도** → 타입은 틀일 뿐. 값은 그 타입으로 만든 [[DATA]] 변수에.
- **타입 이름 충돌** → `ty_` 접두어로 변수(`gv_`)와 구분한다.
- **사용 후 선언** → 타입은 **쓰는 줄보다 먼저(위에)** 선언한다. `DATA gv_x TYPE
  ty_amount.`가 `TYPES ty_amount …`보다 위에 있으면 "모르는 타입" 오류다(위에서
  아래로 해석, [LIKE의 순서 규칙](CH02-L01.html)과 같은 이유).

## 정리

- [[TYPES]]는 **타입(틀)을 정의**할 뿐 값을 담지 않는다. 값은 그 타입으로 만든 [[DATA]] 변수에 담는다.
- 반복되는 타입 정의는 `ty_` 이름 하나로 모은다. **정의가 바뀌어도 고칠 곳은 한 줄**이다.
- 타입은 쓰는 줄보다 **먼저** 선언한다(위→아래 해석).
- 지금 만든 타입은 **이 프로그램 전용**(Local)이다. 전역 공유는 다음 챕터(DDIC)의 몫이다.
