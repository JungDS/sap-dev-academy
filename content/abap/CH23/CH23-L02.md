## RAP의 뿌리 — Root Entity

RAP 앱은 **root entity**(보통 `ZI_*`)에서 시작한다. CH22의 CDS View Entity에 RAP용
요소(키·관리 필드)를 갖춘다. 콘서트 예매(`ZBOOKING`)를 root로 만들자.

```abap
define root view entity ZI_Booking
  as select from zbooking
{
  key booking_id,
      concert_id,
      perf_no,
      customer,
      seats,
      status
}
```

- `define root view entity` — RAP의 루트.
- 키(`booking_id`)가 트랜잭션의 식별자가 된다.
- 필요하면 association으로 공연(`ZI_Concert`)과 연결(CH22).

## 흔한 실수와 점검

- **root 누락** → 트랜잭션 대상 뷰는 `root`로 선언.
- **키 설계** → 키가 곧 OData 엔티티 키. 신중히.
