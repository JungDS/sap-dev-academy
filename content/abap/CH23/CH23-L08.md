## 미래를 향한 규율 — ABAP Cloud

최신 SAP(S/4HANA Cloud)은 **ABAP Cloud** 개발 모델을 권장한다. 핵심은 **Released(공개
승인된) API만** 쓰는 것 — 그래야 업그레이드에도 안 깨지는 **Clean Core**가 된다.

## 무엇이 달라지나

- **Released API만** — `cl_*`·테이블 직접 접근 중 **공개 승인된 것만** 사용(R3 점검).
- **classic 일부 금지** — 화면(Dynpro)·일부 구문은 ABAP Cloud에서 제한된다.
- **3-tier 확장 모델** — 표준을 건드리지 않고 확장(Extension)으로만.

> 이 과정에서 배운 classic(Dynpro·일부 SQL)은 **온프레미스·유지보수**에 여전히 쓰이고,
> RAP·CDS·Released API는 **클라우드·신규**의 표준이다 — 둘 다 안다는 게 강점이다.

## Track-1을 마치며

환경(CH01)부터 RAP(CH23)까지, **읽고 → 모델링하고 → 화면을 만들고 → 객체로 설계하고
→ 트랜잭션 앱**을 짓는 한 바퀴를 돌았다. 콘서트 예매 앱이 그 여정의 산출물이다.

## 흔한 실수와 점검

- **비공개 API 사용** → 클라우드에서 깨진다. Released 여부를 확인.
- **표준 직접 수정** → Clean Core 위반. 확장(Enhancement·BAdI — CH29)으로.
