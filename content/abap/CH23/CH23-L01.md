## 읽기를 넘어 — 쓰기·검증·서비스까지

CH22의 CDS는 데이터를 **읽기/모델링**했다. 그런데 실무 앱은 **생성·수정·삭제·검증**과
**Fiori 화면 연결**까지 필요하다. 이 전부를 현대적으로 묶는 표준이 [[RAP]]
(RESTful Application Programming Model)다.

## RAP의 계층

```text
Fiori Elements (UI)
   │  OData
Service Binding / Service Definition   ← 노출
Behavior Definition / Implementation   ← 동작(CUD·검증·액션)
CDS Projection View (ZC_)              ← 소비 모델
CDS Interface View  (ZI_, root)        ← 기반 모델
DB 테이블 (ZBOOKING …)
```

## managed vs unmanaged

| 시나리오 | 누가 저장하나 |
|---|---|
| **managed** | RAP 런타임이 **자동**으로 DB 저장 처리 |
| **unmanaged** | 기존 로직을 **직접** 연결(레거시 통합) |

신규 개발은 보통 **managed**로 시작한다(코드가 적다).

## 흔한 실수와 점검

- **RAP=CDS로 오해** → CDS는 데이터 모델, RAP은 그 위 트랜잭션·서비스까지.
- **계층 건너뛰기** → 보통 ZI_(root)→ZC_→BDEF→Service 순으로 쌓는다.
