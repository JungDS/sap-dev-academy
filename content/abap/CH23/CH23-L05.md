## 동작의 알맹이

BDEF에서 선언한 동작 중 **표준(create/update/delete)** 은 managed라 RAP이 알아서
한다. 검증·액션 같은 **커스텀 동작**은 **Behavior Pool**(로컬 클래스, `ZBP_*`)에서
구현한다. CH20의 클래스·메서드 지식이 그대로 쓰인다.

```abap
CLASS lhc_booking DEFINITION INHERITING FROM cl_abap_behavior_handler.
  PRIVATE SECTION.
    METHODS validate_seats FOR VALIDATE ON SAVE
      IMPORTING keys FOR Booking~validateSeats.
ENDCLASS.

CLASS lhc_booking IMPLEMENTATION.
  METHOD validate_seats.
    " 키별로 예매 데이터를 읽어 검증(다음 레슨에서 내용 채움)
  ENDMETHOD.
ENDCLASS.
```

- `INHERITING FROM cl_abap_behavior_handler` — RAP 핸들러 클래스.
- `FOR VALIDATE ON SAVE` / `FOR MODIFY` 등으로 어느 동작을 구현하는지 표시.
- 메서드는 **키 집합(keys)** 을 받아 한 번에 처리한다(집합 지향).

## 흔한 실수와 점검

- **단건 사고** → RAP은 여러 키를 한 번에. `keys`를 LOOP로 처리.
- **핸들러 누락** → 커스텀 동작은 behavior pool에 구현해야 동작.
