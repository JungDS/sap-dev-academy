## RAP의 비즈니스 로직 셋

| 종류 | 언제 | 예 |
|---|---|---|
| **Validation** | 저장 전 **검증** | 정원 초과면 거부 |
| **Determination** | 값 **자동 결정** | 예매 상태 기본값·감사필드 |
| **Action** | 사용자가 **명시 실행** | 예약 / 취소 |

## BDEF에 선언

```abap
define behavior for ZI_Booking alias Booking
{
  validation validateSeats on save { field seats; }
  determination setStatus on modify { create; }
  action cancel result [1] $self;            // 취소 액션
}
```

## 정원 초과 Validation (콘서트 thread)

예매 좌석 수가 회차 정원을 넘으면 거부한다 — `ZCX_FULLY_BOOKED`(CH20)의 RAP 버전이다.

```abap
METHOD validateSeats.
  READ ENTITIES OF zi_booking IN LOCAL MODE
    ENTITY Booking FIELDS ( concert_id seats ) WITH CORRESPONDING #( keys )
    RESULT DATA(bookings).
  LOOP AT bookings INTO DATA(b).
    " 회차 정원 조회 후 초과면 APPEND TO failed/reported
  ENDLOOP.
ENDMETHOD.
```

## 흔한 실수와 점검

- **검증을 액션에** → 저장 검증은 `validation`, 사용자 실행은 `action`.
- **failed/reported 누락** → 검증 실패는 `failed`·`reported`로 알려야 RAP이 막는다.
