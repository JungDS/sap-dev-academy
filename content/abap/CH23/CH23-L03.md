## 소비용 뷰

기반(`ZI_Booking`)을 그대로 노출하지 않고, 화면/서비스에 맞춘 **Projection**(`ZC_`)을
둔다(CH22의 ZI_/ZC_ 분리 그대로).

```abap
define root view entity ZC_Booking
  provider contract transactional_query
  as projection on ZI_Booking
{
  key booking_id,
      concert_id,
      perf_no,
      customer,
      seats,
      status
}
```

- `provider contract transactional_query` — 트랜잭션 소비 계약(RAP 표준).
- UI 주석(@UI)·필드 추림을 여기서(또는 Metadata Extension).

## 흔한 실수와 점검

- **contract 누락** → 트랜잭션 소비엔 `transactional_query` 계약.
- **기반 직접 노출** → 소비는 ZC_로, 기반 ZI_는 안정적으로.
