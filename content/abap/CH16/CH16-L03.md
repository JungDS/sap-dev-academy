## 화면을 채우는 요소들

예매 화면을 떠올려 보자 — 공연ID 입력칸, 좌석 수 입력칸, "예매" 버튼. 화면에 올리는
요소는 종류가 정해져 있고, 각자 **같은 이름의 전역 변수**와 연결된다.

| 요소 | 뜻 | 연결 변수 |
|---|---|---|
| Input/Output Field | 값 입력·표시 칸 | 그 타입의 변수 |
| Text Field | 고정 라벨(설명) | — |
| Push Button | 누르면 명령 발생 | FctCode → OK_CODE |
| Checkbox | 켜고 끄기 | `CHAR1`(`'X'`/공백) |
| Radiobutton | 그룹 중 하나 | 그룹 변수 |
| Dropdown | 목록에서 선택 | 값 목록과 연결 |

## 예: 예매 입력 화면

```abap
DATA: p_concert TYPE zconcert-concert_id,   " 공연ID 입력칸
      p_seats   TYPE i,                       " 좌석 수
      ok_code   TYPE sy-ucomm.                " 버튼 명령
```

- 입력칸 `P_CONCERT`에 [[Data Element]]를 쓰면 CH09의 F4·검증이 화면에서도 작동한다.
- "예매" 버튼의 FctCode(예: `SAVE`)가 눌리면 `OK_CODE`로 전달된다(다음 레슨 PAI).

## 흔한 실수와 점검

- **버튼인데 OK_CODE 안 옴** → 버튼에 FctCode를 안 줬다. FctCode를 지정한다.
- **체크박스 값** → 체크=`'X'`, 해제=공백. `CHAR1`로 받는다.
