## 화면을 그리기 직전 — PBO

[[PBO]](Process Before Output)는 화면이 사용자에게 **보이기 직전**에 실행된다.
여기서 초기값을 넣고, 메뉴·툴바(다음 레슨 PF-STATUS)를 켜고, 필드 속성을 조정한다.

```abap
MODULE status_0100 OUTPUT.
  SET PF-STATUS 'ST100'.       " 메뉴·툴바(다음 레슨)
  SET TITLEBAR  'TB100'.       " 제목
ENDMODULE.
```

- `MODULE ... OUTPUT`이 PBO에서 불리는 모듈이다.

## 필드 속성 동적 제어 — LOOP AT SCREEN

상황에 따라 특정 칸을 **읽기 전용/숨김**으로 만들고 싶을 때 `LOOP AT SCREEN`으로
각 필드의 속성을 바꾼다.

```abap
LOOP AT SCREEN.
  IF screen-name = 'P_SEATS' AND p_locked = 'X'.
    screen-input = '0'.        " 입력 불가(읽기 전용)
    MODIFY SCREEN.
  ENDIF.
ENDLOOP.
```

- `screen-input = '0'` → 읽기 전용, `'1'` → 입력 가능. `screen-invisible = '1'` → 숨김.
- 바꾼 뒤 **`MODIFY SCREEN`** 을 해야 반영된다.

## 흔한 실수와 점검

- **MODIFY SCREEN 누락** → 속성을 바꿔도 화면에 반영 안 됨.
- **PBO에서 무거운 조회** → 화면 뜰 때마다 느려진다. 꼭 필요한 준비만.
