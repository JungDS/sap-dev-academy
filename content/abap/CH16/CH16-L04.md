## 화면을 그리기 직전 — PBO

[[PBO]](Process Before Output)는 화면이 사용자에게 **보이기 직전**에 실행된다.
사용자가 버튼을 누른 뒤 준비하면 이미 늦다. 제목·툴바를 붙이고, 드롭다운 목록을 채우고,
특정 조건에서 입력칸을 읽기 전용으로 바꾸는 일은 **화면이 뜨기 전에** 끝나야 한다.

`PROCESS BEFORE OUTPUT` 블록에서 부른 `MODULE ... OUTPUT` 모듈들이 PBO에서 실행된다.

```abap
MODULE status_0100 OUTPUT.        " OUTPUT = PBO에서 실행
  SET PF-STATUS 'ST100'.          " 메뉴·툴바
  SET TITLEBAR  'TB100'.          " 제목
ENDMODULE.
```

> 🔎 **[선행 사용]** `SET PF-STATUS`·`SET TITLEBAR`는 화면 위쪽 **메뉴·툴바·제목**을
> 켜는 문장이다. 여기선 "PBO가 이것도 준비한다"만 알면 된다 — 자세한 문법은
> [PF-STATUS와 TITLEBAR](CH16-L06.html)에서.

## 필드 속성 동적 제어 — LOOP AT SCREEN

상황에 따라 특정 칸을 **읽기 전용/숨김**으로 바꾸려면, 현재 화면의 요소 속성을
`LOOP AT SCREEN`으로 하나씩 돌며 고친다.

```abap
DATA gs_screen TYPE screen.

MODULE modify_screen_0100 OUTPUT.
  LOOP AT SCREEN INTO gs_screen.
    IF gs_screen-name = 'P_SEATS' AND p_locked = 'X'.
      gs_screen-input = '0'.              " 0 = 읽기 전용 (1 = 입력 가능)
      MODIFY SCREEN FROM gs_screen.       " 반드시 반영
    ENDIF.
  ENDLOOP.
ENDMODULE.
```

- `gs_screen-input = '0'` → 읽기 전용, `'1'` → 입력 가능. 바꾼 뒤 **`MODIFY SCREEN FROM
  gs_screen`** 을 해야 화면에 반영된다.
- ⚠️ **읽기 전용만 원하면 `input = 0`** 을 쓴다. `active = 0`은 입력·출력·표시에 **모두**
  영향을 줘 칸이 아예 사라질 수 있다.
- ⚠️ 화면 속성은 **PBO가 돌 때마다 기본값으로 다시 계산**된다. 조건을 계속 유지하려면
  PBO에서 **매번 다시** 설정해야 한다.

> 📌 짧은 `LOOP AT SCREEN.`/`MODIFY SCREEN.`(내장 `screen`) 형식은 공식문서가 피하는
> 표기다 — 명시 work area(`INTO gs_screen` … `FROM gs_screen`)를 쓴다.

## 직접 준비해 보기 — PBO 화면 준비 스텝퍼

PBO를 한 단계씩 진행하며 `sy-pfkey`·`sy-title`이 설정되고, `LOOP AT SCREEN`에서
`P_SEATS`를 잡아 읽기 전용으로 바꾸는 과정을 보자. 특히 **`MODIFY SCREEN`을 누르기 전엔
미리보기가 바뀌지 않는** 것을 확인하자.

## 흔한 실수와 점검

- **`MODIFY SCREEN` 누락** → 속성을 바꿔도 화면에 반영되지 않는다.
- **PBO에서 무거운 DB 조회 반복** → 화면이 뜰 때마다 느려진다. 꼭 필요한 준비만.
- **`active = 0`으로 읽기 전용 시도** → 칸이 사라진다. 읽기 전용은 `input = 0`.
- **한 번 설정했으니 유지될 것이라 기대** → 속성은 PBO마다 reset된다. 매 PBO에서 다시 설정.

## 정리

- **PBO**는 사용자가 보기 전에 화면을 **준비**하는 단계다: 상태·제목·드롭다운·필드 속성.
- 바꾼 화면 속성은 `MODIFY SCREEN FROM gs_screen`으로 반영하고, 속성은 **PBO마다 다시 계산**된다.
