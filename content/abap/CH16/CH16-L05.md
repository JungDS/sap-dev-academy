## 버튼을 누른 뒤 — PAI

[[PAI]](Process After Input)는 사용자가 **버튼·Enter**를 누른 뒤 실행된다. 어떤 버튼이
눌렸는지는 명령 필드 **`OK_CODE`**(화면의 OK field와 같은 이름의 ABAP 변수)로 안다.
`예매`·`뒤로`·`종료`·`취소`는 모두 다른 의미이므로, 이 판단을 제대로 해야 한다.

## 안전한 분기 — save_ok 패턴

공식 권장 패턴은 PAI 처음에 `OK_CODE`를 **보조 변수 `save_ok`로 복사한 뒤 바로
`CLEAR`** 하고, `save_ok`로 분기하는 것이다.

```abap
DATA: ok_code TYPE sy-ucomm,
      save_ok TYPE sy-ucomm.

MODULE user_command_0100 INPUT.
  save_ok = ok_code.            " 보조 변수로 복사
  CLEAR ok_code.               " OK field 즉시 비움

  CASE save_ok.
    WHEN 'SAVE'.
      PERFORM check_booking.   " 예매 검증·저장
    WHEN 'BACK'.
      LEAVE TO SCREEN 0.       " 화면 종료
    WHEN 'EXIT'.
      LEAVE PROGRAM.           " 프로그램 종료
    WHEN 'CANCEL'.
      LEAVE TO SCREEN 0.
  ENDCASE.
ENDMODULE.
```

> 💡 **왜 바로 `CLEAR`?** OK field를 비우지 않으면, 그 다음 Enter를 눌렀을 때 이전
> `'SAVE'`가 **그대로 남아 다시 실행**될 수 있다. 읽자마자 비우는 게 핵심이다.
> (`PERFORM`은 [FORM / PERFORM 기본 호출](CH10-L01.html)에서 본 모듈 호출이다.)

## 화면을 떠나는 문장들

| 문장 | 의미 |
|---|---|
| `LEAVE TO SCREEN 0` | next dynpro를 **0**으로 → 현재 화면 시퀀스 종료(호출자로 복귀) |
| `LEAVE PROGRAM` | 프로그램(트랜잭션) **전체** 종료 — 강한 종료 |
| `SET SCREEN <번호>` | **다음 화면 번호만 지정** — 즉시 안 나가고, 처리 끝에 그 화면 호출 |
| `LEAVE SCREEN` | 현재 화면 처리를 **즉시** 끝내고 다음 화면 호출 |

- ⚠️ `SET SCREEN`은 "다음에 갈 화면"을 정할 뿐 **즉시 떠나지 않는다.** 지금 바로
  떠나려면 `LEAVE SCREEN`(또는 `LEAVE TO SCREEN`)과 함께 쓴다.

## BACK · EXIT · CANCEL — 관례

화면 위 표준 버튼 셋은 역할이 정해져 있다.

- **BACK**(초록 화살표) — 한 단계 뒤로(`LEAVE TO SCREEN 0`).
- **EXIT** — 트랜잭션 종료(`LEAVE PROGRAM`). **한 화면 뒤로 가는 버튼에 함부로 쓰면**
  사용자가 트랜잭션 전체를 잃는다.
- **CANCEL** — 입력 취소하고 뒤로(`LEAVE TO SCREEN 0`).

> 🔶 화면에 **목록**을 보여줄 땐 Table Control 대신 [Grid ALV 기초](CH17-L01.html)를
> 쓴다(이 과정은 Table Control을 다루지 않는다).

## 직접 분기시켜 보기 — OK_CODE 안전 분기 실험실

버튼을 눌러 `ok_code` → `save_ok` 복사 → `CLEAR` → `CASE` 분기를 따라가 보자. 특히
**`CLEAR`를 생략한 채 SAVE 뒤 Enter**를 누르면 이전 `SAVE`가 어떻게 재실행되는지 확인하자.

## 흔한 실수와 점검

- **`OK_CODE`를 안 비움** → 이전 명령이 남아 Enter에 재실행된다. 읽은 뒤 바로 `CLEAR`.
- **읽기 전에 `CLEAR`** → 값을 잃는다. **먼저 `save_ok`로 복사**한 뒤 비운다.
- **EXIT에 `LEAVE TO SCREEN 0`** → EXIT는 트랜잭션 종료(`LEAVE PROGRAM`)다. 화면 종료와 구분.
- **`SET SCREEN`만 쓰고 안 떠남** → 즉시 이동하려면 `LEAVE SCREEN`을 함께.

## 정리

- **PAI**는 사용자의 행동을 해석하는 단계다. `OK_CODE`를 **`save_ok`로 옮기고 바로
  `CLEAR`** 한 뒤 `CASE`로 분기하는 게 기본 패턴이다.
- **화면 종료(`LEAVE TO SCREEN 0`)와 프로그램 종료(`LEAVE PROGRAM`)를 구분**해야 안전한 흐름이 된다.
