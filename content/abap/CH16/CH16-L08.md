## 화면 위 메뉴·툴바 — GUI Status

화면에는 입력칸만 있는 게 아니다. 위쪽에는 **메뉴바·표준 툴바·application toolbar·
function key**가 있고, 사용자는 그 버튼으로 저장하거나 뒤로 가거나 도움말을 연다.
이 상단 버튼들의 function code를 묶은 객체가 [[GUI Status]](PF-STATUS)다.
`SE41`(Menu Painter)에서 만들고, 어떤 버튼을 둘지·각 버튼의 FctCode를 정한다.

## PBO에서 켜기 — SET PF-STATUS · SET TITLEBAR

GUI Status와 제목은 [PBO 처리 흐름](CH16-L06.html)의 `MODULE ... OUTPUT`에서 켠다.

```abap
MODULE status_0100 OUTPUT.
  SET PF-STATUS 'ST100'.     " 이 화면에 쓸 GUI Status
  SET TITLEBAR  'TB100'.     " 제목 표시줄
ENDMODULE.
```

- `SET PF-STATUS`로 상태를 켜면, 그 버튼이 눌릴 때 FctCode가 `OK_CODE`로 온다
  ([PAI 처리 흐름 — OK_CODE와 화면 떠나기](CH16-L07.html)).
- 설정 후 `sy-pfkey`에 현재 status 이름이, `sy-title`에 제목이 들어온다.

## 버튼을 동적으로 숨기기 — EXCLUDING

특정 상황에서 일부 버튼만 비활성화하려면 `EXCLUDING`에 **제외할 function code 목록**을 넘긴다.
(공식문서: `EXCLUDING`된 기능은 화면에서 선택할 수 없게 된다.)

```abap
DATA gt_excl  TYPE STANDARD TABLE OF sy-ucomm.
DATA gv_fcode TYPE sy-ucomm.

MODULE status_0100 OUTPUT.
  CLEAR gt_excl.
  IF gv_locked = 'X'.
    gv_fcode = 'SAVE'.
    APPEND gv_fcode TO gt_excl.          " 잠금 상태면 SAVE 비활성
  ENDIF.

  SET PF-STATUS 'ST100' EXCLUDING gt_excl.
  SET TITLEBAR  'TB100' WITH gv_conc.     " 아래 참고
ENDMODULE.
```

## 제목에 값 넣기 — TITLEBAR ... WITH

제목 텍스트에 `&1`~`&9` 자리표시자를 두고 `WITH`로 값을 채운다. 제목 `TB100`이
**`공연 &1 예매`** 라면, `SET TITLEBAR 'TB100' WITH gv_conc`는 `공연 C001 예매`가 된다.

> 제목은 장식이 아니다 — 사용자가 **지금 어느 공연·어느 업무 화면**에 있는지 알려 주는 안전 장치다.

## 직접 매핑해 보기 — Toolbar와 Function Code 매핑판

툴바 버튼을 눌러 그 **function code가 `OK_CODE`로 전달**되고 PAI `CASE`에서 처리되는지 보자.
잠금으로 `SAVE`를 `EXCLUDING`하면 어떻게 빠지는지, 그리고 **CASE에 없는 버튼(`HELP`)** 을
누르면 어떤 경고가 뜨는지 확인하자.

## 흔한 실수와 점검

- **PF-STATUS 미설정** → 표준 버튼만 떠 내 기능키가 없다. PBO에서 늦지 않게 `SET PF-STATUS`.
- **FctCode 불일치** → Menu Painter의 버튼 코드와 `CASE save_ok` 값이 **같아야** 한다.
  다르면 버튼은 눌렸지만 처리할 branch가 없다.
- **`EXCLUDING` 오타** → 없는 function code를 넣으면 조용히 무시된다. 코드를 점검한다.
- **제목 placeholder 누락** → `&1`이 있는데 `WITH`를 안 주면 자리표시자가 그대로 보인다.

## 정리

- **PF-STATUS**는 화면 위 버튼과 function code의 설계도, **TITLEBAR**는 현재 화면의 업무 맥락이다.
- 둘 다 **PBO에서** 설정하고(`EXCLUDING`으로 동적 비활성화, `WITH`로 제목 값 채움),
  PAI의 `OK_CODE` 처리와 **같은 function code 체계**로 맞춘다.
