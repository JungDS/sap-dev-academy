## 🎫 8단계 — 예매 입력 화면

CH16에서 배운 Module Pool로 **예매 입력 화면**(0100)을 만든다. 공연ID(F4)·회차·좌석수·
예매자를 입력받고, PAI에서 **예매 가능 여부**(CH10-L07의 `can_book`)를 검증한다.

```abap
" 화면 0100 필드: P_CONC(F4)·P_PERF·P_SEATS·P_CUST·OK_CODE
MODULE user_command_0100 INPUT.
  DATA: lv_code TYPE sy-ucomm,
        lv_ok   TYPE abap_bool.
  lv_code = ok_code.  CLEAR ok_code.

  CASE lv_code.
    WHEN 'SAVE'.
      PERFORM can_book USING p_conc p_perf p_seats CHANGING lv_ok.
      IF lv_ok = abap_true.
        MESSAGE '예매 가능 — 저장 단계는 CH24(DML)에서' TYPE 'S'.
      ELSE.
        MESSAGE '좌석이 부족합니다' TYPE 'E'.
      ENDIF.
    WHEN 'BACK' OR 'EXIT' OR 'CANCEL'.
      LEAVE TO SCREEN 0.
  ENDCASE.
ENDMODULE.
```

> 실제 DB 저장(INSERT)·잠금은 **CH24·CH25**에서 붙인다. 여기선 **화면·검증**까지가 목표다.

## 직접 해 볼 과제

1. 화면 0100을 그리고(공연ID에 F4, CH09) T-code로 실행(CH16-L01).
2. 좌석을 과다 입력해 `MESSAGE E`로 막히는지 확인.
3. BACK으로 화면이 닫히는지(`LEAVE TO SCREEN 0`) 확인.

## 흔한 실수와 점검

- **OK_CODE 미초기화** → 읽은 뒤 `CLEAR`(CH16-L05).
- **저장까지 기대** → DML은 CH24. 지금은 화면+검증.
