## 화면에 번호를 붙인다

Dynpro는 각 화면을 **네 자리 번호**(예: `0100`)로 구분한다. 한 Module Pool 안에
여러 화면(0100·0200…)을 두고 번호로 오간다.

## Screen Painter로 그리기

`SE80`(또는 `SE51`) Screen Painter에서 입력칸·버튼 같은 요소를 **끌어다 놓는다.**
각 요소에는 **이름**을 주는데, 그 이름의 전역 변수와 자동으로 연결된다(예: 입력칸
`P_SEATS` ↔ 프로그램의 `DATA p_seats`).

## Flow Logic — PBO / PAI 선언

화면마다 **Flow Logic**에 두 블록이 있고, 각 블록에서 처리 모듈(`MODULE`)을 부른다.

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0100.
PROCESS AFTER INPUT.
  MODULE user_command_0100.
```

- Flow Logic은 ABAP과 조금 다른 **화면 전용 문법**이다(여기서 `MODULE`만 부른다).
- 실제 처리 코드는 프로그램 쪽 `MODULE ... ENDMODULE`에 쓴다.

## 흔한 실수와 점검

- **요소 이름 ≠ 변수 이름** → 이름이 달라 값이 연결 안 됨. 같은 이름으로 맞춘다.
- **Flow Logic에 일반 ABAP** → Flow Logic엔 `MODULE` 등 화면 문법만. 로직은 모듈 안에.
