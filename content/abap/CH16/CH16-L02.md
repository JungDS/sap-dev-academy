## 화면에 번호를 붙인다

Module Pool에는 화면이 하나만 있는 게 아니다. 예매 입력 화면 `0100`, 상세 확인
화면 `0200`, 팝업 `0300`처럼 여러 [[Dynpro]]가 있을 수 있고, 화면마다 layout과
흐름 로직이 **따로** 있다. 그래서 [[Dynpro]]는 **네 자리 번호**로 구분한다 —
번호를 모르면 "지금 내가 어느 화면의 PBO/PAI를 고치는가"를 잃어버린다.

## Screen Painter의 네 조각

`SE80`(또는 `SE51`) [[Screen Painter]]는 "ABAP 코드만 쓰면 화면이 생긴다"가 아니라,
**네 조각이 함께 맞아야** 움직이는 구조다.

| 조각 | 무엇을 담나 | 확인 위치 |
|---|---|---|
| **Layout** | 입력칸·텍스트·버튼의 위치·크기 | Screen Painter Layout |
| **Element List** | 화면 요소 이름·타입·길이·OK field | Element List |
| **Flow Logic** | PBO/PAI 블록 + 호출할 `MODULE` | Flow Logic |
| **Attributes** | next screen·modal 여부·화면 타입 | Screen Attributes |

- 화면 요소에 준 **이름**은 같은 이름의 전역 변수와 자동 연결된다(예: 입력칸
  `GV_SEATS` ↔ 프로그램의 `DATA gv_seats`). 이름이 다르면 값이 운반되지 않는다.
- 사용자 버튼의 function code를 받는 **OK field**는 Element List에서 지정한다(자세히는
  [PAI 처리 흐름 — OK_CODE와 화면 떠나기](CH16-L07.html)).

## Flow Logic — 화면이 부르는 MODULE

화면마다 [[Flow Logic]]에 PBO/PAI 두 블록이 있고, 각 블록에서 처리 모듈(`MODULE`)을 부른다.

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0100.
PROCESS AFTER INPUT.
  MODULE user_command_0100.
```

[[Flow Logic]]은 ABAP과 조금 다른 **화면 전용 문법**이다. 여기서는 `MODULE`을 **부르기만** 한다.

## ABAP 쪽 MODULE 정의 — OUTPUT(PBO) / INPUT(PAI)

실제 처리 코드는 프로그램 쪽 `MODULE ... ENDMODULE`에 쓴다. `OUTPUT`/`INPUT` 추가어가
이 모듈이 **PBO에서** 불릴지 **PAI에서** 불릴지를 정한다.

```abap
MODULE status_0100 OUTPUT.          " OUTPUT = PBO에서 호출
  " 초기값·화면 상태 준비
ENDMODULE.

MODULE user_command_0100 INPUT.     " INPUT = PAI에서 호출
  " 입력·버튼 처리
ENDMODULE.
```

> 🔑 **핵심 구분.** Flow Logic의 `MODULE status_0100.`(호출)과 ABAP의
> `MODULE status_0100 OUTPUT.`(정의)은 **다른 위치**에 있고, **이름이 같아야** 연결된다.
> 이름이 어긋나면 화면 활성화가 실패한다.

## 흔한 실수와 점검

- **Flow Logic 호출 ≠ ABAP 정의** → `MODULE status_0100.`은 있는데
  `MODULE status_0100 OUTPUT.` 정의가 없으면 활성화 오류("호출 대상 없음").
- **요소 이름 ≠ 변수 이름** → 자동 data transport가 안 된다. 같은 이름으로 맞춘다.
- **Flow Logic에 일반 ABAP 로직** → Flow Logic엔 `MODULE` 호출 같은 화면 문법만. 로직은 모듈 안에.
- **OK field 미지정** → 사용자 버튼의 function code를 안정적으로 못 읽는다.
- **next screen `0`을 빈 값으로 오해** → `0`은 화면 sequence **종료**를 뜻한다.

## 정리

- **Screen Number**는 "어느 화면인가"를 가리키는 주소, **Screen Painter**는 그 화면의
  layout·flow logic을 만드는 도구다.
- Dynpro는 **Layout · Element List · Flow Logic · ABAP module** 네 조각이 모두 맞아야 움직인다.
