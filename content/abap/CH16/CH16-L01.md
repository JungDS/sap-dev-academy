## 정형 화면을 넘어서

CH15까지의 선택화면(PARAMETERS·SELECT-OPTIONS)은 SAP이 정해 준 틀이다. 그런데
"공연을 고르고, 좌석 수를 넣고, 예매 버튼을 누르는" **내가 설계한 화면**이 필요하다.
그게 [[Dynpro]](Dynamic Program, 화면)이고, 화면 중심 프로그램이 [[Module Pool]]이다.

## 리포트 vs Module Pool

| | Report(실행형) | Module Pool |
|---|---|---|
| 시작 | `REPORT`·F8 | **T-code**로만 |
| 중심 | 데이터 처리 | **화면(Dynpro)** |
| 흐름 | 이벤트(START-OF-SELECTION…) | **PBO/PAI** |

- Module Pool은 **실행할 T-code가 반드시 있어야** 한다(CH01-L07의 T-code, 이번엔 필수).
- `SE80`(또는 `SE51` Screen Painter)에서 화면을 그린다.

## 화면의 두 박자 — PBO / PAI

화면 하나가 뜨고 처리되는 흐름은 **두 박자**다.

- **PBO**(Process Before Output) — 화면을 **그리기 직전**(초기값·상태 준비).
- **PAI**(Process After Input) — 사용자가 **버튼·Enter를 누른 뒤**(입력 처리).

::embed CH16-L01-S01 | PBO → 화면 표시 → 사용자 입력 → PAI 의 두 박자 흐름::

## 흔한 실수와 점검

- **T-code 없이 실행** → Module Pool은 T-code로만 실행된다(SE93로 생성).
- **리포트 이벤트 기대** → Module Pool은 START-OF-SELECTION이 아니라 PBO/PAI 흐름.
