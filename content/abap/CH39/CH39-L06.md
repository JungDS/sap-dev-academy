## 노출 — Service Definition / Binding

```service
define service ZUI_Booking {
  expose ZC_Booking as Booking;
  expose ZI_Concert as Concert;
  expose ZI_Perf    as Perf;
}
```

- Service Binding을 **OData V4 / UI**로 만들고 **Activate**.

## Fiori Elements로 확인 — 코드 없는 UI

@UI 주석(L03)만으로 Fiori Elements가 화면을 **자동 생성**한다.

- Service Binding의 **Preview**로 List Report(목록)·Object Page(상세)를 바로 띄운다.
- 목록에서 공연·예매를 보고, 상세에서 좌석 수정·**취소 액션** 버튼을 누른다.
- 정원 초과 시 검증 메시지가 화면에 뜬다(L05).

> 🎉 ABAP 코드로 화면을 한 줄도 안 짜고, **모델(@UI)** 만으로 풀 화면이 나온다 —
> CH16의 Dynpro와 비교하면 격세지감이다.

## 흔한 실수와 점검

- **binding 미활성화** → Preview·호출 불가. Activate.
- **@UI 부족** → 목록/상세가 비어 보인다. lineItem·identification 보강.
