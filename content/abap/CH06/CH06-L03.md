## 쌓았으면 다뤄야 한다

내부 테이블의 행을 **하나씩 순회**하고, **특정 행을 찾고**, **고치고 지운다**.
가장 자주 쓰는 동작들이다.

## LOOP — 전체 순회

```abap
LOOP AT lt_person INTO ls_person.
  WRITE: / ls_person-name, ls_person-age.
ENDLOOP.
```

- 한 행씩 [[Work Area]] `ls_person`으로 꺼내 처리한다.
- 현재 행 번호는 `sy-tabix`에 들어온다.

## READ TABLE — 단건 조회

```abap
READ TABLE lt_person INTO ls_person WITH KEY name = 'Kim'.
IF sy-subrc = 0.
  WRITE: / '찾음:', ls_person-age.
ENDIF.
```

- 찾으면 `sy-subrc = 0`, 못 찾으면 `4`. **항상 `sy-subrc`를 확인**한다.

## MODIFY / DELETE — 수정·삭제

```abap
" 순회 중 현재 행 수정
LOOP AT lt_person INTO ls_person.
  ls_person-age = ls_person-age + 1.
  MODIFY lt_person FROM ls_person INDEX sy-tabix.
ENDLOOP.

" 조건으로 삭제
DELETE lt_person WHERE age < 20.
```

## 정렬

```abap
SORT lt_person BY age DESCENDING.
```

## 흔한 실수와 점검

- **sy-subrc 미확인** → READ 실패 후 옛 [[Work Area]] 값을 그대로 쓰는 버그. 꼭 확인.
- **LOOP 안에서 통째 DELETE** → 인덱스가 꼬인다. 조건 삭제는 `DELETE ... WHERE`로.
- **잦은 단건 READ가 느림** → 키 설계(SORTED/HASHED)를 뒤에서 고려.
