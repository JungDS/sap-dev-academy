## 끝나면 사라진다

CH05의 [[Internal Table]]은 강력하지만 **메모리**에만 산다. 프로그램이 끝나면
사라지고, 다른 사용자와 공유도 안 된다. 값을 **디스크에 영구 저장**하고 싶다 —
그게 [[Transparent Table]](투명 테이블)다.

## 투명 테이블이란

[[Transparent Table]]은 DDIC에 정의하는 테이블로, **DB의 실제 물리 테이블과
1:1로 대응**한다. "투명(transparent)"은 DDIC 정의와 DB 테이블이 같은 모양으로
들여다보인다는 뜻.

## SE11에서 만들기

`SE11` → Database Table → `ZTPERSON` → Create. 핵심 입력:

| 항목 | 내용 |
|---|---|
| Delivery Class | 보통 `A`(application data) |
| 필드 | 각 컬럼. 타입은 [[Data Element]]로 |
| Key | 행을 유일하게 식별하는 키 필드 체크 |
| [[MANDT]] | 보통 첫 키 필드 = 클라이언트 |

```text
MANDT  (Key)  → 클라이언트
PERSID (Key)  → ZDE_PERSID
NAME          → ZDE_NAME
AGE           → ZDE_AGE
```

**Technical Settings**(데이터 클래스·크기)까지 채우고 활성화하면, 그 순간 실제
DB 테이블이 생성된다.

## MANDT — 클라이언트 분리

대부분 업무 테이블의 첫 키는 [[MANDT]](클라이언트)다. CH01에서 본 "방(Client)"
별로 데이터가 분리되는 장치다. 보통 직접 다루지 않아도 시스템이 챙긴다.

## 흔한 실수와 점검

- **키 미지정** → 최소 하나의 키 필드가 필요. 보통 MANDT + 업무 키.
- **활성화 안 함** → 활성화해야 DB 테이블이 실제로 생긴다.
- **필드를 표준 타입 직접** → 가능하나 [[Data Element]]로 줘야 라벨·도움말이 산다.

---

그릇(테이블)은 만들었다. 이제 **값을 넣어** 보자 → 데이터 입력(SE16N).
