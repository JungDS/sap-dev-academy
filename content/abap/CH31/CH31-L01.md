## 시스템 간 표준 메시지

[[IDoc]](Intermediate Document)은 SAP 표준의 **메시지 컨테이너**다. 주문·송장 같은
업무 문서를 시스템 간에 **비동기**로 주고받는다(EDI·B2B).

## 3층 구조

| 부분 | 내용 |
|---|---|
| **Control Record**(EDIDC) | 발신/수신·메시지 타입·방향 |
| **Data Records**(EDIDD) | 실제 데이터 — **Segment**들의 묶음 |
| **Status Records**(EDIDS) | 처리 상태 이력 |

- **Basic Type**(예: `ORDERS05`) — IDoc의 구조 정의(어떤 Segment로 구성).
- `WE02`/`WE05` — IDoc 모니터, `WE60` — 구조 문서.

```abap
" Inbound 처리 함수(개념): 표준이 부르며 IDoc 데이터를 넘긴다
" idoc_control(EDIDC), idoc_data(EDIDD) 를 받아 파싱·등록
```

## 흔한 실수와 점검

- **Segment 구조 오해** → `WE60`으로 Basic Type 구조를 먼저 확인.
- **방향 혼동** → Inbound(수신)·Outbound(발신)을 분명히.
