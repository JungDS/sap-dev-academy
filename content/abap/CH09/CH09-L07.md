## 도움말이 여러 개면 뭐가 뜨나

[앞 레슨(F4 부착 지점)](CH09-L06.html)에서 F4가 오는 **세 경로**(고정값·Check Table·Search Help)를 봤다. 그런데
한 입력칸에 이런 장치가 **여러 개** 걸려 있으면? SAP은 정해진 **우선순위**로
**딱 하나**를 골라 띄운다 — "위가 있으면 아래는 안 본다."

## 위에서부터 차례로 — 우선순위 사다리

::embed CH09-L07-S03 | F4 우선순위 사다리 다이어그램 — 위에서부터 검사, 있으면 채택::

1. **코드로 직접 만든 F4** — 프로그램이 화면 이벤트에서 목록을 **직접** 만들어 띄운다. 있으면 이게 최우선이다.
   - 이 "직접 만들기"는 선택화면(`AT SELECTION-SCREEN ON VALUE-REQUEST` → [Report Event·선택화면 심화](CH15-L08.html))과 일반 화면(`PROCESS ON VALUE-REQUEST` → [화면에서 F1·F4 직접 만들기](CH16-L05.html))에서 배운다. **지금은 "맨 위에 이런 게 있다"만 알면 된다**(외울 필요 없음).
2. **화면 칸에 직접 붙인 [[Search Help]]** — 그 입력칸에 `MATCHCODE OBJECT`로 콕 지정한 검색 화면([Elementary](CH09-L04.html)·[Collective](CH09-L05.html)). 선언적 도움말 중 가장 유연하다.
3. **[[Check Table]] · [[Text Table]]** — Foreign Key의 마스터 목록(코드), Text Table이 있으면 코드 옆 이름까지 자동으로 나온다.
4. **Domain [[고정값]](Fixed Values)** — Domain에 정해 둔 몇 개 값. Check Table보다 **아래**다.
5. **타입 기본 도움** — 아무것도 안 걸어도, 날짜 타입이면 **달력**, 시간이면 **시계**가 뜬다.

> 💡 같은 Search Help라도 **화면 칸에 직접** 붙이면 위(2번)로 오지만, **Data Element에만**
> 붙인 경우엔 Check Table보다 아래에 온다. 헷갈리면 `MATCHCODE OBJECT`로 콕 집는 게 안전하다.

::embed CH09-L07-S01 | F4 입력 도움 우선순위 사다리 — 위가 있으면 아래는 안 본다::

## 그래서 — 하나로 명확히

여러 개를 겹쳐 걸면 **의도와 다른 게** 뜰 수 있다(예: Search Help를 원했는데 고정값
목록이 떠 버림). 보통은 우선순위를 이해하고 **하나로 분명히** 한다.

```abap
PARAMETERS pa_id TYPE zde_persid
  MATCHCODE OBJECT zsh_person_coll.   " 이 칸엔 이 Search Help로 못 박기
```

- `MATCHCODE OBJECT`([F4 부착 지점 레슨](CH09-L06.html))로 쓸 Search Help를 **명시**하면 혼선이 없다.

## 직접 해 보기

도움말 후보를 켜고 끄며 F4를 눌러, 여러 개가 걸려 있어도 **가장 높은 하나만** 뜨는 걸 확인해 보자. 상위를 끄면 그제서야 아래가 보인다.

::embed CH09-L07-S02 | F4 우선순위 토글 실험 — 켜고 끄며 무엇이 뜨나::

## 흔한 실수와 점검

- **여러 도움말이 충돌** → 우선순위를 떠올리고, 원하는 하나로 명확히(MATCHCODE 권장).
- **"왜 이 목록이 뜨지?"** → 위 순서를 위에서부터 짚으면 원인이 보인다(상위에 뭔가 걸려 있다).
