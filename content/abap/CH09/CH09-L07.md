## 🎫 콘서트 예매 앱 — 첫 삽

CH09-L01에서 본 콘서트 모델을 **이번 챕터부터 직접 만들어** CH23(RAP)까지 한 단계씩
키운다. 첫 단계는 데이터 토대 — 세 테이블과 관계·도움말이다.

## 1) 세 테이블 (SE11, CH07)

| 테이블 | 키 | 주요 필드 |
|---|---|---|
| `ZCONCERT` | MANDT·CONCERT_ID | ARTIST·VENUE·CAPACITY |
| `ZPERF` | MANDT·CONCERT_ID·PERF_NO | PERF_DATE |
| `ZBOOKING` | MANDT·BOOKING_ID | CONCERT_ID·PERF_NO·CUSTOMER·SEATS·STATUS |

- 필드 타입은 **Data Element**로(라벨·F4 상속, CH03). 정원 CAPACITY·좌석수 SEATS는 정수.
- 활성화하면 실제 DB 테이블이 생긴다(CH07-L01).

## 2) Foreign Key (CH09-L01)

`ZBOOKING-CONCERT_ID` 필드 → Foreign Keys → Check Table `ZCONCERT`. 이제 없는 공연
번호는 입력이 거부되고, 그 칸에서 F4로 공연 목록이 뜬다.

## 3) 공연 Search Help (CH09-L03)

`ZSH_CONCERT`(Elementary) — Selection Method `ZCONCERT`, 공연ID·아티스트·장소를 목록에
보여 주고 CONCERT_ID를 돌려준다. `ZBOOKING-CONCERT_ID`에 붙인다.

## 직접 해 볼 과제

1. 세 테이블을 만들고 **Create Entries**(CH07)로 공연 2개·회차 3개·예매 3건을 넣는다(예매자=정훈영).
2. `ZBOOKING`에 **없는 공연번호**를 넣어 보고 거부되는지 확인.
3. 그 칸에서 **F4**를 눌러 공연 목록이 뜨는지 확인.

> 여기서 만든 테이블·데이터를 다음 챕터부터 계속 쓴다 — 잘 만들어 두자.

## 흔한 실수와 점검

- **활성화 누락** → 테이블·Search Help 모두 활성화해야 동작.
- **타입을 Built-In으로** → Data Element로 줘야 라벨·F4가 산다(CH07-L01).
