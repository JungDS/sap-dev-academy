## 수고가 화면으로 연결되는 지점

지금까지 만든 [[Foreign Key]]·[[고정값]]·[[Search Help]]가 실제 입력 화면
([[PARAMETERS]])에서 **자동으로** F4·검증으로 나타나는 원리를 정리한다.
CH03-L03의 "보상"을 더 깊이 보는 셈.

## 자동 연결 — Data Element 경유

`PARAMETERS p_id TYPE zde_persid.` 처럼 [[Data Element]]로 타입을 주면:

- Data Element/Domain에 붙은 **[[Search Help]]**, **[[고정값]]**, 필드의
  **[[Foreign Key]]**가 그 PARAMETERS의 F4·검증으로 **그대로 따라온다.**
- 별도 코드 없이 화면이 똑똑해진다.

## 명시적으로 붙이기 — MATCHCODE OBJECT

특정 검색 도움말을 직접 지정하고 싶으면:

```abap
PARAMETERS p_id TYPE zde_persid
  MATCHCODE OBJECT zsh_person_coll.
```

- `MATCHCODE OBJECT`로 이 입력칸에 쓸 [[Search Help]]를 명시한다.

## F4가 오는 세 경로

1. **고정값**(Domain) — 짧은 코드 목록.
2. **Foreign Key/Check Table** — 마스터 테이블 값.
3. **Search Help** — 설계된 검색 화면(가장 유연).

## 흔한 실수와 점검

- **F4가 안 떠요** → 표준 타입으로 직접 줬다. Data Element 경유 또는 MATCHCODE로.
- **여러 도움말 충돌** → 우선순위를 이해하고 하나로 명확히(MATCHCODE 권장).
