## 핸들러를 한 곳에

여러 이벤트(더블클릭·hotspot·toolbar·user_command)를 프로그램 곳곳에 흩으면
관리가 어렵다. **전용 핸들러 클래스**(또는 Controller, CH26-MVC)에 모은다.

```abap
CLASS lcl_alv_handler DEFINITION.
  PUBLIC SECTION.
    METHODS: constructor IMPORTING io_grid TYPE REF TO cl_gui_alv_grid,
             on_double_click  FOR EVENT double_click  OF cl_gui_alv_grid IMPORTING e_row,
             on_user_command  FOR EVENT user_command  OF cl_gui_alv_grid IMPORTING e_ucomm,
             on_toolbar       FOR EVENT toolbar        OF cl_gui_alv_grid IMPORTING e_object.
ENDCLASS.

METHOD constructor.
  SET HANDLER on_double_click on_user_command on_toolbar FOR io_grid.
ENDMETHOD.
```

- 생성자에서 **모든 SET HANDLER**를 한 번에.
- CH26의 Controller가 이 핸들러를 들고 Model·View를 조율하면 구조가 깔끔하다.

## 이벤트 → 핸들러 배선 살펴보기

이벤트가 어느 핸들러 메서드로 연결되는지, 그리고 그 배선이 생성자 `SET HANDLER` 한 곳에서
이뤄지는지 확인해 보라.

## 흔한 실수와 점검

- **핸들러 분산** → 한 클래스로 모아 응집도를 높인다.
- **그리드 참조 보관** → 핸들러가 그리드를 알아야 갱신·선택 조회 가능.


## 🧩 도전 과제 (서브퀘스트)

> 선택 과제 — 예매 목록 ALV에 상호작용을 더한다.

1. 예매 행을 **더블클릭**하면 그 예매 상세(또는 공연 정보)를 띄운다.
2. 툴바에 "예매 취소" 버튼(`toolbar`)을 더하고, `user_command`에서 선택 행을 취소 처리한다.
3. (심화) 공연ID 컬럼을 **hotspot**으로 만들어 한 번 클릭으로 공연 상세로 이동.

힌트: L01(double_click)·L02(hotspot)·L03(toolbar)·L04(user_command)·L05(핸들러 클래스).
