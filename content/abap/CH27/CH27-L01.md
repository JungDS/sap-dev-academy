## 표를 클릭하면 반응하게

[Chapter 21 · SALV/Grid ALV 표시 제어 심화](CH21-L01.html)까지 ALV는 **보여주기만** 했다.
업무 화면에서 한 행은 보통 "목록"이고, 사용자가 진짜 보고 싶은 건 **그 행의 상세**다 —
예매 목록에서 한 줄을 더블클릭하면 예매 상세·좌석 변경으로 넘어가는 식.

여기서 핵심은 **ALV가 알아서 상세를 열어 주지 않는다**는 점이다. ALV는 "더블클릭이 일어났고,
클릭된 행·컬럼은 이거야"라고 **이벤트로 알려 줄 뿐**이다. 그 이벤트를 받을 메서드를 만들고
[OO 이벤트 — EVENTS / RAISE EVENT / SET HANDLER](CH20-L09.html)에서 배운 `SET HANDLER`로
ALV에 연결한 뒤, 메서드 안에서 *무슨 업무 반응을 할지*는 내가 정한다.

## 핸들러 메서드 — FOR EVENT

`CL_GUI_ALV_GRID`의 `double_click` 이벤트를 받을 메서드는 세 가지를 갖춘다.

1. 선언에 `FOR EVENT double_click OF cl_gui_alv_grid`
2. 이벤트가 넘기는 `e_row`·`e_column`을 `IMPORTING`으로 받기
3. 실제 ALV 객체에 `SET HANDLER ... FOR go_grid`로 등록

```abap
CLASS lcl_alv_handler DEFINITION.
  PUBLIC SECTION.
    METHODS constructor
      IMPORTING it_booking TYPE ztt_booking.
    METHODS on_double_click
      FOR EVENT double_click OF cl_gui_alv_grid
      IMPORTING e_row e_column.
  PRIVATE SECTION.
    DATA mt_booking TYPE ztt_booking.
ENDCLASS.

CLASS lcl_alv_handler IMPLEMENTATION.
  METHOD constructor.
    mt_booking = it_booking.
  ENDMETHOD.
  METHOD on_double_click.
    READ TABLE mt_booking INTO DATA(ls_booking) INDEX e_row-index.
    IF sy-subrc <> 0.
      MESSAGE '선택한 행을 찾을 수 없습니다.' TYPE 'I'.
      RETURN.
    ENDIF.
    MESSAGE |예매 { ls_booking-booking_id } 상세로 이동| TYPE 'I'.
  ENDMETHOD.
ENDCLASS.
```

핸들러가 내부 테이블 복사본을 오래 들고 있기보다, 실무에선 화면 모델·서비스 객체를 통해
**업무 키로 상세를 조회**하도록 분리하는 편이 좋다.

## 연결 — SET HANDLER

선언만으론 호출되지 않는다. ALV 객체에 **등록**해야 한다.

```abap
DATA(go_handler) = NEW lcl_alv_handler( it_booking = lt_booking ).
SET HANDLER go_handler->on_double_click FOR go_grid.
```

- `e_row-index` — 더블클릭한 **화면 행 번호**. `e_column-fieldname` — 클릭한 컬럼.
- 등록은 표를 그린 **뒤**(`set_table_for_first_display` 이후) 한 번 한다.

## 직접 더블클릭해 보기

아래 예매 ALV의 **행을 더블클릭**하면 핸들러가 받은 `e_row-index`·`e_column`이 이벤트
로그에 쌓인다.

::embed CH27-L01-S01 | Double Click 이벤트 — 행 더블클릭 | 460::

> ⚠️ **`e_row-index`는 업무 키가 아니다.** "3번 행"이란 *사용자가 지금 보는 위치*일 뿐이다.
> 정렬·필터가 걸리면 화면 순서와 내부 테이블 순서가 달라진다. 실습에선 `INDEX`로 흐름을
> 익히되, 실제 상세 조회는 그 행에서 꺼낸 **업무 키**(`booking_id`·`concert_id`)로 한다.

## 흔한 실수와 점검

- **`FOR EVENT`만 선언, `SET HANDLER` 누락** → 메서드는 있지만 호출되지 않는다(이벤트 수업
  최다 오류). 선언 = "이 이벤트를 처리할 수 있다", 등록 = "이 ALV의 이벤트가 나면 이
  메서드를 불러라".
- **핸들러 객체 수명** → 핸들러를 너무 짧은 지역 변수로만 두면 수명이 헷갈린다. 참조를
  전역·컨트롤러 속성처럼 **명확한 곳에 보관**해야 해제·재등록·디버깅이 쉽다.
- **행 번호를 DB 키처럼 사용** → `e_row-index`는 화면 상태에 따라 바뀐다. 행 번호로 내부
  테이블을 읽어 **키를 꺼낸 뒤** 상세로 잇는다.

## 정리

- `double_click` = "사용자가 어떤 ALV 행을 더블클릭했다"는 **신호**.
- `FOR EVENT`로 받을 메서드를 선언하고, `SET HANDLER`로 ALV에 **등록**해야 호출된다.
- `e_row-index`는 화면 위치일 뿐 — 실무 반응은 내부 테이블에서 **업무 키**를 읽어 상세
  조회·화면 이동으로 잇는다.
