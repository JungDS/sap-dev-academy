## 표를 클릭하면 반응하게

CH21까지 ALV는 **보여주기만** 했다. 이제 사용자가 행을 더블클릭하면 **상세 화면**으로
가는 식의 상호작용을 넣는다. CH20-L09의 OO 이벤트(`SET HANDLER`)가 여기 쓰인다.

## 핸들러 메서드 — FOR EVENT

`CL_GUI_ALV_GRID`의 `double_click` 이벤트를 처리할 메서드를 만든다.

```abap
CLASS lcl_handler DEFINITION.
  PUBLIC SECTION.
    METHODS on_double_click
      FOR EVENT double_click OF cl_gui_alv_grid
      IMPORTING e_row e_column.
ENDCLASS.

CLASS lcl_handler IMPLEMENTATION.
  METHOD on_double_click.
    READ TABLE lt_booking INTO DATA(ls) INDEX e_row-index.
    IF sy-subrc = 0.
      " 예매 상세 화면 호출
    ENDIF.
  ENDMETHOD.
ENDCLASS.
```

## 연결 — SET HANDLER

```abap
DATA(lo_h) = NEW lcl_handler( ).
SET HANDLER lo_h->on_double_click FOR go_grid.
```

- `e_row-index` — 클릭한 행 번호. `e_column-fieldname` — 클릭한 컬럼.

## 직접 더블클릭해 보기

아래 ALV 표의 행을 더블클릭하면 핸들러가 받는 `e_row-index`·`e_column`이 이벤트 로그에
찍힌다.

## 흔한 실수와 점검

- **SET HANDLER 누락** → 더블클릭해도 반응 없음(CH20-L09).
- **인덱스 vs 키** → `e_row-index`로 내부 테이블 행을 READ.
