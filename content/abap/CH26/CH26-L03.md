## IF 사다리로 분기가 길어질 때

"일반/VIP/조기예매" 요금 계산을 `IF`/`CASE`로 길게 나누면 추가할 때마다 그 코드를
건드려야 한다. **Strategy**는 각 방식을 **인터페이스 구현**으로 분리해 갈아끼운다.

```abap
INTERFACE zif_price_strategy.
  METHODS calc IMPORTING iv_seats TYPE i RETURNING VALUE(rv) TYPE p.
ENDINTERFACE.

" 사용: 전략을 주입받아 그대로 호출
METHOD checkout.
  DATA(lv_price) = mo_strategy->calc( iv_seats = 2 ).
ENDMETHOD.
```

- 새 요금 정책 = **새 클래스** 추가(기존 코드 불변 — OCP 원칙).
- 어떤 전략을 쓸지는 팩토리·설정으로 주입.

## 흔한 실수와 점검

- **전략 안에 전략 분기** → 전략 내부에서 또 IF 분기하면 의미 없다. 한 전략=한 방식.
- **과설계** → 방식이 1~2개로 고정이면 Strategy가 과할 수 있다.
