## 잠그기 — ENQUEUE

앞 레슨에서 만든 Lock Object를 활성화하면 `ENQUEUE_EZ_BOOKING`·`DEQUEUE_EZ_BOOKING`
함수가 자동으로 생겼다. 우리는 [[Function Module|함수]]를 **호출**([CALL FUNCTION 기본 구조](CH10-L03.html))만
하면 된다. 키를 넘기면 **그 행만** 잠긴다.

```abap
CALL FUNCTION 'ENQUEUE_EZ_BOOKING'
  EXPORTING  mode_zbooking  = 'E'   " 잠금 모드(변경=E)
             booking_id     = lv_id " 이 키만 잠금
             _scope         = '2'   " 잠금 수명(아래 표)
  EXCEPTIONS foreign_lock   = 1     " 이미 남이 잠갔음
             system_failure = 2
             OTHERS         = 3.

IF sy-subrc = 1.
  MESSAGE '다른 사용자가 편집 중입니다' TYPE 'E'.   " 정중히 거절(정상 충돌)
ELSEIF sy-subrc <> 0.
  MESSAGE '잠금 처리 중 시스템 오류' TYPE 'E'.
ENDIF.
```

핵심은 `foreign_lock` 예외다. 이미 **다른 사용자가 잠근** 행을 또 잠그려 하면 여기로
떨어진다(`sy-subrc=1`). `foreign_lock`은 "예외"라는 단어 때문에 비정상처럼 느껴지지만, 업무적으로는
**자주 일어나는 정상 충돌**이다 — 무시하고 변경하면 동시 충돌이 나므로, 반드시 분기해서 "편집 중"이라고
안내하고 멈춰야 한다. (lock은 `sy-subrc`가 아니라 `EXCEPTIONS`로 판정한다.)

## 풀기 — DEQUEUE

```abap
CALL FUNCTION 'DEQUEUE_EZ_BOOKING'
  EXPORTING mode_zbooking = 'E'
            booking_id    = lv_id
            _scope        = '2'.   " ENQUEUE와 같은 _scope로
```

`_scope`는 **잠금의 수명과 해제 주체**를 정한다. ENQUEUE와 DEQUEUE의 `_scope`를 아무렇게나 섞으면
"왜 안 풀리지?"가 된다 — 같은 lock은 같은 값으로 대응시킨다.

| `_scope` | 뜻 | 해제 |
|---|---|---|
| `1` | 현재 프로그램이 관리 | `DEQUEUE` 또는 프로그램 종료로 해제 |
| `2` | SAP LUW/update에 연결(기본적으로 많이 씀) | `COMMIT WORK`/`ROLLBACK WORK` 흐름에서 해제 |
| `3` | 프로그램·update 양쪽 관리 | 양쪽 해제가 모두 맞아야 완전 해제 |

## 두 사용자로 직접 — 누가 이기나?

아래에서 **User A와 B**가 같은 예매(`1001`)를 두고 잠금을 다툰다. A가 먼저 `ENQUEUE`한
뒤 B가 `ENQUEUE`하면 어떻게 되는지, A가 `DEQUEUE`/`COMMIT`하면 B가 언제 가능해지는지
직접 눌러 보라. 가운데 **SM12**(잠금 목록)에 누가 잠갔는지 뜬다.

## 표준 패턴 — 잠금부터 해제까지

1. **ENQUEUE**(잠금) → 2. 읽기 → 3. 변경(DML) → 4. **COMMIT WORK**(확정) → 5. **DEQUEUE**(해제).

> `COMMIT WORK`/`ROLLBACK WORK`는 보통 잠금을 **자동 해제**한다(다음 레슨). 그래도 흐름을
> 명확히 하려고 `DEQUEUE`를 명시하기도 한다.

## 흔한 실수와 점검

- **`foreign_lock` 미처리** → 이미 잠긴 걸 무시하면 충돌. `sy-subrc` 분기로 거절·안내.
- **잠금만 하고 안 풂** → 다른 사용자가 영영 못 쓴다. 반드시 `DEQUEUE`(또는 COMMIT 자동 해제).
- **잠금을 너무 오래 쥠** → 편집 화면을 켜 둔 채 자리를 비우면 남이 막힌다(다음 레슨).

## 정리

- 활성화로 생긴 `ENQUEUE_*`/`DEQUEUE_*`를 **호출**해 잠그고 푼다.
- `foreign_lock`(이미 남이 잠금)은 무시 말고 **정중히 거절**한다.
- 패턴 = ENQUEUE → 읽기 → 변경 → COMMIT → (DEQUEUE).
