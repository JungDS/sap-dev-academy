## 잠그기 — ENQUEUE

Lock Object를 활성화하면 생긴 `ENQUEUE_EZ_BOOKING`을 호출해 잠근다(CH10의 CALL
FUNCTION). 키를 넘기면 그 행만 잠긴다.

```abap
CALL FUNCTION 'ENQUEUE_EZ_BOOKING'
  EXPORTING  booking_id = lv_id
  EXCEPTIONS foreign_lock = 1     " 이미 남이 잠금
             system_failure = 2
             OTHERS = 3.

IF sy-subrc <> 0.
  MESSAGE '다른 사용자가 편집 중입니다' TYPE 'E'.
ENDIF.
```

## 풀기 — DEQUEUE

```abap
CALL FUNCTION 'DEQUEUE_EZ_BOOKING'
  EXPORTING booking_id = lv_id.
```

## 표준 패턴

1. **ENQUEUE**(잠금) → 2. 읽기 → 3. 변경(DML) → 4. **COMMIT** → 5. **DEQUEUE**(해제).

## 흔한 실수와 점검

- **foreign_lock 미처리** → 이미 잠긴 걸 무시하면 충돌. `sy-subrc` 분기로 거절.
- **잠금만 하고 안 풂** → 다른 사용자가 영영 못 쓴다. 반드시 DEQUEUE(또는 COMMIT 시 자동).
