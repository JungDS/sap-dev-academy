## 잠금은 언제 풀리나

- **DEQUEUE** 명시 호출 시.
- **COMMIT WORK / ROLLBACK WORK** 시 — 보통 트랜잭션 종료와 함께 자동 해제된다.
- 세션 종료 시.

`DEQUEUE_ALL`로 현재 프로그램이 건 **모든 잠금**을 한 번에 풀 수도 있다.

## foreign_lock 예외 — 정중히 거절

이미 남이 잠갔으면 `foreign_lock`이 난다. 누가 잠갔는지 알려 주면 친절하다.

```abap
CALL FUNCTION 'ENQUEUE_EZ_BOOKING'
  EXPORTING  booking_id = lv_id
  EXCEPTIONS foreign_lock = 1 OTHERS = 2.

IF sy-subrc = 1.
  MESSAGE |사용자 { sy-msgv1 }가 편집 중입니다| TYPE 'E'.
ENDIF.
```

## 흔한 실수와 점검

- **장시간 잠금** → 사용자가 화면을 켜 둔 채 점심. 가능하면 **짧게** 잠그고 푼다.
- **예외 안 잡음** → foreign_lock을 무시하면 사용자 경험이 나쁘다. 안내한다.
