## 동시에 같은 예매를 고치면

두 사용자가 같은 예매(`ZBOOKING`)를 **동시에** 수정하면, 한쪽 변경이 사라지거나
데이터가 깨진다. CH24의 DML만으로는 못 막는다. **잠금**이 필요하다 — [[Lock Object]]다.

## Lock Object — SE11

`SE11` → Lock Object → `EZ_BOOKING`(관례: `E` + 테이블)을 만든다.

- **Primary Table** — 잠글 테이블(`ZBOOKING`).
- **Lock Argument** — 무엇을 기준으로 잠글지(보통 키 필드: `booking_id`).
- 활성화하면 **ENQUEUE/DEQUEUE 함수**가 자동 생성된다(`ENQUEUE_EZ_BOOKING` 등).

## 잠금 모드

| 모드 | 뜻 | 동시 허용 |
|---|---|---|
| `E` | Exclusive(쓰기) | 한 명만 — 가장 흔함 |
| `S` | Shared(읽기) | 여럿 읽기 OK, 쓰기 차단 |
| `X` | eXclusive non-cumulative | 같은 사용자도 중첩 불가 |

대부분 변경 잠금은 **`E`**(exclusive)다.

## 흔한 실수와 점검

- **Lock Argument 과다** → 너무 넓게 잠그면 동시성↓. 보통 키 단위.
- **잠금 없이 변경** → 동시 충돌 위험. 변경 전 잠근다.
