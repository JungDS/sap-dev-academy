## 고쳤더니 다른 게 깨졌다

기능을 추가·수정하면 멀쩡하던 곳이 깨질 수 있다(회귀). CH26-L05의 **ABAP Unit**을
운영 차원에서 활용해 자동으로 막는다.

- 핵심 로직마다 단위 테스트를 둔다(Mock으로 DB 분리).
- **이송·빌드 게이트**에서 ABAP Unit·ATC를 자동 실행 → 실패면 막는다.
- `gCTS`·CI 파이프라인과 결합해 변경마다 검증.

```abap
" 검증 예
cl_abap_unit_assert=>assert_equals( act = lo->remaining( ) exp = 50 ).
```

## 흔한 실수와 점검

- **테스트 없이 리팩터링** → 회귀를 못 잡는다. 핵심엔 테스트.
- **느린/불안정 테스트** → DB 의존을 Mock으로 끊어 빠르고 안정적으로.

회귀를 막았다. 변경을 운영으로 옮기는 Transport → 다음 레슨.
