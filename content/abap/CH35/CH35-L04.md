## 대량·정기 작업은 배치로

야간 정산·대량 인터페이스(CH24·CH30)는 화면 없이 **정해진 시각에 자동** 실행한다.
그게 [[Background Job]]이다.

- `SM36` — 잡 정의·스케줄(주기·시작 조건).
- `SM37` — 잡 모니터(상태·로그·스풀).
- 프로그램은 보통 `SUBMIT ... VIA JOB ... AND RETURN`으로 잡에 넣거나, 잡 API로 등록.

```abap
" 개념: 잡 열기 → SUBMIT → 닫기
" JOB_OPEN → SUBMIT zprog VIA JOB lv_name NUMBER lv_nr → JOB_CLOSE
```

## 운영 원칙

- **재시작 가능**하게 설계(CH24-L05의 재시작점·로그).
- 실패 시 **알림**(메일·로그)으로 즉시 인지.
- 자원·시간대를 고려해 스케줄(야간·부하 분산).

## 흔한 실수와 점검

- **무한·중복 실행** → 스케줄·잠금으로 중복 방지.
- **실패 방치** → SM37 모니터·알림으로 추적.
