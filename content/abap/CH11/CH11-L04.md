## 처음부터 끝까지 한 번에

지금까지 배운 걸 모아 **DB 조회 → 표 출력**까지 하나의 프로그램으로 완성한다.
classic [Chapter 08 · SELECT](CH08-L03.html) + [[SALV]]의 결합이다.

```abap
REPORT zsalv_person.

DATA: lt_person TYPE TABLE OF ztperson,
      lo_alv    TYPE REF TO cl_salv_table.

START-OF-SELECTION.
  " 1) 데이터 조회 (classic Open SQL)
  SELECT * FROM ztperson INTO TABLE lt_person.

  " 2) SALV 객체 생성
  TRY.
      cl_salv_table=>factory(
        IMPORTING r_salv_table = lo_alv
        CHANGING  t_table      = lt_person ).

      " 3) 표준 기능 + 표시
      lo_alv->get_functions( )->set_all( abap_true ).
      lo_alv->display( ).
    CATCH cx_salv_msg.
      MESSAGE 'ALV 생성 실패' TYPE 'I'.
  ENDTRY.
```

실행(F8)하면 정렬·필터·엑셀 내보내기가 되는 **표 리포트**가 뜬다. 첫 "쓸 만한"
리포트의 완성이다.

## 직접 해 보기

`SELECT → 행 수 확인 → factory → set_all → display` 순서로 눌러, 데이터가 DB→메모리→화면으로 흘러가는 걸 보자. 단계를 건너뛰면 무엇이 빠졌는지 알려 준다.

## 흔한 실수와 점검

- **조회 결과 미확인** → 빈 테이블이면 빈 표. `DESCRIBE TABLE lt_person LINES lv_count`로 행 수를 세어 `IF lv_count = 0`이면 안내 메시지를 띄운다.
- **classic/modern 혼동** → 이 시점 SELECT는 **classic**([Chapter 19 · Modern SQL](CH19-L01.html) 전). `@` 안 쓴다.
