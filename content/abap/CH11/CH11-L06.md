## 🎫 3단계 — 예매 목록을 표로

CH09 데이터를 SELECT해 SALV(CH11)로 깔끔히 띄운다. `WRITE` 리스트와 비교해 보자.

```abap
DATA: lt_book TYPE TABLE OF zbooking,
      lo_alv  TYPE REF TO cl_salv_table.

SELECT * FROM zbooking INTO TABLE lt_book.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = lo_alv
      CHANGING  t_table      = lt_book ).
    lo_alv->get_functions( )->set_all( abap_true ).
    lo_alv->display( ).
  CATCH cx_salv_msg.
ENDTRY.
```

## 직접 해 볼 과제

1. 예매 목록을 SALV로 띄운다.
2. 컬럼 제목이 **Data Element 라벨**(CH03)로 자동으로 뜨는지 확인.
3. (도전) `lo_alv->get_columns( )`로 STATUS 컬럼 제목을 "상태"로 바꿔 본다.

## 흔한 실수와 점검

- **TRY/CATCH 없이 factory** → cx_salv_msg 덤프 위험(CH11-L02).
- **빈 결과** → SELECT 결과가 없으면 빈 표. sy-subrc 확인(CH08).
