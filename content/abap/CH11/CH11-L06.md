## 🎫 3단계 — 예매 목록을 표로

[Chapter 09 · 콘서트 모델](CH09-L09.html) 데이터를 SELECT해 SALV로 깔끔히 띄운다. `WRITE` 리스트와 비교해 보자.

```abap
DATA: lt_book TYPE TABLE OF zbooking,
      lo_alv  TYPE REF TO cl_salv_table.

SELECT * FROM zbooking INTO TABLE lt_book.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = lo_alv
      CHANGING  t_table      = lt_book ).
    lo_alv->get_functions( )->set_all( abap_true ).
    lo_alv->display( ).
  CATCH cx_salv_msg.
ENDTRY.
```

## 직접 해 볼 과제

1. 예매 목록을 SALV로 띄운다.
2. 컬럼 제목이 **Data Element 라벨**([Chapter 03](CH03-L02.html))로 자동으로 뜨는지 확인.
3. 컬럼 제목을 직접 바꾸는 `get_columns( )` 세밀 제어는 [SALV 표시 심화](CH22-L01.html)에서 배운다(지금은 라벨 자동 반영만 확인).

## 흔한 실수와 점검

- **TRY/CATCH 없이 factory** → cx_salv_msg 덤프 위험([factory 메서드](CH11-L02.html)).
- **빈 결과** → SELECT 결과가 없으면 빈 표. `DESCRIBE TABLE lt_book LINES lv_count`로 행 수를 확인해 0이면 안내 메시지.
