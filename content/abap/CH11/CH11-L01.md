## WRITE 리스트는 투박하다

앞선 챕터들에서 데이터를 다뤘지만, `WRITE`로 찍은 리스트는 정렬·필터·열 너비 조정도
안 되고 모양도 단조롭다. 사용자가 쓰기 좋은 **표(그리드)** 로 보여주고 싶다 —
그게 [[ALV]](ABAP List Viewer)이고, 그 **가장 단순한 길**이 [[SALV]]다.

## SALV = 단순·읽기전용 ALV

[[SALV]]는 `CL_SALV_TABLE` 클래스로 표를 그리는 현대적·간결한 ALV다.

- **장점** — 코드 몇 줄로 정렬·필터·합계·엑셀 내보내기 등 표준 기능이 공짜.
- **범위** — 기본은 **읽기 전용** 표시. 셀 편집은 [Editable Grid ALV](CH28-L01.html), 컨테이너형
  표시는 [Grid ALV](CH17-L01.html), 표시 심화는 [SALV 심화](CH21-L01.html)에서 배운다.

| 방식 | 성격 |
|---|---|
| WRITE 리스트 | 투박, 수동 |
| **SALV (CL_SALV_TABLE)** | 간결, 표준 기능 풍부, 읽기전용 |
| Grid ALV (CL_GUI_ALV_GRID) | 강력·편집 가능, 코드 많음 ([Grid ALV](CH17-L01.html)) |

## 왜 SALV부터인가

내부 테이블 하나만 있으면 **몇 줄로** 그럴듯한 리포트가 나온다. 입문 단계의
"결과를 보기 좋게"에 가장 잘 맞는다.
