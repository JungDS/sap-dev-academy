## 뷰 본문과 UI를 분리

UI 주석(`@UI`)이 뷰 정의에 잔뜩 섞이면 본문이 지저분하다. **Metadata Extension**으로
UI 주석만 **별도 파일**에 빼서, 본문은 데이터에 집중하게 한다.

```abap
@Metadata.layer: #CORE
annotate entity ZC_Concert with
{
  @UI.lineItem: [{ position: 10 }]
  artist;

  @UI.lineItem: [{ position: 20 }]
  venue;
}
```

- `annotate entity 뷰 with { ... }` — 기존 뷰에 주석만 덧붙인다.
- `@Metadata.layer` — 여러 계층(CORE/CUSTOMER 등)으로 덮어쓰기 가능.
- 같은 뷰의 UI를 환경별로 다르게 줄 수 있다.

## 흔한 실수와 점검

- **중복 주석** → 본문과 Extension에 같은 주석이 충돌하지 않게.
- **layer 이해** → 상위 layer가 하위를 덮어쓴다.
