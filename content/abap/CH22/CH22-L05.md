## 셀마다 다른 모양·동작 🔘

색뿐 아니라 셀마다 **편집 가능 여부·버튼** 같은 모양을 다르게 줄 수 있다. 색과 똑같이 행 구조에 [[Cell Style|스타일 테이블]](`LVC_T_STYL`)을 품는 deep 방식이다.

```abap
TYPES: BEGIN OF ts_perf_alv,
         "... 기존 표시 필드 ...
         cellstyles TYPE lvc_t_styl,
       END OF ts_perf_alv.

LOOP AT gt_perf_alv ASSIGNING FIELD-SYMBOL(<p>).
  CLEAR <p>-cellstyles.
  IF <p>-booked >= <p>-capacity.                   " 매진 좌석 칸은 비활성으로
    APPEND VALUE lvc_s_styl(
      fieldname = 'BOOKED'
      style     = cl_gui_alv_grid=>mc_style_disabled ) TO <p>-cellstyles.
  ENDIF.
ENDLOOP.

gs_layout-stylefname = 'CELLSTYLES'.               " 스타일 컬럼을 Layout에 연결
```

| style 상수 | 의미 |
|---|---|
| `cl_gui_alv_grid=>mc_style_disabled` | 셀 **비활성** 표시(더 못 건드림) |
| `cl_gui_alv_grid=>mc_style_enabled` | 셀 **편집 가능** 표시(조건부) |
| `cl_gui_alv_grid=>mc_style_button` | 셀을 **버튼**처럼 표시 |

## 색과 무엇이 다른가 🔍

구조는 색과 똑같이 deep지만, **연결하는 layout 속성과 컬럼명 필드가 다르다.**

| | 색(L04) | 모양(L05) |
|---|---|---|
| 행 component | `LVC_T_SCOL` | `LVC_T_STYL` |
| layout 연결 | `ctab_fname` | `stylefname` |
| 컬럼명 필드 | `fname` | `fieldname` |

> 🚧 **버튼처럼 보여도 클릭 처리는 아직 없다.** `mc_style_button`은 *모양*일 뿐, 클릭 이벤트(상세 열기 등)는 이후 ALV 이벤트 챕터에서 구현한다. 마찬가지로 `mc_style_enabled`로 *편집 가능하게 보여도*, 입력값 반영·검증·저장은 이후 편집 챕터의 일이다.

## 흔한 실수와 점검 🚧

- **색과 스타일 혼동** → 색 = `ctab_fname`, 모양·동작 = `stylefname`. 헷갈리면 채웠는데 화면이 안 바뀐다.
- **`fname` vs `fieldname`** → `LVC_T_SCOL`은 `fname`, `LVC_T_STYL`은 `fieldname`. 이름이 비슷해도 구조가 다르다.
- **`stylefname` 미연결** → 스타일 테이블을 채워도 연결 안 하면 화면 그대로(위 체험의 2단계).
- **`CELLSTYLES` 컬럼 노출** → 스타일 테이블도 표시용 기술 컬럼이라, 색의 `CELLCOLORS`처럼 field catalog에서 `no_out`으로 숨긴다([컬럼 숨김](CH22-L03.html)).
- **편집 표시 = 검증 완료 착각** → 비활성처럼 보여도 **저장 전 서버 로직에서 반드시 검증**한다.
