## 뷰를 계층으로

실무 CDS는 보통 두 계층으로 나눈다.

| 계층 | 접두어 | 역할 |
|---|---|---|
| **Interface View** | `ZI_` | 재사용 가능한 **기반**(데이터·관계) |
| **Projection View** | `ZC_` | 특정 화면/서비스용 **소비**(UI에 맞게 추림) |

## Projection View

기반 뷰(`ZI_Concert`)를 **투영**해 소비용으로 다듬는다.

```abap
define view entity ZC_Concert
  as projection on ZI_Concert
{
  key concert_id,
      artist,
      venue,
      capacity
}
```

- `as projection on 기반뷰` — 기반을 투영.
- UI 주석(다음 레슨)·필드 추림을 여기서.
- 이 분리로 **기반은 안정**시키고 **소비는 자유롭게** 바꾼다.

## 흔한 실수와 점검

- **기반에 UI 주석** → UI 관심사는 Projection(ZC_)·Metadata Extension으로.
- **계층 생략** → 작은 건 한 계층도 되지만, 재사용엔 ZI_/ZC_ 분리가 유리.
