## 왜 — 정렬·필터만으론 부족 🎨

정렬·필터를 줘도 긴 컬럼이 잘리고, 제목이 DDIC 기술명 그대로 나오고, 매번 같은 컬럼 순서를 다시 맞춰야 하면 보고서가 불편하다. [[SALV]] layout·variant는 이걸 줄인다. **개발자는 "처음 보기 좋은 기본값"**을 주고, **사용자는 "내 업무 방식 보기"**를 저장한다. [Grid의 Variant 기본 설정](CH17-L06.html)에서 `DISVARIANT`·`i_save`로 하던 그 일을, SALV에서는 `get_layout( )`의 `set_key`·`set_save_restriction`으로 한다.

## 세 층의 표시 설정 🧩

| 층 | 객체 | 역할 |
|---|---|---|
| 표 전체 | `get_display_settings( )` | 제목(list header)·줄무늬 |
| 컬럼 묶음 | `get_columns( )` | 폭 최적화·특정 컬럼 접근 |
| variant 저장 | `get_layout( )` | report별 layout key·저장 허용 |

```abap
DATA(go_disp) = go_salv->get_display_settings( ).
go_disp->set_striped_pattern( abap_true ).
go_disp->set_list_header( '회차별 예매 현황' ).

DATA(go_cols) = go_salv->get_columns( ).
go_cols->set_optimize( abap_true ).               " 컬럼 폭 최적화

TRY.
    DATA(go_cap) = go_cols->get_column( 'CAPACITY' ).
    go_cap->set_short_text(  '정원' ).            " ★ 셋 다 맞춘다
    go_cap->set_medium_text( '정원' ).
    go_cap->set_long_text(   '정원' ).
  CATCH cx_salv_not_found.
    MESSAGE 'CAPACITY 컬럼을 찾을 수 없습니다.' TYPE 'E'.
ENDTRY.
```

> ⚠️ `set_long_text`만 바꾸면 헤더가 그대로일 수 있다. SALV는 컬럼 폭에 따라 **short/medium/long 중 하나를 골라** 쓰므로, 셋을 함께 맞춰야 초보자 혼란이 없다.

## Variant 저장 💾

```abap
DATA(go_layout) = go_salv->get_layout( ).
go_layout->set_key( VALUE salv_s_layout_key( report = sy-repid ) ).   " 어느 리포트의 layout인지
go_layout->set_save_restriction( if_salv_c_layout=>restrict_none ).   " 저장 허용
```

`set_key`로 "이 variant가 어느 리포트의 layout인지" 식별한다(`sy-repid` = 현재 프로그램). 그래야 사용자가 컬럼 순서·숨김·폭을 자기 보기로 저장할 수 있다. `set_save_restriction( restrict_none )`은 저장을 허용한다는 뜻이고, 사용자별/전역 저장을 제한하는 상수도 따로 있다.

> ⚠️ **저장한 variant는 개발자 기본을 덮어쓴다.** 사용자가 variant를 저장하면, 다음에 프로그램이 떠도 개발자가 코드로 준 기본 보기가 아니라 **그 사용자의 variant**가 먼저 적용된다. "코드를 고쳤는데 화면이 그대로"인 대표 원인이니, 위 체험의 *다시 실행*으로 직접 확인하자.

## 정리 📌

- **layout** = 개발자가 주는 *첫 보기*(줄무늬·폭·컬럼 텍스트). **variant** = 사용자가 저장한 *내 보기*.
- 컬럼 텍스트는 short/medium/long을 **함께** 맞춰야 폭에 상관없이 헤더가 바뀐다.
- variant는 **표시 설정만** 저장할 뿐 데이터·권한과는 무관하다.

## 흔한 실수와 점검 🚧

- **`long_text`만 바꿈** → 헤더가 안 바뀔 수 있다. short/medium/long을 **함께**.
- **컬럼명 오타** → `get_column( )`은 화면 제목이 아니라 **내부 필드명**(대문자)으로 접근. 틀리면 `cx_salv_not_found`.
- **레이아웃 키 누락** → variant를 저장하려면 `set_key`로 식별부터.
- **variant ≠ 데이터 변경** → 컬럼 순서·숨김·폭 같은 **표시 설정만** 저장된다. 또 ALV 숨김은 **권한 제어가 아니다** — 보안상 숨길 값은 애초에 조회하지 않는다.
