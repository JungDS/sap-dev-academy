## 트랜잭션의 두 층

- **DB LUW**(Logical Unit of Work) — DB가 보장하는 한 묶음(COMMIT 사이).
- **SAP LUW** — 여러 화면·단계에 걸친 **업무 단위**. 마지막에 한 번 DB에 반영.

SAP은 사용자가 여러 화면을 거치는 동안 변경을 **모아 두었다가**, 마지막 `COMMIT WORK`에
**한꺼번에** DB에 쓴다. 이때 실제 저장을 **Update Module**이 비동기로 처리한다.

```abap
CALL FUNCTION 'Z_SAVE_BOOKING' IN UPDATE TASK
  EXPORTING is_booking = ls_booking.
" ... 다른 처리 ...
COMMIT WORK.        " 이 순간 등록된 update task들이 실행된다
```

- `IN UPDATE TASK` — 지금 당장이 아니라 **COMMIT 시점에** 실행하도록 등록.
- `PERFORM ... ON COMMIT`도 비슷하게 "커밋 때 실행".

## 흔한 실수와 점검

- **즉시 실행 기대** → `IN UPDATE TASK`는 COMMIT 때 실행. 그 전엔 안 됨.
- **표준은 BAPI** → 표준 데이터 변경은 BAPI가 LUW를 알아서 관리.
