## 바꿨다고 끝이 아니다

DML 문은 변경을 **예약**할 뿐, 확정은 [[COMMIT WORK]]가 한다. 중간에 문제가 생기면
[[ROLLBACK WORK]]로 **전부 되돌린다**.

```abap
INSERT zbooking FROM @ls_booking.
INSERT zbooking_item FROM TABLE @lt_items.

IF sy-subrc = 0.
  COMMIT WORK.        " 둘 다 확정
ELSE.
  ROLLBACK WORK.      " 둘 다 취소(부분 저장 방지)
ENDIF.
```

## 왜 중요한가 — 전부 아니면 전무

"예매 헤더는 저장됐는데 항목은 실패" 같은 **반쪽 저장**은 데이터를 깨뜨린다. COMMIT/
ROLLBACK으로 **하나의 묶음(원자성)** 으로 처리한다.

## 흔한 실수와 점검

- **COMMIT 누락** → 변경이 확정 안 돼 사라질 수 있다.
- **남발하는 COMMIT** → 묶여야 할 작업을 쪼개면 반쪽 저장 위험. 논리 단위로.
- **COMMIT WORK AND WAIT** → 후속이 저장 완료를 기다려야 하면 `AND WAIT`.
