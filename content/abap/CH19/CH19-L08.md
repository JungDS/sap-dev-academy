## 🎫 SELECT를 모던으로

CH12·CH13의 조회를 CH19 모던 SQL(`@`·콤마·`@DATA`)로 다듬는다.

```abap
" classic (CH12-L07)
SELECT * FROM zbooking INTO TABLE lt_book
  WHERE concert_id IN s_conc.

" modern (CH19)
SELECT * FROM zbooking
  WHERE concert_id IN @s_conc
  INTO TABLE @DATA(lt_book).
```

공연별 집계도 모던으로(인라인 결과 테이블).

```abap
SELECT c~concert_id, c~artist, c~capacity,
       SUM( b~seats ) AS booked
  FROM zconcert AS c
  LEFT OUTER JOIN zbooking AS b
    ON  c~concert_id = b~concert_id
    AND b~status <> 'C'
  GROUP BY c~concert_id, c~artist, c~capacity
  INTO TABLE @DATA(lt_stat).
```

## 직접 해 볼 과제

1. CH12-L07 조회를 `@`·`@DATA`로 바꾼다.
2. CH13-L08 집계를 콤마·`@DATA`로 바꾼다(필드 사이 콤마 주의).
3. (도전) `CASE WHEN ... THEN 'FULL' ELSE 'OPEN' END AS status`로 매진 상태 컬럼을 계산(CH19-L04).

## 흔한 실수와 점검

- **@ 누락** → 모던 SELECT의 ABAP 변수엔 `@`. 컬럼엔 안 붙인다.
- **콤마/공백 혼용** → 모던은 필드 콤마. classic 공백과 섞지 않는다.
