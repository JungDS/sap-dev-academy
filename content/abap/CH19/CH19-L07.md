## 여기까지의 ABAP SQL

- **콤마·`@`·strict** — classic과 다른 modern SQL의 기본.
- **`@DATA( )`** — 결과 테이블 인라인 선언.
- **SQL 식**(CASE·CAST·COALESCE)·**문자/날짜 함수** — DB에서 값 가공.
- **`SELECT FROM @itab`** — 메모리 테이블도 SQL로.

이제 데이터 접근(읽기·가공)은 현대적이다. 곧 배울 **CDS View**(CH22)는 이 ABAP SQL을
**재사용 가능한 뷰**로 끌어올린 것이다 — 여기서 익힌 SQL 감각이 그대로 쓰인다.

## 다음은 — 코드 구조를 객체로

문법(CH18)·SQL(CH19)을 현대화했다. 그런데 CH17에서 ALV를 만들 때 객체·메서드를
"블랙박스"로 빌려 썼을 뿐, **직접 클래스를 설계**하진 않았다. 이제 코드 구조 자체를
객체로 설계하는 **OO ABAP** → **CH20**.

## 흔한 실수와 점검

- **classic/modern 혼용** → 한 SELECT 안에서 섞지 않는다(콤마면 `@`까지 일관되게).
- **SQL에 업무 로직 과적** → 복잡한 규칙은 ABAP/CDS·클래스로 분리.
