## 결과 그릇을 미리 선언 안 해도 된다

CH18의 인라인 선언을 SELECT에서도 쓴다. 단 SQL이라 **`@DATA(...)`** 형태다(앞 `@`).

```abap
" classic: 먼저 DATA 선언 후 SELECT
DATA lt_flight TYPE TABLE OF sflight.
SELECT * FROM sflight INTO TABLE lt_flight WHERE carrid = 'KE'.

" modern: 결과 타입을 SELECT가 알아서 만들어 준다
SELECT * FROM sflight
  WHERE carrid = 'KE'
  INTO TABLE @DATA(lt_flight).
```

단건은 `INTO @DATA( )`.

```abap
SELECT SINGLE * FROM scarr
  WHERE carrid = 'KE'
  INTO @DATA(ls_carr).
```

- `@DATA(lt)` — 결과 구조에 맞는 테이블을 **자동 선언**.
- 컬럼을 고르면(`SELECT carrid, connid`) 그 컬럼만 가진 구조로 만들어진다.

## 흔한 실수와 점검

- **@ 없이 DATA()** → SQL 안에선 `@DATA()`. 밖에선 `DATA()`(CH18).
- **이미 선언된 이름** → `@DATA()`는 새 이름이어야. 기존 변수면 `@lt_기존`.
