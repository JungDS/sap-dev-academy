## 결과 그릇을 미리 선언 안 해도 된다 🥣

[지난 챕터](CH18-L01.html)에서 배운 인라인 선언(`DATA( )`)을 SELECT 결과에도 쓴다. 단 SQL 문장 안이라 앞에 `@`가 붙어 **`@DATA( )`** 형태다.

```abap
" classic: 먼저 DATA로 선언한 뒤 SELECT
DATA lt_flight TYPE TABLE OF sflight.
SELECT * FROM sflight
  WHERE carrid = 'KE'
  INTO TABLE lt_flight.

" modern: 결과 타입을 SELECT가 알아서 만들어 준다
SELECT * FROM sflight
  WHERE carrid = 'KE'
  INTO TABLE @DATA(lt_flight).
```

`@DATA(lt_flight)`는 **SELECT 결과 구조에 맞는 내부 테이블을 그 자리에서 자동 선언**한다.

## 무엇이 만들어지나 — 모양은 SELECT 목록이 정한다 🧱

`SELECT *`면 원본 테이블 행 구조에 가깝고, **컬럼을 고르면 그 컬럼만 가진** 행 구조가 만들어진다.

```abap
SELECT carrid, connid, seatsmax
  FROM sflight
  WHERE carrid = @lv_carr
  INTO TABLE @DATA(lt_flight).
" → lt_flight 한 행 = carrid · connid · seatsmax
```

> 📦 이렇게 만들어지는 표는 **Standard Table · Empty Key**다. 키로 자주 찾아 읽어야 하는 결과라면, 인라인 대신 **타입을 직접 선언**하고 알맞은 key를 주는 편이 낫다.

단건은 표가 아니라 **Structure**(구조체)로 받는다: `INTO @DATA( )`.

```abap
SELECT SINGLE carrid, carrname
  FROM scarr
  WHERE carrid = @lv_carr
  INTO @DATA(ls_carr).
" → ls_carr = carrid · carrname (구조체)
```

## 계산 컬럼엔 이름(`AS`)을 붙인다 🏷️

값을 계산해 컬럼으로 뽑을 땐 **결과 컬럼에 이름이 없으면** 인라인 표의 구조가 흐릿해진다.

```abap
" 🚫 계산 컬럼에 이름이 없다 — 행 구조가 모호
SELECT carrid,
       seatsmax - seatsocc
  FROM sflight
  INTO TABLE @DATA(lt_bad).

" ✅ AS 로 이름을 준다
SELECT carrid,
       seatsmax - seatsocc AS seats_free
  FROM sflight
  INTO TABLE @DATA(lt_ok).
" → lt_ok 한 행 = carrid · seats_free
```

JOIN에서 같은 이름 컬럼이 겹칠 때도 마찬가지다. **컬럼 이름은 유일**해야 하니 `AS`로 풀어 준다.

## 흔한 실수와 점검 🚧

- **`@` 없이 `DATA( )`** → SQL 문장 안의 target은 `@DATA( )`. `DATA( )`는 SQL 밖 ABAP 문장용([Inline Declaration](CH18-L01.html)).
- **이미 선언된 이름에 `@DATA( )`** → `@DATA( )`는 **새 이름**일 때만. 기존 변수에 받으려면 `@lt_기존`.
- **계산·JOIN 컬럼에 별칭 없음** → 인라인 target은 결과 컬럼 이름이 필요하다. `AS`로 이름을 붙인다.
- **`@FINAL( )`** → 결과를 이후 바꾸지 않을 때만 제한적으로. 입문 실습은 보통 `@DATA( )`로 시작한다.
