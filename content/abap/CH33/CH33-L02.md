## DB 프로시저를 ABAP 메서드로

[[AMDP]](ABAP Managed Database Procedure)는 ABAP 클래스 메서드 안에 **SQLScript**(HANA
프로시저 언어)를 써서, DB에서 직접 실행되게 한다.

```abap
CLASS zcl_booking_amdp DEFINITION PUBLIC.
  PUBLIC SECTION.
    INTERFACES if_amdp_marker_hdb.        " AMDP 표시
    METHODS get_stats
      IMPORTING VALUE(iv_concert) TYPE zconcert-concert_id
      EXPORTING VALUE(et_stats)   TYPE ztt_stats
      AMDP OPTIONS READ-ONLY.             " 읽기 전용 선언
ENDCLASS.

CLASS zcl_booking_amdp IMPLEMENTATION.
  METHOD get_stats BY DATABASE PROCEDURE FOR HDB
                   LANGUAGE SQLSCRIPT
                   USING zbooking.
    et_stats = SELECT concert_id, SUM( seats ) AS total
               FROM zbooking
               WHERE concert_id = :iv_concert
               GROUP BY concert_id;
  ENDMETHOD.
ENDCLASS.
```

- `INTERFACES if_amdp_marker_hdb` — AMDP 클래스 표시.
- `BY DATABASE PROCEDURE FOR HDB LANGUAGE SQLSCRIPT` — 본문이 SQLScript.
- `USING` — 접근할 테이블/뷰를 선언. `:파라미터`로 입력 참조.
- `AMDP OPTIONS READ-ONLY` — 메서드가 **읽기만** 하면 선언에 붙인다. DB가 병렬·읽기복제본으로 더 빠르게 처리하고, 실수로 쓰기를 넣으면 문법 검사가 막아준다(이 예시는 SELECT뿐이라 READ-ONLY).

## 흔한 실수와 점검

- **USING 누락** → 쓰는 테이블을 USING에 선언해야 한다.
- **ABAP 문법 혼입** → 본문은 SQLScript다(ABAP 문 못 씀).
