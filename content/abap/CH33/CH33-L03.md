## DB에 직접 말 걸기

[[ADBC]](ABAP Database Connectivity)는 **DB 고유 Native SQL**을 ABAP에서 직접 실행하는
객체지향 API다(옛 `EXEC SQL`의 현대판).

```abap
TRY.
    DATA(lo_sql) = NEW cl_sql_statement( ).
    DATA(lo_res) = lo_sql->execute_query(
      |SELECT carrid, COUNT(*) FROM sflight GROUP BY carrid| ).
    lo_res->set_param_table( REF #( lt_result ) ).
    lo_res->next_package( ).
  CATCH cx_sql_exception INTO DATA(lx).
    " DB 오류 처리
ENDTRY.
```

- Open SQL이 못 하는 **DB 고유 기능**이 꼭 필요할 때만.
- **client 자동 종속·구문 검사 없음** → 직접 책임(MANDT·SQL injection 주의).

## 흔한 실수와 점검

- **client 누락** → Native SQL은 MANDT를 자동으로 안 챙긴다. 직접 조건에.
- **문자열 SQL 주입** → 파라미터 바인딩으로(직접 연결 금지).
