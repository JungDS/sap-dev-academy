## 양식이 안 나온다

- **데이터 누락** → Form Interface에 넘긴 데이터부터 확인(디버거 CH04).
- **렌더링 실패**(Adobe) → ADS 연결·로그 확인.
- **출력 결정 안 됨** → Output Control 조건(L03) 점검.
- **인쇄 큐** → `SP01`(스풀)에서 출력 상태 확인.

## 변경 대응

- 양식 레이아웃·로고·법적 문구는 자주 바뀐다 → **이송·버전 관리**(CH01 이송요청).
- 데이터 구조가 바뀌면 Form Interface도 함께 — 활성화·재생성.
- 운영 양식 변경은 **테스트 출력**(샘플 데이터)으로 검증 후 적용.

## 흔한 실수와 점검

- **운영 직접 변경** → 양식도 코드처럼 이송으로. 직접 수정 금지.
- **테스트 없이 배포** → 깨진 양식이 고객에게 나간다. 샘플 출력 검증.


## 🧩 도전 과제 (서브퀘스트)

> 선택 과제 — 예매 확인서(티켓)를 출력한다.

1. 예매 확인서 양식을 **Adobe Forms**(또는 Smart Forms)로 만든다(공연·회차·좌석·예매자).
2. 생성된 **PDF(XSTRING)** 를 파일로 저장하거나 메일 첨부한다(BIN).
3. (심화) 예매 확정 시 **Output Control**로 확인서가 자동 출력되게 결정 규칙을 설계한다.

힌트: L01~L02(Forms)·L03(Output Control)·L04(PDF 다운로드)·L05(오류/변경).
