## PDF 네이티브 양식 — Adobe Forms

[[Adobe Forms]](`SFP`)는 Adobe(LiveCycle) 기반의 현대 양식으로, **PDF 출력·인터랙티브
폼**에 강하다. 신규는 보통 Smart Forms보다 Adobe Forms를 쓴다.

## 구조

- **Interface** — 양식에 넘길 데이터 정의(Smart Forms의 Form Interface와 유사).
- **Form Context** — 인터페이스 데이터를 레이아웃에 연결.
- **Layout** — Adobe LiveCycle Designer로 디자인.
- 호출은 생성 함수(`FP_FUNCTION_MODULE_NAME`)를 얻어 `FP_JOB_OPEN`/`CLOSE` 사이에서.

```abap
CALL FUNCTION 'FP_JOB_OPEN' CHANGING ie_outputparams = ls_out.
CALL FUNCTION lv_fm EXPORTING is_booking = ls_booking
  IMPORTING /1bcdwb/formoutput = ls_formout.   " PDF 바이트 포함
CALL FUNCTION 'FP_JOB_CLOSE'.
```

## 흔한 실수와 점검

- **ADS 미설정** → Adobe Document Services가 있어야 렌더링된다.
- **JOB_OPEN/CLOSE 짝** → 출력은 OPEN~CLOSE 사이에서.

---

두 양식 도구를 봤다. 언제 어떤 양식이 자동으로 나가나 — Output Control → 다음 레슨.
