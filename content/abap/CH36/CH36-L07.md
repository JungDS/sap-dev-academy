## Draft — 미저장 변경 보관

L04의 `draft table`로, 사용자가 입력하다 만 변경이 **초안**으로 보관된다. 다른 기기에서
이어 하거나, 실수로 닫아도 복구된다. managed RAP이 초안 처리를 대부분 자동으로 해 준다.

## 권한 — DCL과 인스턴스 권한

- **DCL**(CH22-L06)로 "누가 어떤 공연의 예매를 볼지" 행 단위 제어.
- BDEF의 **authorization**(global/instance)으로 동작별 권한(예매 생성·취소 권한).

```bdef
define behavior for ZI_Booking
{
  authorization master ( global, instance );
  // ... determinations / validations / actions ...
}
```

## 운영 마무리

- **ATC·ABAP Unit** 통과(CH35) → **이송**(DEV→QAS→PRD).
- **Application Log**로 추적, **Clean Core**(Released API만, CH23-L08·CH29).

## 🎓 여정의 끝 — 그리고 시작

정훈영의 첫 `WRITE 'Hello'`(CH01)에서 출발해, 변수·타입·구조·테이블·SQL을 익히고,
화면과 ALV를 만들고, 모던 문법·OO·CDS·RAP으로 **콘서트 예매 풀스택 앱**을 완성했다.
Track-2에서 트랜잭션·동시성·인터페이스·성능·운영까지 — **현업 개발자의 한 바퀴**를 돌았다.

> 여기서 끝이 아니다. 실제 프로젝트, UI5/Fiori 심화, 새 릴리스의 기능 — 배운 토대 위에
> 계속 쌓아 가면 된다. **0부터 시작한 당신은, 이제 개발자다.** 🚀

## 흔한 실수와 점검
