## 기본값 — Determination

생성 시 상태·감사필드를 자동 채운다(CH24 감사 stamp의 RAP 버전).

```abap
METHOD setDefaults.
  READ ENTITIES OF zi_booking IN LOCAL MODE
    ENTITY Booking ALL FIELDS WITH CORRESPONDING #( keys ) RESULT DATA(bks).
  MODIFY ENTITIES OF zi_booking IN LOCAL MODE
    ENTITY Booking UPDATE FIELDS ( status created_by )
    WITH VALUE #( FOR b IN bks (
      %tky        = b-%tky
      status      = 'N'
      created_by  = sy-uname ) ).
ENDMETHOD.
```

## 정원 검증 — Validation

좌석 합계가 회차 정원을 넘으면 거부한다(`ZCX_FULLY_BOOKED`의 RAP 표현).

```abap
METHOD validateCapacity.
  " 키별 예매 + 회차 정원·기존 합계를 읽어 비교
  " 초과 시: APPEND ... TO failed-booking / reported-booking (메시지)
ENDMETHOD.
```

## 취소 — Action

```abap
METHOD cancelBooking.
  MODIFY ENTITIES OF zi_booking IN LOCAL MODE
    ENTITY Booking UPDATE FIELDS ( status )
    WITH VALUE #( FOR k IN keys ( %tky = k-%tky status = 'C' ) ).
  " result에 변경된 인스턴스 반환
ENDMETHOD.
```

- 검증 실패는 **failed/reported**로 알려 RAP이 저장을 막는다(CH23-L07).
- 액션은 사용자가 버튼으로 실행(취소).

## 흔한 실수와 점검

- **IN LOCAL MODE 누락** → 핸들러 내부 읽기/수정은 보통 `IN LOCAL MODE`.
- **failed/reported 누락** → 검증 실패를 안 알리면 안 막힌다.
