## 소비 모델 — ZC_ + UI

기반을 투영하고, Fiori가 읽을 **@UI 주석**으로 화면을 선언한다.

```cds
@Metadata.allowExtensions: true
define root view entity ZC_Booking
  provider contract transactional_query
  as projection on ZI_Booking
{
  @UI.facet: [{ purpose: #STANDARD, type: #IDENTIFICATION_REFERENCE, position: 10 }]
  key booking_id,

  @UI: { lineItem: [{ position: 10 }], identification: [{ position: 10 }] }
  @EndUserText.label: '공연'
      concert_id,

  @UI: { lineItem: [{ position: 20 }] }
      customer,

  @UI: { lineItem: [{ position: 30 }] }
      seats,

  @UI: { lineItem: [{ position: 40 }] }
      status,
      _Concert,
      _Perf
}
```

- `@UI.lineItem` — 목록(List Report) 컬럼.
- `@UI.identification`/`facet` — 상세(Object Page) 폼.
- 주석이 많으면 Metadata Extension(CH22-L05)으로 분리.

## 흔한 실수와 점검

- **UI 주석 위치** → lineItem(목록)·identification(상세)을 구분.
- **contract 누락** → 트랜잭션 소비엔 `transactional_query`.
