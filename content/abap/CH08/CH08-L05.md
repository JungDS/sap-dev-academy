## 조건을 정교하게

`WHERE`로 필요한 행만 추린다. 비교 대상이 ABAP 변수면 classic은 **그대로** 적는다
(현대식 escape는 CH19).

```abap
DATA gv_carr TYPE sflight-carrid VALUE 'KE'.

SELECT * FROM sflight INTO TABLE gt_flight
  WHERE carrid = gv_carr.        " classic: host 변수 그대로
```

## 비교·논리 연산자

`=`(EQ) · `<>`(NE) · `<`(LT) · `>`(GT) · `<=`(LE) · `>=`(GE), 그리고 `AND`/`OR`/`NOT`·괄호.

## 범위·목록·패턴

```abap
SELECT * FROM sflight INTO TABLE gt_flight
  WHERE fldate BETWEEN '20260101' AND '20261231'   " 범위
    AND carrid IN ('KE', 'LH', 'AA')               " 목록
    AND connid LIKE '07%'.                          " 패턴
```

- `BETWEEN a AND b` — a~b 범위.
- `IN ( ... )` — 목록 중 하나.
- `LIKE` — 패턴. `%`=임의 길이, `_`=한 글자. (`'07%'` = 07로 시작)

## IS NULL

값이 채워지지 않은(NULL) 행을 가린다.

```abap
SELECT * FROM spfli INTO TABLE gt_spfli WHERE fltime IS NULL.
```

## 흔한 실수와 점검

- **modern escape 습관** → 이 챕터는 classic이라 host 변수를 그대로 쓴다(현대식은 CH19).
- **LIKE wildcard 혼동** → `%`=여러 글자, `_`=한 글자.
- **단일값 vs 범위** → 범위·다중은 `IN`/`BETWEEN`. SELECT-OPTIONS의 Range는 CH12.
