## 딱 한 건이면 충분할 때

특정 사람 한 명, 특정 주문 하나처럼 **키로 단건**만 필요할 때가 많다. 이럴 땐
[[SELECT SINGLE]]이 맞다.

## SELECT SINGLE — 한 행을 Work Area로

```abap
DATA: lv_id     TYPE ztperson-persid VALUE '0001',
      ls_person TYPE ztperson.

SELECT SINGLE * FROM ztperson
  INTO ls_person
  WHERE persid = lv_id.

IF sy-subrc = 0.
  WRITE: / ls_person-name, ls_person-age.
ELSE.
  WRITE: / '없음'.
ENDIF.
```

- 결과는 [[Work Area]] 한 건(`INTO ls_`, `INTO TABLE` 아님).
- **키 전체**로 조회하면 정확히 한 행이 보장된다.
- `sy-subrc = 0`이면 찾음, `4`면 없음 — **반드시 확인**.

## 필요한 컬럼만 (classic)

```abap
SELECT SINGLE name age          " 공백 구분(classic)
  FROM ztperson
  INTO CORRESPONDING FIELDS OF ls_person
  WHERE persid = lv_id.
```

## 흔한 실수와 점검

- **부분 키로 SINGLE** → 여러 행 중 임의의 한 행이 온다. 단건은 **전체 키**로.
- **sy-subrc 미확인** → 없음(4)일 때 옛 [[Work Area]] 값을 쓰는 버그.
- **INTO TABLE과 혼동** → SINGLE은 한 건이라 `INTO ls_`(구조체).
