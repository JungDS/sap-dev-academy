## 결과가 한 건? 여러 건?

읽는 결과의 모양에 따라 SELECT 형태가 다르다.

## SELECT SINGLE — 키로 딱 한 건

특정 한 행이 필요하면 [[SELECT SINGLE]]. 결과는 [[Work Area]] 하나(`INTO ls_`).

```abap
DATA ls_flight TYPE sflight.

SELECT SINGLE * FROM sflight
  INTO ls_flight
  WHERE carrid = 'KE' AND connid = '0701' AND fldate = '20260623'.

IF sy-subrc = 0.
  WRITE: / ls_flight-seatsocc, '/', ls_flight-seatsmax.
ENDIF.
```

- **키 전체**로 조회하면 정확히 한 행이 보장된다.

## INTO TABLE — 여러 건 한 번에

여러 행은 `INTO TABLE`로 내부 테이블에 통째로 담는다(= Array Fetch, 빠르다).

```abap
SELECT * FROM sflight INTO TABLE lt_flight WHERE carrid = 'KE'.
```

## SELECT … ENDSELECT — 한 행씩 (지양)

`INTO ls_`로 받으면 한 행씩 반복 처리하는 `SELECT…ENDSELECT` 루프가 된다.

```abap
SELECT * FROM sflight INTO ls_flight WHERE carrid = 'KE'.
  WRITE: / ls_flight-fldate.
ENDSELECT.
```

> ⚠️ `SELECT…ENDSELECT`는 DB를 여러 번 왕복해 **느리다.** 가능하면 `INTO TABLE`로
> 한 번에 가져와 `LOOP`로 처리한다. **중첩 SELECT는 절대 금지**(성능 최악).

## UP TO n ROWS — 줄여 읽기

처음 몇 건만 필요하면 행 수를 제한한다.

```abap
SELECT * FROM sflight INTO TABLE lt_flight UP TO 10 ROWS.
```

## 흔한 실수와 점검

- **부분 키로 SINGLE** → 임의의 한 행이 온다. 단건은 **전체 키**로.
- **ENDSELECT 남용** → 대량은 `INTO TABLE`. 중첩 SELECT 금지.
- **sy-subrc 미확인** → SINGLE도 없음(4)일 수 있다.

---

형태는 골랐다. 그런데 결과를 **어디에 어떻게** 담을지(INTO 대상)도 여러 가지다 → 다음 레슨.
