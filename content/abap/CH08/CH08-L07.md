## 결과가 없을 수도 있다

조회는 늘 성공하지 않는다. 우리 `ZGUGUDAN`에는 2·3단만 넣었으니, **5단**을 조회하면
한 건도 안 나온다. 이때 빈 화면만 두면 사용자는 영문을 모른다 — **알려 줘야** 한다.

## sy-subrc로 분기

```abap
SELECT * FROM zgugudan INTO TABLE gt_gugu WHERE dan = 5.

IF sy-subrc <> 0.
  WRITE: / '5단 데이터가 없습니다.'.
ELSE.
  LOOP AT gt_gugu INTO gs_line.
    WRITE: / gs_line-dan, 'x', gs_line-mul, '=', gs_line-result.
  ENDLOOP.
ENDIF.
```

## MESSAGE — 사용자에게 알리기 (맛보기)

`WRITE`로 화면에 찍는 것보다, **MESSAGE**로 알림을 띄우는 게 더 적절할 때가 많다.

```abap
IF sy-subrc <> 0.
  MESSAGE '해당 단의 데이터가 없습니다.' TYPE 'S'.   " S = 상태바 알림
ENDIF.
```

- `TYPE 'S'`(Status, 상태바)·`TYPE 'I'`(Information, 팝업)가 가벼운 알림이다.
- 여기선 **맛보기**만 — 메시지 타입 전체(I/S/W/E/A/X)와 메시지 클래스는 **CH15**에서
  정식으로 배운다.

## 흔한 실수와 점검

- **결과 미확인 진행** → 빈 테이블로 다음 로직을 돌리면 사고. 항상 `sy-subrc` 분기.
- **모든 걸 E로** → 단순 "없음"은 S/I가 적절. E(오류)는 흐름을 막는다(CH15).
