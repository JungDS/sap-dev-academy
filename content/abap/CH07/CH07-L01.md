## 저장한 걸 다시 읽고 싶다

CH06에서 데이터를 [[Transparent Table]]에 영구 저장했다. 이제 프로그램에서 그
값을 **읽어** 와야 쓸모가 있다. DB에서 읽는 언어가 [[Open SQL]]이다.

> 이 챕터는 **classic 구문**으로 쓴다(필드를 공백으로 나열, host 변수 그대로).
> 현대 `@`/`@DATA()` 구문은 CH18에서 별도로 배운다 — 지금 섞지 않는다.

## SELECT … INTO TABLE — 여러 행 읽기

DB 테이블의 행들을 [[Internal Table]]에 한 번에 담는다.

```abap
DATA lt_person TYPE TABLE OF ztperson.

SELECT * FROM ztperson
  INTO TABLE lt_person.

IF sy-subrc = 0.
  WRITE: / lines( lt_person ), '건 읽음'.
ENDIF.
```

- `INTO TABLE`은 결과 전체를 내부 테이블에 채운다.
- 읽은 행 수는 `sy-dbcnt`, 성공 여부는 `sy-subrc`(0 = 1건 이상).

## 필요한 컬럼만 (classic — 공백 구분)

전체(`*`)보다 필요한 컬럼만 읽는 게 빠르다. **classic은 필드를 공백으로** 나열한다.

```abap
DATA lt_person TYPE TABLE OF ztperson.

SELECT persid name age          " 콤마 없이 공백 구분 (classic)
  FROM ztperson
  INTO CORRESPONDING FIELDS OF TABLE lt_person.
```

- 대상 구조의 **같은 이름 필드**에 담으려면 `INTO CORRESPONDING FIELDS OF TABLE`.

## 흔한 실수와 점검

- **콤마로 필드 나열** → classic에선 오류. 콤마·`@`는 CH18(modern)부터.
- **결과 미확인** → 빈 결과일 수 있다. `sy-subrc`/`lines( )`로 확인.
- **큰 테이블 전체 SELECT** → 성능 주의. 다음 레슨의 WHERE로 줄인다.

---

전부 읽어왔다. 그런데 보통은 **조건에 맞는 행만** 필요하다 → WHERE(다음 레슨).
