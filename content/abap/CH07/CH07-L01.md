## 저장한 걸 다시 읽고 싶다

CH06에서 데이터를 [[Transparent Table]](디스크에 영구 저장되는 DB 테이블)에 넣었다.
이제 프로그램에서 그 값을 **읽어** 와야 쓸모가 있다. DB에서 읽는 언어가 [[Open SQL]]이다.

> 📌 이 챕터는 **classic 구문**으로 쓴다 — 필드를 **공백**으로 나열하고, 비교 값에 `@`를 붙이지 않는다.
> 현대 `@`/`@DATA()` 구문은 **CH18**에서 별도로 배운다. 지금 섞으면 경계가 무너진다.

## SELECT … INTO TABLE — 여러 행 읽기

DB 테이블의 행들을 [[Internal Table]](메모리에 잠깐 두는 표)에 한 번에 담는다.

```abap
DATA lt_person TYPE TABLE OF ztperson.

SELECT * FROM ztperson
  INTO TABLE lt_person.

IF sy-subrc = 0.
  WRITE: / lines( lt_person ), '건 읽음'.
ENDIF.
```

- `INTO TABLE`은 결과 **전체**를 내부 테이블에 채운다.
- 성공 여부는 [[sy-subrc]](0 = 1건 이상), 읽은 행 수는 `sy-dbcnt`.

## 직접 조회해 보자

조회는 **두 가지를 고르는 일**이다 — 어떤 **컬럼**을 읽을지(projection), 어떤 **행**만 읽을지(WHERE, selection).
아래 시뮬레이터에서 컬럼과 조건을 바꾸면 **classic SELECT 코드가 실시간으로** 만들어지고,
**▶ 실행(F8)** 을 누르면 `ZTPERSON`에서 조건에 맞는 행이 `LT_PERSON`으로 담긴다.

::embed code-learning/select-query-simulator.html | SELECT 조회 시뮬레이터 — 컬럼·조건을 바꿔 실행 | 620::

> 🖱️ 예제 칩 **"컬럼만 (name age)"** 을 누르면 코드가 `SELECT name age …`로 바뀌고 결과 표의 **열이 줄어든다**.
> **"조건 (age >= 33)"** 은 원본 테이블에서 **맞는 행만 초록색**으로 강조되는 걸 볼 수 있다.

## 필요한 컬럼만 (classic — 공백 구분)

전체(`*`)보다 필요한 컬럼만 읽는 게 빠르다. **classic은 필드를 공백으로** 나열한다(콤마 ✕).

```abap
SELECT persid name age          " 콤마 없이 공백 구분 (classic)
  FROM ztperson
  INTO CORRESPONDING FIELDS OF TABLE lt_person.
```

- 컬럼 일부만 읽을 땐, 같은 이름 필드에 맞춰 담는 **`INTO CORRESPONDING FIELDS OF TABLE`** 을 쓴다.
- 시뮬레이터에서 "컬럼 선택"을 켜면 `INTO TABLE` → `INTO CORRESPONDING FIELDS OF TABLE`로 **자동으로 바뀌는** 걸 확인해 보라.

## 결과는 반드시 확인한다

조회가 **항상 성공하는 건 아니다.** 조건에 맞는 행이 하나도 없으면 [[sy-subrc]]가 `4`가 된다.

| 결과 | sy-subrc | sy-dbcnt |
|---|---|---|
| 1건 이상 읽음 | `0` | 읽은 행 수 |
| 한 건도 없음 | `4` | `0` |

> 🔎 시뮬레이터에서 `WHERE age > 100` 처럼 아무도 안 맞는 조건을 주면 **`sy-subrc = 4`** 가 뜬다.
> 결과를 안 보고 다음 로직으로 가면, 빈 테이블을 가지고 일하다 사고가 난다.

## 흔한 실수와 점검

> ⚠️ **콤마로 필드 나열** — `SELECT persid, name`은 classic에서 오류. 콤마·`@`는 CH18(modern)부터.

> ⚠️ **결과 미확인** — 빈 결과일 수 있다. 항상 [[sy-subrc]]·`lines( )`로 확인.

> ⚠️ **큰 테이블 전체 SELECT** — `SELECT *`로 다 읽으면 성능 위험. 다음 레슨의 **WHERE**로 줄인다.

## 정리

- 저장된 데이터를 읽는 언어가 [[Open SQL]] — 이 챕터는 **classic**(공백 구분·`@` 없음).
- `SELECT … INTO TABLE`로 여러 행을 [[Internal Table]]에 담는다.
- 컬럼 일부만 읽으면 `INTO CORRESPONDING FIELDS OF TABLE`.
- 조회 결과는 **반드시 [[sy-subrc]]** 로 확인한다(없으면 `4`).

---

전부 읽어왔다. 그런데 보통은 **조건에 맞는 행만** 필요하다 → WHERE(다음 레슨).
