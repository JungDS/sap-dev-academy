## 빈 테이블 채우기

`ZGUGUDAN`을 만들었으니 값을 넣어야 조회도 의미가 있다. 학습 단계에선 SE11의
**Create Entries**로 한 건씩 손으로 입력한다.

## SE11 Create Entries — 한 건씩 손입력

`SE11` → `ZGUGUDAN` 표시 → 메뉴 **Utilities → Table Contents → Create Entries**.
입력 화면이 뜨면 DAN·MUL·RESULT를 채우고 저장한다.

2단과 3단을 손으로 넣어 보자 — 2×1 … 2×9, 3×1 … 3×9, **총 18행**이다.

```text
DAN  MUL  RESULT
 2    1     2
 2    2     4
 ...
 3    9    27
```

::embed CH07-L02-S01 | Create Entries 입력 훈련기 — 손입력·키 중복·클라이언트 분리::

> 😵 18행을 손으로 넣는 것도 지친다. [Chapter 06 · 구구단 캡스톤](CH06-L06.html)에서는 같은
> 걸 **프로그램으로 81행**을 순식간에 만들었다. "손입력은 학습·소량 확인용, 대량은
> 프로그램"이라는 감을 잡아 둔다.

## 넣은 값 다시 보기 — Table Contents

`SE11` → `ZGUGUDAN` → **Utilities → Table Contents → Display**로 방금 넣은 18행을
조회한다. 조건을 주면 해당 행만 나온다.

## 키 중복은 안 된다

키가 같은 행을 두 번 넣을 수 없다. `MANDT + DAN + MUL`이 키라면 같은 (단, 곱수)
조합은 하나뿐이다.

## 흔한 실수와 점검

- **입력 권한/설정** → 교육 시스템 안내를 따른다. 활성화된 테이블이라야 입력된다.
- **클라이언트 혼동** → 다른 Client에서 넣은 데이터는 안 보인다([[MANDT]] 분리).
- **키 중복** → 같은 키 조합 재입력은 거부된다.
- **DB가 값까지 맞춰 줄 거란 착각** → 키는 **중복만** 막는다. `2×2`인데 `RESULT=5`처럼
  계산이 틀린 값도 손입력으로는 들어간다. 값이 업무적으로 맞는지는 별도 검증의 몫이다.
