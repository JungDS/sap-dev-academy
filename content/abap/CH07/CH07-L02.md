## 다 읽을 필요는 없다

테이블이 커지면 전체를 읽는 건 낭비다. **조건에 맞는 행만** 가져오자 — `WHERE`다.

## WHERE — host 변수 (classic)

비교 대상이 ABAP 변수면 **그대로** 적는다(classic). 현대의 `@` escape는 아직 안 쓴다.

```abap
DATA: lv_age    TYPE i VALUE 20,
      lt_person TYPE TABLE OF ztperson.

SELECT * FROM ztperson
  INTO TABLE lt_person
  WHERE age >= lv_age.          " classic: host 변수 lv_age 그대로
```

> CH18(modern)에서는 같은 조건을 `WHERE age >= @lv_age`처럼 `@`로 escape한다.
> 지금은 classic이라 `@` 없이 쓴다 — 섞으면 경계가 무너진다.

## 자주 쓰는 조건

```abap
SELECT * FROM ztperson INTO TABLE lt_person
  WHERE age >= lv_age
    AND name = lv_name.
```

- `AND` / `OR`, 비교 연산자(`=`, `<>`, `>=` …), `LIKE`, `BETWEEN`, `IN` 등.
- 범위·다중 값 조건은 CH11(SELECT-OPTIONS)에서 본격적으로 다룬다.

## 흔한 실수와 점검

- **`@lv_age` 사용** → 이 챕터는 classic이라 `@` 없이. modern은 CH18.
- **WHERE 없이 큰 테이블** → 성능·메모리 위험. 필요한 만큼만 조건으로.
- **대소문자/공백** → 문자 비교는 저장된 형식과 정확히 일치해야 한다.
