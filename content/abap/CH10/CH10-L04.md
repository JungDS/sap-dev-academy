## 처음부터 끝까지 한 번에

지금까지 배운 걸 모아 **DB 조회 → 표 출력**까지 하나의 프로그램으로 완성한다.
classic SELECT(CH07) + [[SALV]](CH10)의 결합이다.

```abap
REPORT zsalv_person.

DATA: lt_person TYPE TABLE OF ztperson,
      lo_alv    TYPE REF TO cl_salv_table.

START-OF-SELECTION.
  " 1) 데이터 조회 (classic Open SQL)
  SELECT * FROM ztperson INTO TABLE lt_person.

  " 2) SALV 객체 생성
  TRY.
      cl_salv_table=>factory(
        IMPORTING r_salv_table = lo_alv
        CHANGING  t_table      = lt_person ).

      " 3) 표준 기능 + 표시
      lo_alv->get_functions( )->set_all( abap_true ).
      lo_alv->display( ).
    CATCH cx_salv_msg.
      MESSAGE 'ALV 생성 실패' TYPE 'I'.
  ENDTRY.
```

실행(F8)하면 정렬·필터·엑셀 내보내기가 되는 **표 리포트**가 뜬다. 첫 "쓸 만한"
리포트의 완성이다.

## 흔한 실수와 점검

- **조회 결과 미확인** → 빈 테이블이면 빈 표. `lines( )`로 확인 후 메시지도 가능.
- **classic/modern 혼동** → 이 시점 SELECT는 **classic**(CH18 전). `@` 안 쓴다.
