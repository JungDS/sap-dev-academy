## 값을 주고받기

서브루틴이 바깥 값을 받아 처리하고 결과를 돌려주려면 **[[파라미터]]**가 필요하다.
`FORM`에서는 `USING`과 `CHANGING`으로 한다.

## USING(입력) / CHANGING(입출력)

```abap
FORM add_tax
  USING    iv_amount TYPE p
  CHANGING cv_result TYPE p.
  cv_result = iv_amount * '1.1'.
ENDFORM.

START-OF-SELECTION.
  DATA lv_out TYPE p LENGTH 8 DECIMALS 2.
  PERFORM add_tax USING 1000 CHANGING lv_out.
  WRITE lv_out.
```

- `USING` — 주로 **입력**값(관례). `CHANGING` — **돌려줄** 값.
- 접두어 관례: `iv_`(input), `cv_`(changing).

> 엄밀히 `USING`도 참조로 넘어가 바뀔 수 있다. **값 보호**가 필요하면
> `USING value(iv_x)`로 값 전달(by value). 관례상 입력은 USING, 출력은 CHANGING로
> **의도를 표현**한다.

## 세 가지 전달 방식

| 방식 | 문법 | 뜻 |
|---|---|---|
| Call by Reference | `USING iv_x` · `CHANGING cv_x` | 원본을 가리킴(빠름, 바뀔 수 있음) |
| Call by Value | `USING value(iv_x)` | 값 복사 — 원본 보호(입력 전용) |
| Call by Value and Result | `CHANGING value(cv_x)` | 복사해 처리 후 **정상 종료 시** 되돌려씀 |

- **입력만** → `USING value(...)`로 원본을 보호한다.
- **돌려줄 값** → 보통 `CHANGING`(reference).
- **타입 명시** → `USING iv_x TYPE p`처럼 타입을 적으면 안전하다(권장).

## 빠져나오기 · 값 유지 — RETURN · STATICS

- `RETURN` — 서브루틴(또는 메서드)을 **즉시 빠져나간다**(남은 코드 건너뜀).
- `STATICS` — 서브루틴 안에서 선언하되 **호출이 끝나도 값이 유지**되어 다음 호출에
  누적된다(일반 지역변수 `DATA`는 호출마다 사라진다).

> 메서드/함수에는 파라미터를 **선택적으로**(`OPTIONAL`)·**기본값**(`DEFAULT`)으로 줄
> 수도 있다(호출 시 일부를 생략하게). 메서드 본격 설계는 [Chapter 20 · OO 기초](CH21-L01.html)에서 배운다.

## 직접 해 보기

전달 방식을 바꿔 가며 FORM이 값을 바꿨을 때 **원본이 보호되는지/바뀌는지** 비교해 보자. `RETURN`(조기 종료)·`STATICS`(값 유지)도 직접 눌러 보자.

## 흔한 실수와 점검

- **USING/CHANGING 순서·타입 불일치** → 호출과 정의의 파라미터가 맞아야 한다.
- **출력을 USING으로** → 의도가 흐려진다. 돌려줄 값은 CHANGING.
