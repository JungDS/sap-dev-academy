## 왜 여기서 시작하나

코드를 한 줄 쓰기도 전에, 우리는 먼저 **SAP 시스템에 들어가야** 한다.
ABAP은 내 PC가 아니라 **서버(애플리케이션 서버)** 위에서 돌아가고,
그 서버에 접속하는 창구가 바로 [[SAPGUI]]다. 이 레슨의 목표는 단 하나 —
**로그온해서 개발을 시작할 화면 앞에 앉는 것.**

## SAP GUI와 접속 구조

[[SAPGUI]]는 서버에 연결하는 데스크톱 클라이언트다. 실행하면 "SAP Logon"
창이 뜨고, 접속할 시스템 목록이 보인다. 하나의 시스템은 보통
**System ID(SID)** 와 접속 정보로 식별된다.

- **Application Server / Instance** — 어느 서버에 붙을지
- **System ID (SID)** — 시스템을 구분하는 3글자 코드 (예: `DEV`, `NPL`)

접속 항목이 없다면 관리자(또는 교육용 시스템 안내)에 따라 새 항목을 등록한다.

## 로그온하기

접속 항목을 더블클릭하면 로그온 화면이 나온다. 네 가지를 입력한다.

- **Client** — 같은 시스템 안의 독립된 작업 공간(3자리 숫자, 예: `100`)
- **User** — 사용자 ID
- **Password** — 비밀번호 (최초 로그온 시 변경 요구될 수 있음)
- **Language** — `KO`(한국어) 또는 `EN`(영어)

::embed CH01-L01-S01 | SAP 로그온 화면::

> **Client가 뭐죠?** 한 시스템 안에 여러 개의 분리된 환경이 있다고 보면 된다.
> Client 100과 200은 데이터가 서로 보이지 않는 별개의 방이다. 로그온할 때
> **어느 방에 들어갈지** 고르는 것.

## SAP Easy Access — 첫 화면

로그온에 성공하면 [[SAP Easy Access]] 메뉴가 나타난다. 여기서 두 가지만
기억하면 된다.

::embed CH01-L01-S02 | SAP Easy Access 첫 화면::

1. **메뉴 트리** — 기능들이 폴더처럼 정리돼 있다. 하지만 우리는 이걸 잘
   안 쓴다(느리다).
2. **명령창(Command field)** — 화면 **왼쪽 위**의 입력칸. 여기에 코드를
   쳐서 어디든 바로 간다. 다음 레슨의 주인공이다.

## 흔한 실수와 점검

- **로그온이 안 돼요** → Client 번호가 맞는지, Language가 지원되는지 확인.
- **비밀번호 잠김** → 관리자에게 초기화 요청. 교육 시스템은 보통 안내 문서가 있다.
- **화면이 영어로 떠요** → Language에 `KO`를 넣고 다시 로그온.
