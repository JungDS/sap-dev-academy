## 화면이 아직 비어 있다

환경도 갖췄고([[REPORT]] 한 줄로) 프로그램 뼈대도 세웠다. 그런데 실행하면 **화면은 여전히 백지**다.
지금 가장 하고 싶은 건 딱 하나 — **"뭐라도 눈으로 보고 싶다."** 그 한 줄을 책임지는 게 [[WRITE]]다.
이번 레슨에서 처음으로 개발 루프(작성 → 활성화 → 실행)를 **끝까지 한 바퀴** 돌려 본다.

> 🎯 다른 언어에서 `print("Hello")` 한 줄로 시작했던 그 순간을, ABAP에서도 만든다.

## WRITE — ABAP의 print

[[WRITE]]는 **값을 화면(리스트)에 찍는 가장 기본 출력문**이다. 다른 언어의 출력문과 같은 역할이다.

| 언어 | 출력문 |
|---|---|
| Python | `print('Hello')` |
| JavaScript | `console.log('Hello')` |
| Java | `System.out.println("Hello")` |
| **ABAP** | **`WRITE 'Hello'.`** |

```abap
REPORT zhello.

WRITE 'Hello, ABAP!'.
```

작성 → 저장 → **활성화(Ctrl+F3)** → **실행(F8)**. 그러면 리스트 화면에 `Hello, ABAP!`이 찍힌다.

## 직접 출력해 보자

말로 듣는 것보다 **직접 고쳐 실행**하는 게 빠르다. 아래 에디터의 코드를 바꾸고 **▶ 실행(F8)** 을 누르면
SAP 리스트 화면에 결과가 그려진다. 위쪽 **예제 칩**으로 네 가지 출력 패턴을 갈아 끼우며,
`/`를 넣고 빼면 줄이 어떻게 달라지는지 눈으로 확인해 보자.

::embed code-learning/write-output-simulator.html | WRITE 출력 시뮬레이터 — 코드를 고쳐 F8로 실행 | 500::

> 🖱️ 예제 칩 **"이어 붙기 (/ 없음)"** 와 **"줄바꿈 (/)"** 을 번갈아 눌러 보면, 같은 세 줄의 코드가
> **한 줄로 붙는지 / 세 줄로 나뉘는지**가 한눈에 비교된다.

## 같은 줄? 새 줄? — `/`의 규칙

[[WRITE]]는 기본적으로 **앞 출력에 이어 같은 줄**에 찍는다. 줄을 바꾸려면 값 앞에 `/`를 붙인다.

| 코드 | 결과 |
|---|---|
| `WRITE 'A'.`<br>`WRITE 'B'.` | `A B` (같은 줄에 이어 붙음) |
| `WRITE 'A'.`<br>`WRITE / 'B'.` | `A`<br>`B` (`/`에서 새 줄) |

```abap
WRITE 'first line'.
WRITE / 'second line'.
```

> 💡 `/`는 "**여기서 줄을 바꿔라**"는 신호다. 빠뜨리면 다음 출력이 옆에 붙어 버린다 — 위 시뮬레이터에서
> `/`를 지웠다 살렸다 해 보면 바로 체감된다.

## 여러 값 한 번에 — 콜론(`:`) 체인

같은 키워드를 반복하기 귀찮을 때, 콜론(`:`) 뒤에 콤마(`,`)로 나열하면 **한 문장처럼 묶어** 쓸 수 있다.

```abap
WRITE: / 'A', 'B', 'C'.
```

이 한 줄은 아래 세 문장과 **완전히 같다**. (`/`는 그게 붙은 값 하나에만 적용된다.)

```abap
WRITE / 'A'.
WRITE 'B'.
WRITE 'C'.
```

<details>
<summary>콜론 체인은 WRITE 전용이 아니다 (더 보기)</summary>

콜론(`:`) + 콤마 체인은 ABAP **공통 문법**이다. 앞으로 배울 `DATA:`, `PARAMETERS:` 등에서도
똑같이 "키워드 한 번 + 콤마로 나열"이 가능하다. 지금은 [[WRITE]]로 그 감만 잡아 두면 된다.
</details>

## 문자열 리터럴

작은따옴표로 감싼 `'...'`가 **문자(텍스트) [[리터럴]]** 이다 — 코드에 값을 **직접 적어 둔 것**.

```abap
WRITE '값을 직접 적은 것이 리터럴'.
```

지금은 출력할 값을 코드에 직접 박아 넣고 있다. 따옴표 안 글자가 그대로 화면에 나온다.
시뮬레이터에서 따옴표 없이 `lv_name` 같은 걸 적으면 "변수"로 취급되는데 — 그건 다음 챕터의 이야기다.

## 흔한 실수와 점검

세 가지 모두 위 시뮬레이터에서 **직접 재현**해 볼 수 있다(에러 메시지가 그대로 뜬다).

> ⚠️ **따옴표 짝 안 맞음** — `'`를 열고 안 닫으면 문자열이 끝나지 않아 오류. (`WRITE 'oops.` 실행해 보기)

> ⚠️ **줄바꿈이 안 됨** — `/`를 빠뜨렸다. 새 줄로 내리려면 `WRITE / ...`.

> ⚠️ **마침표 누락** — 앞 레슨의 그 규칙. 여전히 1번 단골 에러다. 모든 문장은 `.`으로 끝난다.

## 정리

- [[WRITE]]는 ABAP의 기본 출력문 — 다른 언어의 `print`에 해당한다.
- **같은 줄**에 이어 붙는 게 기본, 줄을 바꾸려면 값 앞에 **`/`**.
- 콜론(`:`) + 콤마로 여러 [[WRITE]]를 **한 문장처럼 체인**할 수 있다.
- 따옴표로 감싼 `'...'`는 코드에 직접 적은 **[[리터럴]]**.
