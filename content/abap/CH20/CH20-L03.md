## 객체가 태어날 때

`NEW`로 객체를 만드는 순간 **자동으로** 실행되는 메서드가 [[CONSTRUCTOR]](생성자)다.
초기값을 받아 속성을 채우기 좋다.

```abap
CLASS zcl_booking_manager DEFINITION PUBLIC.
  PUBLIC SECTION.
    METHODS constructor IMPORTING iv_concert TYPE zconcert-concert_id.
  PRIVATE SECTION.
    DATA mv_capacity TYPE i.
ENDCLASS.

CLASS zcl_booking_manager IMPLEMENTATION.
  METHOD constructor.
    SELECT SINGLE capacity FROM zconcert
      WHERE concert_id = @iv_concert INTO @mv_capacity.   " 정원 로드
  ENDMETHOD.
ENDCLASS.
```

객체 생성 시 생성자 인자를 넘긴다.

```abap
DATA(lo_mgr) = NEW zcl_booking_manager( iv_concert = 'C001' ).
```

## 정적 생성자 — CLASS_CONSTRUCTOR

클래스가 **처음 쓰일 때 딱 한 번** 실행되는 `CLASS_CONSTRUCTOR`도 있다(공용 초기화).

## 흔한 실수와 점검

- **생성자에 무거운 작업** → 객체 생성이 느려진다. 꼭 필요한 초기화만.
- **인스턴스 vs 정적 생성자** → 객체마다=`constructor`, 클래스 1회=`class_constructor`.
