## 같은 규약, 다른 구현

"출력할 수 있다", "검증할 수 있다" 같은 **규약(약속)** 만 정하고, 실제 방법은 각
클래스가 알아서 — 그게 [[Interface]](인터페이스)다. 서로 다른 클래스를 **같은 방식으로
다룰** 수 있게 해 준다(다형성의 기반).

```abap
INTERFACE zif_printable.
  METHODS print.
ENDINTERFACE.
```

## 클래스에서 구현 — INTERFACES

```abap
CLASS zcl_booking DEFINITION PUBLIC.
  PUBLIC SECTION.
    INTERFACES zif_printable.       " 이 규약을 따른다
ENDCLASS.

CLASS zcl_booking IMPLEMENTATION.
  METHOD zif_printable~print.       " 인터페이스~메서드
    WRITE: / '예매 내역 출력'.
  ENDMETHOD.
ENDCLASS.
```

- `INTERFACES zif_printable` — 규약을 받아들인다.
- 구현은 `인터페이스~메서드` 이름으로.
- 호출: `lo_obj->zif_printable~print( )` 또는 인터페이스 참조로.

## 흔한 실수와 점검

- **메서드 구현 누락** → 인터페이스의 모든 메서드를 구현해야 활성화된다.
- **`~` 빼먹음** → 구현·호출 시 `인터페이스~메서드` 형식.
