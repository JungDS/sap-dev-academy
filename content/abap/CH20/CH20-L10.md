## 🎫 9단계 — 예약 로직을 클래스로

CH10-L07의 잔여석·예매판정을, 이제 **클래스**(CH20)로 캡슐화한다. 정원 초과는
**예외**(`ZCX_FULLY_BOOKED`)로 알린다.

```abap
CLASS zcl_booking_manager DEFINITION PUBLIC.
  PUBLIC SECTION.
    METHODS constructor IMPORTING iv_concert TYPE zconcert-concert_id
                                  iv_perf    TYPE zbooking-perf_no.
    METHODS remaining RETURNING VALUE(rv) TYPE i.
    METHODS book IMPORTING iv_seats TYPE i
                 RAISING   zcx_fully_booked.
  PRIVATE SECTION.
    DATA: mv_concert TYPE zconcert-concert_id,
          mv_perf    TYPE zbooking-perf_no,
          mv_cap     TYPE i.
ENDCLASS.

CLASS zcl_booking_manager IMPLEMENTATION.
  METHOD constructor.
    mv_concert = iv_concert.  mv_perf = iv_perf.
    SELECT SINGLE capacity FROM zconcert
      WHERE concert_id = @iv_concert INTO @mv_cap.
  ENDMETHOD.

  METHOD remaining.
    SELECT COALESCE( SUM( seats ), 0 ) FROM zbooking
      WHERE concert_id = @mv_concert AND perf_no = @mv_perf
        AND status <> 'C' INTO @DATA(lv_sum).
    rv = mv_cap - lv_sum.
  ENDMETHOD.

  METHOD book.
    IF iv_seats > me->remaining( ).
      RAISE EXCEPTION TYPE zcx_fully_booked.
    ENDIF.
    " 실제 INSERT는 CH24(DML)
  ENDMETHOD.
ENDCLASS.
```

호출부:

```abap
DATA(lo_mgr) = NEW zcl_booking_manager( iv_concert = 'C001' iv_perf = '01' ).
TRY.
    lo_mgr->book( iv_seats = 2 ).
  CATCH zcx_fully_booked.
    MESSAGE '매진입니다' TYPE 'I'.
ENDTRY.
```

## 직접 해 볼 과제

1. `ZCX_FULLY_BOOKED` 예외 클래스(`CX_STATIC_CHECK` 상속)를 만든다.
2. 정원보다 많은 좌석으로 `book`을 불러 예외가 잡히는지 확인.
3. (도전) `cancel( iv_booking )` 메서드를 더해 상태를 'C'로 바꾼다.

## 흔한 실수와 점검

- **예외 미선언** → `book`에 `RAISING zcx_fully_booked`를 선언해야 호출부가 잡는다.
- **상태 캡슐화** → 정원·회차는 PRIVATE, 외부는 메서드로만(CH20-L02).
