## 왜 — 누가 들을지 몰라도 알린다 📣

예약 관리자가 정원 초과를 *예외로 던지는 것*은 **실패 처리**다. 반면 "방금 매진됐다"는 실패가 아니라 **사건**이다. 누군가는 이 사건을 듣고 ALV 행을 빨갛게 칠하거나, 로그를 남기거나, 알림을 띄울 수 있다.

메서드 호출은 호출자가 대상을 알고 직접 부른다. **이벤트는 반대다** — 발생시키는 객체는 *누가 들을지 모른다*. 관심 있는 객체가 미리 handler를 등록해 두고, 사건이 나면 반응한다. (ALV의 더블클릭·버튼 같은 상호작용이 모두 이 구조 — 이후 ALV 챕터에서 그대로 쓴다.)

## 선언 · 발생 — EVENTS / RAISE EVENT 🔔

```abap
CLASS zcl_booking_manager DEFINITION PUBLIC CREATE PUBLIC.
  PUBLIC SECTION.
    EVENTS sold_out                         " 이벤트 선언(전달값 포함)
      EXPORTING VALUE(iv_concert) TYPE zconcert-concert_id
                VALUE(iv_perf)    TYPE zbooking-perf_no.
ENDCLASS.

METHOD book.
  IF iv_seats = me->remaining( ).
    RAISE EVENT sold_out                     " 사건 발생 + 값 전달
      EXPORTING iv_concert = mv_concert
                iv_perf    = mv_perf.
  ENDIF.
ENDMETHOD.
```

## 반응 등록 — Handler / SET HANDLER 🔌

반응하는 쪽은 `FOR EVENT`로 handler 메서드를 선언한다(전달값 + `sender`를 받음).

```abap
CLASS zcl_booking_monitor DEFINITION PUBLIC CREATE PUBLIC.
  PUBLIC SECTION.
    METHODS on_sold_out
      FOR EVENT sold_out OF zcl_booking_manager
      IMPORTING iv_concert iv_perf sender.
ENDCLASS.

CLASS zcl_booking_monitor IMPLEMENTATION.
  METHOD on_sold_out.
    MESSAGE |{ iv_concert }/{ iv_perf } 매진| TYPE 'I'.
  ENDMETHOD.
ENDCLASS.
```

마지막으로 `SET HANDLER`로 **어떤 발생 객체의 이벤트를 들을지** 연결한다.

```abap
DATA(lo_mgr)     = NEW zcl_booking_manager( iv_concert = 'C001' iv_perf = '01' ).
DATA(lo_monitor) = NEW zcl_booking_monitor( ).

SET HANDLER lo_monitor->on_sold_out FOR lo_mgr.   " 이 매니저의 sold_out을 듣는다
```

::embed CH20-L09-S01 | Event Wiring Panel (SET HANDLER 등록 · RAISE EVENT)::

흐름은 **네 단계**다: ① `EVENTS` 선언 → ② `FOR EVENT` handler 선언 → ③ `SET HANDLER`로 발생 객체에 등록 → ④ `RAISE EVENT`가 실행되면 등록된 handler가 **동기적으로**(다음 문장 전에) 호출된다.

## 흔한 실수와 점검 🚧

- **`SET HANDLER` 누락** → 이벤트가 나도 아무도 안 듣는다. 선언만으로 연결되지 않는다 — **등록**해야 한다.
- **이벤트를 예외처럼** → 실패를 중단·책임 전가하면 예외, 단지 *알리는* 것이면 이벤트. 둘은 용도가 다르다.
- **handler 순서 의존** → 여러 handler가 등록될 수 있고 **호출 순서는 정해지지 않는다**. 순서가 꼭 필요하면 한 handler 안에서 명시적으로 구성한다.
- **handler 객체 수명** → handler 객체도 참조로 등록된다. 불필요한 등록을 오래 유지하지 않는다.
