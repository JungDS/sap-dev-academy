## 객체가 "사건"을 알린다

"예약이 매진됐다" 같은 사건이 일어났을 때, 객체가 **이벤트**를 발생시키고 관심 있는
다른 객체가 **반응**하게 한다. ALV의 더블클릭·버튼 같은 상호작용이 모두 이 구조다
(CH21·CH27에서 활용).

## 이벤트 선언 · 발생 — EVENTS / RAISE EVENT

```abap
CLASS zcl_booking_manager DEFINITION PUBLIC.
  PUBLIC SECTION.
    EVENTS sold_out.                 " 이벤트 선언
ENDCLASS.

METHOD book.
  IF me->remaining( ) = 0.
    RAISE EVENT sold_out.            " 사건 발생
  ENDIF.
ENDMETHOD.
```

## 반응 등록 — Handler / SET HANDLER

```abap
CLASS zcl_monitor DEFINITION.
  PUBLIC SECTION.
    METHODS on_sold_out FOR EVENT sold_out OF zcl_booking_manager.
ENDCLASS.

" 연결: 이 매니저의 sold_out이 나면 monitor가 반응
SET HANDLER lo_monitor->on_sold_out FOR lo_mgr.
```

- `EVENTS` 선언 → `RAISE EVENT`로 발생 → 핸들러(`FOR EVENT`)가 처리 → `SET HANDLER`로 연결.

## 흔한 실수와 점검

- **SET HANDLER 누락** → 이벤트가 나도 아무도 안 듣는다. 핸들러를 연결해야.
- **핸들러 시그니처** → `FOR EVENT ... OF 클래스`가 정확해야 연결된다.
