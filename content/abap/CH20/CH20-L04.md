## 객체가 필요한가, 아닌가

- **인스턴스 멤버**(`METHODS`·`DATA`) — 객체마다 따로. 객체로 `->` 호출.
- **정적 멤버**(`CLASS-METHODS`·`CLASS-DATA`) — 클래스 공용. 객체 없이 `=>` 호출.

```abap
" 인스턴스 — 객체 필요
DATA(lo_mgr) = NEW zcl_booking_manager( iv_concert = 'C001' ).
lo_mgr->book( iv_seats = 2 ).

" 정적 — 클래스로 직접
DATA(lv) = zcl_booking_manager=>format_id( 'C001' ).
```

## me-> — 자기 자신

메서드 안에서 자기 객체의 다른 멤버를 부를 때 `me->`를 쓴다(보통 생략 가능, 명확히 할 때).

```abap
METHOD book.
  IF iv_seats > me->remaining( ).
    " 정원 초과 처리
  ENDIF.
ENDMETHOD.
```

## 언제 정적?

- 객체 상태가 필요 없는 **유틸리티**(포맷·계산) → 정적.
- 객체별 상태(잔여석 등)를 다루면 → 인스턴스.

## 흔한 실수와 점검

- **정적에서 인스턴스 멤버** → 정적 메서드는 `me`가 없다(객체 없음). 인스턴스 멤버 못 씀.
- **`=>` vs `->`** → 정적 `=>`, 인스턴스 `->`.

---

호출 방식을 정리했다. 여러 클래스가 **같은 규약**을 따르게 하는 Interface → 다음 레슨.
