## 오류를 객체로

"정원이 초과됐다" 같은 오류 상황을 **[[예외]] 객체**로 던지고 받는다. CH11에서 `cx_salv_msg`를
`TRY/CATCH`로 잡았던 그 메커니즘을, 이제 정식으로 + 내 예외도 만든다.

## 내 예외 만들기 · 던지기

`ZCX_FULLY_BOOKED`(예외 클래스, SE24에서 `CX_STATIC_CHECK` 상속)를 만들어 던진다.

```abap
METHOD book.
  IF iv_seats > me->remaining( ).
    RAISE EXCEPTION TYPE zcx_fully_booked.
  ENDIF.
ENDMETHOD.
```

## 잡기 — TRY / CATCH / CLEANUP

```abap
TRY.
    lo_mgr->book( iv_seats = 100 ).
  CATCH zcx_fully_booked INTO DATA(lx).
    MESSAGE lx->get_text( ) TYPE 'I'.
  CLEANUP.
    " 예외로 빠져나갈 때 뒷정리(자원 해제 등)
ENDTRY.
```

## CX 계층 — 세 갈래

| 상위 | 성격 |
|---|---|
| `CX_STATIC_CHECK` | 시그니처에 선언·반드시 처리(checked) |
| `CX_DYNAMIC_CHECK` | 처리 권장(미처리 시 런타임) |
| `CX_NO_CHECK` | 처리 강제 안 함(심각·어디서든) |

모두 최상위 `CX_ROOT`에서 내려온다. `get_text( )`로 메시지를 얻는다. 드물게
`RESUME`으로 예외 지점부터 재개할 수도 있다.

## 흔한 실수와 점검

- **예외 무시** → `TRY/CATCH` 없이 던지면 덤프. 적절히 잡는다.
- **너무 광범위 CATCH `cx_root`** → 의도한 예외만 좁게 잡는 게 안전.
