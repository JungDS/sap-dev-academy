## 무엇을 보여줄까

그리드는 **내부 테이블**의 내용을 표로 그린다(CH06). 그러니 보여줄 데이터를 먼저
[[Internal Table]]에 담아야 한다. 예매 목록을 그려 보자.

```abap
DATA lt_booking TYPE TABLE OF zbooking.

SELECT * FROM zbooking INTO TABLE lt_booking
  WHERE concert_id = p_concert.       " 이 공연의 예매들
```

- 행 타입(`ZBOOKING`)이 [[Data Element]]로 타입팅돼 있으면 컬럼 제목이 자동으로 붙는다(CH11처럼).
- 보여줄 컬럼·순서를 더 세밀히 정하려면 **Field Catalog**(다음 레슨)를 쓴다.

## 흔한 실수와 점검

- **빈 테이블** → 오류는 아니나 빈 그리드. 데이터 유무를 `sy-subrc`로 확인(CH08).
- **행 타입 불명** → 그리드가 컬럼을 못 잡는다. 명확한 구조 타입의 테이블로.

---

데이터를 담았다. 이제 **어떤 컬럼을 어떻게** 보여줄지 정하는 Field Catalog → 다음 레슨.
