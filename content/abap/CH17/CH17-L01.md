## 화면 안에 표를 박고 싶다

[Chapter 11 · SALV 1차 (간단 ALV)](CH11-L02.html)의 [[SALV]]는 전체 화면을 차지하는
간단한 표였다. 그런데 [Custom Control과 Container · Tabstrip · Subscreen](CH16-L07.html)에서
만든 **내 화면 안의 한 영역**에 표를 박고, 버튼·입력칸과 함께 두고 싶다. 그러려면 더
세밀한 **Grid ALV**([[CL_GUI_ALV_GRID]])와, 그것이 살 **컨테이너**가 필요하다.

> 🔎 **[선행 사용]** 이 챕터는 객체(`TYPE REF TO`·`CREATE OBJECT`)와 메서드 호출을 많이 쓴다.
> 본격 OO는 [Global Class 생성과 객체](CH20-L01.html)에서 배우니, 지금은 "표준 클래스로
> 객체를 만들어 메서드를 부른다"는 흐름만 따라가면 된다(분석·암기 불필요).

## 세 가지 이름을 구분하자

이 레슨에서 헷갈리기 쉬운 이름이 **세 개**다. 미리 갈라 두자.

| 이름 | 어디에 있나 | 의미 |
|---|---|---|
| `CONT100` | Screen Painter | 화면에 그려 둔 **Custom Control 영역**의 이름 |
| `go_cont` | ABAP 전역 변수 | `CONT100` 영역을 붙잡는 **object reference** |
| `CL_GUI_CUSTOM_CONTAINER` | SAP 표준 클래스 | 화면 영역과 Control Framework를 잇는 **클래스** |

> 💡 여기서 **컨테이너**는 데이터를 담는 내부 테이블이 아니라, **화면 컨트롤을 담는
> 프레젠테이션 영역**이다. 이름이 비슷해 헷갈리기 쉬우니 주의.

## Custom Container 만들기

[Custom Control과 Container · Tabstrip · Subscreen](CH16-L07.html)에서 화면에 둔
Custom Control 영역(`CONT100`)에 [[Custom Container]] 객체를 연결한다.

```abap
DATA go_cont TYPE REF TO cl_gui_custom_container.

CREATE OBJECT go_cont
  EXPORTING container_name = 'CONT100'.
```

- `container_name`은 ABAP 변수명이 아니라 **Screen Painter Custom Control의 이름**이다.
  실제 화면 요소 이름과 **정확히** 같아야 붙는다.
- 이 `go_cont` 위에 다음 레슨에서 [[CL_GUI_ALV_GRID]]를 얹는다.

## 직접 연결해 보기 — 컨테이너 바인딩 스텝퍼

이름을 맞추거나 틀리게 두고, **Screen Painter 확인 → PBO 실행 → CREATE OBJECT**를 한
단계씩 진행해 보자. `go_cont`가 `initial`에서 `bound`로 바뀌는지, 이름이 틀리면 왜 못
붙는지 확인한다.

## 흔한 실수와 점검

- **이름 불일치** → 화면에서 `CC_ALV`로 만들고 코드에 `CONT100`을 넘기면 연결되지 않는다.
  화면 Custom Control 이름과 `container_name`이 같아야 한다.
- **PBO마다 생성** → 컨테이너·그리드는 화면 컨트롤이라, 매 PBO마다 새로 만들면 중복 생성·
  깜박임·Control Framework 오류가 난다. `IF go_cont IS INITIAL`로 **한 번만**.

## 정리

- 이 레슨은 "화면에 빈 영역을 만들었다"에서 **"ABAP 코드가 그 영역을 붙잡았다"** 로 넘어가는 단계다.
- 아직 표는 없지만, 이 단계가 없으면 다음 단계의 **Grid ALV가 들어갈 자리**가 없다.
