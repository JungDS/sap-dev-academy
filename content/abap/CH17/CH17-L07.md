## 드디어 화면에

앞 레슨들에서 조각을 만들었다 — 자리(컨테이너)·그리드 객체·데이터·컬럼 설명서·표 모양·Variant.
하지만 조각이 준비됐다고 표가 **자동으로** 나타나진 않는다. 그리드에게 "이 데이터와 이
컬럼 설명서와 이 layout으로 **처음** 표를 그려라"라고 말하는 호출이 필요하다 —
`set_table_for_first_display`다.

```abap
go_grid->set_table_for_first_display(
  EXPORTING is_layout       = ls_layout
            is_variant      = ls_variant
            i_save          = 'A'
  CHANGING  it_outtab       = lt_booking      " 표시할 데이터
            it_fieldcatalog = lt_fcat         " 컬럼 정의
  EXCEPTIONS invalid_parameter_combination = 1
             program_error                 = 2
             too_many_lines                = 3
             OTHERS                        = 4 ).

IF sy-subrc <> 0.
  MESSAGE 'ALV Grid를 처음 표시하지 못했습니다.' TYPE 'E'.
ENDIF.
```

> 💡 **`EXPORTING` vs `CHANGING`을 역할로 나눠 보자.**
> - `EXPORTING`(`is_layout`·`is_variant`·`i_save`) = **어떻게 보일지**(설정).
> - `CHANGING`(`it_outtab`·`it_fieldcatalog`) = **무엇을 어떤 컬럼으로** 그릴지(데이터·정의).

- `it_outtab` — 보여줄 [[Internal Table]]. `it_fieldcatalog` — 컬럼 정의([Field Catalog 기초](CH17-L04.html)).
- `is_layout`·`is_variant` — 모양·저장([Layout 기본 설정](CH17-L05.html)·[Variant 기본 설정](CH17-L06.html)).

아래에서 컨테이너 → 그리드 → field catalog → `set_table_for_first_display` 4단계를 차례로
밟아, 마지막에 표가 화면에 그려지는 걸 확인하자.

## 흔한 실수와 점검

- **`EXPORTING`/`CHANGING` 섞기** → 데이터·fieldcat은 **`CHANGING`**, 설정은 `EXPORTING`이다.
- **준비 없이 호출** → `it_outtab`·`it_fieldcatalog`를 채우고 호출한다. 빈 fieldcat이면 표가 비어 보인다.
- **PBO마다 반복 호출** → 이건 **처음 한 번**의 호출이다. 이미 표시된 뒤 데이터만 바뀌면
  다음 레슨의 refresh를 쓴다.

## 정리

- 이 호출이 **이 챕터의 첫 완성 순간** — 데이터가 화면의 Grid ALV로 바뀐다.
- `EXPORTING`=어떻게 보일지, `CHANGING`=무엇을 그릴지로 나눠 기억하고, `sy-subrc`로 성공을 확인한다.
