## 컨테이너 위에 그리드를 얹는다

컨테이너는 **자리만** 준비한다. 실제로 표를 그리고, 정렬 버튼을 주고, 컬럼 너비를
조절하고, 나중에 색·이벤트까지 다룰 주인공은 [[CL_GUI_ALV_GRID]]다. 만들 때 `i_parent`에
**어느 컨테이너에 들어갈지**(앞 레슨의 `go_cont`)를 넘긴다.

```abap
DATA go_grid TYPE REF TO cl_gui_alv_grid.

CREATE OBJECT go_grid
  EXPORTING i_parent = go_cont.    " 이 그리드는 go_cont 안에 들어간다
```

- `go_grid`는 표 컨트롤 객체, `go_cont`는 그 표가 들어갈 **부모 영역**이다.
- `go_cont`가 아직 없거나 `initial`이면 `go_grid`를 화면에 정상적으로 붙일 수 없다.

## SALV와 무엇이 다른가

| 관점 | SALV | Grid ALV |
|---|---|---|
| 목적 | 빠른 목록 표시 | 화면 안 표 **컨트롤** 구성 |
| 시작 코드 | 짧음([factory](CH11-L02.html)) | 컨테이너·그리드·field catalog·layout 필요 |
| 화면 배치 | 단독 목록 화면 | **Dynpro 안 특정 영역**에 배치 |
| 후속 확장 | 제한적 | 색·[이벤트](CH30-L01.html)·[편집](CH31-L01.html)·툴바까지 |

## 그리드를 만들었는데 왜 표가 안 보이지?

`CREATE OBJECT go_grid`까지 했는데도 화면에 표가 **안 보일 수 있다.** 이건 실패가
아니라 **정상**이다. 무엇을(데이터), 어떻게(Field Catalog) 보여줄지, 그리고 표시
명령(`set_table_for_first_display`)이 **아직 없기** 때문이다.

아래에서 컨테이너 → 그리드를 차례로 만들고, 오른쪽 **준비 상태 체크리스트**에서 무엇이
아직 비어 있는지 확인하자. `i_parent`를 비워 두면 왜 연결이 깨지는지도 보라.

::embed CH17-L02-S01 | 그리드 준비 상태 체크리스트 | 560::

## 흔한 실수와 점검

- **`i_parent`에 엉뚱한 reference** → `go_cont`가 아니면 화면 영역 연결이 깨진다.
- **그리드만 만들고 끝** → Grid는 컨트롤 객체일 뿐. 실제 표 출력은
  [SET_TABLE_FOR_FIRST_DISPLAY](CH17-L07.html)에서 일어난다.

## 정리

- 이 레슨은 "표가 들어갈 자리(컨테이너)" 위에 **"표를 그릴 컨트롤(그리드)"** 을 얹는 단계다.
- 그리드를 만들었다고 표가 바로 보이는 게 아니다. 데이터·Field Catalog·display가 더 필요하다.
