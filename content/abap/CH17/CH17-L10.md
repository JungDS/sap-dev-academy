## 조각을 하나의 화면으로

지금까지는 조각을 하나씩 배웠다: 자리·그리드·데이터·Field Catalog·Layout·Variant·display·refresh·색.
실무에선 조각이 아니라 **하나의 화면**이 필요하다. 사용자가 공연 번호를 입력하고,
[Module Pool 프로그램 구조](CH16-L01.html)의 화면 0100 안 `CONT100` 영역에서 예매 목록을
보고, 새로고침해도 화면이 안정적으로 유지되어야 한다.

```text
[Screen 0100]
-------------------------------------------------
| 공연번호 gv_concert: C001       [조회] [뒤로]   |
|                                               |
| [Custom Control CONT100]                      |
|   BOOKING_ID | CUSTOMER | SEATS | STATUS       |
|   5001       | 정훈영   | 2     | N            |
|   5002       | 손흥민   | 4     | N            |
-------------------------------------------------
```

## 전체 골격 — 책임을 FORM으로 나눈다

긴 코드를 한 덩어리로 보면 어느 줄이 어떤 책임인지 놓친다. **FORM으로 나누면 조립 순서가
함수 이름에 드러난다.** PBO에서 **처음 한 번만** ALV를 만들고, 이후엔 refresh만 한다.

```abap
DATA: go_cont    TYPE REF TO cl_gui_custom_container,
      go_grid    TYPE REF TO cl_gui_alv_grid,
      gt_booking TYPE STANDARD TABLE OF zbooking,
      gt_fcat    TYPE lvc_t_fcat,
      gs_layout  TYPE lvc_s_layo,
      gs_variant TYPE disvariant.

MODULE status_0100 OUTPUT.
  IF go_grid IS INITIAL.
    PERFORM build_alv.        " 처음 한 번 — 조립
  ELSE.
    PERFORM refresh_alv.      " 이후 — 갱신만
  ENDIF.
ENDMODULE.

FORM build_alv.
  PERFORM select_booking.     " 1. 데이터
  PERFORM create_grid.        " 2. 컨테이너 + 그리드
  PERFORM build_fieldcat.     " 3. 컬럼 설명서
  PERFORM build_layout.       " 4. 표 모양 + variant
  PERFORM display_grid.       " 5. 처음 띄우기
ENDFORM.
```

## 각 단계 FORM

```abap
FORM select_booking.
  SELECT * FROM zbooking INTO TABLE gt_booking
    WHERE concert_id = gv_concert.
  IF sy-subrc <> 0.
    MESSAGE '해당 공연의 예매 데이터가 없습니다.' TYPE 'S'.   " 빈 결과는 정상일 수 있음
  ENDIF.
ENDFORM.

FORM create_grid.
  CREATE OBJECT go_cont EXPORTING container_name = 'CONT100'.
  CREATE OBJECT go_grid EXPORTING i_parent = go_cont.
ENDFORM.

FORM build_fieldcat.
  DATA ls_fcat TYPE lvc_s_fcat.
  CALL FUNCTION 'LVC_FIELDCATALOG_MERGE'
    EXPORTING i_structure_name = 'ZBOOKING'
    CHANGING  ct_fieldcat      = gt_fcat
    EXCEPTIONS OTHERS          = 1.
  IF sy-subrc <> 0.
    MESSAGE 'Field Catalog를 생성하지 못했습니다.' TYPE 'E'.
  ENDIF.
  LOOP AT gt_fcat INTO ls_fcat.
    IF ls_fcat-fieldname = 'SEATS'.
      ls_fcat-coltext = '좌석수'. ls_fcat-outputlen = 8.
      MODIFY gt_fcat FROM ls_fcat INDEX sy-tabix.
    ENDIF.
  ENDLOOP.
ENDFORM.

FORM build_layout.
  gs_layout-zebra      = abap_true.
  gs_layout-grid_title = '예매 목록'.
  gs_layout-cwidth_opt = abap_true.
  gs_variant-report    = sy-repid.
ENDFORM.

FORM display_grid.
  go_grid->set_table_for_first_display(
    EXPORTING is_layout  = gs_layout
              is_variant = gs_variant
              i_save     = 'A'
    CHANGING  it_outtab       = gt_booking
              it_fieldcatalog = gt_fcat
    EXCEPTIONS OTHERS          = 1 ).
  IF sy-subrc <> 0.
    MESSAGE 'ALV Grid를 처음 표시하지 못했습니다.' TYPE 'E'.
  ENDIF.
ENDFORM.

FORM refresh_alv.
  DATA ls_stable TYPE lvc_s_stbl.
  ls_stable-row = abap_true.
  ls_stable-col = abap_true.
  go_grid->refresh_table_display( EXPORTING is_stable = ls_stable
                                  EXCEPTIONS OTHERS   = 1 ).
ENDFORM.
```

## 완성 체크리스트

| 체크 | 질문 |
|---|---|
| 화면 영역 | 0100 화면에 `CONT100`이 있는가 |
| 생성 가드 | `go_grid IS INITIAL`일 때만 build하는가 |
| 데이터 | `gt_booking` 행 수와 화면 행 수가 맞는가 |
| 컬럼 | `gt_fcat` 변경이 화면 컬럼 제목에 반영되는가 |
| 모양 | zebra·title·width가 반영되는가 |
| 갱신 | 두 번째 PBO부터 display가 아니라 **refresh**가 호출되는가 |

## 직접 해 볼 과제

1. **매진 회차 행 색상** — [행 색상 기초](CH17-L09.html)의 `info_fname`에 색 컬럼을 더한다.
2. **좌석수 합계** — Field Catalog의 `SEATS`에 `do_sum = abap_true`([Grid ALV Column 제어 심화](CH22-L03.html) 맛보기).
3. **위치 보존 새로고침** — 데이터 변경 후 [Refresh와 Stable Refresh 기초](CH17-L08.html)의 `is_stable`로 위치를 지킨다.

> 막히면 해당 조각 레슨으로 돌아가 확인한다. "어느 조각이 어디서 쓰이는지"를 손으로 맞춰
> 보는 게 이 실습의 목적이다.

## 흔한 실수와 점검

- **"처음 구성"과 "이후 갱신" 섞기** → `set_table_for_first_display`는 처음 한 번, `refresh_table_display`는 이후. 이 원칙을 화면 흐름에 고정한다.
- **Custom Control 이름 불일치** → 화면과 코드의 `container_name`이 같아야 한다.
- **merge 실패 무시** → `LVC_FIELDCATALOG_MERGE`가 실패했는데 `gt_fcat`을 확인 안 하고 display로 넘어가지 않는다.
- **0건을 오류로** → 조회 결과 없음은 정상 업무 상황일 수 있다.

## 정리

- 이 챕터는 "표를 보여 주는 코드"가 아니라 **"화면 안 표 컨트롤을 조립하는 코드"** 다.
- 컨테이너·그리드·데이터·field catalog·layout·variant·display·refresh가 **각자 맡은 책임**을
  알아야, 이후의 표시 제어·이벤트·편집 Grid를 흔들리지 않고 배운다.
