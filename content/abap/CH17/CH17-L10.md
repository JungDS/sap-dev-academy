## 조각을 하나로 — 실행되는 리포트

CH17에서 컨테이너·그리드·Field Catalog·Layout·display·refresh·색을 **따로따로** 배웠다.
이제 그 조각을 **하나의 완성된 프로그램**으로 모아, 콘서트 **예매 목록**을 그리드로
띄운다. 화면 0100에 Custom Control `CONT100`을 둔 Module Pool(CH16) 기준이다.

## 전체 골격

```abap
DATA: go_cont    TYPE REF TO cl_gui_custom_container,
      go_grid    TYPE REF TO cl_gui_alv_grid,
      lt_booking TYPE TABLE OF zbooking,
      lt_fcat    TYPE lvc_t_fcat,
      ls_layout  TYPE lvc_s_layo.

MODULE status_0100 OUTPUT.
  IF go_grid IS INITIAL.
    PERFORM build_alv.                 " 처음 한 번만 만든다
  ELSE.
    go_grid->refresh_table_display( ). " 이후엔 갱신만
  ENDIF.
ENDMODULE.

FORM build_alv.
  " 1) 데이터
  SELECT * FROM zbooking INTO TABLE lt_booking WHERE concert_id = p_concert.

  " 2) 컨테이너 + 그리드
  CREATE OBJECT go_cont EXPORTING container_name = 'CONT100'.
  CREATE OBJECT go_grid EXPORTING i_parent = go_cont.

  " 3) Field Catalog (자동 생성 후 다듬기)
  CALL FUNCTION 'LVC_FIELDCATALOG_MERGE'
    EXPORTING i_structure_name = 'ZBOOKING'
    CHANGING  ct_fieldcat      = lt_fcat.

  " 4) Layout
  ls_layout-zebra      = abap_true.
  ls_layout-cwidth_opt = abap_true.
  ls_layout-grid_title = '예매 목록'.

  " 5) 띄우기
  go_grid->set_table_for_first_display(
    EXPORTING is_layout       = ls_layout
    CHANGING  it_outtab       = lt_booking
              it_fieldcatalog = lt_fcat ).
ENDFORM.
```

## 직접 해 볼 과제

1. **매진 회차를 행 빨강**으로 — `ls_layout-info_fname`(CH17-L09)에 색 컬럼을 더해 본다.
2. **좌석수 합계** — Field Catalog에서 `SEATS` 컬럼에 `do_sum = abap_true`(CH21-L03 맛보기).
3. **새로고침 유지** — 데이터 변경 후 `refresh_table_display`에 `is_stable`을 줘 위치를 보존한다(CH17-L08).

> 막히면 CH17-L01~L09의 해당 조각으로 돌아가 확인한다. "어느 조각이 어디서 쓰이는지"를
> 손으로 맞춰 보는 게 이 실습의 목적이다.

## 흔한 실수와 점검

- **CREATE OBJECT 중복** → 컨테이너·그리드는 처음 한 번만(`IS INITIAL` 가드).
- **fieldcat 비움** → merge를 안 부르면 컬럼이 안 그려진다.
- **refresh 누락** → 데이터만 바꾸고 화면은 그대로. PBO에서 refresh.
