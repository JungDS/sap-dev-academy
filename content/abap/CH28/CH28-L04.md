## 행마다 다른 입력 규칙

[Editable Field Catalog 설정](CH28-L01.html)의 `edit`는 **컬럼 전체**를 여닫는 설정이다.
그런데 실무에선 *같은 컬럼이라도 행마다* 규칙이 다르다 — "매진 회차는 좌석 수 수정 불가,
그 외는 가능"처럼. 이건 [Deep Structure 기반 Cell Style](CH21-L05.html)의 **Cell
Style**(`LVC_T_STYL`)로 셀 단위로 제어한다.

```abap
LOOP AT lt_perf ASSIGNING FIELD-SYMBOL(<ls_perf>).
  DATA lv_style TYPE i.
  IF <ls_perf>-seatsocc >= <ls_perf>-capacity.
    lv_style = cl_gui_alv_grid=>mc_style_disabled.  " 매진=잠금
  ELSE.
    lv_style = cl_gui_alv_grid=>mc_style_enabled.   " 그외=편집
  ENDIF.
  <ls_perf>-cellstyles = VALUE #(
    ( fieldname = 'SEATS' style = lv_style ) ).
ENDLOOP.

ls_layout-stylefname = 'CELLSTYLES'.
```

구조는 **2단계**다. 먼저 Field Catalog에서 `SEATS` 컬럼을 편집 가능하게 열고, 그다음 행별
`cellstyles`로 "이 행의 SEATS는 잠글지 열지"를 정한다 — **전체 허용 + 예외 제어**의 분리다.

## 매진 토글로 확인

행을 **매진**으로 바꾸면 그 행의 `SEATS` 셀만 읽기전용으로 잠기는 걸 확인해 보라(같은 컬럼,
셀마다 다른 스타일).

::embed CH28-L04-S01 | Cell Style — 매진 행 입력 잠금 | 360::

## 흔한 실수와 점검

- **fieldcat `edit`을 꺼 둠** → cell style만으론 입력 제어가 기대처럼 안 보인다. *기본은
  컬럼을 열고*, 예외 셀만 disabled로 덮는다.
- **style을 한 번만 계산** → 행 상태가 바뀌면(잔여석 0이 됨 등) `cellstyles`를 **다시
  계산**해야 한다. 안 그러면 매진인데도 셀이 열려 보인다.
- **셀 잠금을 보안으로 착각** → Cell Style은 *UI 입력 제어*일 뿐이다. 악의적 조작·동시
  변경·다른 프로그램의 변경은 UI 잠금으로 못 막는다. **저장 전 전체 검증**(이 챕터 마지막)이
  따로 필요하다.

## 정리

- Field Catalog `edit` = **컬럼 단위** 허용, Cell Style = **셀 단위** 예외 제어.
- 같은 `SEATS`라도 매진 행은 disabled, 일반 행은 enabled로 만든다(`stylefname`으로 연결).
- 데이터 상태가 바뀌면 style도 다시 계산 — **UI 잠금은 저장 전 검증을 대신하지 않는다.**
