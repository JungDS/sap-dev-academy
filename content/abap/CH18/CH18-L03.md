## 같은 데이터를 다른 모양으로

업무 프로그램은 같은 데이터를 다른 모양으로 옮기는 일이 많다. DB 행을 화면 표시용
구조로, 입력 구조를 저장 구조로. classic은 같은 이름 필드를 자동으로 옮기는
`MOVE-CORRESPONDING`을 썼다([Local Structure (BEGIN OF ~ END OF)](CH05-L01.html)).

```abap
" classic
MOVE-CORRESPONDING ls_source TO ls_target.
```

modern에서는 **값을 만드는 식** 흐름 안에서 같은 일을 한다: `CORRESPONDING` constructor.

## 같은 이름은 자동, 다른 이름은 MAPPING, 제외는 EXCEPT

표시용 구조에 이름이 다른 필드(`display_status`)와 원본에 없는 필드(`audit_user`)가 있다고 하자.

```abap
TYPES: BEGIN OF ts_booking_ui,
         booking_id     TYPE zbooking-booking_id,
         customer       TYPE zbooking-customer,
         seats          TYPE zbooking-seats,
         display_status TYPE zbooking-status,   " 이름 다름
         audit_user     TYPE syuname,           " 원본에 없음
       END OF ts_booking_ui.

DATA(ls_ui) = CORRESPONDING ts_booking_ui(
  ls_booking
  MAPPING display_status = status      " 대상 = 원본
  EXCEPT  audit_user ).
```

- `booking_id`·`customer`·`seats` — **같은 이름**이라 자동으로 옮긴다.
- `MAPPING display_status = status` — 이름이 다른 필드 연결. 문법은 **`대상필드 = 원본필드`** 다.
- `EXCEPT audit_user` — **대상 필드** `audit_user`를 결과에서 채우지 않는다(초기값으로 둠).

내부 테이블도 같은 규칙이 각 행에 적용된다.

```abap
DATA(lt_ui) = CORRESPONDING tt_booking_ui(
  lt_booking MAPPING display_status = status EXCEPT audit_user ).
```

> 💡 `CORRESPONDING`은 **모든 것을 복사하는 도구가 아니다.** **대상 타입 기준**으로 옮길 수
> 있는 필드만 결과에 들어간다. 원본에만 있는 필드(`created_by` 등)는 **조용히 버려진다.**

## 직접 매핑해 보기 — Field Mapping Board

원본·대상 구조의 필드가 어떻게 연결되는지 보자. 같은 이름은 자동, `status`→`display_status`는
`MAPPING`으로, `audit_user`는 `EXCEPT`로. 생성되는 `CORRESPONDING` 코드와 결과 Structure가 어떻게 바뀌는지 확인한다.

## 흔한 실수와 점검

- **`MAPPING` 방향 혼동** → 문법은 `대상 = 원본`이다. `display_status = status`는 대상 `display_status`에 원본 `status`를 넣는다.
- **`EXCEPT`를 원본 기준으로 생각** → `EXCEPT`는 **대상 필드** 기준이다.
- **이름 같다고 의미도 같다고 가정** → 한 구조의 `status`(예매 상태)와 다른 구조의 `status`(처리 상태)는 의미가 다를 수 있다. 이름은 같고 의미가 다르면 자동 복사는 위험하다.

> 💡 `CORRESPONDING`은 변환 로직을 **숨기기 쉽다.** 단순 이름 매칭엔 좋지만, 값 변환·조건부
> 매핑이 많아지면 `LOOP`와 명시 대입이 더 읽기 쉽다. 이 챕터에선 "같은 이름 또는 간단한 이름 변경" 범위에서만 쓴다.

## 정리

- `CORRESPONDING`은 **같은 이름 필드**를 기준으로 Structure·테이블을 새 타입으로 옮기는 constructor다.
- `MAPPING`은 이름이 다른 필드를 연결(`대상 = 원본`), `EXCEPT`는 **대상 필드**를 결과에서 제외한다.
- 강력하지만, **이름은 같고 의미가 다른** 구조에는 주의해야 한다.
