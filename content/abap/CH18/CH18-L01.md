## 위에서 선언, 아래서 사용 — 번거롭다

지금까지는 변수를 **먼저 `DATA`로 선언**하고, 한참 아래에서 **사용**했다. 타입도
일일이 적었다. 모던 ABAP은 **쓰는 자리에서 바로 선언**하게 해 준다 — 인라인 선언이다.

> 📌 이 챕터(CH18)부터 **모던 문법**을 쓴다. CH17까지는 일부러 classic만 썼다 — 이제
> 경계를 넘었다.

## DATA() — 쓰면서 선언

```abap
" classic
DATA ls_book TYPE zbooking.
READ TABLE lt_booking INTO ls_book WITH KEY booking_id = '0001'.

" modern — 타입은 우변에서 추론
READ TABLE lt_booking INTO DATA(ls_book) WITH KEY booking_id = '0001'.

LOOP AT lt_booking INTO DATA(ls_row).
  WRITE: / ls_row-customer.
ENDLOOP.

DATA(lv_total) = ls_book-seats * 10000.   " 타입 자동 추론
```

## FINAL() — 다시 안 바뀌는 값

값을 한 번만 정하고 **이후 변경을 막고** 싶으면 `FINAL()`로 선언한다(안전성↑).

```abap
FINAL(lv_max) = 100.    " 이후 lv_max 재대입 시 문법 오류
```

## 주의 — SELECT 인라인은 다음 챕터

**SELECT 안의 인라인 선언**은 별도 escape가 필요한 **New Open SQL**이라 **CH19**에서
배운다. 이 챕터의 인라인은 LOOP·READ·대입 등 **SQL 밖**에서 쓴다.

## 흔한 실수와 점검

- **이미 선언된 변수에 DATA()** → 같은 이름을 또 `DATA()`로 쓰면 충돌. 한 번만.
- **SELECT에 DATA()** → SQL 안의 인라인은 CH19(New Open SQL). 여기선 LOOP/READ 등에.
