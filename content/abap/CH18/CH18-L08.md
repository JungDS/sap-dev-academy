## 합계 하나 만들려고 LOOP를 또 쓴다

[[Internal Table]]을 배운 뒤로 우리는 반복해서 [[LOOP]]를 썼다. 전체 좌석 수를 합산하거나,
상태가 `N`인 예매만 모으는 코드는 매우 흔하다. `LOOP`는 명확하지만, **"합계 하나"** 나
**"조건에 맞는 행만 추출"** 처럼 패턴이 단순하면 준비 코드가 많다.

## REDUCE — 여러 값을 하나로 줄인다

[[REDUCE]]는 **초기값**을 만들고, 반복하며 누적 변수에 값을 더해 **최종 결과 하나**를 만든다.

```abap
DATA(lv_total_seats) = REDUCE i(
  INIT sum = 0
  FOR ls_book IN lt_booking
  NEXT sum = sum + ls_book-seats ).
```

이건 아래 classic loop와 **같은 목적**이다.

```abap
DATA lv_total_seats TYPE i.
LOOP AT lt_booking INTO DATA(ls_book).
  lv_total_seats = lv_total_seats + ls_book-seats.
ENDLOOP.
```

- `INIT sum = 0` — 누적을 **시작할 값**.
- `FOR ls_book IN lt_booking` — 한 행씩 꺼낸다.
- `NEXT sum = ...` — 다음 누적값을 계산한다(반복마다).

## FILTER — 조건에 맞는 행만 새 테이블로

[[FILTER]]는 조건에 맞는 행만 뽑아 **새 Internal Table**을 만든다. `FILTER`는 테이블의
**Key**와 조건 관계가 중요해서, 여기서는 Key가 분명한 [[Sorted Table]]로 익힌다.

```abap
TYPES tt_booking TYPE SORTED TABLE OF ts_booking
  WITH NON-UNIQUE KEY status.

DATA(lt_open) = FILTER tt_booking( lt_booking WHERE status = 'N' ).
```

`EXCEPT`를 붙이면 조건에 **맞지 않는** 행을 가져온다.

```abap
DATA(lt_not_open) = FILTER tt_booking( lt_booking EXCEPT WHERE status = 'N' ).
```

## 흔한 실수와 점검

- **`REDUCE`를 처음부터 복잡하게** → 초보에게 가장 읽기 어려운 표현식 중 하나다. **단순
  합계**부터 시작한다. 누적 중간에 검증·메시지가 많으면 `LOOP`가 낫다.
- **`FILTER`에서 Key 무시** → `FILTER`는 테이블 타입과 Key를 의식해야 한다. Key가 분명한
  테이블에서 시작한다.
- **`FILTER ... EXCEPT`를 단순 `NOT`으로** → 항상 같은 사고방식은 아니다. 결과 행을 직접 확인한다.

## 정리

- `REDUCE`는 테이블을 **합계나 하나의 결과로 줄이는** 표현식이다.
- `FILTER`는 **조건에 맞는 행만 새 테이블로** 만드는 표현식이다.
- 둘 다 **단순한 반복 패턴**을 줄이는 도구다. 복잡한 업무 흐름은 여전히 `LOOP`가 나을 수 있다.
