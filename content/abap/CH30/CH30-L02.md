## 한 번 클릭하는 링크

[Double Click Event](CH30-L01.html)는 **행 전체**를 상세로 보냈다. 그런데 모든 셀을
더블클릭 대상으로 만들면 사용자는 어디를 눌러야 할지 헷갈린다. 예매 목록에서
*공연ID·고객번호*처럼 "눌러서 이동할 값"만 **링크처럼** 보여 주는 게 낫다. 그게
**Hotspot**이다.

핵심은 **핸들러만 만들면 끝이 아니라는 점**이다. 먼저 [Field Catalog](CH17-L04.html)에서
그 컬럼의 `hotspot` 속성을 켜야 화면에서 링크처럼 보이고, 사용자의 **한 번 클릭**이
`hotspot_click` 이벤트로 이어진다.

## 컬럼을 hotspot으로 — Field Catalog

```abap
LOOP AT lt_fcat REFERENCE INTO DATA(lr_fcat).
  IF lr_fcat->fieldname = 'CONCERT_ID'.
    lr_fcat->hotspot = abap_true.        " 이 컬럼을 링크처럼
  ENDIF.
ENDLOOP.
```

## 핸들러 — hotspot_click

`hotspot_click`은 클릭된 **행과 컬럼**을 넘긴다. `e_column_id-fieldname`으로 *의도한
컬럼일 때만* 반응한다(핸들러는 앞 레슨처럼 생성자로 주입받은 예매 테이블 `mt_booking`을 든다).

```abap
METHODS on_hotspot_click
  FOR EVENT hotspot_click OF cl_gui_alv_grid
  IMPORTING e_row_id e_column_id.

METHOD on_hotspot_click.
  IF e_column_id-fieldname <> 'CONCERT_ID'.
    RETURN.                              " 다른 컬럼이면 무시
  ENDIF.
  READ TABLE mt_booking INTO DATA(ls_booking) INDEX e_row_id-index.
  IF sy-subrc <> 0.
    RETURN.
  ENDIF.
  MESSAGE |공연 { ls_booking-concert_id } 상세로 이동| TYPE 'I'.
ENDMETHOD.
```

- `e_column_id-fieldname` — 클릭한 컬럼명. `e_row_id-index` — 클릭한 화면 행.
- 연결은 앞 레슨처럼 `SET HANDLER ... FOR go_grid`로 등록한다.

## 직접 클릭해 보기

`concert_id`가 밑줄 링크로 보인다. **한 번** 클릭하면 `hotspot_click`이 행·컬럼과 함께
실행되는 걸 확인해 보라(더블클릭 아님).

::embed CH30-L02-S01 | Hotspot 이벤트 — 링크 셀 클릭 | 460::

> 🔁 **더블클릭 ↔ Hotspot** — `double_click`은 **행** 전체를 대상으로 상세 조회에, Hotspot은
> *특정 컬럼 값*이 링크임을 명확히 보일 때 어울린다. 같은 값에 둘 다 강하게 걸면 "한 번/두
> 번"이 헷갈리니 하나만.

## 흔한 실수와 점검

- **`hotspot` 안 켜고 핸들러만** → 사용자는 셀이 클릭 가능한지 모르고, 이벤트도 안 난다.
  *Field Catalog에서 `hotspot` 먼저* 켠다.
- **컬럼 구분 없이 반응** → 핸들러가 모든 클릭에 반응하면 엉뚱한 컬럼에도 동작한다.
  `e_column_id-fieldname`으로 의도한 컬럼만 처리한다.
- **`e_row_id-index`를 업무 키로** → 앞 레슨과 같은 이유로 화면 행 위치일 뿐. 키
  (`concert_id`)로 상세를 잇는다. 또 Hotspot은 **탐색**용 — 셀 값을 바꾸는 입력은
  [Editable Grid ALV와 입력 검증](CH31-L01.html)에서 다룬다.

## 정리

- Hotspot = 특정 셀을 **링크처럼** 보이게 하고 **한 번 클릭**으로 `hotspot_click`을 일으킨다.
- **Field Catalog에서 `hotspot=abap_true`를 먼저** 켜야 링크가 되고 이벤트가 난다.
- 핸들러는 `e_row_id`·`e_column_id`로 *클릭 위치*를 확인 — 더블클릭은 행 중심, Hotspot은
  컬럼 값 중심 탐색.
