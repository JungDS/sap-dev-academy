## BAPI가 없을 때 — 화면을 자동으로

표준 BAPI가 없는 트랜잭션은, **사용자가 화면에 입력하는 과정을 흉내** 내 자동 등록한다.
그게 [[BDC]](Batch Data Communication)다. 화면·필드·값을 `BDCDATA`에 채워 트랜잭션을 돌린다.

```abap
DATA lt_bdc TYPE TABLE OF bdcdata.

" 화면·필드·값을 차곡차곡(헬퍼 FORM으로)
PERFORM fill USING 'X' 'SAPMXXXX' '0100'.   " 화면
PERFORM fill USING ' ' 'FIELD-X'  '값'.      " 필드

CALL TRANSACTION 'XX01' USING lt_bdc
  MODE 'N' UPDATE 'S'                         " N=무화면, S=동기
  MESSAGES INTO lt_msg.
```

- `MODE`: `A`(전체화면)·`E`(오류만)·`N`(무화면, 배치).
- 결과 메시지를 받아 성공/실패를 판정(L05의 재처리와 결합).
- **BAPI가 있으면 BAPI 우선** — BDC는 화면 변경에 취약하다.

## 흔한 실수와 점검

- **화면 흐름 변경** → 표준 화면이 바뀌면 BDC가 깨진다. BAPI 가능하면 그쪽.
- **MODE 'A'로 대량** → 배치는 `N`. `A`는 디버깅용.
