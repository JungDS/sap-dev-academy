## 표준 데이터는 BAPI로

CH24에서 "표준 테이블은 직접 DML 금지, BAPI 경유"라 했다. [[BAPI]](Business API)는
SAP이 제공하는 **검증·후속처리까지 포함한** 표준 기능이다(예: `BAPI_SALESORDER_CREATEFROMDAT2`).

```abap
DATA lt_return TYPE TABLE OF bapiret2.

CALL FUNCTION 'BAPI_..._CREATE'
  EXPORTING  is_header = ls_header
  TABLES     return    = lt_return.

" 결과 확인 — 오류(E/A)가 있나?
IF line_exists( lt_return[ type = 'E' ] ) OR line_exists( lt_return[ type = 'A' ] ).
  CALL FUNCTION 'BAPI_TRANSACTION_ROLLBACK'.
ELSE.
  CALL FUNCTION 'BAPI_TRANSACTION_COMMIT' EXPORTING wait = abap_true.
ENDIF.
```

- `BAPIRET2` 테이블의 `type`(E/A=오류)을 반드시 확인.
- BAPI는 보통 **스스로 COMMIT 안 함** → `BAPI_TRANSACTION_COMMIT`을 호출해야 확정.

## 흔한 실수와 점검

- **COMMIT 누락** → BAPI만 부르고 끝내면 저장 안 됨.
- **Return 미확인** → 오류를 무시하고 COMMIT하면 깨진 데이터.
