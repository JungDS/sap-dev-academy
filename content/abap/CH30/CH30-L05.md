## 서버 파일과 자동화

PC 파일(L04)과 달리, 배치 인터페이스는 **서버 파일**을 읽고 쓴다.

```abap
OPEN DATASET lv_path FOR INPUT IN TEXT MODE ENCODING UTF-8.
DO.
  READ DATASET lv_path INTO DATA(lv_rec).
  IF sy-subrc <> 0. EXIT. ENDIF.
  " 파싱·검증·등록
ENDDO.
CLOSE DATASET lv_path.
```

## 인터페이스 공통 — 검증·로그·재처리

BAPI·RFC·BDC·File 모두 같은 골격이 필요하다(CH24-L04와 같은 정신).

1. **읽기/수신** → 2. **검증** → 3. **등록**(BAPI/DML) → 4. **결과 로그** → 5. **실패분 재처리**.

- 성공/실패를 **인터페이스 로그 테이블**에 남긴다.
- 실패 건은 보관 후 원인 해결 시 **재처리 잡**으로.
- 멱등성(중복 수신 방지: 메시지ID·키 체크)을 고려.

## 흔한 실수와 점검

- **CLOSE DATASET 누락** → 파일 핸들 누수.
- **재처리 없음** → 일시 오류로 누락된 데이터가 영영 안 들어온다.


## 🧩 도전 과제 (서브퀘스트)

> 선택 과제 — 예매 데이터를 외부와 주고받는다.

1. 예매 조회를 **RFC** 함수(Remote-Enabled)로 노출해 외부에서 호출 가능하게 한다.
2. 엑셀 예매 파일을 업로드해 **검증 후** 등록(BAPI/DML)하고 실패 건을 로그에 남긴다.
3. (심화) 표준 트랜잭션 등록은 **BDC Session**(`SHDB`로 녹화)으로 대량 처리한다.

힌트: L01(BAPI)·L02(RFC)·L03(BDC/SHDB)·L04(엑셀)·L05(서버파일/재처리).
