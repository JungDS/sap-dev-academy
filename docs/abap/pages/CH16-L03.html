Screen Programming / Dynpro 기초

화면 요소 — 입력·버튼·체크박스·라디오·드롭다운

화면 요소를 세 가지 행동(값 운반·function code·표시)으로 구분해 변수와 잇는다.

화면을 채우는 요소들

예매 화면을 떠올려 보자. 공연ID 입력칸, 좌석 수 입력칸, "예매" 버튼, 상태 드롭다운. 사용자는 코드가 아니라 요소와 상호작용한다. 개발자는 각 요소가 ABAP 변수와 어떻게 연결되고, 어떤 요소가 PAI를 발생시키는지 알아야 한다.

요소 역할 ABAP 연결
Input/Output Field 값 입력·표시 같은 이름의 전역 변수
Text Field 고정 라벨·안내 보통 변수 운반 없음
Checkbox 켜고 끄기 CHAR1('X'/공백)
Radiobutton 그룹 중 하나 그룹 안에서 하나
Push Button 누르면 PAI 발생 FctCode → OK_CODE
Dropdown Listbox 고정 목록에서 선택 key가 화면 field로
Frame 요소 묶음 표시 표시 전용

요소는 세 가지 "행동"으로 나뉜다

이 레슨의 핵심은 모양 암기가 아니라 행동 구분이다.

  • 🟦 값을 운반 — Input/Output Field·Checkbox·Radiobutton·Dropdown. 사용자가 고른 값이 ABAP 변수로 들어온다.
  • 🟧 function code를 발생 — Push Button. 값이 아니라 어떤 버튼을 눌렀는지를 OK_CODE로 보낸다.
  • ⬜ 화면 이해만 도움 — Text Field·Frame. 운반하는 값이 없다(라벨·테두리).

이 구분이 되어야 PAI 처리 흐름 — OK_CODE와 화면 떠나기에서 입력을 제대로 해석할 수 있다.

DATA: gv_conc  TYPE zconcert-concert_id,   " 공연ID 입력칸

  • 화면 요소 이름을 GV_CONC로 만들고 ABAP에 같은 이름 변수가 있으면, 이름 기준으로 값이 오간다: PBO 끝에 변수 → 화면, PAI 시작에 화면 → 변수.
  • 입력칸에 를 쓰면 Elementary Search Help의 F4·검증이 화면에서도 작동한다.

Dropdown은 어떻게 채우나 — Listbox(VRM)

Search Help가 "검색 팝업"이라면, Dropdown(Listbox) 은 화면에서 바로 펼쳐지는 고정 목록이다. 값이 몇 개로 정해져 있을 때(상태 = 예약/대기/취소) F4보다 간편하다. 목록은 VRM(Value Request Manager) 함수 VRM_SET_VALUES로 코드에서 채운다.

DATA gt_values TYPE vrm_values.
DATA gs_value  LIKE LINE OF gt_values.

MODULE fill_status_list OUTPUT.        " PBO에서 목록을 채운다
  CLEAR gt_values.
  gs_value-key = 'R'. gs_value-text = '예약'. APPEND gs_value TO gt_values.
  gs_value-key = 'W'. gs_value-text = '대기'. APPEND gs_value TO gt_values.
  gs_value-key = 'C'. gs_value-text = '취소'. APPEND gs_value TO gt_values.

  CALL FUNCTION 'VRM_SET_VALUES'        " 화면 field에 목록 전달
    EXPORTING
      id     = 'GV_STAT'
      values = gt_values.
ENDMODULE.
  • key는 변수에 들어가는 실제 값(R), text는 화면에 보이는 글자(예약)다. 둘은 다르다.
  • CALL FUNCTION은 CALL FUNCTION 기본 구조에서 배운 그 호출이다.

💡 많고 검색이 필요 → Search Help(F4). 적고 고정 → Dropdown(Listbox/VRM).

흔한 실수와 점검

  • 버튼인데 OK_CODE 안 옴 → 버튼에 FctCode를 안 줬다. FctCode를 지정한다.
  • 체크박스를 Boolean처럼 → 화면 운반 값은 'X'/공백인 character다(CHAR1).
  • Text Field로 값 받기 → Text Field는 라벨이다. 값은 Input/Output Field로 받는다.
  • 긴 목록을 Dropdown으로 → 검색이 필요할 만큼 길면 F4/Search Help가 맞다.

정리

  • Dynpro 요소는 보이는 모양과 운반되는 값이 다르다.
  • Input/Output·Checkbox·Radiobutton·Dropdown은 값을, Push Button은 function code를 보내고, Text Field·Frame은 화면 이해를 돕는다.

요소를 올렸다. 그런데 DDIC에 이미 라벨·타입·F4까지 완성된 필드를 화면에 얹으면 프로그램과 어떻게 이어질까 → Dictionary 화면 필드와 TABLES 운반.
