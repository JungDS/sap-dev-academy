Grid ALV 기초

SET_TABLE_FOR_FIRST_DISPLAY

데이터·fieldcat·layout을 묶어 그리드를 처음 띄운다.

Track 1Chapter 17Lesson 7중급

드디어 화면에

지금까지 만든 데이터·Field Catalog·Layout·Variant를 그리드의 set_table_for_first_display 메서드에 넘기면 표가 화면에 그려진다.

go_grid->set_table_for_first_display(
  EXPORTING is_layout       = ls_layout
            is_variant      = ls_variant
            i_save          = 'A'
  CHANGING  it_outtab       = lt_booking      " 표시할 데이터
            it_fieldcatalog = lt_fcat ).      " 컬럼 정의
  • it_outtab — 보여줄 .
  • it_fieldcatalog — 컬럼 정의(CH17-L04).
  • is_layout·is_variant — 모양·레이아웃 저장(L05·L06).
  • 이건 처음 한 번의 호출이다. 데이터가 바뀐 뒤 다시 그릴 땐 refresh(다음 레슨).

흔한 실수와 점검

  • CHANGING/EXPORTING 구분 → 데이터·fieldcat은 CHANGING으로 넘긴다.
  • 빈 fieldcat → 컬럼 정의가 없으면 표가 비어 보인다.

표를 처음 띄웠다. 그런데 데이터가 바뀌면 어떻게 다시 그릴까? → Refresh(다음 레슨).
