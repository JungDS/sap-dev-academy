Open SQL 2차: JOIN·집계

FOR ALL ENTRIES 사용 기준

내부 테이블을 조건으로 DB 조회 — FOR ALL ENTRIES의 함정과 규칙.

Track 1Chapter 13Lesson 6중급

이미 가진 목록으로 조회

화면이나 앞 단계에서 만든 내부 테이블의 값들을 조건으로 DB를 조회하고 싶다. (FAE)가 그 도구다 — 단, 함정이 많아 규칙을 지켜야 한다.

기본형 (classic)

ABAP
1
2
3
4
5

" 전제: lt_dept (조회 기준이 될 내부 테이블)에 dept_id들이 있음
IF lt_dept IS NOT INITIAL.          " ← 필수!
  SELECT persid name dept_id
    FROM ztperson
    INTO CORRESPONDING FIELDS OF TABLE lt_person
    FOR ALL ENTRIES IN lt_dept
    WHERE dept_id = lt_dept-dept_id.
ENDIF.

반드시 지킬 규칙

  • 빈 테이블 금지 → lt_dept가 비어 있으면 FAE는 WHERE를 무시하고 전체 조회한다(대형 사고). 항상 IS NOT INITIAL 확인.
  • 중복 자동 제거 → FAE 결과는 중복 행이 제거된다. 중복이 의미 있으면 주의.
  • WHERE에 기준 테이블 필드 → lt_dept-dept_id처럼 기준 테이블 필드를 조건에.

빈 목록일 때 무슨 일이? — 직접 실험

lt_dept를 비운 채 안전장치를 끄고 실행하면, FAE는 WHERE를 통째로 무시하고 전체 테이블을 조회한다. 운영 DB에서는 화면이 멈추거나 배치 시간이 폭증하는 사고로 이어진다. 안전장치(IS NOT INITIAL)가 왜 선택이 아니라 필수인지 직접 보자.

⚠️ 아래는 시뮬레이션이다. 실제 운영 DB에서 빈 목록 + 안전장치 OFF를 실험하면 안 된다.

흔한 실수와 점검

  • IS NOT INITIAL 누락 → 빈 테이블 → 전체 조회 폭탄. 1순위 점검.
  • 중복 제거 간과 → 결과 건수가 기대와 다를 수 있다.
