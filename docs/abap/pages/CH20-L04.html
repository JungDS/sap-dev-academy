Advanced ABAP SQL

UNION · INTERSECT · EXCEPT — 결과 집합 다루기

JOIN이 컬럼을 옆으로 붙인다면, 집합 연산은 행을 위아래로 합·교·차.

Track 1Chapter 20Lesson 4고급

왜 — 두 목록을 비교하고 싶다 🔀

업무에선 두 조회 결과를 비교할 때가 많다.

  • 전체 공연과 예약된 공연을 합치고 싶다.
  • 전체 공연 중 예약도 있는 공연만.
  • 전체 공연 중 아직 예약이 없는 공연만.

이 요구는 JOIN보다 집합 연산으로 더 직접 표현될 때가 있다.

무엇 — 합 · 교 · 차 🧮

(두 SELECT 결과를 집합처럼 합치는 연산)은 여러 SELECT 결과를 하나로 병합한다.

연산 의미 업무 질문
UNION 중복 제거하고 합침(기본이 DISTINCT) "A 또는 B에 있는 공연"
UNION ALL 중복 유지하고 합침 "두 출처의 행을 그대로 모두"
INTERSECT DISTINCT 양쪽에 모두 있는 행만 "전체 공연이면서 예약도 있는"
EXCEPT DISTINCT 왼쪽에 있고 오른쪽엔 없는 행만 "전체 공연 중 예약 없는"

예약이 없는 공연을 차집합으로 찾아보자. 두 SELECT는 같은 컬럼(공연 ID) 을 낸다.

SELECT concert_id
  FROM zconcert
EXCEPT DISTINCT
SELECT concert_id
  FROM zbooking
  WHERE status <> 'C'
INTO TABLE @DATA(lt_without_booking).

전체 공연에서 "취소 아닌 예약이 있는 공연"을 빼면 예약 없는 공연이 남는다. 교집합으로 바꾸면 반대로 "예약 있는 공연"이 나온다.

어떻게 확인 — 벤 다이어그램처럼 ✅

집합 연산은 왼쪽·오른쪽 집합을 따로 써 본 뒤 결과를 예상한다.

집합 값
전체 공연 A C001, C002, C003
예약 있는 공연 B C001, C003, C004
연산 예상 결과
A UNION DISTINCT B C001, C002, C003, C004
A UNION ALL B C001, C002, C003, C001, C003, C004
A INTERSECT DISTINCT B C001, C003
A EXCEPT DISTINCT B C002

흔한 실수와 점검 🚧

  • 컬럼 개수·타입 불일치 → 좌우 SELECT의 컬럼 개수가 같고 타입이 호환돼야 한다. *는 못 쓰고 컬럼을 명시한다.
  • EXCEPT 방향 뒤바꿈 → 차집합은 방향이 있다. A EXCEPT B(A에서 B를 뺌)와 B EXCEPT A는 다르다.
  • UNION ALL의 중복 → ALL은 중복을 남긴다. 중복 제거가 목적이면 기본 UNION(DISTINCT).
  • ORDER BY·INTO 위치 → 개별 SELECT가 아니라 병합 결과 맨 뒤에 둔다(INTO도 전체 문장 끝).

집합으로 합치고 뺐다. 이제 상세 행을 유지하면서 그룹 계산을 붙이는 window로 → Window Expression.
