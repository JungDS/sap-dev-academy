CH02 · 변수와 표준 타입 (+ Local Type)

Complete 타입(STRING·I·F·D·T)

길이를 따로 안 적어도 되는 완전한 표준 타입부터.

"길이를 안 적어도 되는" 타입

타입 중에는 그 자체로 완전해서 길이를 추가로 지정할 필요가 없는 것들이 있다. 이를 이라 한다. 입문 단계에서 가장 자주 쓴다.

대표 Complete 타입

타입 뜻 예
가변 길이 문자열 '안녕', 긴 텍스트
I 정수(integer) 42, -7
F 부동소수점(float) 3.14
D 날짜(YYYYMMDD, 8자리) '20260618'
T 시간(HHMMSS, 6자리) '143000' 
DATA: lv_text  TYPE string,
      lv_qty   TYPE i,
      lv_today TYPE d.

lv_qty   = 10 + 5.
lv_today = sy-datum.   " 시스템 현재 날짜
WRITE: / lv_qty, / lv_today.

sy-datum, sy-uzeit은 시스템 필드(System field)로 현재 날짜·시간을 담고 있다. D·T 변수에 바로 넣어 쓸 수 있다.

정수 연산의 함정

I는 정수라 나눗셈 결과가 반올림된다.

DATA lv_r TYPE i.
lv_r = 7 / 2.     " 3.5 → 4 (반올림)
WRITE lv_r.

소수를 정확히 다루려면 다음 레슨의 P(packed) 타입이 필요하다.

흔한 실수와 점검

  • 날짜를 순수 숫자로 착각 → D는 내부적으로 8자리 문자다. '20260618'처럼 다룬다.
  • 정수 나눗셈 반올림 → 7 / 2가 4가 된다. 의도와 다르면 P 타입을 쓴다.

여기까진 길이를 안 적었다. 그런데 "이름은 최대 10글자", "금액은 소수 2자리" 처럼 길이·자릿수를 내가 정해야 하는 타입이 있다 → 다음 레슨(Incomplete).
