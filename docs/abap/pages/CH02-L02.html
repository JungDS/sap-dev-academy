Complete 타입(STRING·I·F·D·T)

"길이를 안 적어도 되는" 타입

타입 중에는 그 자체로 완전해서 길이를 추가로 지정할 필요가 없는 것들이 있다. 이를 이라 한다. 입문 단계에서 가장 자주 쓴다.

대표 Complete 타입

타입 뜻 예
가변 길이 문자열 '안녕', 긴 텍스트
I 정수(integer) 42, -7
F 부동소수점(float) 3.14
D 날짜(YYYYMMDD, 8자리) '20260618'
T 시간(HHMMSS, 6자리) '143000'
ABAP

DATA: lv_text  TYPE string,
      lv_qty   TYPE i,
      lv_today TYPE d.

lv_qty   = 10 + 5.
lv_today = '20260623'.   " 날짜는 따옴표 안 8자리(YYYYMMDD)
WRITE: / lv_qty, / lv_today.

💡 "오늘 날짜"를 자동으로 채워 주는 시스템 값(sy-datum)도 있는데, 그건 흐름 제어를 배우는 CH04에서 다른 시스템 값들과 함께 정리한다. 지금은 날짜를 직접 적어 넣는다.

숫자를 적는 법

숫자 리터럴은 따옴표 없이 그대로 적는다 — 정수 42, 음수 -7. 그런데 소수는 따옴표로 감싼다: '123.45'. ABAP에서 마침표 .는 문장의 끝이라, 따옴표 없이 123.45라고 쓰면 123 다음에서 문장이 끊겨 버리기 때문이다.

ABAP
1
2
3
4
DATA lv_price TYPE p LENGTH 8 DECIMALS 2.

lv_price = '123.45'.    " O — 소수는 따옴표 안에
" lv_price = 123.45.    " X — '.'이 문장 끝으로 해석됨

타입이 다른 값을 대입하면 ABAP이 자동으로 변환한다(예: 문자 '10' → 정수 10). 다만 받는 변수가 짧으면 잘릴 수 있으니 길이에 주의한다.

정수 연산의 함정

I는 정수라 나눗셈 결과가 반올림된다.

ABAP
1
2
3
DATA lv_r TYPE i.
lv_r = 7 / 2.     " 3.5 → 4 (반올림)
WRITE lv_r.

소수를 정확히 다루려면 다음 레슨의 P(packed) 타입이 필요하다.

흔한 실수와 점검

  • 날짜를 순수 숫자로 착각 → D는 내부적으로 8자리 문자다. '20260618'처럼 다룬다.
  • 정수 나눗셈 반올림 → 7 / 2가 4가 된다. 의도와 다르면 P 타입을 쓴다.

여기까진 길이를 안 적었다. 그런데 "이름은 최대 10글자", "금액은 소수 2자리" 처럼 길이·자릿수를 내가 정해야 하는 타입이 있다 → 다음 레슨(Incomplete).
