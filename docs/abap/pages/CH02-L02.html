Complete 타입(STRING·I·F·D·T)

길이를 따로 안 적어도 되는 완전한 표준 타입부터.

Track 1Chapter 2Lesson 2입문

"길이를 안 적어도 되는" 타입

타입 중에는 그 자체로 완전해서 길이를 추가로 지정할 필요가 없는 것들이 있다. 이를 이라 한다. 입문 단계에서 가장 자주 쓴다.

대표 Complete 타입

타입 뜻 크기 · 범위 예
가변 길이 문자열 길이 가변 — 내용만큼 자람 '안녕', 긴 텍스트
I 정수(integer) 4 · -2,147,483,648 ～ +2,147,483,647 (약 ±21억) 42, -7
F (float) 8바이트 · 약 ±1.8 × 10³⁰⁸까지 · 근삿값(유효숫자 15～16자리) 3.14
D 날짜 8자리 문자 · YYYYMMDD (0001～9999년) '20260618'
T 시간 6자리 문자 · HHMMSS (000000～235959) '143000'

💡 왜 Complete인가 — I·F·D·T는 길이가 고정돼 있고 STRING은 가변(내용만큼 자람)이라, 어느 쪽도 길이를 따로 적을 필요가 없다. 그래서 Complete다. (±21억을 넘는 정수가 필요하면 INT8(8바이트)도 있다.)

DATA: gv_text  TYPE string,
      gv_qty   TYPE i,
      gv_today TYPE d.

gv_qty   = 10 + 5.
gv_today = '20260623'.   " 날짜는 따옴표 안 8자리(YYYYMMDD)
WRITE: / gv_qty, / gv_today.

💡 "오늘 날짜"를 자동으로 채워 주는 시스템 값(sy-datum)도 있는데, 그건 흐름 제어를 배우는 CH04에서 다른 시스템 값들과 함께 정리한다. 지금은 날짜를 직접 적어 넣는다.

숫자를 적는 법

숫자 은 따옴표 없이 그대로 적는다 — 정수 42, 음수 -7. 그런데 소수는 따옴표로 감싼다: '123.45'. ABAP에서 마침표 .는 문장의 끝이라, 따옴표 없이 123.45라고 쓰면 123 다음에서 문장이 끊겨 버리기 때문이다.

ABAP
1
2
3
4
DATA gv_price TYPE p LENGTH 8 DECIMALS 2.

gv_price = '123.45'.    " O — 소수는 따옴표 안에
" gv_price = 123.45.    " X — '.'이 문장 끝으로 해석됨

타입이 다른 값을 대입하면 ABAP이 자동으로 변환한다(예: 문자 '10' → 정수 10). 다만 받는 변수가 짧으면 잘릴 수 있으니 길이에 주의한다.

정수 연산의 함정

I는 정수라 나눗셈 결과가 반올림된다.

ABAP
1
2
3
DATA gv_r TYPE i.
gv_r = 7 / 2.     " 3.5 → 4 (반올림)
WRITE gv_r.

소수를 정확히 다루려면 다음 레슨의 P(packed) 타입이 필요하다.

흔한 실수와 점검

  • 날짜를 순수 숫자로 착각 → D는 내부적으로 8자리 문자다. '20260618'처럼 다룬다.
  • 정수 나눗셈 반올림 → 7 / 2가 4가 된다. 의도와 다르면 P 타입을 쓴다.

여기까진 길이를 안 적었다. 그런데 "이름은 최대 10글자", "금액은 소수 2자리" 처럼 길이·자릿수를 내가 정해야 하는 타입이 있다 → 다음 레슨(Incomplete).
