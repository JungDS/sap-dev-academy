Constructor와 객체 초기화

왜 — 초기화를 깜빡하면 🫥

를 만든 뒤 매번 초기화 를 따로 불러야 한다면 실수가 생긴다.

DATA(lo_mgr) = NEW zcl_booking_manager( ).
lo_mgr->set_concert( iv_concert = 'C001' ).
lo_mgr->load_capacity( ).            " ← 이걸 빼먹으면?

누군가 load_capacity( )를 빼먹으면, 예약 관리자는 정원도 모른 채 잔여석을 계산하려 든다. 객체는 만들어졌지만 아직 쓸 준비가 안 된 상태다.

CONSTRUCTOR — 태어날 때 자동 실행 🐣

NEW로 를 만드는 순간 자동으로 실행되는 가 (생성자)다. 꼭 필요한 값을 IMPORTING으로 받아, 객체를 즉시 사용 가능한 상태로 만든다.

CLASS zcl_booking_manager DEFINITION PUBLIC CREATE PUBLIC.
  PUBLIC SECTION.
    METHODS constructor IMPORTING iv_concert TYPE zconcert-concert_id
                                  iv_perf    TYPE zbooking-perf_no.
  PRIVATE SECTION.
    DATA mv_concert TYPE zconcert-concert_id.
    DATA mv_perf    TYPE zbooking-perf_no.
    DATA mv_cap     TYPE i.
ENDCLASS.

CLASS zcl_booking_manager IMPLEMENTATION.
  METHOD constructor.
    mv_concert = iv_concert.          " 입력값 → 객체 상태로 보관
    mv_perf    = iv_perf.
    SELECT SINGLE capacity FROM zconcert
      WHERE concert_id = @iv_concert INTO @mv_cap.   " 정원 로드
  ENDMETHOD.
ENDCLASS.

객체를 만들 때 생성자 인자를 넘긴다.

ABAP
1
DATA(lo_mgr) = NEW zcl_booking_manager( iv_concert = 'C001' iv_perf = '01' ).

💡 iv_concert는 생성할 때 전달된 입력값, mv_concert는 객체가 이후 계속 보관할 상태다. 둘은 다른 것이라, 입력값을 속성에 대입해 줘야 객체가 그 값을 기억한다.

정적 생성자 — CLASS_CONSTRUCTOR 🏷️

(class_constructor)도 있다. 가 처음 쓰일 때 딱 한 번 실행되는 클래스 공용 초기화다.

CLASS zcl_booking_manager DEFINITION PUBLIC CREATE PUBLIC.
  PUBLIC SECTION.
    CLASS-METHODS class_constructor.
  PRIVATE SECTION.
    CLASS-DATA gv_version TYPE string.
ENDCLASS.

CLASS zcl_booking_manager IMPLEMENTATION.
  METHOD class_constructor.
    gv_version = '1.0'.
  ENDMETHOD.
ENDCLASS.

constructor는 별 초기화(객체마다 실행), class_constructor는 클래스 공용 초기화(최초 1회)다. 위 체험에서 객체를 둘 만들면 constructor는 두 번, class_constructor는 한 번만 실행되는 걸 볼 수 있다.

흔한 실수와 점검 🚧

  • 생성자에 무거운 작업 → 만드는 모든 곳이 느려진다. 정원 조회처럼 꼭 필요한 초기 상태만. 대량 통계·ALV 준비까지 넣지 않는다.
  • 입력값을 속성에 대입 안 함 → iv_concert만 받고 mv_concert에 안 넣으면 입력값이 사라지고 객체 상태가 빈다.
  • class_constructor로 객체 초기화 → 전체에 한 번 실행되니 객체마다 다른 공연 ID·회차를 담을 수 없다. 객체별 값은 constructor로.
  • 없는 공연 ID로 객체 생성 → 이 예제는 공연이 실제로 있다는 전제다. 입력값이 올바른지는 선택화면 단계에서 이미 걸러진 것으로 본다.

객체를 초기화했다. 이제 객체가 필요한 것()과 필요 없는 것(정적)을 나누자 → Static Method와 Instance Method.
