버튼이 눌렸을 때

추가한 버튼(ZCANCEL)이 눌리면 user_command 가 온다. 명령 코드로 분기해 처리한다.

METHODS on_user_command
  FOR EVENT user_command OF cl_gui_alv_grid
  IMPORTING e_ucomm.

METHOD on_user_command.
  CASE e_ucomm.
    WHEN 'ZCANCEL'.
      " 선택한 행들을 가져와 취소 처리
      go_grid->get_selected_rows( IMPORTING et_index_rows = DATA(lt_rows) ).
      LOOP AT lt_rows INTO DATA(ls_row).
        READ TABLE lt_booking INTO DATA(ls) INDEX ls_row-index.
        " ls-booking_id 취소(DML+잠금: CH24/25)
      ENDLOOP.
      go_grid->refresh_table_display( ).
  ENDCASE.
ENDMETHOD.
  • get_selected_rows( ) — 사용자가 고른 행들.
  • 처리 후 refresh_table_display( )(CH17/21)로 갱신.
  • 실제 변경은 CH24(DML)·CH25(잠금) 패턴으로.

선택 → 취소 → 갱신 체험

행을 체크해 고르고 예매 취소 버튼을 누르면 user_command가 ZCANCEL로 분기해 선택 행을 취소하고 화면을 갱신한다. 아무것도 안 고르고 누르면 어떻게 되는지도 확인해 보라.

흔한 실수와 점검

  • 선택 없음 처리 → 아무 행도 안 골랐을 때 안내.
  • 갱신 누락 → 처리 후 refresh 안 하면 화면이 옛 상태.

버튼 처리까지 했다. 이 핸들러들을 깔끔히 모으는 설계 → 다음 레슨.
