Internal Table

Table Type(Local·DDIC)

내부 테이블의 타입도 로컬 TYPES와 전역 DDIC Table Type으로.

Track 1Chapter 5Lesson 2초급

테이블 모양에도 이름을

TYPE TABLE OF ty_person을 여러 곳에 반복하면, 구조체 때와 같은 불편이 온다. 테이블 타입에도 이름을 붙여 재사용한다.

로컬 Table Type — TYPES

ABAP
1
2
3
TYPES ty_person_tab TYPE STANDARD TABLE OF ty_person.

DATA lt_person TYPE ty_person_tab.
  • STANDARD TABLE은 가장 기본 종류. (TABLE OF만 써도 기본은 STANDARD.)

전역 DDIC Table Type — SE11

SE11 → Data Type → Table Type → ZTT_PERSON → 행 타입에 구조 ZST_PERSON 지정 → 활성화.

ABAP
1
DATA lt_person TYPE ztt_person.

여러 프로그램이 같은 테이블 타입을 공유한다(구조·라벨까지 일관).

종류 살짝 맛보기

종류 특징 언제
STANDARD 입력 순서 유지, 인덱스 접근 기본
SORTED 키로 자동 정렬·빠른 탐색 정렬 유지 필요
HASHED 키로 초고속 단건 접근 유일 키 대량

지금은 STANDARD로 충분하다. 나머지는 성능이 절실해질 때 다시 본다.

흔한 실수와 점검

  • 행 타입 혼동 → Table Type의 "행"은 구조(Structure)다. 행 타입을 먼저 정한다.
  • 과한 선택 → 처음부터 SORTED/HASHED 남발 금지. 기본은 STANDARD.

타입까지 정리했다. 이제 쌓은 행들을 읽고 고치는 법이다 → LOOP·READ·MODIFY.
