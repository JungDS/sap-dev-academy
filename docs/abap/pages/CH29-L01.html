표준을 직접 고치면 안 된다

SAP 표준 프로그램을 직접 수정하면 업그레이드 때 덮어써지고, 지원도 못 받는다. SAP은 표준 안에 확장할 자리를 미리 마련해 둔다. 그 옛 방식이 User/Customer Exit다.

종류 형태
User Exit 표준이 부르는 빈 FORM(주로 SD/MM)
Customer Exit FUNCTION EXIT_...(SMOD/CMOD로 활성화)

FUNCTION exit_saplxxxx_001.

ENDFUNCTION.
  • SMOD(Exit 정의 조회)·CMOD(프로젝트로 활성화).
  • 오래된 방식이라 신규는 BAdI(L03)·Enhancement(L02)를 선호.

자리에 끼워 보기

표준이 마련한 Exit 자리에 내 코드를 넣고 빼 보라 — 표준 소스는 한 줄도 안 바뀌는데 동작만 확장된다.

흔한 실수와 점검

  • 표준 직접 수정 → 절대 금지. 제공된 Exit/Enhancement만.
  • 활성화 누락 → CMOD 프로젝트를 활성화해야 동작.

옛 방식을 봤다. 이제 코드 어디에나 끼우는 Enhancement Point → 다음 레슨.
