Grid ALV 기초

Variant 기본 설정

사용자가 컬럼 배치를 저장·재사용하는 Display Variant.

Track 1Chapter 17Lesson 6중급

사용자마다 다른 표 모양

같은 표라도 사람마다 보고 싶은 컬럼·순서가 다르다. ALV는 사용자가 컬럼을 바꿔 Variant(레이아웃) 로 저장하고 다시 불러 쓰게 해 준다.

ABAP
1
2
3
DATA ls_variant TYPE disvariant.

ls_variant-report = sy-repid.    " 이 프로그램 기준으로 Variant 구분
  • report = sy-repid — 어느 프로그램의 Variant인지 식별.
  • display 호출 시 is_variant로 넘기면, 사용자가 툴바에서 레이아웃을 저장/선택할 수 있다.

흔한 실수와 점검

  • report 미설정 → Variant 저장/불러오기가 꼬인다. sy-repid를 넣는다.
  • save 옵션 → 저장 범위(사용자/전역)는 display의 i_save로 정한다.

저장 가능한 표까지 만들었다. 이제 준비한 모든 것을 한 번에 띄우는 display 호출 → 다음 레슨.
