Grid ALV 기초

Variant 기본 설정

사용자가 표시 방식(컬럼 순서·필터)을 저장·재사용하는 Display Variant.

Track 1Chapter 17Lesson 6중급

사용자마다 다른 표 모양

현업 사용자는 같은 표를 같은 방식으로 보지 않는다. 회계 담당자는 금액 컬럼을 앞에, 운영 담당자는 상태·변경일을 먼저 보고 싶을 수 있다. 매번 컬럼을 드래그하고 필터를 다시 잡게 하면 화면은 불편한 도구가 된다. ALV의 는 사용자가 자기 보기 방식을 저장하고 다시 불러오게 해 준다.

ABAP

DATA ls_variant TYPE disvariant.

ls_variant-report = sy-repid.    " 현재 프로그램 이름 → 프로그램별 Variant 구분
  • sy-repid는 현재 실행 중인 프로그램 이름이다. Variant를 프로그램별로 구분하려면 필요하다.
  • display 호출에서 is_variant와 i_save를 함께 넘긴다(i_save = 'A' = 저장 허용 — 교육용).

go_grid->set_table_for_first_display(
  EXPORTING is_variant = ls_variant
            i_save     = 'A'
            is_layout  = ls_layout
  CHANGING  it_outtab       = lt_booking
            it_fieldcatalog = lt_fcat ).

💡 Variant는 데이터 저장이 아니다. 예매 데이터가 저장되는 게 아니라, 컬럼 순서· 필터·정렬 같은 표시 방식이 저장된다.

직접 저장·복원해 보기 — 사용자별 보기

사용자 A는 예매자·좌석수를 앞에, 사용자 B는 상태·예매번호를 앞에 두고 저장한다. 저장한 Variant를 다시 열면 그 컬럼 순서가 복원된다. report를 비우면 왜 기준이 약해지는지도 보자.

흔한 실수와 점검

  • report 미설정 → 어느 프로그램의 Variant인지 기준이 약해져 저장/조회가 꼬인다. sy-repid를 넣는다.
  • 저장 범위를 함부로 → 운영에서 전역 Variant를 아무나 바꾸게 하면 모든 사용자의 화면이 바뀐 것처럼 느낀다. i_save 범위(사용자/전역)는 프로젝트 규칙에 맞춘다.

정리

  • 이 레슨은 사용자가 표를 자기 방식으로 다시 볼 수 있게 하는 장치()를 연결한다.
  • DISVARIANT-report = sy-repid로 프로그램을 식별하고, is_variant·i_save를 display에 넘긴다.

저장 가능한 표까지 만들었다. 이제 데이터·field catalog·layout·variant를 한 번에 넘겨 실제 표를 띄우자 → SET_TABLE_FOR_FIRST_DISPLAY.
