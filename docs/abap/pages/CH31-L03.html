IDoc 오류 추적과 재처리

실패한 IDoc을 찾아 고치고 다시 처리한다.

Track 2Chapter 31Lesson 3고급

상태 코드로 추적

IDoc은 처리 단계마다 상태 코드를 남긴다. 모니터에서 실패를 찾아 원인을 본다.

상태 뜻
53 Inbound 성공
51 Inbound 오류(애플리케이션)
03 / 12 Outbound 전송됨
02 / 26 Outbound 오류
  • WE02/WE05 — IDoc 목록·상태 조회.
  • BD87 — 실패 IDoc을 재처리(원인 해결 후 다시 등록).

재처리 구조

CH24-L04·CH30-L05와 같은 정신 — 실패는 남기고, 원인 해결 후 다시.

  • 데이터 오류(51) → 마스터/매핑 수정 후 BD87 재처리.
  • 일시 오류 → 자동 재처리 잡(예약).

흔한 실수와 점검

  • 상태 51 방치 → 업무 문서가 누락된다. 정기 모니터링·알림.
  • 무한 재처리 → 데이터 오류는 고치기 전엔 계속 실패. 원인부터.

표준 메시지 연계를 다뤘다. 이제 웹·모바일과 잇는 게이트웨이 → 다음 레슨.
