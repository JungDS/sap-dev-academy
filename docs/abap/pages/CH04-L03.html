Structure (Local · DDIC)

구조체 다루기

구조체 복사·초기화·동일 이름 필드 옮기기(MOVE-CORRESPONDING).

Track 1Chapter 4Lesson 3초급

묶었으면 다뤄야 한다

구조체를 통째로 복사하거나, 비우거나, 이름이 같은 필드만 골라 옮기는 일이 자주 생긴다. 기본기 몇 가지를 익힌다.

통째로 대입 / 초기화

같은 타입이면 구조체끼리 통째로 대입된다. CLEAR로 전체를 비운다.

DATA: ls_a TYPE zst_person,
      ls_b TYPE zst_person.

ls_a-name = 'Kim'.
ls_b = ls_a.        " 모든 컴포넌트 복사
CLEAR ls_a.         " ls_a를 초기값으로

MOVE-CORRESPONDING — 이름이 같은 필드만

타입이 다른 두 구조 사이에서 이름이 같은 컴포넌트만 골라 옮긴다.

MOVE-CORRESPONDING ls_source TO ls_target.
  • 두 구조의 공통 이름 필드만 복사되고, 나머지는 건드리지 않는다.
  • 화면·테이블·DB 구조가 조금씩 다를 때 아주 유용하다.

현대 구문에서는 ls_target = CORRESPONDING #( ls_source ). 로도 쓴다. (현대 ABAP 구문은 CH17에서 본격적으로 다룬다 — 지금은 고전형으로 충분.)

흔한 실수와 점검

  • 타입 다른데 통째 대입 → 컴포넌트 구성이 다르면 오류/엉뚱한 결과. 다르면 .
  • CLEAR vs REFRESH → 구조체는 CLEAR. (여러 행을 담는 내부 테이블 비우기는 다음 챕터에서.)

한 건(구조체)은 자유자재로 다룬다. 그런데 사람이 한 명이 아니라 여러 명이면? 구조 하나로는 부족하다 — 같은 모양의 행을 여러 개 담고 싶다 → CH05(Internal Table).
