연산자와 흐름 제어

산술 연산과 대입 · 날짜 산술

값을 더하고 빼고 곱하고 나눈다 — 그리고 날짜도 계산한다.

Track 1Chapter 4Lesson 1입문

값을 받았지만, 계산을 못 한다

로 숫자를 입력받는 데까지 왔다(CH03). 그런데 막상 그 값으로 무언가를 계산하려니 방법이 없다. 입력받은 두 수를 더하고 싶고, 금액에 부가세를 곱하고 싶다. 변수에 값을 담는 것과, 그 값으로 계산하는 것은 다른 일이다.

이번 레슨에서 ABAP의 산술 연산자부터 익힌다.

사칙연산 — + - * / 와 제곱 **

연산 결과를 =로 변수에 담는다. =는 "오른쪽을 계산해서 왼쪽에 넣어라"라는 뜻이다.

ABAP
1
2
3
4
5
6
7
8
DATA gv_result TYPE i.

gv_result = 7 + 3.      " 10
gv_result = 7  3.      " 4
gv_result = 7 * 3.      " 21
gv_result = 2 ** 10.    " 1024 (2의 10제곱)

WRITE: / gv_result.

⚠️ 연산자 앞뒤에는 공백이 필수다. 7+3(X) → 7 + 3(O). ABAP은 토큰을 공백으로 끊어 읽기 때문에 붙여 쓰면 오류가 난다.

계산 순서 — 우선순위와 괄호

여러 연산이 한 줄에 섞이면 정해진 순서로 계산된다. 제곱 **이 가장 먼저, 그다음 *·/, 마지막으로 +·- 순이다. 같은 등급끼리는 왼쪽부터 차례로 한다.

ABAP
1
2
3
4
DATA gv_n TYPE i.

gv_n = 2 + 3 * 4.        " 14  — 곱하기가 먼저: 3 * 4 = 12, 거기에 + 2
gv_n = ( 2 + 3 ) * 4.    " 20  — 괄호가 먼저: 5 * 4

순서를 바꾸고 싶으면 괄호 ( )로 묶는다 — 묶은 부분이 먼저 계산된다. 이때도 괄호 안팎에 공백을 둔다: (2+3)(X) → ( 2 + 3 )(O).

나눗셈은 소수까지 — 타입에 주의

정수() 변수에 나눗셈 결과를 담으면 반올림되어 소수가 사라진다. 소수가 필요하면 (Packed) 타입을 쓴다.

ABAP
1
2
3
4
5
DATA gv_i   TYPE i.
DATA gv_dec TYPE p LENGTH 8 DECIMALS 2.

gv_i   = 10 / 4.    " 3  (2.5가 반올림됨!)
gv_dec = 10 / 4.    " 2.50

⚠️ 0으로 나누지 말 것 — 나누는 수가 0이면 나눗셈 자체가 정의되지 않아 프로그램이 런타임 오류(실행 도중 멈추는 오류)로 중단된다. 나누는 값이 0이 될 수 있는지 늘 확인한다.

정수 몫과 나머지 — DIV · MOD

/는 소수까지 계산하지만, 때로는 나눈 몫(정수) 과 나머지 만 따로 알고 싶다. 그때 DIV(몫)와 MOD(나머지)를 쓴다. 둘 다 + - * /와 같은 갈래의 산술 연산자다.

ABAP
1
2
3
4
5
DATA gv_q TYPE i.
DATA gv_r TYPE i.

gv_q = 17 DIV 5.     " 몫 → 3
gv_r = 17 MOD 5.     " 나머지 → 2

💡 MOD는 짝수·홀수 판별에 자주 쓴다 — n MOD 2가 0이면 짝수, 1이면 홀수다.

문장형 연산 — ADD · SUBTRACT · MULTIPLY · DIVIDE

= 말고, 문장처럼 읽히는 classic 연산문도 있다. 현업 코드에 자주 보이니 눈에 익혀 둔다.

ABAP
1
2
3
4
5
6
DATA gv_n TYPE i VALUE 100.

ADD 10 TO gv_n.          " gv_n = gv_n + 10  → 110
SUBTRACT 30 FROM gv_n.   " gv_n = gv_n - 30  → 80
MULTIPLY gv_n BY 2.      " gv_n = gv_n * 2   → 160
DIVIDE gv_n BY 4.        " gv_n = gv_n / 4   → 40

💡 ADD 10 TO gv_n 은 gv_n = gv_n + 10 과 똑같다. 둘 중 편한 쪽을 쓰면 된다.

대입과 초기화 — = 와 CLEAR

=로 값을 옮긴다. 변수를 처음 상태로 비우려면 를 쓴다(숫자는 0, 문자는 공백으로 돌아간다).

ABAP
1
2
3
DATA gv_amount TYPE i VALUE 500.

CLEAR gv_amount.     " gv_amount = 0

옛 코드에는 MOVE a TO b(= b = a)도 보이는데, 지금은 =가 표준이다. MOVE는 굳이 새로 쓰지 않는다(legacy).

날짜도 숫자다 — D 타입 산술

(날짜) 타입은 내부적으로 YYYYMMDD 8자리 숫자다. 그래서 날짜에 숫자를 더하면 그만큼 뒤의 날짜가 되고, 두 날짜를 빼면 그 사이의 일수가 된다.

ABAP
1
2
3
4
5
6
DATA gv_start TYPE d VALUE '20260101'.
DATA gv_due   TYPE d.
DATA gv_days  TYPE i.

gv_due  = gv_start + 30.        " 30일 뒤 → 20260131
gv_days = gv_due  gv_start.    " 두 날짜 차이 → 30

'20260101'처럼 따옴표 안 8자리로 날짜 을 적는다. 시간()도 같은 방식(HHMMSS 6자리)으로 더하고 뺄 수 있다.

흔한 실수와 점검

  • 연산자 공백 누락 → a+b가 아니라 a + b. 괄호도 ( a + b )처럼 띄운다.
  • 계산 순서 착각 → 2 + 3 * 4는 20이 아니라 14(곱하기 먼저). 원하는 순서는 괄호로.
  • 나눗셈 소수가 사라짐 → 결과 변수를 대신 (DECIMALS)로.
  • 0으로 나누기 → 런타임 오류로 멈춘다. 나누는 수가 0이 아닌지 확인.
  • 날짜 형식 → '20260101'(YYYYMMDD), 8자리·따옴표.

정리

값을 계산할 수 있게 됐다 — 사칙연산·ADD류·날짜 산술. 다음은 비슷한 결의 도구, 문자열을 다루는 법이다(잇고·자르고·바꾸기) → 다음 레슨(CH04-L02).
