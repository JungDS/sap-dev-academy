셋의 자리

수단 강점 언제
Open SQL / Modern SQL 이식성·간결·검증 ··식 — 대부분
CDS View 재사용·의미·시맨틱 공유 모델·뷰 계층
AMDP SQLScript 절차 로직 임시테이블·복잡 알고리즘

선택 순서

  1. Open/Modern SQL로 되면 그걸로.
  2. 재사용·모델이면 CDS.
  3. 절차적·복잡 로직이 DB에 꼭 필요하면 AMDP.
  4. DB 고유 기능 불가피하면 ADBC(최후).

위로 갈수록 이식성·유지보수·클라우드 친화. AMDP/ADBC는 강력하지만 DB 종속이라 신중히.

흔한 실수와 점검

  • AMDP 과용 → 대부분 CDS/SQL로 충분. 복잡할 때만.
  • CDS 무시 → 재사용 가능한 가공은 CDS로 끌어올린다.

수단을 골랐다. 마지막으로 운영 리스크와 DB 종속성 → 다음 레슨.
