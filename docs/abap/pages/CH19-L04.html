New Open SQL / Modern ABAP SQL

SQL Expression — CASE / CAST / COALESCE

SELECT 안에서 값을 계산·변환·치환한다 — DB가 직접.

왜 — DB에 계산을 시킨다 ⚙️

지금까지는 DB에서 데이터를 읽어 ABAP으로 가져온 뒤, LOOP를 돌며 상태 문구나 계산 값을 만들었다.

ABAP

LOOP AT lt_flight INTO DATA(ls).
  IF ls-seatsocc >= ls-seatsmax.
    " status = 'FULL'
  ELSE.
    " status = 'OPEN'
  ENDIF.
ENDLOOP.

로직이 ABAP에 보여 좋지만, 단순한 계산·상태 분류까지 전부 ABAP LOOP로 가져오면 DB가 잘할 일을 애플리케이션 서버로 떠넘기는 셈이다. SQL expression은 그 계산을 SELECT 안에서(즉 DB에서) 하게 한다. 결과 컬럼을 만들어 ABAP으로 넘겨 준다.

CASE — 조건에 따라 컬럼을 만든다 🔀

ABAP
1
2
3
4
5
6
SELECT carrid, connid, seatsocc, seatsmax,
       CASE WHEN seatsocc >= seatsmax THEN 'FULL'
            ELSE 'OPEN' END AS seat_status
  FROM sflight
  WHERE carrid = @lv_carr
  INTO TABLE @DATA(lt_status).

seat_status는 DB에서 계산된 결과 컬럼이다. ABAP에서 다시 LOOP를 돌며 상태를 붙일 필요가 없다. (특정 값을 비교하는 simple CASE, 조건식을 쓰는 searched CASE 두 형태가 있다.)

CAST — 타입을 바꾼다 🔁

ABAP
1
2
3
4
5
SELECT carrid,
       CAST( seatsmax AS CHAR( 5 ) ) AS seatsmax_text
  FROM sflight
  WHERE carrid = @lv_carr
  INTO TABLE @DATA(lt_text).

숫자 seatsmax를 문자형 결과 컬럼으로 만든다. 단, 길이를 의식해야 한다. CHAR( 5 )가 너무 짧으면 큰 값이 잘릴 수 있다.

COALESCE — null이면 대체값 🕳️

COALESCE는 첫 번째 non-null 값을 돌려준다. left outer join에서 오른쪽 테이블에 매칭이 없으면 그 컬럼은 null이 되는데, 그때 보여 줄 대체 값을 준다.

ABAP
1
2
3
4
5
SELECT c~carrid, c~carrname,
       COALESCE( p~cityto, 'NO ROUTE' ) AS city_to_text
  FROM scarr AS c
  LEFT OUTER JOIN spfli AS p ON p~carrid = c~carrid
  INTO TABLE @DATA(lt_route).

⚠️ COALESCE가 다루는 null은 DB 개념이다. ABAP의 빈 값(initial — 빈 문자열·0·초기 날짜)과는 다른 개념이니 혼동하지 말자.

흔한 실수와 점검 🚧

  • 계산 컬럼에 별칭 누락 → 결과 컬럼엔 AS 이름을 줘야 받기 쉽다.
  • 같은 SELECT list의 별칭을 그 안에서 재사용 → ... AS booked 만든 걸 같은 목록의 다른 식에서 booked로 쓸 수 없다. 식을 반복하거나 단계를 나눈다.
  • CASE 결과 타입 불일치 → THEN 'FULL'(문자)과 ELSE 0(숫자)을 섞지 않는다. 같은 계열로.
  • COALESCE ≠ initial 대체 → DB null을 다루는 함수다. ABAP initial과 구분.
  • 과한 SQL 가공 → 복잡한 업무 규칙·메시지·권한 분기는 ABAP/이후 장의 코드로. SQL 식은 단순 가공에.

DB에서 값을 가공했다. 이제 문자열·날짜 전용 SQL 함수도 보자 → SQL String / Date Function.
