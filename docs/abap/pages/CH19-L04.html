DB가 계산해서 돌려준다

modern SQL은 SELECT 목록에서 계산·분기·변환을 할 수 있다. ABAP으로 다시 LOOP 돌며 가공할 필요가 줄어든다(DB에서 처리 = 보통 더 빠름).

SELECT carrid, connid,
       seatsocc,
       seatsmax,
       CASE WHEN seatsocc >= seatsmax THEN 'FULL'
            ELSE 'OPEN' END AS status      " 컬럼을 계산
  FROM sflight
  INTO TABLE @DATA(lt_flight).

CAST · COALESCE

SELECT carrid,
       CAST( seatsmax AS CHAR ) AS seats_text,   " 타입 변환
       COALESCE( cityto, 'N/A' ) AS dest         " NULL이면 대체값
  FROM spfli
  INTO TABLE @DATA(lt).
  • CASE WHEN ... THEN ... ELSE ... END — 조건 컬럼.
  • CAST( x AS 타입 ) — 타입 변환.
  • COALESCE( a, b ) — a가 NULL이면 b.
  • 계산 컬럼엔 **AS 별칭**으로 이름을 준다.

흔한 실수와 점검

  • 별칭 누락 → 계산 컬럼엔 AS로 이름을 줘야 받기 쉽다.
  • 과한 SQL 가공 → 복잡한 업무 로직은 ABAP/CDS로. SQL 식은 단순 가공에.
