METHOD on_data_changed.
  LOOP AT er_data_changed->mt_good_cells INTO DATA(c).
    CASE c-fieldname.
      WHEN 'SEATS'.
        IF c-value > '10'.
          er_data_changed->add_protocol_entry(
            i_msgid = 'ZMSG' i_msgty = 'E' i_msgno = '002'
            i_msgv1 = '1인 최대 10석'
            i_fieldname = c-fieldname i_row_id = c-row_id ).
        ENDIF.
    ENDCASE.
  ENDLOOP.
  " 오류가 있으면 ALV가 자동으로 프로토콜 팝업·셀 강조
  er_data_changed->display_protocol( ).
ENDMETHOD.
  • add_protocol_entry — 오류 메시지+위치 등록.
  • display_protocol( ) — 오류 목록을 사용자에게 표시.

흔한 실수와 점검

  • 검증 후 통과 처리 → 오류 없으면 값이 그대로 반영된다(정상).
  • 메시지 → i_msgid는 SE91 메시지 클래스(CH15).

입력 단계 검증을 했다. 마지막으로 DB에 쓰기 직전 최종 검증 → 다음 레슨.
