Transparent Table (SE11)

Transparent ↔ Structure ↔ Table Type 비교

같은 DDIC 모양이 쓰임에 따라 작업영역·내부테이블·영속테이블이 된다.

Track 1Chapter 7Lesson 3초급

같은 모양, 다른 쓰임

지금까지 "구구단 한 줄"(dan·mul·result)이라는 같은 모양을 세 가지로 썼다. 헷갈리기 쉬우니 한 번에 정리한다.

무엇 어떻게 사는 곳 용도
Structure(Work Area) TYPE ty_line 메모리(한 건) 한 줄을 담아 다룸
Internal Table TYPE TABLE OF ty_line 메모리(여러 건) 여러 줄을 모아 처리
Transparent Table SE11 DB 테이블 디스크(영속) 영구 저장·공유

핵심: 앞의 둘은 메모리라 프로그램이 끝나면 사라지고, Transparent Table만 남는다.

DDIC 구조를 TYPE로 / TABLE OF로

전역 DDIC Structure(예: ZST_LINE) 하나를 만들어 두면, 프로그램에서 작업영역으로도 내부 테이블로도 쓸 수 있다.

ABAP
1
2
DATA ls_line TYPE zst_line.            " 작업영역(한 건)
DATA lt_line TYPE TABLE OF zst_line.   " 내부 테이블(여러 건)

DB에 영구 저장이 필요하면 같은 모양을 Transparent Table로 만든다. 즉 모양은 공유, 쓰임(메모리냐 디스크냐)만 다르다.

Transparent Table 본래 목적 = 영속

Structure·Internal Table은 "지금 이 실행 동안" 쓰는 작업 도구다. Transparent Table은 "실행이 끝나도 남아야 하는 업무 데이터"를 위한 것이다 — 이게 본래 목적이다.

흔한 실수와 점검

  • 메모리 테이블에 영속 기대 → 내부 테이블은 끝나면 사라진다. 영구 저장은 DB 테이블.
  • 셋을 같은 것으로 → 모양만 같을 뿐 사는 곳·수명이 다르다.
