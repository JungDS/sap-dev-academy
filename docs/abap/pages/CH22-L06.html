데이터 행 단위 권한

CDS는 DCL(Data Control Language)로 "누가 어떤 행을 볼 수 있는지"를 선언한다. 프로그램마다 권한 코드를 짜는 대신, 뷰에 권한을 선언적으로 붙인다.

@MappingRole: true
define role ZI_Concert_Role {
  grant select on ZI_Concert
    where ( venue ) = aspect pfcg_auth( Z_VENUE_AUTH, VENUE, ACTVT = '03' );
}
  • define role — 접근 규칙.
  • grant select on 뷰 where 조건 — 어떤 행을 허용할지.
  • aspect pfcg_auth(...) — 기존 권한 (PFCG)와 연결.
  • 뷰에 @AccessControl.authorizationCheck: #CHECK를 주면 DCL이 적용된다.

행 단위 보안을 코드가 아니라 모델에 둬, 모든 소비처에 일관 적용한다.

흔한 실수와 점검

  • authorizationCheck 미설정 → DCL을 만들어도 뷰가 #NOT_REQUIRED면 적용 안 됨.
  • 권한 없음 → pfcg_auth가 참조할 권한 객체가 있어야 한다.

CDS로 데이터를 현대적으로 모델링·노출·보호했다. 배운 CDS를 콘서트앱에 적용해 ZI_Concert/ZC_Concert를 만들자 → CH22-L07(실습).
