색을 줄 세 단위 🎨

색을 배운 뒤 흔한 실수는 모든 걸 색으로 해결하려는 것이다. 좋은 화면은 색을 많이 쓰는 화면이 아니라 색의 단위가 정확한 화면이다.

단위 방법 언제 콘서트 앱 예
행 색 layout info_fname(행 색상 기초) 한 행 전체의 상태가 중요할 때 지난 회차 전체를 회색
컬럼 색 fieldcat emphasize 특정 컬럼이 항상 중요할 때 잔여석 컬럼을 늘 강조
셀 색 layout ctab_fname + LVC_T_SCOL 행마다 한 칸만 조건부일 때 잔여석 0인 그 셀만 빨강

세 단위는 연결하는 곳이 다르다.

" 컬럼 색 — field catalog의 그 컬럼에 emphasize를 준다
LOOP AT gt_fcat ASSIGNING FIELD-SYMBOL(<fc>).
  IF <fc>-fieldname = 'SEATS_LEFT'.
    <fc>-emphasize = 'C500'.        " 잔여석 컬럼을 늘 강조
  ENDIF.
ENDLOOP.
ABAP
1
2
" 셀 색 — layout에 LVC_T_SCOL 컬럼명을 연결(이전 레슨)
gs_layout-ctab_fname = 'CELLCOLORS'.

💡 emphasize 값은 'C' + 색 번호(1～7) + 진하게(0/1) + 반전(0/1) 형식이다. 'C500' = 색 5(초록 계열)·연하게·정상. 잔여석처럼 "있으면 좋은 값"은 초록, 위험 신호는 'C600'(빨강)처럼 색 번호를 골라 쓴다.

🎯 콘서트 앱: 지난 회차 전체는 행 색, 매진·임박처럼 한 칸의 상태는 잔여석 셀 색으로 나눠 쓸 수 있다. (다음 실습은 매진·임박을 셀 색으로 강조한다.)

화면에서 이렇게 확인한다 🔍

위 비교기의 버튼을 눌러 같은 표가 어떻게 달라지는지 본다.

  1. 행 색 → 지난 회차 줄 전체가 회색이 되는지(줄 단위).
  2. 컬럼 색 → SEATS_LEFT 컬럼 전체가 강조되는지(컬럼 단위).
  3. 셀 색 → 잔여 0인 그 칸만 빨강이 되는지(칸 단위).
  4. 세 화면을 번갈아 보며 "지금 강조하려는 게 줄인가·컬럼인가·한 칸인가"를 스스로 물어본다.

정리 📌

  • 색은 많이 쓰는 게 아니라 단위가 정확해야 한다: 줄 전체=행 색, 늘 중요한 컬럼=컬럼 색, 행마다 다른 한 칸=셀 색.
  • 셀 색은 가장 세밀하지만 deep 구조·계산 비용이 크니, 행/컬럼으로 충분하면 그쪽을.
  • 같은 의미의 색은 화면이 바뀌어도 일관되게, 그리고 색만으로 전달하지 말고 텍스트·아이콘을 함께.

흔한 실수와 점검 🚧

  • 과한 색 → 다 칠하면 강조가 무의미. 정말 중요한 것만.
  • 같은 의미를 행+셀 중복 강조 → 전체 행이 문제인지, 한 셀만 문제인지 결정해서 하나로.
  • 색만으로 전달 → 시스템 테마에 따라 색감이 달라진다. STATUS_TEXT·아이콘·tooltip을 함께 두어 접근성을 챙긴다(색만 의존 ✕).
  • 셀 색 남용 → deep 구조·계산 비용이 든다. 행/컬럼으로 충분하면 그쪽을.

색의 단위를 정확히 골랐다. 마지막으로 데이터를 갱신해도 보던 위치를 지키는 Stable Refresh → Stable Refresh와 표시 상태 보존.
