개발 패키지와 이송요청 입문

$TMP의 한계를 넘어, 개발 객체를 패키지에 보관하고 이송요청으로 관리하는 첫걸음.

$TMP의 한계

작성 예정: 불편 회수 — Local Object()는 다른 시스템으로 이송 못 하고 팀 공유도 안 된다.

개발 패키지란

작성 예정: 개발 객체를 묶는 그릇. Z/Y 네임스페이스. SE80/SE21에서 생성.

이송요청(Transport Request)이 뜨는 이유

작성 예정: 패키지에 객체를 저장하면 (Workbench Request)에 기록된다 — 변경 추적·이송 단위.

Transport Organizer 맛보기 (SE09/SE10)

작성 예정: 내 요청 목록 확인까지만. 깊은 Release/Import는 뒤로 미룬다(→ CH34).

흔한 실수와 점검

작성 예정: $TMP에 둔 객체를 나중에 패키지로 옮기기, 요청 번호 혼동.
