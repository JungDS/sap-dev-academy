프로그램을 넘어 재사용

은 한 프로그램 안에 갇힌다. 여러 프로그램이 공유하는 재사용 블록이 (함수 모듈)이다. SE37에서 만들고 관리한다.

CALL FUNCTION 'Z_ADD_TAX'
  EXPORTING
    iv_amount = 1000          " 넘겨줄 값
  IMPORTING
    ev_result = lv_out        " 돌려받을 값
  EXCEPTIONS
    invalid   = 1
    OTHERS    = 2.

IF sy-subrc <> 0.
  " 예외 처리
ENDIF.
  • EXPORTING — 함수에 넘기는 값(호출자 입장).
  • IMPORTING — 함수에서 돌려받는 값.
  • CHANGING — 넘겼다가 수정되어 돌아오는 값(입출력).
  • TABLES — 내부 테이블 전달(레거시 방식).
  • EXCEPTIONS — 오류를 sy-subrc로 받는다.

방향이 헷갈린다: 호출자 기준이다. 내가 함수로 EXPORT(내보냄) → 함수가 받음.

어디서 찾나

표준 함수도 많다. SE37에서 검색하거나, BAPI(Chapter 30 · BAPI)처럼 표준 인터페이스로 제공된다.

직접 해 보기

정상/음수 금액을 호출해 sy-subrc가 어떻게 바뀌는지, EXPORTING·IMPORTING을 호출자 기준으로 읽는다는 게 무슨 뜻인지 확인해 보자.

흔한 실수와 점검

  • EXPORTING/IMPORTING 방향 혼동 → 호출자 기준으로 생각.
  • EXCEPTIONS 무시 → sy-subrc 확인 누락은 단골 버그.

함수 모듈은 절차적 재사용의 표준이었다. 그런데 현대 ABAP의 중심은 클래스다 — 가볍게 맛보자 → Local Class(다음 레슨).
