여정의 종착점 — 내 앱을 완성한다

CH01의 첫 WRITE부터 여기까지, 콘서트 예매(ZCONCERT/ZPERF/ZBOOKING)를 한 단계씩 키웠다. 이제 RAP + Fiori로 전체를 하나의 실전 앱으로 완성한다.

시나리오

사용자(정훈영)가 Fiori 화면에서 공연·회차를 보고 예매한다. 정원을 넘으면 막고, 매진을 표시하며, 취소도 가능하다. 변경은 저장 전 Draft로 보관된다.

무엇을 쓰나 — 통합 지도

계층 기술 챕터
데이터 테이블·DDIC CH03·07·09
모델 CDS(ZI_/ZC_·Association) CH23
동작 RAP(BDEF/BIMP·Action·Validation) CH24
서비스 Service Binding(OData) CH24
UI Fiori Elements CH39-L06
운영 권한·이송·로그 CH32·35

흔한 실수와 점검

  • 한 번에 다 만들기 → 계층을 순서대로(데이터→모델→동작→서비스→UI) 쌓는다.
  • 범위 과욕 → 핵심 흐름(예매·취소·검증)부터 완성하고 확장.

만들 것을 정의했다. 토대인 Interface View부터 → 다음 레슨.
