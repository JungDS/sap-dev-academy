Structure (Local · DDIC)

구조 재사용 — 중첩 · .INCLUDE · .APPEND

구조를 구조로 — 끼워 넣고 펼쳐 담아 재사용한다.

Track 1Chapter 5Lesson 3초급

구조를 또 짜기 싫다

주소(시·도·우편번호)나 감사필드(생성자·생성일)처럼 같은 필드 묶음을 여러 구조에 매번 다시 적기는 번거롭다. DDIC는 이미 만든 구조를 다른 구조에 끼워 넣어 재사용한다. 방법이 세 가지인데, "어떻게 들어가느냐"가 다르다.

① 중첩 — 구조를 컴포넌트로

의 타입에 다른 구조를 주면, 구조 안에 구조가 들어간다(중첩 구조).

DATA: BEGIN OF gs_order,
        head TYPE zst_order_head,   " 구조를 통째로 한 컴포넌트로
        qty  TYPE i,
      END OF gs_order.

gs_order-head-customer = '정훈영'.   " 한 단계 더 -로 파고든다
  • 접근이 gs_order-head-customer처럼 단계가 생긴다(-를 두 번).
  • "고객 정보는 head 안에"처럼 묶음이 그대로 보존된다.

② .INCLUDE — 펼쳐서 담기

DDIC Structure 편집 화면에서 다른 구조를 .INCLUDE로 포함하면, 그 구조의 필드들이 평평하게(직접 필드처럼) 들어온다.

  • 중첩은 ls-head-customer로 들어가지만, .INCLUDE는 펼쳐져 **ls-customer**로 바로 접근한다.
  • "묶음 티 안 나게, 내 필드인 것처럼" 쓰고 싶을 때.
중첩() .INCLUDE
접근 ls-head-customer ls-customer(펼쳐짐)
느낌 묶음 유지 내 필드처럼

③ .APPEND — 표준에 안전하게 더하기

.APPEND(Append Structure)는 SAP 표준 구조·테이블을 직접 고치지 않고, 내 필드를 별도 구조로 덧붙인다. 표준 원본을 안 건드려 시스템 업그레이드에 안전하다(확장 주제 — 자세히는 CH29).

끼워 넣기(.INCLUDE)=남의 필드를 내 것처럼 펼쳐 담기, 덧붙이기(.APPEND)=표준에 안전하게 더하기. 묶음을 유지하려면 중첩.

흔한 실수와 점검

  • 중첩인데 - 한 번 → 중첩은 ls-head-customer(단계마다 -). 단계를 빠뜨리지 않는다.
  • 표준 구조 직접 수정 → 업그레이드 때 덮어쓰여 사라진다. 표준 확장은 .APPEND로.

구조를 재사용하는 법까지 익혔다. 이제 묶은 구조체를 다루는 요령(복사·초기화)을 보자 → 다음 레슨.
