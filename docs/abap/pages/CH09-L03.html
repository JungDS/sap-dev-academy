DDIC 관계와 입력도움말(F4)

Text Table — 코드 옆 이름표

코드만 든 마스터에 사람이 읽을 이름을 — 언어별 Text Table.

Track 1Chapter 9Lesson 3초급

코드만으론 못 읽는다

마스터 테이블은 보통 코드(예: 공연번호 C001)만 키로 든다. 그런데 화면·목록엔 사람이 읽을 이름(공연명)이 필요하다. 게다가 이름은 언어마다 다르다(한국어 "콘서트", 영어 "Concert"). 이 이름을 별도 테이블에 두는 게 (텍스트 테이블)다.

Text Table의 모양

핵심은 키 구성이다 — 마스터 키 + 언어(SPRAS).

테이블 키 담는 것
ZCONCERT(마스터) CONCERT_ID 정원·장소 등
ZCONCERT_T(Text) CONCERT_ID + SPRAS 공연명(언어별)
  • 같은 C001이라도 언어(SPRAS)가 다르면 다른 줄 → 언어별 이름을 따로 보관.
  • 마스터와는 (CH09-L01)로 잇고, SE11에서 그 관계를 로 지정한다.

무엇이 좋아지나

  • 화면··가 코드 옆에 이름을 자동으로 보여 준다(코드만 보면 무슨 공연인지 모르니까).
  • 사용자의 로그인 언어에 맞는 이름이 자동으로 뜬다.

💡 표준 예: 많은 마스터가 코드(마스터 테이블)와 언어별 텍스트(텍스트 테이블)를 나눠 둔다. "코드는 마스터, 이름은 "로 기억하자.

직접 해 보기

로그인 언어를 KO/EN으로 바꿔 F4 목록의 이름이 어떻게 달라지는지, SPRAS를 빼면 왜 다국어가 깨지는지 확인해 보자.

흔한 실수와 점검

  • 언어 키(SPRAS) 누락 → 텍스트 테이블의 키엔 반드시 언어가 들어간다. 빠지면 다국어가 안 된다.
  • 마스터에 이름 직접 → 다국어가 필요하면 텍스트는 로 분리한다.

코드 옆 이름표까지 붙였다. 이제 사용자가 목록에서 골라 입력하게 하는 로 → 다음 레슨.
