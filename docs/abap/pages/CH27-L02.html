한 번 클릭하는 링크

Double Click Event는 행 전체를 상세로 보냈다. 그런데 모든 셀을 더블클릭 대상으로 만들면 사용자는 어디를 눌러야 할지 헷갈린다. 예매 목록에서 공연ID·고객번호처럼 "눌러서 이동할 값"만 링크처럼 보여 주는 게 낫다. 그게 Hotspot이다.

핵심은 핸들러만 만들면 끝이 아니라는 점이다. 먼저 Field Catalog에서 그 컬럼의 hotspot 속성을 켜야 화면에서 링크처럼 보이고, 사용자의 한 번 클릭이 hotspot_click 로 이어진다.

컬럼을 hotspot으로 — Field Catalog

LOOP AT lt_fcat REFERENCE INTO DATA(lr_fcat).
  IF lr_fcat->fieldname = 'CONCERT_ID'.
    lr_fcat->hotspot = abap_true.        " 이 컬럼을 링크처럼
  ENDIF.
ENDLOOP.

핸들러 — hotspot_click

hotspot_click은 클릭된 행과 컬럼을 넘긴다. e_column_id-fieldname으로 의도한 컬럼일 때만 반응한다(핸들러는 앞 레슨처럼 생성자로 주입받은 예매 테이블 mt_booking을 든다).

METHODS on_hotspot_click
  FOR EVENT hotspot_click OF cl_gui_alv_grid
  IMPORTING e_row_id e_column_id.

METHOD on_hotspot_click.
  IF e_column_id-fieldname <> 'CONCERT_ID'.
    RETURN.                              " 다른 컬럼이면 무시
  ENDIF.
  READ TABLE mt_booking INTO DATA(ls_booking) INDEX e_row_id-index.
  IF sy-subrc <> 0.
    RETURN.
  ENDIF.
  MESSAGE |공연 { ls_booking-concert_id } 상세로 이동| TYPE 'I'.
ENDMETHOD.
  • e_column_id-fieldname — 클릭한 컬럼명. e_row_id-index — 클릭한 화면 행.
  • 연결은 앞 레슨처럼 SET HANDLER ... FOR go_grid로 등록한다.

직접 클릭해 보기

concert_id가 밑줄 링크로 보인다. 한 번 클릭하면 hotspot_click이 행·컬럼과 함께 실행되는 걸 확인해 보라(더블클릭 아님).

🔁 더블클릭 ↔ Hotspot — double_click은 행 전체를 대상으로 상세 조회에, Hotspot은 특정 컬럼 값이 링크임을 명확히 보일 때 어울린다. 같은 값에 둘 다 강하게 걸면 "한 번/두 번"이 헷갈리니 하나만.

흔한 실수와 점검

  • hotspot 안 켜고 핸들러만 → 사용자는 셀이 클릭 가능한지 모르고, 도 안 난다. Field Catalog에서 hotspot 먼저 켠다.
  • 컬럼 구분 없이 반응 → 핸들러가 모든 클릭에 반응하면 엉뚱한 컬럼에도 동작한다. e_column_id-fieldname으로 의도한 컬럼만 처리한다.
  • e_row_id-index를 업무 키로 → 앞 레슨과 같은 이유로 화면 행 위치일 뿐. 키 (concert_id)로 상세를 잇는다. 또 Hotspot은 탐색용 — 셀 값을 바꾸는 입력은 Editable Grid ALV와 입력 검증에서 다룬다.

정리

  • Hotspot = 특정 셀을 링크처럼 보이게 하고 한 번 클릭으로 hotspot_click을 일으킨다.
  • Field Catalog에서 hotspot=abap_true를 먼저 켜야 링크가 되고 가 난다.
  • 핸들러는 e_row_id·e_column_id로 클릭 위치를 확인 — 더블클릭은 행 중심, Hotspot은 컬럼 값 중심 탐색.

링크를 만들었다. 이제 표 위 툴바에 내 버튼을 추가하자 → Toolbar Event(다음 레슨).
