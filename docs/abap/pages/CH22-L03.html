왜 — 컬럼이 다 보이면 표가 아니라 덤프다 🧾

회차 목록을 그냥 띄우면 MANDT 같은 기술 필드가 첫 칸을 차지하고, 좌석수 합계는 눈으로 세야 하고, 상태 코드는 왼쪽에 붙어 잘 안 읽힌다. 이건 DB를 고칠 문제가 아니라 화면에 어떻게 보일지의 문제다. Grid ALV는 그 규칙을 field catalog로 지시한다.

Field Catalog = 화면 표시 지시서 📋

Field Catalog 기초에서 만든 를 다시 꺼낸다. 핵심은 LVC_T_FCAT은 DB 테이블 구조가 아니라, ALV에게 "이 컬럼을 화면에 어떻게 보일지" 알려 주는 지시서라는 점이다. 각 행이 한 컬럼의 표시 규칙을 담는다.

속성 의미 예
fieldname 적용할 output table(표시용 Internal Table) 필드명 STATUS
no_out 기본 화면에서 숨김 MANDT 숨김
do_sum 숫자 컬럼 합계 SEATS 합계
just 정렬 방향 (L/C/R) STATUS 가운데
key 키 컬럼처럼 고정·강조 PERF_NO
edit 입력 가능한 모양 맛보기만(실제 편집은 이후 챕터)

LOOP AT lt_fcat ASSIGNING FIELD-SYMBOL(<fc>).
  CASE <fc>-fieldname.
    WHEN 'MANDT'.   <fc>-no_out = abap_true.    " 숨김
    WHEN 'SEATS'.   <fc>-do_sum = abap_true.    " 합계
    WHEN 'STATUS'.  <fc>-just   = 'C'.          " 가운데 정렬
    WHEN 'PERF_NO'. <fc>-key    = abap_true.    " 키 고정·강조
  ENDCASE.
ENDLOOP.

🔍 확인: 표가 이상하면 화면부터 보지 말고, 표시 전에 lt_fcat을 디버거로 열어 그 컬럼 행의 no_out·do_sum·just 값을 먼저 확인한다. 화면은 이 지시서의 결과일 뿐이다.

카탈로그를 바꿔도 데이터는 그대로 🧱

lt_fcat을 바꾼다고 DB 테이블 구조나 데이터가 바뀌지 않는다. MANDT(클라이언트 구분용 기술 필드)를 no_out으로 숨겨도 Internal Table엔 값이 그대로 남아 있고, SEATS에 do_sum을 켜도 DB에 합계 행이 생기지 않는다. 화면 표시 규칙일 뿐이다.

흔한 실수와 점검 🚧

  • fieldname ≠ 화면 제목 → 반드시 output table의 component 이름(보통 대문자). DDIC 설명·화면 제목을 쓰면 적용 안 된다.
  • 문자 컬럼에 do_sum → 합계는 숫자 컬럼에만 의미. 상태 코드·이름·날짜엔 안 먹는다.
  • no_out vs 카탈로그에서 삭제 → no_out은 다시 표시 가능한 숨김(카탈로그엔 있음), 삭제는 ALV가 그 컬럼을 아예 모르는 상태.
  • 숨김 = 권한 제어 착각 → 사용자가 variant/layout으로 다시 볼 수 있다. 민감 정보는 애초에 조회하지 않는다.

컬럼을 제어했다. 이제 한 줄 전체가 아니라 셀 하나하나에 색을 입혀 위험을 정확히 강조하자 → Deep Structure 기반 Cell Color.
