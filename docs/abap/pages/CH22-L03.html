JOIN을 관계로

CH13의 JOIN을 매번 쓰는 대신, CDS는 뷰 사이 관계(Association) 를 선언해 두고 필요할 때 경로로 따라간다. 공연↔회차↔예매 관계를 모델에 새긴다.

define view entity ZI_Concert
  as select from zconcert
  association [0..*] to ZI_Perf as _Perf
    on $projection.concert_id = _Perf.concert_id
{
  key concert_id,
      artist,
      capacity,
      _Perf            // 관계를 노출(소비 측에서 따라갈 수 있게)
}
  • association [카디널리티] to 대상 as _이름 on 조건 — 관계 선언.
  • _Perf를 { }에 넣어 노출하면, 소비 측에서 _Perf로 회차에 접근한다.

흔한 실수와 점검

  • 노출 누락 → association을 선언만 하고 { }에 안 넣으면 소비 측에서 못 따라간다.
  • on 조건 → $projection.필드 = _대상.필드로 키를 정확히 연결.

관계를 새겼다. 이제 뷰에 의미를 부여하는 Annotation → 다음 레슨.
