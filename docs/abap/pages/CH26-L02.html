Singleton Pattern

인스턴스를 단 하나만 — Singleton.

Track 2Chapter 26Lesson 2고급

단 하나여야 하는 객체

설정··연결처럼 프로그램에 하나만 있어야 하는 객체가 있다. 매번 NEW하면 상태가 분산된다. Singleton은 인스턴스를 하나로 보장한다.

CLASS zcl_config DEFINITION PUBLIC CREATE PRIVATE.
  PUBLIC SECTION.
    CLASS-METHODS get_instance RETURNING VALUE(ro) TYPE REF TO zcl_config.
  PRIVATE SECTION.
    CLASS-DATA go_instance TYPE REF TO zcl_config.
ENDCLASS.

CLASS zcl_config IMPLEMENTATION.
  METHOD get_instance.
    IF go_instance IS INITIAL.
      go_instance = NEW #( ).        " 최초 1회만 생성
    ENDIF.
    ro = go_instance.
  ENDMETHOD.
ENDCLASS.
  • CREATE PRIVATE — 외부에서 NEW 금지(생성 통제).
  • get_instance( ) — 항상 같은 인스턴스를 돌려준다.

흔한 실수와 점검

  • CREATE PRIVATE 누락 → 외부에서 NEW 가능해 '하나' 보장이 깨진다.
  • 남용 → 전역 상태는 테스트·동시성을 어렵게 한다. 정말 하나여야 할 때만.

하나로 통제했다. 이제 동작을 갈아끼우는 Strategy → 다음 레슨.
