Editable Grid ALV와 입력 검증

DATA_CHANGED_FINISHED Event

변경이 모두 반영된 뒤 — 합계·연동 갱신.

Track 2Chapter 28Lesson 3고급

변경이 끝난 뒤에 할 일

data_changed는 검증용(바뀌기 직전), data_changed_finished는 변경이 내부 테이블에 반영된 뒤 온다. 합계·잔여석 같은 연동 값 재계산에 적합하다.

METHODS on_changed_finished
  FOR EVENT data_changed_finished OF cl_gui_alv_grid
  IMPORTING e_modified.

METHOD on_changed_finished.
  IF e_modified = abap_true.
    " 좌석 합계 등 파생값 재계산 후
    go_grid->refresh_table_display(
      EXPORTING is_stable = VALUE #( row = abap_true col = abap_true ) ).
  ENDIF.
ENDMETHOD.
  • e_modified — 실제로 바뀐 게 있나.
  • 여기서 파생 컬럼을 다시 계산하고 stable refresh(CH21).

흔한 실수와 점검

  • 두 이벤트 혼동 → 검증=data_changed, 후처리=data_changed_finished.
  • 무한 갱신 → 재계산이 또 변경을 유발하지 않게 주의.

후처리까지 했다. 이제 셀 단위 입력 가능 여부를 동적으로 → Cell Style(다음 레슨).
