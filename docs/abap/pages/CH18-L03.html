Modern ABAP Syntax

CORRESPONDING과 구조 매핑

이름이 같은 필드끼리 한 식으로 옮긴다 — CORRESPONDING

Track 1Chapter 18Lesson 3중급

MOVE-CORRESPONDING의 식 버전

CH05에서 이름이 같은 필드만 옮기는 MOVE-CORRESPONDING을 배웠다. 모던에서는 식으로 쓴다 — CORRESPONDING #( ).

ABAP
1
2
3
4
5
" classic
MOVE-CORRESPONDING ls_source TO ls_target.

" modern
DATA(ls_target) = CORRESPONDING ty_target( ls_source ).

내부 테이블도 통째로 매핑된다.

ABAP
1
DATA(lt_target) = CORRESPONDING ty_target_tab( lt_source ).

이름이 다를 때 — MAPPING · EXCEPT

ABAP
1
2
3
ls_target = CORRESPONDING #( ls_source
  MAPPING tgt_field = src_field    " 이름이 다른 필드 연결
          EXCEPT  skip_field ).    " 제외할 필드
  • MAPPING a = b — 이름이 다른 필드를 직접 연결.
  • EXCEPT f — 복사에서 제외.

흔한 실수와 점검

  • 이름 다른데 MAPPING 없음 → 안 옮겨진다. MAPPING으로 연결.
  • 깊은 구조 → 중첩 매핑은 DEEP 옵션 등 추가 고려.

구조 매핑까지 식으로 했다. 이제 테이블의 한 행을 대괄호로 콕 집는 Table Expression → 다음 레슨.
