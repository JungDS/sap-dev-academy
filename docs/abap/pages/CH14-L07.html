Classic DDIC View·유지보수 객체

SE16N 데이터 브라우저

테이블 내용을 빠르게 조회하는 만능 브라우저 — SE16N.

Track 1Chapter 14Lesson 7중급

빠르게 들여다보는 만능 도구

지금까지 데이터를 SE11 Create Entries(한 건씩, Chapter 7 · Transparent Table)· TMG/SM30(표준 유지보수 화면, Chapter 14 · TMG/SM30)으로 다뤘다. 그런데 "아무 테이블이나 빠르게 내용만 보고 싶다"면? 그게 SE16N(Data Browser)이다.

SE16N로 조회

SE16N → 테이블명 입력 → 조건(선택) → 실행(F8). 어떤 테이블이든 별도 화면 없이 행을 바로 조회한다.

  • 컬럼별 조건·정렬·레이아웃 저장 등 편의 기능이 풍부하다.
  • ZCONCERT·ZBOOKING·SFLIGHT 등 무엇이든 테이블명만 알면 본다.

아래에서 SM30으로 H03을 저장한 뒤 SE16N으로 확인해 보자. 조건 없이/Key/오타로 조회하고, 원본 테이블과 Database View 결과가 왜 다를 수 있는지도 보자.

왜 '마지막에' 배우나

SE16N은 너무 편해서, 먼저 보여 주면 앞에서 배운 Maintenance View·TMG를 만들 이유가 사라져 보인다. 하지만 SE16N은 개발자용 빠른 점검 도구이고, 실제 사용자가 쓰는 정식 입력 화면은 Maintenance View·전용 프로그램이다. 역할이 다르다.

⚠️ 운영 데이터 직접 편집 주의 — SE16N으로 운영 테이블을 함부로 고치면 위험하다 (검증·이력 우회). 보통 조회 위주로 쓰고, 편집은 권한·상황을 가린다.

흔한 실수와 점검

  • 조건 없이 대용량 조회 → 조건을 안 넣고 큰 테이블을 실행하면 시스템 부담이 크다. 항상 Key나 조건을 넣어 좁혀 조회하는 습관을 들인다.
  • SE16N을 정식 입력 화면으로 → 개발/점검용이다. 사용자 입력은 Maintenance/전용 화면.
  • 권한 → 편집은 권한에 따라 제한된다(교육 시스템 안내).

DDIC 뷰와 유지보수·조회 도구를 마쳤다. 그런데 이 클래식 뷰들은 현대의 CDS View로 진화한다. 그 비교를 준비하자 → 다음 레슨.
