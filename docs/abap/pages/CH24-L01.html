CH08은 읽기였다 — 이제 쓰기

CH08의 Open SQL은 읽기 전용이었다. 실무는 데이터를 바꿔야 한다. DB를 직접 변경하는 네 문장이 (Data Manipulation Language)이다.

문 동작
INSERT 새 행 추가(이미 있으면 오류)
UPDATE 기존 행 수정
MODIFY 있으면 수정, 없으면 추가(upsert)
DELETE 행 삭제

INSERT zbooking FROM @ls_booking.
UPDATE zbooking SET status = 'X' WHERE booking_id = @lv_id.
MODIFY zbooking FROM @ls_booking.
DELETE FROM zbooking WHERE booking_id = @lv_id.

⚠️ 표준 테이블은 직접 쓰지 않는다

SCARR·SFLIGHT 같은 SAP 표준 테이블은 INSERT/UPDATE로 직접 바꾸면 안 된다 (검증·후속 로직 우회 → 데이터 깨짐). 반드시 BAPI/표준 API를 경유한다. 직접 DML은 내 테이블(ZBOOKING 등)에만.

흔한 실수와 점검

  • INSERT 중복키 → 이미 있으면 덤프. upsert가 필요하면 MODIFY.
  • WHERE 없는 UPDATE/DELETE → 전체가 바뀐다. 조건을 반드시.
  • 표준 테이블 직접 DML → 금지. BAPI 경유.
