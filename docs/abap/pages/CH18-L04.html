READ TABLE의 식 버전

CH06의 READ TABLE을 모던에서는 대괄호로 더 짧게 쓴다.

" classic
READ TABLE lt_booking INTO ls_book WITH KEY booking_id = '0001'.

" modern — 키로 콕 집기
DATA(ls_book) = lt_booking[ booking_id = '0001' ].

" 인덱스로
DATA(ls_first) = lt_booking[ 1 ].

없으면 예외가 난다

찾는 행이 없으면 CX_SY_ITAB_LINE_NOT_FOUND 예외가 난다(classic의 sy-subrc=4와 다르다). 그래서 존재를 먼저 확인하거나 예외를 잡는다.

ABAP
1
2
3
4
5
IF line_exists( lt_booking[ booking_id = '0001' ] ).
  DATA(ls) = lt_booking[ booking_id = '0001' ].
ENDIF.

DATA(lv_idx) = line_index( lt_booking[ booking_id = '0001' ] ).  " 위치(없으면 0)

예외 처리(TRY/CATCH)는 CH20에서 정식으로 — 지금은 line_exists( )로 안전하게.

흔한 실수와 점검

  • 없는 행 직접 접근 → 예외 덤프. line_exists( )로 먼저 확인.
  • classic과 혼동 → 표현식은 sy-subrc를 안 세운다(예외로 알린다).

행을 식으로 집었다. 이제 문자열을 우아하게 만드는 String Template → 다음 레슨.
