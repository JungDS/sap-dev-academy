엑셀로 받은 데이터 등록

업무에서 자주 — 사용자가 엑셀로 준 데이터를 읽어 등록한다. 보통 엑셀을 CSV/탭 텍스트로 저장해 읽거나, 표준 변환 함수를 쓴다.

DATA lt_raw TYPE TABLE OF string.

cl_gui_frontend_services=>gui_upload(
  EXPORTING filename = lv_path filetype = 'ASC'
  CHANGING  data_tab = lt_raw
  EXCEPTIONS OTHERS = 1 ).

LOOP AT lt_raw INTO DATA(lv_line).
  SPLIT lv_line AT cl_abap_char_utilities=>horizontal_tab
    INTO DATA(lv_a) DATA(lv_b).      " 탭 구분 파싱
  " 검증 후 내부 테이블에 APPEND
ENDLOOP.
  • cl_gui_frontend_services=>gui_upload — PC 파일 읽기.
  • 한 줄씩 SPLIT으로 컬럼 분리(CH04 문자열).
  • 읽은 뒤 반드시 검증(타입·필수·중복) 후 등록(BAPI/DML).

💡 .xls를 직접 셀 단위로 읽는 옛 표준 함수 ALSM_EXCEL_TO_INTERNAL_TABLE도 있다 (행·열·값으로 반환). 다만 신형 포맷·대량엔 CSV/탭 텍스트나 XCO/abap2xlsx가 더 안정적이다.

흔한 실수와 점검

  • 헤더 행 포함 → 첫 줄(제목)은 건너뛴다.
  • 무검증 등록 → 외부 데이터는 신뢰 금지. 철저히 검증.
