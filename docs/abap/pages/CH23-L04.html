무엇을 할 수 있는지 선언

(BDEF)은 이 엔티티에 어떤 동작(생성·수정·삭제·액션)이 가능한지 선언한다. root 뷰(ZI_Booking)에 붙인다.

managed implementation in class zbp_i_booking unique;

define behavior for ZI_Booking alias Booking
persistent table zbooking
lock master
{
  create;
  update;
  delete;

  field ( readonly ) booking_id;          // 키는 읽기전용
  mapping for zbooking { booking_id = booking_id; }
}
  • managed implementation in class ... — 동작 구현 (다음 레슨).
  • create/update/delete — 표준 동작.
  • persistent table — 실제 저장 테이블. lock master — 잠금 단위.

흔한 실수와 점검

  • mapping 누락 → CDS 필드와 DB 컬럼 이름이 다르면 mapping 필요.
  • lock 미지정 → 동시성 위해 lock master 지정.

동작을 선언했다. 이제 그 동작의 실제 코드(Behavior Implementation) → 다음 레슨.
