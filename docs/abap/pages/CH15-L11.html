Report Event·Selection Screen 심화

여러 선택화면 — 화면번호·CALL·Variant

선택화면을 여러 개 두고 골라 부른다 — 화면번호·팝업 호출·입력값 저장.

Track 1Chapter 15Lesson 11중급

화면 하나로는 부족할 때

지금까지 PARAMETERS·SELECT-OPTIONS는 하나의 기본 선택화면에 쌓였다. 그 기본 화면의 번호가 1000이다(executable report가 자동으로 갖는 표준 화면). 그런데 모든 조건을 1000 하나에 몰아넣으면 화면이 커지고 복잡해진다. 고급 조건·추가 필터는 별도 선택화면으로 분리해 필요할 때만 열고 싶을 때가 있다.

PARAMETERS pa_conc TYPE zconcert-concert_id.   " 표준 화면(1000)

SELECTION-SCREEN BEGIN OF SCREEN 1100 AS WINDOW TITLE TEXT-010.
  PARAMETERS pa_max TYPE i DEFAULT 100.         " 고급: 최대 건수
  PARAMETERS pa_log AS CHECKBOX.                " 고급: 로그 남기기
SELECTION-SCREEN END OF SCREEN 1100.
  • 기본 화면(1000) 외에 번호를 가진 선택화면을 따로 선언한다.
  • AS WINDOW — 대화상자(작은 창)로 띄울 화면. (팝업이면 붙이는 게 좋다.)

불러오기 — CALL SELECTION-SCREEN

CALL SELECTION-SCREEN 1100 STARTING AT 10 5.   " (열,행)에 modal 팝업
IF sy-subrc = 0.
  " 사용자가 Execute로 나옴 — pa_max·pa_log 적용
ELSE.
  " Back/Exit/Cancel로 나옴 (sy-subrc = 4) — 적용 안 함
ENDIF.
  • <번호> — 그 화면을 띄운다.
  • STARTING AT 열 행 — modal 대화상자로 그 위치에 띄운다(없으면 전체 화면).
  • sy-subrc로 사용자가 Execute(0) 했는지 취소(4) 했는지 판단한다(공식 기준).

본문 PUSHBUTTON·툴바 FUNCTION KEY(Selection Screen UI 구성)가 눌렸을 때 AT SELECTION-SCREEN에서 sscrfields-ucomm을 보고 CALL SELECTION-SCREEN 1100을 호출하면, 메인 화면 위에 옵션 팝업이 뜬다.

Selection Screen Variant — 입력값 저장

매번 같은 조건을 손으로 넣기는 번거롭다. 사용자가 채운 선택화면 값을 로 저장해 두고 다음에 불러 쓴다.

  • 선택화면에서 저장(Save as Variant)하면 그 조건 묶음이 이름으로 보관된다.
  • 다음 실행 때 Variant를 고르면 값이 자동으로 채워진다.
  • Chapter 35 · Background Job 운영에서 특히 중요 — 잡은 화면 입력이 없으니, 미리 만든 Variant로 조건을 준다.

⚠️ Variant는 입력값 세트를 저장하는 기능일 뿐, 검증·권한 확인을 대신하지 않는다.

직접 호출하고 저장해 보기 — 보조 선택화면과 Variant 금고

고급조건을 눌러 보조 화면(1100)을 팝업으로 열고, Execute로 닫으면 값이 적용(sy-subrc=0), Cancel로 닫으면 적용 안 됨(sy-subrc=4)을 확인하자. 를 저장·불러오며 값이 어떻게 복원되는지도 본다.

흔한 실수와 점검

  • 보조 선택화면을 CALL SCREEN으로 호출 → 선택화면은 CALL SELECTION-SCREEN 으로 부른다. (공식문서: CALL SCREEN으로는 선택화면을 호출할 수 없다.)
  • sy-subrc 무시 → 사용자가 취소했는데도 값을 적용한 것처럼 처리하게 된다. sy-subrc로 분기.
  • CALL 했는데 값이 안 보임 → 호출한 화면의 항목은 그 화면(BEGIN/END OF SCREEN) 안에서 선언해야 한다.
  • 로 검증을 대신 → Variant는 값 저장일 뿐, 검증·권한은 여전히 필요하다.

정리

  • 선택화면은 하나만 있는 게 아니다. 표준 1000 외에 보조 화면을 만들어 필요할 때 호출한다.
  • 보조 화면은 CALL SELECTION-SCREEN(≠CALL SCREEN)으로 부르고, sy-subrc (0=실행/4=취소)로 사용자의 선택·취소를 안전하게 처리한다.
  • 는 반복 입력을 줄여 주지만 검증을 대신하지 않는다.

선택화면을 여러 개 두고 골라 부르는 것까지 익혔다. 이제 배운 ·검증·구성을 콘서트앱에 적용해 예매현황 리포트를 만들자 → 실습 — 예매현황 리포트 (이벤트·검증).
