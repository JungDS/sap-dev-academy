같은 JOIN을 또 짜기 싫다

CH12에서 사람⨝부서 JOIN을 코드로 짰다. 같은 조인을 여러 프로그램이 반복하면 낭비다. 그 JOIN을 DDIC에 객체로 박아 테이블처럼 쓰고 싶다 — 다.

Database View

는 여러 테이블을 Foreign Key 관계로 조인해 정의한 읽기전용 DDIC 뷰다. 만든 뒤엔 테이블처럼 SELECT한다.

  FROM zv_person_dept
  INTO CORRESPONDING FIELDS OF TABLE lt_out.

View vs 코드 JOIN

Database View Open SQL JOIN
정의 위치 DDIC(재사용) 코드(그때그때)
조인 기준 보통 Foreign Key ON 자유
성격 읽기전용 조회 구문
  • 여러 곳이 같은 조인 → View로 재사용.
  • 한 번뿐/유연한 조인 → 코드 JOIN.

흔한 실수와 점검

  • View로 쓰기 시도 → Database View는 읽기전용.
  • FK 없으면 조인 정의 까다로움 → 보통 Foreign Key 기반.

여러 테이블 조인 뷰는 봤다. 한 테이블의 일부 필드만 노출하는 뷰는? → Projection View(다음 레슨).
