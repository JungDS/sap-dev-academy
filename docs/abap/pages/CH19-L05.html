DB에서 문자열·날짜도 가공 🧰

modern SQL은 문자열·날짜용 함수도 제공한다. 결과 목록에 보여 줄 간단한 route code·대문자 도시명·기준일 같은 값은, ABAP으로 다 가져와 LOOP 돌기보다 DB에서 만들어 가져오는 편이 자연스러울 때가 있다.

SELECT carrid, connid,
       CONCAT( carrid, connid )  AS route,    " 잇기
       SUBSTRING( connid, 1, 2 ) AS prefix,   " 1번째부터 2글자
       UPPER( cityfrom )         AS city_up   " 대문자
  FROM spfli
  WHERE carrid = @lv_carr
  INTO TABLE @DATA(lt_route).

connid = '0400'이면 SUBSTRING( connid, 1, 2 )는 '04'다.

⚠️ SQL SUBSTRING의 시작 위치는 1부터다. 지난 챕터의 ABAP substring( off = 0 … )는 0부터였다. 이름이 비슷해도 실행 위치(DB↔ABAP)와 인자 의미가 다르다. 같은 글자를 집으려면 SQL pos = ABAP off + 1.

CONCAT·SUBSTRING 외에 UPPER/LOWER(대/소문자)·LENGTH(길이) 등이 있다.

날짜 함수 📅

SELECT carrid, connid, fldate,
       DATS_ADD_DAYS( fldate, 7 )              AS due,        " 7일 뒤
       DATS_DAYS_BETWEEN( @sy-datum, fldate )  AS days_gap    " 두 날짜 차이
  FROM sflight
  WHERE carrid = @lv_carr
  INTO TABLE @DATA(lt_due).
  • DATS_ADD_DAYS( fldate, 7 ) — ABAP SQL에선 인자 두 개(date, days)로 쓴다. fldate가 20260626이면 20260703.
  • DATS_DAYS_BETWEEN( @sy-datum, fldate ) — 두 날짜의 차이를 정수(일)로. @sy-datum은 ABAP 시스템 날짜를 host variable로 넘긴 것.

흔한 실수와 점검 🚧

  • SUBSTRING 위치 혼동 → SQL은 1부터. 0을 넣지 않는다(ABAP off와 다름).
  • 함수 결과에 별칭 누락 → AS route·AS due로 이름을 줘야 결과 구조가 읽기 쉽다.
  • null 전파 → 문자열 함수는 인자가 null이면 결과 전체가 null이 될 수 있다. outer join과 함께면 COALESCE와 같이 생각한다.
  • 날짜 유효성 → DATS 값은 YYYYMMDD 유효 날짜여야 한다. 잘못된 날짜는 함수마다 초기화·오류가 날 수 있다.
  • 함수 릴리스 차이 → DATS_* 같은 SQL 함수는 시스템 릴리스에 따라 없을 수도 있다. 교육 자료는 대표 함수만 보이니, 내 시스템에서 syntax check·F1 문서로 지원 여부를 확인한다.
  • 표시 문구를 전부 SQL로 → 사용자 언어·번역·Text Symbol·메시지는 ABAP/UI 계층에서. SQL 함수는 결과 컬럼의 간단한 가공에 집중.

DB에서 문자·날짜까지 가공했다. 이제 이미 에 있는 내부 테이블을 SELECT 소스로 쓰는 법을 보자 → SELECT FROM @itab 기초.
