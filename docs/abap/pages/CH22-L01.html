SALV/Grid ALV 표시 제어 심화

SALV Sort / Filter / Function 제어

SALV 객체로 정렬·필터·툴바 기능을 코드로 — display 전에 지정한다.

Track 1Chapter 22Lesson 1고급

왜 — display( )만으론 부족하다 🧰

SALV의 목적과 CL_SALV_TABLE 개요에서 는 내부 테이블을 빨리 보여 주는 데 충분했다. 그런데 실제 업무 표는 그냥 보이기만 하면 부족하다. 날짜가 뒤섞여 있거나, 지난 회차가 섞여 있거나, 사용자가 엑셀·합계 기능을 못 찾으면 "덜 만든 표"처럼 느껴진다.

이제 객체지향을 배웠으니, SALV 에게 display( ) 전에 "표준 기능을 켜라", "이 컬럼으로 먼저 정렬하라", "이 상태는 빼라"를 지시할 수 있다.

SALV 하위 객체 — 표시 조건을 꺼내 쓴다 🔧

는 여러 하위 설정 객체를 돌려준다.

하위 객체 대표 의미
functions get_functions( )->set_all( ) 표준 툴바 기능(정렬·필터·인쇄·엑셀)
sorts get_sorts( )->add_sort( ) 첫 표시 때 적용할 기본 정렬(기본 오름차순, 내림은 sequence = if_salv_c_sort=>sort_down)
filters get_filters( )->add_filter( ) 첫 표시 때 적용할 기본 필터
aggregations get_aggregations( )->add_aggregation( ) 숫자 컬럼의 합계/(표준 툴바 Σ로도 가능)

⚠️ CL_SALV_TABLE=>FACTORY는 값을 돌려주는 식이 아니라 static method 호출이다. SALV 객체는 **IMPORTING r_salv_table**으로 받는다(DATA(lo_salv) = factory(...)가 아니다).

PARAMETERS pa_conc TYPE zperf-concert_id.

DATA lt_perf TYPE STANDARD TABLE OF zperf.
SELECT * FROM zperf WHERE concert_id = @pa_conc INTO TABLE @lt_perf.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = DATA(lo_salv)
      CHANGING  t_table      = lt_perf ).

    " 표준 기능 켜기
    lo_salv->get_functions( )->set_all( abap_true ).

    " 공연일 오름차순
    lo_salv->get_sorts( )->add_sort( 'PERF_DATE' ).

    " 지난 회차는 빼고 오늘 이후만
    lo_salv->get_filters( )->add_filter(
      columnname = 'PERF_DATE' sign = 'I' option = 'GE' low = sy-datum ).

    lo_salv->display( ).
  CATCH cx_salv_msg cx_salv_not_found INTO DATA(lx_salv).
    MESSAGE lx_salv->get_text( ) TYPE 'E'.
ENDTRY.

화면에서 이렇게 확인한다 🔍

위 체험의 버튼을 순서대로 눌러 보며 실제 화면과 맞춰 본다.

  1. 표준 기능 켜기 → 표 위 툴바에 정렬·필터·합계·엑셀 아이콘이 생기는지.
  2. PERF_DATE 정렬 → 행이 공연일 오름차순으로 다시 서는지.
  3. 지난 회차 빼기 → 오늘 이전 회차가 화면에서 사라지는지(보이는 행 수 감소).
  4. 필드명 오타 실행 → 없는 컬럼명을 주면 cx_salv_not_found가 나는지.

정리 📌

  • 는 display( ) 한 줄이 아니라, 그 전에 하위 (get_functions·get_sorts·get_filters)로 첫 화면을 설계한다.
  • 정렬·필터의 컬럼명은 화면 제목이 아니라 내부 필드명이다.
  • 표시 필터는 이미 읽은 데이터를 거를 뿐, DB 부하를 줄이려면 SELECT … WHERE에서 먼저.

흔한 실수와 점검 🚧

  • display( ) 후에 설정 → 정렬·기능·필터는 화면을 띄우기 전에 지정한다.
  • 컬럼명을 화면 제목으로 → add_sort·add_filter의 컬럼명은 내부 테이블 필드명(보통 대문자 PERF_DATE). 틀리면 가 컬럼을 못 찾는다.
  • 필터를 DB 조건으로 착각 → 여기 필터는 이미 읽은 내부 테이블을 표시 단계에서 거른다. 데이터를 줄이려면 SELECT … WHERE에서 먼저.
  • 표준 기능을 전부 켜기 → 도구는 많아지지만 화면 목적이 흐려질 수 있다. 업무 화면은 필요한 기능만 켜는 설계도 고려.
