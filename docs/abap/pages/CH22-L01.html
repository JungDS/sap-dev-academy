Classic View의 한계를 넘어

CH14의 Database View는 GUI로 정의하는 classic 방식이라 계산 컬럼·재사용·버전 관리가 약했다. 현대 SAP은 CDS(Core Data Services)로 DB 계층에서 강력하게 모델링한다. 그 단위가 다.

기본 구조

CDS는 ABAP이 아니라 DDL(데이터 정의 언어)로, ADT(Eclipse)에서 만든다.

@AccessControl.authorizationCheck: #NOT_REQUIRED
define view entity ZI_Concert
  as select from zconcert
{
  key concert_id,
      artist,
      venue,
      capacity
}
  • define view entity 이름 as select from 소스 — 뷰 정의.
  • { } 안에 노출할 필드. key로 키 지정.
  • CH19의 ABAP SQL 감각(필드 선택·식)이 그대로 쓰인다.

define view entity ZI_Perf
  as select from zperf
{
  key concert_id,
  key perf_no,
      perf_date,
      cast( perf_date as abap.dats ) as event_date
}

흔한 실수와 점검

  • 활성화 → CDS도 활성화해야 뷰가 생긴다(ADT에서).
  • classic View와 혼동 → CDS는 DDL 소스 기반·재사용·확장에 강하다.
