AMDP / ADBC / Pushdown

DB Pushdown 판단 기준

연산을 DB로 내릴지 ABAP에 둘지 고른다.

Track 2Chapter 33Lesson 1고급

데이터를 끌어오지 말고, 코드를 내려보낸다

전통 ABAP은 데이터를 ABAP 서버로 끌어와 처리했다(Data-to-Code). HANA 시대는 연산을 DB로 내려보내(Code-to-Data) 데이터가 있는 곳에서 처리한다 — 훨씬 빠르다.

무엇으로 내리나 (우선순위)

  1. Open SQL / Modern SQL(CH19) — ·식. 1순위(이식성↑).
  2. CDS View(CH22) — 재사용 모델·계산.
  3. AMDP(L02) — SQLScript가 필요한 복잡 로직.
  4. ADBC / Native SQL(L03) — 최후의 수단(DB 종속).

판단 기준

  • 표준 SQL·CDS로 되면 그걸로(이식성·유지보수).
  • 절차적·복잡 로직(루프·임시테이블)이 꼭 DB에서 필요하면 AMDP.
  • DB 고유 기능이 불가피할 때만 Native SQL.

흔한 실수와 점검

  • 무조건 AMDP → 대부분 Open SQL/CDS로 충분. 복잡할 때만 AMDP.
  • 이식성 무시 → Native SQL은 DB를 갈아탈 때 발목. 최소화.
