DDIC 관계와 입력도움말(F4)

Input Help 호출 우선순위

F4를 누르면 어떤 도움말이 뜨나 — 위에서부터 정해지는 순서.

Track 1Chapter 9Lesson 7초급

도움말이 여러 개면 뭐가 뜨나

CH09-L06에서 F4가 오는 세 경로(고정값·Check Table·Search Help)를 봤다. 그런데 한 입력칸에 이런 장치가 여러 개 걸려 있으면? SAP은 정해진 우선순위로 딱 하나를 골라 띄운다 — "위가 있으면 아래는 안 본다."

        ▼ 없으면
④  타입 기본 도움        ← 날짜=달력, 숫자=계산기
  1. 코드로 직접 만든 F4 — 프로그램이 화면 이벤트에서 목록을 직접 만들어 띄운다. 있으면 이게 최우선이다.
    • 이 "직접 만들기"는 선택화면(AT SELECTION-SCREEN ON VALUE-REQUEST → CH15)과 일반 화면(PROCESS ON VALUE-REQUEST → CH16)에서 배운다. 지금은 "맨 위에 이런 게 있다"만 알면 된다(외울 필요 없음).
  2. — CH09-L04/L05에서 만든 검색 화면. 선언적 도움말 중 가장 유연하다.
  3. · · — Foreign Key의 마스터 목록, 코드 옆 이름(Text Table), Domain의 Fixed Values에서 자동으로 목록이 나온다.
  4. 타입 기본 도움 — 아무것도 안 걸어도, 날짜 타입이면 달력, 숫자면 계산기가 뜬다.

그래서 — 하나로 명확히

여러 개를 겹쳐 걸면 의도와 다른 게 뜰 수 있다(예: Search Help를 원했는데 고정값 목록이 떠 버림). 보통은 우선순위를 이해하고 하나로 분명히 한다.

ABAP
1
2
PARAMETERS p_id TYPE zde_persid
  MATCHCODE OBJECT zsh_person_coll.   " 이 칸엔 이 Search Help로 못 박기
  • MATCHCODE OBJECT(CH09-L06)로 쓸 Search Help를 명시하면 혼선이 없다.

흔한 실수와 점검

  • 여러 도움말이 충돌 → 우선순위를 떠올리고, 원하는 하나로 명확히(MATCHCODE 권장).
  • "왜 이 목록이 뜨지?" → 위 순서를 위에서부터 짚으면 원인이 보인다(상위에 뭔가 걸려 있다).

도움말이 어떻게 정해지는지까지 알았다. 그럼 프로그램 코드의 검증은 어디까지 해야 하나? 선언(DDIC)과 코드의 역할을 가르자 → 다음 레슨.
