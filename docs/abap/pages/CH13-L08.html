🎫 5단계 — 공연별 현황 한눈에

지금까지 예매를 행 단위로 봤다. 이제 공연별로 묶어 예매 좌석합계를 한다 (CH13 JOIN·GROUP BY). 정원과 비교하면 잔여 현황이 된다.

TYPES: BEGIN OF ty_stat,
         concert_id TYPE zconcert-concert_id,
         artist     TYPE zconcert-artist,
         capacity   TYPE zconcert-capacity,
         booked     TYPE i,
       END OF ty_stat.

DATA: lt_stat TYPE TABLE OF ty_stat,
      lo_alv  TYPE REF TO cl_salv_table.

SELECT c~concert_id c~artist c~capacity SUM( b~seats ) AS booked
  FROM zconcert AS c
  LEFT OUTER JOIN zbooking AS b
    ON c~concert_id = b~concert_id AND b~status <> 'C'
  INTO TABLE lt_stat
  GROUP BY c~concert_id c~artist c~capacity.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = lo_alv CHANGING t_table = lt_stat ).
    lo_alv->display( ).
  CATCH cx_salv_msg.
ENDTRY.

직접 해 볼 과제

  1. 공연별 예매 좌석합계가 맞게 나오는지 확인.
  2. 결과 구조에 잔여석(capacity − booked) 컬럼을 더해 LOOP로 계산해 본다.
  3. (도전) HAVING으로 "예매가 정원의 80%를 넘은 공연"만 추린다.

흔한 실수와 점검

  • LEFT JOIN 이유 → 예매 없는 공연도 0으로 보이게(INNER면 빠진다).
  • classic → SUM( )·GROUP BY는 classic OK. modern @·콤마는 CH19.
  • 합계 매핑 → SELECT 필드 순서와 ty_stat 순서를 맞춘다.

공연별 현황까지 만들었다. 이제 공연 마스터를 표준 화면으로 등록·유지보수하자 → CH14.
