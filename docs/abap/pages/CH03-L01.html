Domain(얕게)

로컬 타입의 벽

CH02의 는 편했지만 그 프로그램 안에서만 산다. 회사 전체가 쓰는 "금액", "통화코드" 같은 타입을 매 프로그램이 또 정의하는 건 낭비고, 정의가 갈리면 사고다.

😵 A 프로그램은 금액을 DEC 8,2, B 프로그램은 DEC 11,2로 정의했다면? 같은 "금액"인데 자릿수가 달라 데이터가 어긋난다. 한 곳에서 한 번만 정의하고 싶다.

이제 타입을 전역으로 올린다 — 그 시작이 의 이다.

Domain의 역할 — 기술 속성만

(SAP Dictionary)는 시스템 전체의 데이터 정의를 한 곳에서 관리하는 중앙 사전이다. 그 안에서 은 순수한 기술 속성만 담는다.

Domain이 아는 것 Domain이 모르는 것
데이터 타입·길이·소수·대소문자·허용값 이 값이 "금액"인지 "단가"인지 (의미)
(선택) 허용 값 = 또는 화면에 뜰 라벨

의미·라벨은 다음 레슨의 가 맡는다. 기술은 Domain, 의미는 Data Element — 역할이 갈린다.

⚠️ Domain은 변수 타입으로 직접 못 쓴다. DATA lv TYPE z도메인.은 오류다. 변수는 항상 를 거쳐 선언한다 → DATA lv TYPE zde_amount.(다음 레슨). Domain은 그 Data Element 뒤에서 기술 속성을 대는 한 겹 아래 층이다.

Domain의 속성, 하나씩

Domain을 만들 때 채우는 칸들이다. 하나씩 차분히 보자.

① 데이터 타입 — 값이 "어떻게 생겼는지". 자주 쓰는 것만:

DDIC 데이터 타입 뜻 ABAP 변수로 선언하면
CHAR 문자(텍스트) TYPE c
NUMC 숫자만 담는 문자(자릿수 고정, 앞 0 유지) TYPE n
DEC 십진수(소수 가능 · 금액 등) TYPE p
INT4 정수 TYPE i
DATS 날짜(YYYYMMDD, 8자리) TYPE d

⚠️ 헷갈리기 쉬운 점 — 문자 타입이 DDIC(도메인)에서는 CHAR, 그런데 ABAP 변수를 선언할 땐 키워드가 c 다. 같은 "문자"인데 이름이 다르다. (CH02에서 DATA lv TYPE c를 봤다면, 그 도메인 버전이 CHAR.)

② 그 밖의 속성:

속성 뜻
길이(Length) 담을 수 있는 자릿수 (예: CHAR 10 = 글자 10칸)
출력 길이(Output Length) 화면에 표시될 칸 너비. 보통 길이와 같지만, 부호·천단위 구분기호 때문에 더 길 수 있다
소수(Decimals) 소수점 이하 자릿수 (DEC 전용, 길이보다 작아야 함)
대소문자(Lower Case) 체크 안 하면 입력이 자동으로 대문자로 바뀐다(CHAR 기본). 소문자를 살리려면 체크

값을 제한하는 두 가지 — 고정값 vs 값 테이블

특정 값만 받게 하려면 두 가지 방법이 있다. 상황에 따라 둘 중 하나를 쓴다.

(Fixed Values) (Value Table)
무엇 Domain 안에 직접 적은 허용 값 목록 허용 값이 든 다른 테이블을 가리킴
잘 맞는 곳 거의 안 바뀌는 소수의 값 자주 바뀌는 목록
예 O=Open, C=Closed 항공사 목록 테이블 SCARR
이어짐 화면 ·입력검증(CH09) 외래 키(Foreign Key)의 바탕(CH09)

💡 둘 다 "허용 값을 제한"하지만 동시에 쓰지 않는다. 값이 고정이면 고정값, 값이 자라나는 목록이면 값 테이블.

SE11에서 직접 만들기

이제 속성을 알았으니 직접 만들어 보자. SE11(Dictionary) → Domain → 이름 → Create. 아래 시뮬레이터에서 값을 채우고 💾 저장 → ✓ 검사 → 🔥 활성화 순서로 눌러, DDIC 객체가 살아나는 과정을 따라가 보라.

🖱️ 예제 칩을 바꿔 보라 — 금액(DEC 8,2), 상태(CHAR 1 + O/C), 항공사 코드(CHAR 3 + SCARR). 데이터 타입을 바꾸면 위에 "ABAP 변수로는 TYPE …" 대응이 뜨고, CHAR일 때만 대소문자 옵션이 나타난다.

활성화 — DDIC의 철칙

방금 봤듯, 저장만 하면 객체는 "비활성(inactive)" 상태라 런타임에서 쓸 수 없다. (Ctrl+F3)를 해야 런타임 객체가 생성되고, 비로소 다른 객체가 이 Domain을 참조할 수 있다.

상태 의미
비활성 (저장만 함) 작업 중인 초안 — 런타임에서 못 씀
활성 (활성화 완료) 런타임 객체 생성됨 — 참조·실행 가능

⚠️ 활성화한 뒤 정의를 바꾸면 다시 비활성이 된다. 반영하려면 재활성화해야 한다. (시뮬레이터에서 활성화 후 길이를 고치면 배지가 바로 "비활성"으로 돌아간다.)

흔한 실수와 점검

⚠️ CHAR ↔ c 혼동 — Domain 데이터 타입은 CHAR, 변수 선언 키워드는 c. 둘은 같은 "문자"지만 쓰는 자리가 다르다.

⚠️ 이름 규칙 — 전역 객체도 Z/Y로 시작. (시뮬레이터에서 AMOUNT로 검사하면 막힌다.)

⚠️ 저장만 하고 끝 — 비활성이라 아무도 못 쓴다. 반드시 까지.

⚠️ + 동시 사용 — 보통 둘 중 하나만. 값이 고정이면 고정값, 목록이면 값 테이블.

정리

  • 같은 타입을 프로그램마다 다시 정의하는 낭비를 막으려 타입을 전역으로 올린다 → 의 .
  • 은 기술 속성(데이터 타입·길이·출력길이·소수·대소문자·허용값)만 담는다. 의미·라벨은 몫.
  • 문자 타입은 DDIC=CHAR, 변수=c. 허용 값은 또는 (둘 중 하나).
  • DDIC 객체는 해야 런타임에 산다 — 저장만 하면 비활성.

기술 속성은 정의했다. 그런데 이 값이 화면에 어떤 이름표로 뜨고 무슨 의미인지는 누가 정하나? → (다음 레슨).
