연산자와 흐름 제어

문자열 다루기

문자열을 잇고, 자르고, 찾고, 바꾸고, 다듬는다.

Track 1Chapter 4Lesson 2입문

숫자 다음은 글자

숫자를 계산했으니, 이번엔 문자열이다. 이름과 성을 한 줄로 잇고, "성:이름"을 쪼개고, 특정 글자를 찾아 바꾸는 일은 업무 프로그램에서 끝없이 나온다.

잇기 — CONCATENATE 와 &&

여러 문자열을 하나로 잇는다. 는 사이 구분자를 SEPARATED BY로 지정할 수 있다.

ABAP
1
2
3
4
DATA gv_full TYPE string.

CONCATENATE '정훈영' '님' INTO gv_full.                  " 정훈영님
CONCATENATE '2026' '06' '23' INTO gv_full SEPARATED BY '-'. " 2026-06-23

짧게 이을 땐 && 연산자가 더 간단하다(아주 자주 쓰여 미리 익혀 둔다).

ABAP
1
gv_full = '정훈영' && '님'.       " 정훈영님

💡 CONCATENATE는 구분자(SEPARATED BY)까지 챙길 수 있어 여러 조각을 정교하게 이을 때 좋고, &&는 두어 개를 짧게 이을 때 간편하다. 상황에 맞게 고른다.

쪼개기 · 길이 — SPLIT · STRLEN

SPLIT은 구분자를 기준으로 문자열을 나눈다. strlen( )은 글자 수를 센다.

ABAP
1
2
3
4
5
6
7
DATA gv_id   TYPE string VALUE 'KR:정훈영'.
DATA gv_cty  TYPE string.
DATA gv_name TYPE string.
DATA gv_len  TYPE i.

SPLIT gv_id AT ':' INTO gv_cty gv_name.   " gv_cty='KR', gv_name='정훈영'
gv_len = strlen( gv_name ).               " 3

strlen( )처럼 괄호 안에 값을 넣어 결과를 돌려주는 것을 라 한다. 숫자용 내장함수도 있다: abs( -5 )(절댓값 5), sqrt( 9 )(3), ipow( base = 2 exp = 8 )(256), gv MOD 3(나머지), gv DIV 3(몫).

찾고 바꾸기 — FIND · REPLACE

ABAP
1
2
3
4
DATA gv_text TYPE string VALUE 'SAP ABAP'.

FIND 'ABAP' IN gv_text.        " 찾으면 sy-subrc = 0
REPLACE 'ABAP' IN gv_text WITH 'HANA'.   " gv_text = 'SAP HANA'

옛 코드의 SEARCH문은 더 쓰지 않는다(obsolete). **찾기는 FIND**가 표준이다.

다듬기 — CONDENSE · 부분 접근(offset)

는 앞뒤 공백을 없애고 중간 연속 공백을 한 칸으로 줄인다. 문자열의 일부만 떼어 보려면 CH02에서 배운 offset 접근 gv+시작(길이)을 쓴다.

ABAP
1
2
3
4
5
DATA gv_raw  TYPE string VALUE '   정훈영   '.
DATA gv_date TYPE c LENGTH 8 VALUE '20260623'.

CONDENSE gv_raw.                 " '정훈영'
WRITE gv_date+4(2).              " 5번째부터 2글자 → '06' (월)

이 밖에 SHIFT(밀기)·TRANSLATE(대소문자/치환)·OVERLAY(겹쳐쓰기)· WRITE 값 TO 변수(서식을 입혀 변수로 받기)도 같은 갈래의 도구다.

흔한 실수와 점검

  • SPLIT 대상 변수 개수 → 쪼갠 조각 수만큼 받을 변수를 적는다.
  • && 양옆 공백 → a && b처럼 띄운다.
  • CONDENSE로 안 지워질 때 → 중간 모든 공백 제거는 CONDENSE gv NO-GAPS.

정리

문자열을 잇고·쪼개고·찾고·다듬었다. 계산도 글자 처리도 되는데 — 프로그램은 아직 언제나 똑같이 행동한다. "값이 양수면 이렇게, 아니면 저렇게" 갈라지게 하고 싶다 → CH04-L03(IF와 조건식).
