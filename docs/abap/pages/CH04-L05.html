연산자와 흐름 제어

DO / WHILE · 루프 제어

같은 일을 되풀이한다 — 그리고 멈추고 건너뛴다.

Track 1Chapter 4Lesson 5입문

같은 일을 9번 반복하려면?

"2단을 출력하라"는 WRITE를 9번 복붙하면 된다. 하지만 9단까지면 81줄을 복붙해야 한다. 사람이 할 짓이 아니다. 반복을 컴퓨터에 맡기는 게 ·이다.

DO — 정해진 횟수만큼

DO n TIMES. 는 안쪽을 n번 반복하고 ENDDO로 닫는다. 지금 몇 번째 반복인지는 시스템이 에 자동으로 넣어 준다(1, 2, 3 …).

WHILE — 조건이 참인 동안

횟수가 아니라 조건으로 돌릴 땐 을 쓴다. 조건이 거짓이 되면 멈춘다.

⚠️ WHILE에서는 조건을 언젠가 거짓으로 만드는 코드(gv_n = gv_n + 1)가 반드시 있어야 한다. 없으면 무한루프에 빠진다.

루프 제어 — EXIT · CONTINUE · CHECK

문 뜻
EXIT 반복을 즉시 완전히 빠져나간다
CONTINUE 이번 회차만 건너뛰고 다음 회차로
CHECK 조건 조건이 거짓이면 이번 회차를 건너뛴다(CONTINUE의 조건부 버전)

SY — 시스템이 채워 주는 변수들

처럼 시스템이 자동으로 값을 채워 주는 변수 묶음을 (System Fields)라 한다. 프로그램 어디서나 읽을 수 있고, 자주 쓰는 것들은 외워 두면 좋다.

필드 담긴 값 쓰임 예
sy-index 현재 반복 회차 DO/ 안에서 몇 번째인지
sy-datum 오늘 날짜 날짜 입력칸 기본값
sy-uzeit 현재 시각 처리 시각 기록
sy-uname 로그인 사용자 누가 실행했는지
sy-mandt 현재 클라이언트(집) —
sy-langu 로그인 언어 다국어 처리
sy-tcode 현재 어느 화면에서 왔는지
sy-repid 현재 프로그램 이름 로그·메시지
sy-subrc 직전 명령의 성공/실패 0이면 성공(뒤에서 핵심)

WRITE: / '오늘은', sy-datum, '실행자', sy-uname.

💡 주의: SY 변수는 읽어서 쓰는 값이다. 여기에 내 데이터를 저장하면 안 된다 (시스템이 언제든 덮어쓴다). 예: 날짜 PARAMETERS 기본값을 sy-datum(오늘)으로 주는 식. 그리고 sy-subrc처럼 명령이 세팅하는 값은 그 명령 바로 다음 줄에서 확인한다 — 사이에 끼어든 문장이 그 값을 덮을 수 있어서, 늦게 읽으면 엉뚱한 결과가 된다. sy-subrc(직전 결과)는 CH06·CH08에서 본격적으로 활용한다.

흔한 실수와 점검

  • 무한루프 → WHILE 조건을 바꾸는 코드를 빠뜨림. DO n TIMES는 안전.
  • ENDDO/ENDWHILE 짝 → DO는 ENDDO, WHILE은 ENDWHILE.
  • EXIT vs CONTINUE → EXIT=완전 탈출, CONTINUE=이번 회차만 건너뜀.

정리

반복(DO·)과 그 제어(EXIT·CONTINUE·CHECK), 그리고 SY 변수까지 익혔다. 그런데 "반복하는 동안 변수가 정말 1,2,3으로 변하나?"를 눈으로 확인하고 싶다 → 코드를 한 줄씩 멈춰 보는 (CH04-L06).
