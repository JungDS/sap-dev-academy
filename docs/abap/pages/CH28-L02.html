Editable Grid ALV와 입력 검증

DATA_CHANGED Event

셀이 바뀌는 순간 검증한다 — DATA_CHANGED.

Track 2Chapter 28Lesson 2고급

바뀌는 순간 끼어들기

Editable Field Catalog 설정에서 셀을 입력 가능하게 만들었다. 그런데 사용자가 99·0·빈칸·문자를 넣어도 저장할 때까지 아무 말이 없으면, 여러 칸을 틀리게 입력한 뒤 한꺼번에 오류를 맞는다. data_changed 는 셀 값이 바뀐 직후 발생해, 그 자리에서 즉시 검증한다.

METHODS on_data_changed
  FOR EVENT data_changed OF cl_gui_alv_grid
  IMPORTING er_data_changed.       " 변경 프로토콜 객체

METHOD on_data_changed.
  LOOP AT er_data_changed->mt_good_cells INTO DATA(ls_cell).
    IF ls_cell-fieldname <> 'SEATS'.
      CONTINUE.
    ENDIF.
    TRY.
        DATA(lv_seats) = CONV i( ls_cell-value ).
      CATCH cx_sy_conversion_no_number.
        er_data_changed->add_protocol_entry(
          i_msgid = 'ZMSG' i_msgty = 'E' i_msgno = '001'
          i_fieldname = ls_cell-fieldname
          i_row_id    = ls_cell-row_id ).
        CONTINUE.
    ENDTRY.
    IF lv_seats < 1 OR lv_seats > 10.
      er_data_changed->add_protocol_entry(
        i_msgid = 'ZMSG' i_msgty = 'E' i_msgno = '002'
        i_fieldname = ls_cell-fieldname
        i_row_id    = ls_cell-row_id ).
    ENDIF.
  ENDLOOP.
ENDMETHOD.
  • er_data_changed — 변경 프로토콜(바뀐 셀·유효성·오류 메시지를 ALV에 알리는 통로).
  • mt_good_cells — 이름과 달리 "이미 좋은 셀"이 아니다. ALV가 넘긴 변경 후보를 순회하며 업무 규칙으로 다시 판단한다.
  • 오류는 셀 색을 직접 바꾸지 말고 add_protocol_entry로 등록 → ALV가 그 셀을 빨갛게 표시·막는다.
  • 화면 입력은 문자라 숫자 변환이 실패할 수 있다 — 예외 처리로 덤프 대신 셀 오류로 바꾼다.

직접 고쳐 보기

SEATS를 고칠 때마다 data_changed가 발생해 검증하는 걸 확인해 보라(99·0·빈칸·문자처럼 1~10 밖이면 셀이 빨개진다).

흔한 실수와 점검

  • MESSAGE … TYPE 'E'로만 처리 → 문장은 보여도 어느 셀이 문제인지 ALV가 모른다. 셀 위치+메시지를 함께 주는 add_protocol_entry가 필요하다.
  • 매번 전체 내부 테이블 검사 → data_changed는 지금 바뀐 셀을 빠르게 보는 자리다. 정원 합계·중복·여러 행 관계 같은 무거운 규칙은 저장 전 검증(이 챕터 뒤)에서 다룬다.
  • 문자 입력 미고려 → 화면 값은 사용자가 타이핑한 것이라 숫자 변환이 깨질 수 있다. 변환 를 프로토콜 메시지로 바꿔 덤프 대신 셀 오류를 보여 준다.

정리

  • data_changed = 셀이 바뀐 직후 발생 → 그 자리에서 즉시 검증.
  • 오류는 add_protocol_entry로 ALV에 알려 셀을 빨갛게(직접 막지 않음).
  • mt_good_cells는 "검증된 셀"이 아니라 변경 후보 — 프로그램이 규칙으로 다시 판단한다.

바뀌는 순간을 잡았다. 이제 모든 변경이 끝난 뒤 한 번 처리하는 DATA_CHANGED_FINISHED → DATA_CHANGED_FINISHED Event(다음 레슨).
