Transparent Table (SE11)

Create Entries로 구구단 입력 · 데이터 조회

만든 테이블에 손으로 데이터를 넣고(SE11 Create Entries) 확인한다.

Track 1Chapter 7Lesson 2초급

빈 테이블 채우기

ZGUGUDAN을 만들었으니 값을 넣어야 조회도 의미가 있다. 학습 단계에선 SE11의 Create Entries로 한 건씩 손으로 입력한다.

SE11 Create Entries — 한 건씩 손입력

SE11 → ZGUGUDAN 표시 → 메뉴 Utilities → Table Contents → Create Entries. 입력 화면이 뜨면 DAN·MUL·RESULT를 채우고 저장한다.

2단과 3단을 손으로 넣어 보자 — 2×1 … 2×9, 3×1 … 3×9, 총 18행이다.

😵 18행을 손으로 넣는 것도 지친다. CH06에서는 같은 걸 프로그램으로 81행을 순식간에 만들었다. "손입력은 학습·소량 확인용, 대량은 프로그램"이라는 감을 잡아 둔다.

넣은 값 다시 보기 — Table Contents

SE11 → ZGUGUDAN → Utilities → Table Contents → Display로 방금 넣은 18행을 조회한다. 조건을 주면 해당 행만 나온다.

키 중복은 안 된다

키가 같은 행을 두 번 넣을 수 없다. MANDT + DAN + MUL이 키라면 같은 (단, 곱수) 조합은 하나뿐이다.

흔한 실수와 점검

  • 입력 권한/설정 → 교육 시스템 안내를 따른다. 활성화된 테이블이라야 입력된다.
  • 클라이언트 혼동 → 다른 Client에서 넣은 데이터는 안 보인다( 분리).
  • 키 중복 → 같은 키 조합 재입력은 거부된다.

이제 18행이 디스크에 영구히 남았다. 그런데 같은 DDIC 모양을 프로그램에선 작업 영역으로도, 내부 테이블로도 썼다 — 셋은 어떻게 다를까? 정리하고 가자 → 다음 레슨.
