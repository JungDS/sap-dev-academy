Report Event·Selection Screen 심화

Selection Screen 고급 — 블록·그룹 검증과 커스텀 F1·F4

필드 하나를 넘어 — 블록/라디오그룹 검증과, 코드로 직접 만드는 F1 도움말·F4 입력 도움.

필드 하나를 넘어선 검증·도움말

CH15-L04에서 AT SELECTION-SCREEN ON <필드>로 한 필드를 검증했다. 실무는 더 넓은 단위(블록·라디오그룹)나, 코드로 직접 만드는 F1·F4가 필요할 때가 있다.

AT SELECTION-SCREEN ON BLOCK b1.
  " 블록 b1 전체의 조합 규칙 검증
  IF p_from > p_to.
    MESSAGE '기간이 올바르지 않습니다' TYPE 'E'.
  ENDIF.

AT SELECTION-SCREEN ON RADIOBUTTON GROUP g1.
  " 라디오 그룹 g1 선택에 따른 검증
  • ON BLOCK <블록> — 한 블록(CH15-L09의 BLOCK)의 여러 필드를 묶어 검증.
  • ON RADIOBUTTON GROUP <그룹> — 라디오 선택과 다른 입력의 정합성 검증.

코드로 만드는 F1 도움말 — ON HELP-REQUEST

DDIC 문서(Data Element)로 부족할 때, 특정 필드의 F1을 코드로 만든다.

ABAP
1
2
3
AT SELECTION-SCREEN ON HELP-REQUEST FOR p_carr.
  " 직접 만든 도움말 화면/팝업 표시
  MESSAGE '항공사 코드 2자리를 입력하세요' TYPE 'I'.

코드로 만드는 F4 — ON VALUE-REQUEST

CH09의 선언적 Search Help로 부족할 때(목록을 그때그때 DB에서 만들어야 할 때), F4 입력 도움을 코드로 채운다. 실무에서 매우 자주 쓰는 패턴이다.

DATA lt_list TYPE TABLE OF scarr.

AT SELECTION-SCREEN ON VALUE-REQUEST FOR p_carr.
  " 1) 목록 데이터를 조회 (classic — CH15는 CH18 이전)
  SELECT carrid carrname FROM scarr
    INTO CORRESPONDING FIELDS OF TABLE lt_list.

  " 2) F4 팝업으로 보여 주고, 고른 값을 화면 필드에 반영
  CALL FUNCTION 'F4IF_INT_TABLE_VALUE_REQUEST'
    EXPORTING retfield    = 'CARRID'      " 돌려줄 컬럼
              dynpprog    = sy-repid
              dynpnr      = sy-dynnr
              dynprofield = 'P_CARR'      " 값을 채울 화면 필드
    TABLES    value_tab   = lt_list.      " 보여 줄 목록

  • value_tab — 팝업에 띄울 목록(코드에서 만든다).

  • retfield/dynprofield — 고른 값을 어느 화면 필드에 돌려줄지.

  • (심화) 고른 값으로 연관 필드를 추가 조회해 같이 채울 수도 있다(예: 항공사 고르면 노선 목록 갱신).

  • 선언적(CH09) 우선 — DDIC Search Help로 되면 그걸 쓰고, 동적·특수할 때만 코드 F4.

  • 검증(ON·ON BLOCK)과 도움말(F1·F4)은 목적이 다르다 — 막는 것 vs 돕는 것.

흔한 실수와 점검

  • 블록 이름 불일치 → ON BLOCK의 이름은 정의한 BLOCK 이름과 같아야.
  • 코드 F4 남용 → 대부분 DDIC Search Help(CH09)로 충분. 코드 F4는 동적일 때만.
