Grid ALV 기초

종합 실습 — 예매 목록 Grid ALV 완성

컨테이너부터 색까지, 배운 조각을 책임별 FORM으로 나눠 하나의 화면으로 모은다.

조각을 하나의 화면으로

지금까지는 조각을 하나씩 배웠다: 자리·그리드·데이터·Field Catalog·Layout·Variant·display·refresh·색. 실무에선 조각이 아니라 하나의 화면이 필요하다. 사용자가 공연 번호를 입력하고, Module Pool 프로그램 구조의 화면 0100 안 CONT100 영역에서 예매 목록을 보고, 새로고침해도 화면이 안정적으로 유지되어야 한다.

전체 골격 — 책임을 FORM으로 나눈다

긴 코드를 한 덩어리로 보면 어느 줄이 어떤 책임인지 놓친다. FORM으로 나누면 조립 순서가 함수 이름에 드러난다. PBO에서 처음 한 번만 ALV를 만들고, 이후엔 refresh만 한다.

DATA: go_cont    TYPE REF TO cl_gui_custom_container,
      go_grid    TYPE REF TO cl_gui_alv_grid,
      gt_booking TYPE STANDARD TABLE OF zbooking,
      gt_fcat    TYPE lvc_t_fcat,
      gs_layout  TYPE lvc_s_layo,
      gs_variant TYPE disvariant.

MODULE status_0100 OUTPUT.
  IF go_grid IS INITIAL.
    PERFORM build_alv.        " 처음 한 번 — 조립
  ELSE.
    PERFORM refresh_alv.      " 이후 — 갱신만
  ENDIF.
ENDMODULE.

FORM build_alv.
  PERFORM select_booking.     " 1. 데이터
  PERFORM create_grid.        " 2. 컨테이너 + 그리드
  PERFORM build_fieldcat.     " 3. 컬럼 설명서
  PERFORM build_layout.       " 4. 표 모양 + variant
  PERFORM display_grid.       " 5. 처음 띄우기
ENDFORM.

FORM select_booking.
  SELECT * FROM zbooking INTO TABLE gt_booking
    WHERE concert_id = gv_concert.
  IF sy-subrc <> 0.
    MESSAGE '해당 공연의 예매 데이터가 없습니다.' TYPE 'S'.   " 빈 결과는 정상일 수 있음
  ENDIF.
ENDFORM.

FORM create_grid.
  CREATE OBJECT go_cont EXPORTING container_name = 'CONT100'.
  CREATE OBJECT go_grid EXPORTING i_parent = go_cont.
ENDFORM.

FORM build_fieldcat.
  DATA ls_fcat TYPE lvc_s_fcat.
  CALL FUNCTION 'LVC_FIELDCATALOG_MERGE'
    EXPORTING i_structure_name = 'ZBOOKING'
    CHANGING  ct_fieldcat      = gt_fcat
    EXCEPTIONS OTHERS          = 1.
  IF sy-subrc <> 0.
    MESSAGE 'Field Catalog를 생성하지 못했습니다.' TYPE 'E'.
  ENDIF.
  LOOP AT gt_fcat INTO ls_fcat.
    IF ls_fcat-fieldname = 'SEATS'.
      ls_fcat-coltext = '좌석수'. ls_fcat-outputlen = 8.
      MODIFY gt_fcat FROM ls_fcat INDEX sy-tabix.
    ENDIF.
  ENDLOOP.
ENDFORM.

FORM build_layout.
  gs_layout-zebra      = abap_true.
  gs_layout-grid_title = '예매 목록'.
  gs_layout-cwidth_opt = abap_true.
  gs_variant-report    = sy-repid.
ENDFORM.

FORM display_grid.
  go_grid->set_table_for_first_display(
    EXPORTING is_layout  = gs_layout
              is_variant = gs_variant
              i_save     = 'A'
    CHANGING  it_outtab       = gt_booking
              it_fieldcatalog = gt_fcat
    EXCEPTIONS OTHERS          = 1 ).
  IF sy-subrc <> 0.
    MESSAGE 'ALV Grid를 처음 표시하지 못했습니다.' TYPE 'E'.
  ENDIF.
ENDFORM.

FORM refresh_alv.
  DATA ls_stable TYPE lvc_s_stbl.
  ls_stable-row = abap_true.
  ls_stable-col = abap_true.
  go_grid->refresh_table_display( EXPORTING is_stable = ls_stable
                                  EXCEPTIONS OTHERS   = 1 ).
ENDFORM.

스스로 답해 보기

조각을 다 배웠다면 아래를 말로 답할 수 있어야 한다(막히면 해당 레슨으로).

개념 자가점검 (펼치기)
  • CONT100과 CL_GUI_CUSTOM_CONTAINER는 어떻게 다른가? go_cont는 그중 무엇인가?
  • 그리드(go_grid)만 만들면 왜 표가 안 보이나? 무엇이 더 필요한가?
  • Field Catalog와 출력용 Internal Table은 각각 무슨 역할인가?
  • Layout과 Variant는 무엇이 다른가?
  • set_table_for_first_display와 refresh_table_display는 언제 각각 쓰나?
  • Stable Refresh는 무엇을 지켜 주나?
  • 행 색상이 안 보일 때 점검할 네 가지는?
  • 셀 색상··편집 Grid는 왜 이 챕터가 아니라 뒤 챕터인가?

직접 해 볼 과제

  1. 매진 회차 행 색상 — 행 색상 기초의 info_fname에 색 컬럼을 더한다.
  2. 좌석수 합계 — Field Catalog의 SEATS에 do_sum = abap_true(Grid ALV Column 제어 심화 맛보기).
  3. 위치 보존 새로고침 — 데이터 변경 후 Refresh와 Stable Refresh 기초의 is_stable로 위치를 지킨다.

막히면 해당 조각 레슨으로 돌아가 확인한다. "어느 조각이 어디서 쓰이는지"를 손으로 맞춰 보는 게 이 실습의 목적이다.

흔한 실수와 점검

  • "처음 구성"과 "이후 갱신" 섞기 → set_table_for_first_display는 처음 한 번, refresh_table_display는 이후. 이 원칙을 화면 흐름에 고정한다.
  • Custom Control 이름 불일치 → 화면과 코드의 container_name이 같아야 한다.
  • merge 실패 무시 → LVC_FIELDCATALOG_MERGE가 실패했는데 gt_fcat을 확인 안 하고 display로 넘어가지 않는다.
  • 0건을 오류로 → 조회 결과 없음은 정상 업무 상황일 수 있다.

정리

  • 이 챕터는 "표를 보여 주는 코드"가 아니라 "화면 안 표 컨트롤을 조립하는 코드" 다.
  • 컨테이너·그리드·데이터·field catalog·layout·variant·display·refresh가 각자 맡은 책임을 알아야, 이후의 표시 제어··편집 Grid를 흔들리지 않고 배운다.

조각을 하나로 완성했다. 그런데 이 classic 코드는 길다. 이제 같은 일을 더 짧고 현대적으로 쓰는 길로 → Inline Declaration.
