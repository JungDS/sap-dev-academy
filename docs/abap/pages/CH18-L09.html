Modern ABAP Syntax

LET으로 표현식 속 이름 읽기

표현식 안에서만 사는 보조 이름 — LET으로 중간값을 한 번만 이름 붙인다.

Track 1Chapter 18Lesson 9중급

표현식이 길어지면 길을 잃는다

Modern syntax를 배우면 한 줄 안에서 값 생성·조건·반복·변환이 함께 보인다. 처음엔 짧아 좋지만, 계산식이 조금만 길어져도 "어떤 값이 먼저 만들어지고, 어디에 쓰였는지" 를 놓치기 쉽다. 특히 같은 계산을 여러 자리에서 반복할 때 그렇다.

표현식 밖에 임시 변수를 만들 수도 있지만, 그 변수가 그 표현식 밖에서는 의미가 없다면 범위를 좁히는 편이 낫다. 그게 이다.

LET — 표현식 안에서만 사는 이름표

LET 이름 = 값 IN ... 꼴이다. ABAP은 LET을 먼저 계산한 뒤 IN 뒤를 계산한다. 그 이름은 그 표현식 안에서만 읽을 수 있다(바깥에서 다시 쓰는 일반 변수가 아니다).

COND 안에서 같은 계산을 한 번만 이름 붙이는 예다.

DATA(lv_status_text) = COND string(
  LET lv_remaining = lv_capacity - lv_reserved IN
  WHEN lv_remaining = 0  THEN '매진'
  WHEN lv_remaining < 10 THEN |마감 임박: { lv_remaining }석|
  ELSE                        |예매 가능: { lv_remaining }석| ).

lv_capacity - lv_reserved를 WHEN마다 반복하지 않고 lv_remaining 하나로 읽는다.

역할이 다른 이름들을 구분한다

REDUCE 안에서 LET 보조값과 INIT 누적값은 역할이 다르다. 섞으면 어려워진다.

ABAP
1
2
3
4
5
DATA(lv_total_price) = REDUCE i(
  LET lv_price = 10000 IN          " 보조값(고정)
  INIT sum = 0                     " 누적값(반복마다 바뀜)
  FOR ls_book IN lt_booking
  NEXT sum = sum + ls_book-seats * lv_price ).
위치 역할 값이 바뀌나
LET lv_price = 10000 표현식 안 보조값 보통 안 바뀜
INIT sum = 0 누적 시작값 반복마다 바뀜
FOR ls_book IN ... 반복 대상 행 행마다 바뀜
NEXT sum = ... 다음 누적값 반복마다 실행

흔한 실수와 점검

  • 일반 DATA 선언으로 오해 → LET 이름은 표현식 안 보조 이름이다. 표현식 뒤에서 WRITE lv_remaining.처럼 바깥에서 쓰면 접근할 수 없다.
  • 이름 충돌 → 이미 쓰는 변수 이름을 LET 보조 이름으로 다시 쓰면 헷갈린다.
  • LET 남용 → 쓰면 무조건 읽기 좋아지는 게 아니다. 같은 계산을 한 번 이름 붙여야 더 명확할 때만 쓴다. 검증·메시지·예외가 섞이면 IF/LOOP/일반 변수로 분리한다.

🔎 공식 문서엔 COND/SWITCH 결과 자리에 THROW(예외 발생)를 둘 수 있지만, 예외 클래스와 TRY/CATCH 이해가 필요하다. ELSE THROW ... 형태는 Exception Class — TRY/CATCH/CX 계층에서 다시 배운다.

정리

  • LET ... IN은 modern 표현식 안에서만 쓰는 작은 이름표다. 먼저 보조값을 계산하고, IN 뒤에서 그 이름으로 결과를 만든다.
  • 복잡한 표현식도 " 보조값 · FOR 반복값 · INIT 누적값 · THEN/ELSE 결과값" 을 구분하면 순서대로 읽힌다.
