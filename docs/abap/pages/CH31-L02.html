메시지 흐름을 설계

(Application Link Enabling)는 IDoc을 어느 시스템이 어느 시스템에 어떤 메시지를 보낼지 설계·운영하는 틀이다.

  • Distribution Model(BD64) — 발신↔수신·메시지 타입 관계 정의.
  • Partner Profile(WE20) — 파트너별 인바운드/아웃바운드 설정(처리 함수·포트).
  • Port(WE21) — 전송 경로(tRFC·파일 등).
  • Logical System(BD54) — 시스템 논리 이름.

흔한 실수와 점검

  • Partner Profile 누락 → IDoc이 생성/수신돼도 처리 안 됨.
  • 모델 미배포 → BD64에서 모델을 배포(distribute)해야 양쪽 일치.

흐름을 설계했다. 그런데 IDoc이 실패하면? 오류 추적·재처리 → 다음 레슨.
