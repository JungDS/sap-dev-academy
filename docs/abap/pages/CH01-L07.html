개발 환경과 첫 프로그램

T-code 생성 기초 (SE93)

내 프로그램을 짧은 코드 하나로 실행되게 만든다 — SE93.

Track 1Chapter 1Lesson 7입문

🖥️이번 Lesson에서 다루는 트랜잭션 코드

사용자에게 "SE38 열고 프로그램명 입력하세요"라고 할 순 없다

지금까지 내 프로그램을 에서 열고 F8로 실행했다. 하지만 실제 사용자에게 "SE38을 열고, 프로그램 이름을 정확히 입력하고, 실행하세요"라고 시킬 수는 없다. 사용자는 짧은 코드 하나 — 예를 들어 ZHELLO — 만 입력하면 바로 실행되길 원한다. 그 짧은 코드가 (트랜잭션 코드)이고, 만드는 곳이 이다.

T-code 다시 보기

는 화면 위쪽 명령창에 입력하는 기능 바로가기다(앞 레슨). SE38, SE11처럼 SAP이 기본 제공하는 것도 있지만, 내 프로그램에도 붙일 수 있다.

SE93로 만들기

명령창에 SE93을 입력해 들어간 뒤:

  1. 만들 트랜잭션 코드를 입력한다 — 고객용은 Z 또는 Y로 시작(ZHELLO).
  2. Create(생성) → 유형에서 "Program and selection screen (report transaction)" (실행형 리포트용)을 고른다.
  3. 연결할 프로그램 이름을 적는다(내가 만든 ZHELLO).
  4. 저장·활성화하면 끝 — 이제 명령창에 ZHELLO만 쳐도 프로그램이 실행된다.

왜 T-code가 필요한가 — 사용자는 SE38을 못 쓴다

SE38은 개발자 전용 화면이다. 현업(업무) 사용자에게는 SE38 권한 자체가 없어서, "에서 프로그램을 열어 실행"하는 길이 막혀 있다. 그래서 프로그램을 로 감싸 사용자가 명령창에 짧은 코드만 쳐도 실행되게 만든다. 즉 T-code는 있으면 편한 것이 아니라 사용자가 내 프로그램에 접근하는 사실상 유일한 통로다.

  • 실행 통로 → 사용자는 오직 T-code로만 내 프로그램을 실행한다(SE38 불가).
  • 메뉴·즐겨찾기 등록 → T-code가 있어야 Area Menu(SE43)에 넣거나 SAP Easy Access 즐겨찾기에 등록할 수 있다.
  • 화면 프로그램(Module Pool) 은 시작 T-code가 아예 필수 — 없으면 화면 자체를 띄울 수 없다. 화면은 Chapter 16 · Screen Programming / Dynpro 기초에서 배운다.

흔한 실수와 점검

  • 이름 충돌·네임스페이스 → 고객 는 Z/Y로 시작. SAP 표준 코드 이름은 못 쓴다.
  • 프로그램명 오타 → 연결한 프로그램 이름이 틀리면 실행 시 오류. 정확히 입력.
  • 활성화 안 함 → 만든 뒤 활성화해야 동작한다(activate — 프로그램 구조와 주석 참고).

정리

이제 환경 접속부터 출력·보관·실행()까지 개발의 한 바퀴를 다 돌았다. 그런데 돌아보면 값을 늘 '...' 따옴표에 직접 적어 왔다. 같은 값을 재사용하거나 계산 결과를 보관하려면 어딘가에 담아 둬야 한다 → Chapter 2 · 변수와 타입.
