Refresh와 Stable Refresh 기초

데이터가 바뀐 뒤 표를 다시 그린다 — 스크롤·선택을 유지하며.

Track 1Chapter 17Lesson 8중급

데이터가 바뀌면 다시 그린다

예매가 추가/취소돼 내부 테이블이 바뀌면, 그리드를 다시 그려야 화면에 반영된다. refresh_table_display를 부른다.

ABAP
1
go_grid->refresh_table_display( ).

Stable Refresh — 위치를 지키며

그냥 refresh하면 스크롤 위치·행 선택이 초기화돼 사용자가 보던 자리를 잃는다. Stable 옵션을 주면 스크롤·선택을 유지한 채 다시 그린다.

ABAP
1
2
3
4
5
6
DATA ls_stable TYPE lvc_s_stbl.
ls_stable-row = abap_true.
ls_stable-col = abap_true.

go_grid->refresh_table_display(
  EXPORTING is_stable = ls_stable ).

흔한 실수와 점검

  • refresh 안 함 → 내부 테이블만 바꾸고 화면은 그대로. refresh 호출 필요.
  • stable 미사용 → 새로고침마다 맨 위로 튄다. 편집·갱신이 잦으면 is_stable.

표를 갱신하는 법까지 익혔다. 마지막으로 행에 색을 입혀 강조해 보자 → 다음 레슨.
