CLASS ltcl_test IMPLEMENTATION.

    cl_abap_unit_assert=>assert_equals( act = lo->remaining( ) exp = 50 ).
  ENDMETHOD.
ENDCLASS.
  • FOR TESTING 에 테스트 를 둔다.
  • cl_abap_unit_assert로 기대값 검증.
  • Mock(가짜 의존성)으로 DB 없이 로직만 빠르게 검증.

테스트를 돌려 보기

테스트를 실행하면 초록 ✓가 뜬다. 그 상태에서 버그 주입을 켜면(로직이 잘못된 값을 반환) 해당 테스트가 빨갛게 실패하는 걸 보라 — 이렇게 회귀를 자동으로 잡는 것이 테스트의 가치다.

흔한 실수와 점검

  • DB 직접 의존 → 테스트가 느리고 불안정. 로 분리·Mock.
  • 테스트 없음 → 리팩터링이 두려워진다. 핵심 로직엔 단위 테스트를.

🧩 도전 과제 (서브퀘스트)

선택 과제 — 예매 로직을 OO 패턴으로 다듬는다.

  1. ZCL_BOOKING_MANAGER(CH20-L10)를 Factory로 생성하게 바꾼다(일반/VIP 예매).
  2. 요금 계산을 Strategy()로 분리해 정책을 갈아끼운다.
  3. (심화) remaining() 로직에 ABAP Unit 테스트를 작성하고 DB는 Mock으로 분리한다.

힌트: L01(Factory)·L03(Strategy)·L05(ABAP Unit/의존성 주입).
