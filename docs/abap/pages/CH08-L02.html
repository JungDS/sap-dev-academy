SELECT의 네 가지 핵심

📌 이 챕터는 classic 구문 — 필드를 공백으로 나열한다(콤마 ✕). 현대식 Open SQL 구문은 CH19에서 따로 배운다. 섞으면 경계가 무너진다.

SELECT 한 문장은 네 가지를 정하는 일이다.

  1. FROM — 어느 테이블에서
  2. 필드 — 어느 컬럼을 (* = 전부, 또는 필드 나열)
  3. WHERE — 어느 행만 (다음 레슨들)
  4. INTO — 어디에 담을지 (Work Area / Internal Table)

손입력한 구구단을 되찾기

CH07에서 손으로 넣은 18행을, 이제 SELECT로 한 번에 되찾는다.

DATA lt_gugu TYPE TABLE OF zgugudan.

SELECT * FROM zgugudan
  INTO TABLE lt_gugu.

IF sy-subrc = 0.
  WRITE: / lines( lt_gugu ), '건 읽음'.   " 18
ENDIF.
  • INTO TABLE은 결과 전체를 내부 테이블에 채운다().
  • 성공 여부는 (0 = 1건 이상), 읽은 행 수는 sy-dbcnt.

* vs 필드 (projection)

전부(*)보다 필요한 컬럼만 읽는 게 빠르다. classic은 필드를 공백으로 나열한다(콤마 ✕).

ABAP
1
2
3
SELECT dan mul result          " 콤마 없이 공백 구분 (classic)
  FROM zgugudan
  INTO CORRESPONDING FIELDS OF TABLE lt_gugu.
  • 컬럼 일부만 읽을 땐, 같은 이름 필드에 맞춰 담는 INTO CORRESPONDING FIELDS OF TABLE.
  • 여러 테이블을 다룰 땐 테이블~필드(예: sflight~carrid)로 어느 테이블 컬럼인지 밝힌다.

결과는 반드시 확인

결과 sy-subrc sy-dbcnt
1건 이상 0 읽은 행 수
한 건도 없음 4 0

흔한 실수와 점검

  • 콤마로 필드 나열 → SELECT dan, mul은 classic에서 오류. 콤마 구분은 CH19(modern)부터.
  • 결과 미확인 → 빈 결과일 수 있다. 항상 ·lines( ) 확인.
  • SELECT * 남용 → 큰 테이블 전체는 성능 위험. WHERE로 줄인다.

읽을 테이블·필드를 정했다. 그런데 SELECT에는 여러 형태가 있다 — 한 건만, 여러 건, 줄여 읽기 → 다음 레슨.
