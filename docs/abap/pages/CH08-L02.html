Value Table과 Foreign Key의 차이

Domain의 Value Table은 '제안', Foreign Key는 '실제 관계'.

비슷해 보이는 둘

을 만들 때 Value Table을 지정할 수 있고, 테이블 필드엔 를 건다. 둘 다 "어떤 테이블의 값으로 제한"하는 것 같은데 뭐가 다른가? 헷갈리기 쉬운 지점이라 분명히 짚는다.

Value Table = Domain의 '제안'

  • 은 Domain 수준에서 "이 도메인 값은 보통 이 테이블에서 온다"고 제안하는 것이다.
  • 그 자체로는 검증을 강제하지 않는다. 어디까지나 기본 제안.

Foreign Key = 테이블 필드의 '실제 관계'

  • 는 테이블 필드 수준에서 거는 실제 관계다.
  • 이게 있어야 입력검증·F4가 실제로 동작한다.
  • Foreign Key를 만들 때, Domain에 Value Table이 있으면 시스템이 그걸 Check Table로 자동 제안한다 → 둘은 협력 관계.
Value Table Foreign Key

흔한 실수와 점검

  • Value Table만 믿기 → 그것만으론 검증 안 된다. 필드에 Foreign Key를 걸어야.
  • Check Table ≠ Value Table → 보통 같게 쓰지만, Foreign Key에서 다른 Check Table을 지정할 수도 있다.
