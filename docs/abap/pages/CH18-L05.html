Modern ABAP Syntax

String Template과 내장 함수

출력 문장 모양 그대로 — | … { } … | 템플릿과 함수형 문자열 함수.

Track 1Chapter 18Lesson 5중급

CONCATENATE를 넘어서 — 문장 모양 그대로

문자열은 사용자가 보는 마지막 결과인 경우가 많다(메시지·ALV 문구·로그). classic은 문자열 다루기의 CONCATENATE(·&&)로 조각을 이었다. 간단하면 괜찮지만 변수와 문장이 여러 번 섞이면 출력 문장을 머릿속에서 다시 조립해야 한다.

String Template(| … |)은 값이 들어갈 자리를 { 표현식 }으로 표시해, 출력될 문장 모양 그대로 쓰게 해 준다.

ABAP
1
2
3
4
5
" classic
CONCATENATE lv_name '님 환영합니다' INTO lv_msg.

" modern — 최종 문장이 코드에 그대로 보인다
DATA(lv_msg) = |{ lv_name }님 환영합니다|.

구구단처럼 값이 여러 개 섞일수록 차이가 크다.

ABAP
1
DATA(lv_eq) = |{ ls_line-dan } x { ls_line-mul } = { ls_line-result }|.   " "2 x 3 = 6"

⚠️ { } 안쪽 공백은 문법 요소다. |{ lv_name }|처럼 중괄호 안쪽에 공백을 두는 형태를 기본으로 삼는다.

서식까지 한 번에 — DATE = USER · NUMBER = USER

날짜·숫자를 그냥 { sy-datum }·{ lv_total }로 넣으면 사용자가 기대하는 구분자·형식과 다를 수 있다. 화면 출력용이면 형식 옵션을 의식적으로 쓴다.

ABAP
1
2
DATA(lv_line) =
  |오늘은 { sy-datum DATE = USER }, 합계 { lv_total NUMBER = USER }원입니다.|.

함수형 문자열 함수

문자열 다루기의 명령형 처리를, modern에서는 값을 돌려주는 함수로 표현식 흐름에 쓴다.

ABAP
1
2
3
DATA(lv_up)  = to_upper( lv_name ).
DATA(lv_sub) = substring( val = lv_text off = 0 len = 3 ).   " 앞 3글자
DATA(lv_len) = strlen( lv_text ).

직접 조립해 보기 — Message Composer

CONCATENATE와 String Template을 나란히 비교하고, 사용자 형식을 켜면 날짜·숫자가 어떻게 바뀌는지 보자. substring의 off·len을 바꿔 보고, 문자열 길이보다 긴 범위를 요청하면 어떤 일이 생기는지도 확인한다.

흔한 실수와 점검

  • { } 누락 → 변수는 { }로 감싸야 값이 들어간다. 안쪽 공백도 지킨다.
  • 숫자·날짜 raw 임베딩 → 사람이 기대하는 표시와 다를 수 있다. NUMBER = USER·DATE = USER 검토.
  • substring 범위 초과 → 문자열 길이보다 긴 off/len은 오류가 난다. 입력 길이를 확인한다.
  • template 안 복잡한 계산 → 다시 읽기 어렵다. 복잡한 값은 먼저 DATA( )로 계산해 이름을 붙이고 template엔 그 이름만.

💡 String template은 읽을 문장을 코드와 가깝게 만들지만, 모든 텍스트 관리를 대체하진 않는다. 번역이 필요한 고정 문구는 Text Symbol·메시지 클래스를 계속 고려한다(다국어 설계는 별도 주제).

정리

  • String Template은 사람이 읽을 문장과 코드의 모양을 가깝게 만든다 — { 표현식 }으로 값을 넣고, 날짜·숫자는 형식 옵션을.
  • to_upper·substring·strlen 같은 함수는 표현식 흐름에 잘 맞지만, 범위와 가독성은 계속 확인한다.
