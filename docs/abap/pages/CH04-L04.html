IF 사다리가 길어질 때

"등급이 A면 …, B면 …, C면 …, 그 외엔 …" — 하나의 값을 여러 경우로 나눌 때 IF/ELSEIF를 길게 늘어놓으면 읽기 힘들다. 이럴 때 가 더 깔끔하다.

CASE … WHEN … ENDCASE

CASE 뒤에 검사할 값을 한 번 적고, WHEN마다 경우를 나열한다. 어디에도 안 맞으면 WHEN OTHERS로 받는다. ENDCASE로 닫는다.

PARAMETERS p_grade TYPE c LENGTH 1.

CASE p_grade.
  WHEN 'A'.
    WRITE: / '우수'.
  WHEN 'B'.
    WRITE: / '양호'.
  WHEN 'C'.
    WRITE: / '보통'.
  WHEN OTHERS.
    WRITE: / '등급 없음'.
ENDCASE.

같은 처리를 묶으려면 WHEN에 값을 OR로 나열한다.

  WHEN OTHERS.
    WRITE: / '불합격'.

IF냐 CASE냐 — 도구 선택

  • 한 변수를 여러 고정값과 비교 → CASE (깔끔함).
  • 범위·복합 조건(p_amt > 1000 AND …) → IF (유연함).

는 =(같다) 비교만 한다. "1000보다 크냐" 같은 범위 판단은 IF의 몫이다.

흔한 실수와 점검

  • ENDCASE 누락 → CASE는 ENDCASE로 닫는다(ENDIF 아님).
  • WHEN OTHERS 빼먹음 → 예상 못 한 값이 들어오면 아무것도 안 한다. 안전하게 WHEN OTHERS를 둔다.
  • 범위 비교를 로 → 는 "같다"만 본다. 범위는 IF로.

정리

값을 보고 갈라지는 두 도구(IF·)를 익혔다. 이제 같은 일을 여러 번 되풀이하고 싶다 — 구구단처럼 1단부터 9단까지 → CH04-L05(DO/WHILE).
