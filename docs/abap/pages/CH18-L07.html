COND·SWITCH 조건 표현식

조건에 따라 값 하나를 만드는 표현식 — 흐름을 나누는 IF/CASE와 구분한다.

값 하나 정하려고 IF를 여러 줄 쓴다

와 는 흐름을 나눈다. 그런데 실무 코드에는 흐름 전체를 나누기보다 "상태 문구 하나", "색 코드 하나" 를 조건에 따라 정하고 싶은 경우가 많다. 이때 IF로 변수를 채우면, 정작 하려는 일(값 하나 정하기)보다 대입 과정이 더 크게 보인다.

DATA lv_status_text TYPE string.
IF lv_remaining = 0.
  lv_status_text = '매진'.
ELSEIF lv_remaining < 10.
  lv_status_text = '마감 임박'.
ELSE.
  lv_status_text = '예매 가능'.
ENDIF.

COND — 값을 만드는 IF

는 조건을 차례로 검사해 첫 번째로 참인 결과를 돌려준다. IF에 가깝다.

ABAP
1
2
3
4
DATA(lv_status_text) = COND string(
  WHEN lv_remaining = 0  THEN '매진'
  WHEN lv_remaining < 10 THEN '마감 임박'
  ELSE                        '예매 가능' ).

SWITCH — 값을 만드는 CASE

는 하나의 값이 어떤 상수와 같은지 비교해 결과를 돌려준다. CASE에 가깝다.

ABAP
1
2
3
4
DATA(lv_status_text) = SWITCH string( lv_status
  WHEN 'N' THEN '예약'
  WHEN 'C' THEN '취소'
  ELSE          '알 수 없음' ).

💡 꼭 구분할 점 — COND/SWITCH는 문장 묶음을 실행하는 도구가 아니라 값을 만드는 도구다. 조건마다 저장·화면 호출·복잡한 처리를 넣고 싶다면 표현식이 아니라 IF/CASE가 맞다.

흔한 실수와 점검

  • 모든 IF를 COND로 → 짧다고 다 바꾸지 않는다. 흐름을 나누는 건 여전히 IF/CASE다.
  • SWITCH에 복잡한 조건 → SWITCH는 하나의 값을 상수들과 같은지 비교할 때만 맞다. 범위·복합 조건은 COND.
  • ELSE 생략 → 조건에 안 맞으면 결과가 초기값이 된다(빈 문자·0). 필요하면 ELSE를 둔다.
  • 결과 타입 불일치 → THEN 뒤 결과들은 결과 타입으로 변환 가능해야 한다.

정리

  • COND는 값을 만드는 IF, SWITCH는 값을 만드는 다.
  • 흐름을 나누는 IF/CASE를 대체하는 게 아니라, 단순 조건 대입을 더 가까운 위치에 드러내는 도구다.

조건으로 값을 골랐다. 이제 테이블 여러 행을 하나로 줄이거나 걸러 내는 표현식으로 가자 → REDUCE·FILTER 테이블 표현식.
