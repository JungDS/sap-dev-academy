구구단 전체를 메모리에 담는다

CH05에서 구구단 한 줄을 구조체로 묶었지만 한 번에 한 줄뿐이었다. 이제 같은 모양의 행을 내부 테이블에 쌓으면 81줄을 통째로 들고 있을 수 있다.

TYPES: BEGIN OF ty_line,
         dan    TYPE i,
         mul    TYPE i,
         result TYPE i,
       END OF ty_line.

DATA: ls_line TYPE ty_line,
      lt_gugu TYPE TABLE OF ty_line.

DO 8 TIMES.
  ls_line-dan = sy-index + 1.            " 2~9단
  DO 9 TIMES.
    ls_line-mul    = sy-index.
    ls_line-result = ls_line-dan * ls_line-mul.
    APPEND ls_line TO lt_gugu.           " 한 줄씩 쌓는다
  ENDDO.
ENDDO.

이제 lt_gugu에는 구구단 81행이 전부 들어 있다 — 출력 후에도 사라지지 않고 정렬·검색에 쓸 수 있다.

정렬해서 출력 — SORT + LOOP

SORT lt_gugu BY result DESCENDING.       " 결과 큰 순서로

LOOP AT lt_gugu INTO ls_line.
  WRITE: / ls_line-dan, 'x', ls_line-mul, '=', ls_line-result.
ENDLOOP.

한 줄씩 출력만 하던 CH05와 달리, 모아 두었기에 "결과가 큰 순서"처럼 마음대로 재배열할 수 있다.

디버거로 행들 보기

APPEND 다음 줄에 BREAK-POINT를 걸고 lt_gugu를 펼치면, 행이 하나씩 쌓여 2개, 4개 … 81개가 되는 모습이 보인다(CH04-L06 디버거).

정리 — 그리고 다음 불편

구구단 전체를 메모리에서 자유롭게 다뤘다. 그런데 프로그램이 끝나면 lt_gugu는 흔적도 없이 사라진다. 다시 실행하면 또 처음부터 만들어야 한다. 값을 영구히 저장해 두고 싶다 → CH07(Transparent Table).
