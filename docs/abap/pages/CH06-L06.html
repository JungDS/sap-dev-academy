Track 1Chapter 6Lesson 6초급

구구단 전체를 메모리에 담는다

Chapter 5 · 구구단 캡스톤에서 구구단 한 줄을 Structure로 묶었지만 한 번에 한 줄뿐이었다. 이제 같은 모양의 행을 에 쌓으면 81줄을 통째로 들고 있을 수 있다. 이 실습은 이번 챕터 전체의 결론이다.

배운 개념 구구단에서의 역할
Line Type dan·mul·result를 묶은 행 모양(ts_line)
Work Area 지금 만들고 있는 한 줄(gs_line)
Internal Table 81줄을 담는 표(gt_gugu)
APPEND 한 줄씩 쌓기
SORT 결과 기준 재배열
LOOP 쌓인 81줄 출력

TYPES: BEGIN OF ts_line,
         dan    TYPE i,
         mul    TYPE i,
         result TYPE i,
       END OF ts_line.

DATA: gs_line TYPE ts_line,
      gt_gugu TYPE TABLE OF ts_line,
      gv_cnt  TYPE i.

DO 8 TIMES.
  gs_line-dan = sy-index + 1.            " 2~9단
  DO 9 TIMES.
    gs_line-mul    = sy-index.
    gs_line-result = gs_line-dan * gs_line-mul.
    APPEND gs_line TO gt_gugu.           " 한 줄씩 쌓는다
  ENDDO.
ENDDO.

DESCRIBE TABLE gt_gugu LINES gv_cnt.     " gv_cnt = 81

이제 gt_gugu에는 구구단 81행이 전부 들어 있다. 출력 후에도 사라지지 않고 정렬·검색에 쓸 수 있다.

정렬해서 출력 — SORT + LOOP

SORT gt_gugu BY result DESCENDING.       " 결과 큰 순서로

LOOP AT gt_gugu INTO gs_line.
  WRITE: / gs_line-dan, 'x', gs_line-mul, '=', gs_line-result.
ENDLOOP.

한 줄씩 출력만 하던 지난 챕터의 구구단과 달리, 모아 두었기에 "결과가 큰 순서" 같은 원하는 기준으로 마음대로 재배열할 수 있다.

디버거로 행들 보기

APPEND 다음 줄에 BREAK-POINT를 걸고(디버깅 입문) gt_gugu를 펼치면, 행이 하나씩 쌓여 2개, 4개 … 81개가 되는 모습이 보인다.

흔한 실수와 점검

  • 바깥 sy-index와 안쪽 sy-index 혼동 → 안쪽 루프에 들어가면 는 안쪽 회차다. 그래서 바깥 단수는 안쪽 루프에 들어가기 전에 gs_line-dan에 저장해 둔다.
  • APPEND 위치 → mul·result가 채워진 뒤에 실행돼야 한다. 안쪽 루프 밖에 두면 단마다 한 줄만 쌓인다.
  • 정렬 후 원래 순서 기대 → SORT 후에는 순서가 바뀐다. 원래 입력 순서가 필요하면 정렬 전에 기준이 되는 필드를 따로 보존한다.
  • 담았으니 저장됐다는 착각 → 에 담긴 것과 DB에 저장된 것은 다르다.

정리 — 그리고 다음 불편

구구단 전체를 에서 자유롭게 다뤘다. 그런데 프로그램이 끝나면 gt_gugu는 흔적도 없이 사라진다. 다시 실행하면 또 처음부터 만들어야 한다. 값을 영구히 저장해 두고 싶다 → Chapter 7 · Transparent Table.
