개발 환경과 첫 프로그램

WRITE로 문자열 출력

첫 출력 — WRITE로 화면에 글자를 찍어 개발 루프를 완성한다.

Track 1Chapter 1Lesson 4입문

드디어, 화면에 출력

여기까지 왔으면 이제 결과를 눈으로 볼 차례다. ABAP의 가장 기본 출력문 로 화면에 글자를 찍고, 개발 루프(작성→활성화→실행)를 처음으로 끝까지 돌려본다.

WRITE로 한 줄 출력

REPORT zhello.

WRITE 'Hello, ABAP!'.

작성 → 저장 → 활성화(Ctrl+F3) → 실행(F8). 그러면 리스트 화면에 Hello, ABAP!이 출력된다. 다른 언어의 print / console.log에 해당한다.

여러 줄과 줄바꿈

는 기본적으로 같은 줄에 이어 출력한다. 줄을 바꾸려면 /를 쓴다.

WRITE 'first line'.
WRITE / 'second line'.

여러 값을 한 번에 출력할 땐 콜론(:)으로 문장을 체인한다.

WRITE: / 'A', 'B', 'C'.

WRITE: 뒤에 콤마로 나열하면 WRITE를 반복해 쓴 것과 같다.

문자열 리터럴

작은따옴표로 감싼 '...'가 문자 리터럴이다.

WRITE '값을 직접 적은 것이 리터럴'.

지금은 출력할 값을 코드에 직접 적고 있다. 따옴표 안 글자가 그대로 나온다.

흔한 실수와 점검

  • 따옴표 짝 안 맞음 → '를 열고 안 닫으면 문자열이 끝나지 않아 오류.
  • 줄바꿈이 안 됨 → /를 빠뜨렸다. WRITE / ....
  • 마침표 누락 → 앞 레슨의 그 규칙. 여전히 1번 단골이다.

축하한다 — 첫 프로그램이 돌아갔다. 그런데 곧 답답해진다. 출력할 값을 매번 따옴표에 직접 적자니 바꾸기도 번거롭고 재사용도 안 된다. "값을 어딘가에 담아 두고 다루고 싶다" — 그게 다음 챕터 CH02(변수와 타입) 의 출발점이다.
