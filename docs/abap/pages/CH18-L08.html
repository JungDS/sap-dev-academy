Modern ABAP Syntax

REDUCE·FILTER 테이블 표현식

여러 행을 하나로 줄이는 REDUCE, 조건 행만 새 테이블로 뽑는 FILTER.

Track 1Chapter 18Lesson 8중급

합계 하나 만들려고 LOOP를 또 쓴다

을 배운 뒤로 우리는 반복해서 를 썼다. 전체 좌석 수를 합산하거나, 상태가 N인 예매만 모으는 코드는 매우 흔하다. LOOP는 명확하지만, "합계 하나" 나 "조건에 맞는 행만 추출" 처럼 패턴이 단순하면 준비 코드가 많다.

REDUCE — 여러 값을 하나로 줄인다

는 초기값을 만들고, 반복하며 누적 변수에 값을 더해 최종 결과 하나를 만든다.

ABAP
1
2
3
4
DATA(lv_total_seats) = REDUCE i(
  INIT sum = 0
  FOR ls_book IN lt_booking
  NEXT sum = sum + ls_book-seats ).

이건 아래 classic loop와 같은 목적이다.

ABAP
1
2
3
4
DATA lv_total_seats TYPE i.
LOOP AT lt_booking INTO DATA(ls_book).
  lv_total_seats = lv_total_seats + ls_book-seats.
ENDLOOP.
  • INIT sum = 0 — 누적을 시작할 값.
  • FOR ls_book IN lt_booking — 한 행씩 꺼낸다.
  • NEXT sum = ... — 다음 누적값을 계산한다(반복마다).

FILTER — 조건에 맞는 행만 새 테이블로

는 조건에 맞는 행만 뽑아 새 을 만든다. FILTER는 테이블의 Key와 조건 관계가 중요해서, 여기서는 Key가 분명한 로 익힌다.

ABAP
1
2
3
4
TYPES tt_booking TYPE SORTED TABLE OF ts_booking
  WITH NON-UNIQUE KEY status.

DATA(lt_open) = FILTER tt_booking( lt_booking WHERE status = 'N' ).

EXCEPT를 붙이면 조건에 맞지 않는 행을 가져온다.

ABAP
1
DATA(lt_not_open) = FILTER tt_booking( lt_booking EXCEPT WHERE status = 'N' ).

흔한 실수와 점검

  • REDUCE를 처음부터 복잡하게 → 초보에게 가장 읽기 어려운 표현식 중 하나다. 단순 합계부터 시작한다. 누적 중간에 검증·메시지가 많으면 LOOP가 낫다.
  • FILTER에서 Key 무시 → FILTER는 테이블 타입과 Key를 의식해야 한다. Key가 분명한 테이블에서 시작한다.
  • FILTER ... EXCEPT를 단순 NOT으로 → 항상 같은 사고방식은 아니다. 결과 행을 직접 확인한다.

정리

  • REDUCE는 테이블을 합계나 하나의 결과로 줄이는 표현식이다.
  • FILTER는 조건에 맞는 행만 새 테이블로 만드는 표현식이다.
  • 둘 다 단순한 반복 패턴을 줄이는 도구다. 복잡한 업무 흐름은 여전히 LOOP가 나을 수 있다.

표현식이 점점 길어진다. 그 안의 중간값에 이름을 붙여 읽기 쉽게 하는 LET으로 가자 → LET으로 표현식 속 이름 읽기.
