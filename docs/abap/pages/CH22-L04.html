데이터에 의미를 입힌다

CDS의 강점 중 하나는 Annotation(@로 시작하는 메타정보)으로 필드·뷰에 의미를 선언하는 것이다. UI·검색·분석 도구가 이 의미를 읽어 자동으로 동작한다.

@EndUserText.label: '공연'
define view entity ZC_Concert
  as projection on ZI_Concert
{
  @EndUserText.label: '공연 ID'
  key concert_id,

  @UI.lineItem: [{ position: 10 }]      // 목록 표시 위치
  @EndUserText.label: '아티스트'
      artist,

      capacity
}
  • @EndUserText.label — 라벨(다국어).
  • @UI.* — Fiori 화면 자동 구성 힌트(목록·폼 위치 등).
  • @Semantics.* — 금액·수량·통화 같은 의미. 단 수량/금액 의미(@Semantics.quantity.unitOfMeasure·@Semantics.amount.currencyCode)는 단위·통화 필드를 함께 가리켜야 하므로 그 짝 필드가 있는 모델에서 쓴다.

흔한 실수와 점검

  • 주석 위치 → 뷰 전체 주석은 뷰 위, 필드 주석은 그 필드 위.
  • UI 주석 과다 → 많아지면 Metadata Extension으로 분리(다음 레슨).

의미를 달았다. UI 주석이 많아지면 따로 빼는 Metadata Extension → 다음 레슨.
