Deep Structure 기반 Cell Color

행이 아니라 셀 하나만 — 행 구조에 색 정보 테이블을 품는다.

Track 1Chapter 22Lesson 4고급

행이 아니라 셀 하나만 🎯

행 색상 기초에서 행 전체 색을 봤다. 그런데 공연명·날짜·정원은 정상인데 잔여석만 위험하다면, 행 전체를 칠하는 것보다 그 셀 하나만 칠하는 게 정확하다.

방법은 행 구조에 (LVC_T_SCOL)을 컬럼으로 품는 것이다. 한 행 안에서도 여러 셀에 각각 다른 색이 붙을 수 있으니, 색 정보는 단일 값이 아니라 목록이다. 여기서 Deep Structure 개념()이 쓰인다.

🧱 flat ↔ deep 분리. DB는 flat table로 조회한다(deep component를 가진 work area는 SQL에서 제한). 표시용 internal table을 따로 두고 거기에 색 deep component를 채운다.

TYPES: BEGIN OF ts_perf_alv,
         concert_id TYPE zperf-concert_id,
         perf_no    TYPE zperf-perf_no,
         capacity   TYPE zconcert-capacity,  " 정원은 공연 마스터에서
         booked     TYPE i,                  " 예약 좌석 합계(ZBOOKING 집계)
         seats_left TYPE i,                  " 잔여석 = capacity - booked
         cellcolors TYPE lvc_t_scol,         " 셀 색 정보(테이블 in 행)
       END OF ts_perf_alv.
DATA gt_perf_alv TYPE STANDARD TABLE OF ts_perf_alv.

각 행에서 색을 줄 셀을 지정한다.

LOOP AT gt_perf_alv ASSIGNING FIELD-SYMBOL(<perf>).
  CLEAR <perf>-cellcolors.
  IF <perf>-booked >= <perf>-capacity.              " 매진
    APPEND VALUE lvc_s_scol(
      fname     = 'BOOKED'                            " 어느 셀
      color-col = col_negative                        " 빨강 계열(색 번호)
      color-int = 1 ) TO <perf>-cellcolors.           " 진하게(1) / 연하게(0)
  ENDIF.
ENDLOOP.

색 한 칸(lvc_s_scol)은 어느 셀에(fname) 무슨 색을(color) 줄지 담는다. color는 다시 세 부분이다.

요소 뜻 예
color-col 색 번호(1～7). col_negative(빨강)·col_total(노랑)·col_positive(초록) 같은 상수로 쓴다 col_negative
color-int 진하게 표시(1) / 연하게(0) 1
color-inv 글자색↔배경색 반전(1/0) 0

핵심 — 연결해야 보인다 🔌

cellcolors에 색 줄을 넣기만 하면 ALV가 자동으로 읽지 않는다. layout에 "이 컬럼이 셀 색 테이블"이라고 알려야 한다.

ABAP
1
2
DATA gs_layout TYPE lvc_s_layo.
gs_layout-ctab_fname = 'CELLCOLORS'.     " 셀색 컬럼 이름을 Layout에 연결
  • LVC_T_SCOL 한 줄 = (어느 컬럼 fname + 색 color).
  • ctab_fname = 그 색 목록이 든 component 이름을 ALV layout에 알려 주는 연결선.

흔한 실수와 점검 🚧

  • ctab_fname 미지정 → 색 컬럼을 둬도 연결 안 하면 색이 그냥 안 보이는 데이터. (위 체험의 2단계.)
  • fname 오타 → SEATS_OCC처럼 틀리면 색 정보는 있어도 적용 안 된다. 컬럼 제목이 아니라 component 이름(SEATSOCC).
  • deep로 직접 SELECT → DB는 flat으로 읽고, 표시용 deep table로 옮긴 뒤 색을 채운다.
  • CELLCOLORS가 화면에 노출 → 기술 컬럼이라 field catalog에서 숨긴다(no_out).
  • 색 남용 → 위험·경고·확정 같은 제한된 의미에만. 다 칠하면 오히려 안 읽힌다.

셀에 색을 줬다. 이제 같은 deep 방식으로 셀의 모양·동작(비활성·버튼)을 정하는 Cell Style → Deep Structure 기반 Cell Style.
