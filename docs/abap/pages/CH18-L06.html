Modern ABAP Syntax

CONV·EXACT 변환 표현식

임시 변수 없이 그 자리에서 타입을 바꾸는 CONV, 손실을 막는 EXACT.

Track 1Chapter 18Lesson 6중급

변환 한 번 하려고 변수를 또 만든다

숫자·문자·날짜·금액은 서로 다른 타입이라고 여러 번 봤다. classic ABAP에서는 타입을 바꿀 때 임시 변수를 하나 만들어 대입한 뒤 그 변수를 다시 넘기는 방식이 흔했다.

ABAP
1
2
3
DATA lv_count_text TYPE string VALUE '12'.
DATA lv_count      TYPE i.
lv_count = lv_count_text.        " 변환해서 담을 변수를 따로 만듦

값 하나 바꾸자고 변수를 또 선언하니, 정작 하려던 일보다 준비 과정이 더 커 보인다.

CONV — 그 자리에서 타입을 바꾼다

는 값을 지정한 타입으로 바꾼 결과를 그 자리에서 만든다. CONV 타입( 값 ) 꼴이다.

ABAP
1
DATA(lv_count) = CONV i( '12' ).   " 문자 '12' → 정수 12, 임시 변수 없이

메서드나 함수가 특정 타입을 요구할 때 임시 변수를 줄이는 데도 좋다.

ABAP
1
2
DATA(lv_name) = '정훈영'.
DATA(lv_line) = |예매자: { CONV string( lv_name ) }|.

EXACT — 손실이 나면 막는다

는 손실 없는 변환을 요구하는 표현식이다. 값이 잘리거나 반올림돼 의미가 달라질 수 있으면 변환을 막는다(예외가 난다). 금액·수량처럼 한 자리도 틀리면 안 되는 값에 쓴다.

ABAP
1
DATA(lv_price) = EXACT i( '120000' ).   " 손실 없이 딱 맞아야 통과

🔎 EXACT가 손실을 만나면 예외가 발생한다. 예외를 잡아서 처리하는 법은 Exception Class — TRY/CATCH/CX 계층에서 배운다. 여기서는 "손실을 강하게 막고 싶을 때 쓰는 도구"라는 것까지만 알면 된다.

흔한 실수와 점검

  • 타입을 속이는 문법으로 오해 → CONV는 실제 변환 규칙을 따른다. '12A'를 정수로 바꾸려 하면 안전하게 변환되지 않는다.
  • 이미 같은 타입인데 CONV → 불필요하면 문법 검사 경고가 날 수 있다.
  • 금액·날짜를 무턱대고 변환 → 업무 의미가 큰 값은 원본 타입과 값 범위를 먼저 확인한다.
  • 변환을 과하게 중첩 → 읽기 어려우면 이름 있는 변수로 분리한다.

정리

  • CONV 타입( 값 )은 임시 변환 변수를 줄이고 그 자리에서 타입 의도를 드러낸다.
  • EXACT는 손실 없는 변환을 요구한다(실패 시 예외 → CH21).
  • 변환은 편의 문법이 아니라 데이터 의미를 바꾸는 행위다. 항상 값의 안전성을 함께 생각한다.

값을 바꾸는 표현식을 봤다. 이제 조건에 따라 값 하나를 고르는 표현식으로 넘어가자 → COND·SWITCH 조건 표현식.
