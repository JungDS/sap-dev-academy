Screen Programming / Dynpro 기초

Screen Number와 Screen Painter

화면에 번호를 붙이고, Screen Painter의 네 조각으로 화면을 그린다.

Track 1Chapter 16Lesson 2중급

화면에 번호를 붙인다

Module Pool에는 화면이 하나만 있는 게 아니다. 예매 입력 화면 0100, 상세 확인 화면 0200, 팝업 0300처럼 여러 가 있을 수 있고, 화면마다 layout과 흐름 로직이 따로 있다. 그래서 Dynpro는 네 자리 번호로 구분한다 — 번호를 모르면 "지금 내가 어느 화면의 PBO/PAI를 고치는가"를 잃어버린다.

Screen Painter의 네 조각

SE80(또는 SE51) 는 "ABAP 코드만 쓰면 화면이 생긴다"가 아니라, 네 조각이 함께 맞아야 움직이는 구조다.

조각 무엇을 담나 확인 위치
Layout 입력칸·텍스트·버튼의 위치·크기 Screen Painter Layout
Element List 화면 요소 이름·타입·길이·OK field Element List
PBO/PAI 블록 + 호출할 MODULE Flow Logic
Attributes next screen·modal 여부·화면 타입 Screen Attributes
  • 화면 요소에 준 이름은 같은 이름의 전역 변수와 자동 연결된다(예: 입력칸 P_SEATS ↔ 프로그램의 DATA p_seats). 이름이 다르면 값이 운반되지 않는다.
  • 사용자 버튼의 function code를 받는 OK field는 Element List에서 지정한다(자세히는 PAI 처리 흐름 — OK_CODE와 화면 떠나기).

Flow Logic — 화면이 부르는 MODULE

화면마다 에 PBO/PAI 두 블록이 있고, 각 블록에서 처리 모듈(MODULE)을 부른다.

ABAP

2
3
4
PROCESS BEFORE OUTPUT.
  MODULE status_0100.
PROCESS AFTER INPUT.
  MODULE user_command_0100.

Flow Logic은 ABAP과 조금 다른 화면 전용 문법이다. 여기서는 MODULE을 부르기만 한다.

ABAP 쪽 MODULE 정의 — OUTPUT(PBO) / INPUT(PAI)

실제 처리 코드는 프로그램 쪽 MODULE ... ENDMODULE에 쓴다. OUTPUT/INPUT 추가어가 이 모듈이 PBO에서 불릴지 PAI에서 불릴지를 정한다.

MODULE status_0100 OUTPUT.          " OUTPUT = PBO에서 호출
  " 초기값·화면 상태 준비
ENDMODULE.

MODULE user_command_0100 INPUT.     " INPUT = PAI에서 호출
  " 입력·버튼 처리
ENDMODULE.

🔑 핵심 구분. 의 MODULE status_0100.(호출)과 ABAP의 MODULE status_0100 OUTPUT.(정의)은 다른 위치에 있고, 이름이 같아야 연결된다. 이름이 어긋나면 화면 활성화가 실패한다.

직접 연결해 보기 — Screen Painter 연결 지도

Layout · Element List · · ABAP Source 네 패널이 어떻게 연결되는지 보고, 연결을 하나씩 깨뜨려 어떤 활성화 오류가 나는지 확인하자.

흔한 실수와 점검

  • 호출 ≠ ABAP 정의 → MODULE status_0100.은 있는데 MODULE status_0100 OUTPUT. 정의가 없으면 활성화 오류("호출 대상 없음").
  • 요소 이름 ≠ 변수 이름 → 자동 data transport가 안 된다. 같은 이름으로 맞춘다.
  • Flow Logic에 일반 ABAP 로직 → Flow Logic엔 MODULE 호출 같은 화면 문법만. 로직은 모듈 안에.
  • OK field 미지정 → 사용자 버튼의 function code를 안정적으로 못 읽는다.
  • next screen 0을 빈 값으로 오해 → 0은 화면 sequence 종료를 뜻한다.

정리

  • Screen Number는 "어느 화면인가"를 가리키는 주소, 는 그 화면의 layout·flow logic을 만드는 도구다.
  • 는 Layout · Element List · · ABAP module 네 조각이 모두 맞아야 움직인다.

화면과 흐름의 골격을 만들었다. 이제 화면에 올릴 요소들(입력칸·버튼·체크박스·라디오· 드롭다운)을 보자 → 화면 요소 — 입력·버튼·체크박스·라디오·드롭다운.
