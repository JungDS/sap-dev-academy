SELECT-OPTIONS와 Range Table

WHERE … IN (classic range)

Range Table을 조회 조건으로 — classic WHERE … IN.

Track 1Chapter 11Lesson 3중급

범위를 조회에 꽂기

가 채운 을 WHERE ... IN으로 SELECT에 바로 꽂는다. 한 줄로 범위·다중 조건이 적용된다.

이 챕터는 classic이다. WHERE age IN s_age처럼 @ 없이 쓴다. modern(WHERE age IN @s_age)은 CH18부터.

WHERE … IN (classic)

DATA lt_person TYPE TABLE OF ztperson.

SELECT * FROM ztperson
  INTO TABLE lt_person
  WHERE age IN s_age.        " classic: @ 없이 Range Table 직접
  • IN s_age 하나로 사용자가 입력한 모든 범위·목록 조건이 적용된다.
  • 비어 있으면(아무 입력 없음) 조건 없음으로 동작 → 전체 조회. 의도와 다르면 검증을 추가한다.

여러 SELECT-OPTIONS 결합

SELECT * FROM ztperson INTO TABLE lt_person
  WHERE age  IN s_age
    AND name IN s_name.

흔한 실수와 점검

  • @s_age 사용 → 이 챕터는 classic이라 @ 없이. modern은 CH18.
  • 빈 SELECT-OPTIONS = 전체 → 무조건 조회가 의도가 아니면 입력 필수 처리.

조건은 됐다. 그런데 화면의 다중 선택 팝업(여러 값·제외)은 어떻게 다루나 → 다음 레슨.
