Internal Table

Internal Table 기초

같은 모양의 행을 여러 개 — 내부 테이블 선언과 행 추가.

Track 1Chapter 6Lesson 1초급

한 건으로는 부족하다

CH05의 구조체는 사람 한 명을 담는다. 그런데 명단은 보통 여러 명이다. 같은 모양(구조)의 행을 여러 개 쌓아 두고 싶다 — 그게 (내부 테이블)다.

⚠️ Internal Table ≠ Database Table — 가장 흔한 입문 오해다. 내부 테이블은 프로그램이 도는 동안만 메모리에 있다가 끝나면 사라진다. 영구 저장은 DB 테이블(CH07)의 몫이다. 이 둘을 구별하지 못하면 로직 설계 전체가 흔들린다.

선언 — TYPE TABLE OF

행의 모양(구조 타입) 위에 TABLE OF를 붙인다.

ABAP
1
2
3
4
5
6
7
TYPES: BEGIN OF ty_person,
         name TYPE string,
         age  TYPE i,
       END OF ty_person.

DATA: ls_person TYPE ty_person,          " 한 건(Work Area)
      lt_person TYPE TABLE OF ty_person. " 여러 건(Internal Table)
  • lt_ 접두어가 관례(local table). 은 메모리에만 존재한다.
  • 한 건을 담는 (ls_)와 짝지어 쓴다.

행 추가 — APPEND

에 값을 채우고 APPEND로 테이블에 쌓는다.

ABAP
1
2
3
4
5
6
ls_person-name = '정훈영'. ls_person-age = 30.
APPEND ls_person TO lt_person.

CLEAR ls_person.
ls_person-name = '홍길동'. ls_person-age = 25.
APPEND ls_person TO lt_person.

이제 lt_person에는 행이 둘이다. lines( lt_person )으로 개수를 센다.

비우기 — CLEAR / REFRESH

ABAP
1
2
3
CLEAR   lt_person.   " 모든 행 제거(메모리는 유지)
REFRESH lt_person.   " 행 전체 삭제(내부 테이블 전용·CLEAR와 사실상 동일)
FREE    lt_person.   " 행 삭제 + 차지했던 메모리까지 반환
  • CLEAR/REFRESH — 행을 모두 비우되 잡아 둔 메모리는 유지.
  • FREE — 행을 비우고 메모리까지 반환(대용량 테이블을 다 쓴 뒤 권장).
  • ⚠️ 옛 Header Line(테이블에 같은 이름의 작업영역이 딸리는 단축 방식)이 붙어 있으면 CLEAR lt_가 그 헤더만 비울 수 있어 혼란스럽다. 우리는 헤더라인 없이 INTO ls_ 방식을 쓰므로 CLEAR lt_는 행 전체를 비운다(헤더라인은 옛 코드에서 알아보는 용도로만).

흔한 실수와 점검

  • APPEND 전에 CLEAR 안 함 → 이전 값이 남아 섞일 수 있다. 새 행 전에 CLEAR ls_person 습관.
  • 헤더라인 옛 방식 → 오래된 코드의 "header line" 대신, 지금은 INTO ls_ 작업영역 방식을 쓴다(명확·안전).

행을 쌓았다. 그런데 이 테이블 타입도 로컬·전역으로 나뉜다 → 다음 레슨(Table Type).
