Structure (Local · DDIC)

Local Structure (BEGIN OF ~ END OF)

관련된 값을 하나로 — BEGIN OF ~ END OF로 구조체를 만든다.

Track 1Chapter 5Lesson 1초급

구구단 한 줄에 변수가 셋

CH04에서 구구단을 만들 때 gv_dan(단)·gv_mul(곱하는 수)·gv_res(결과) 변수 세 개를 따로 들고 다녔다. 이 셋은 사실 "구구단 한 줄"이라는 한 묶음인데, 코드에는 뿔뿔이 흩어져 보인다. 함수에 넘기거나 초기화할 때도 셋을 일일이 챙겨야 한다. 관련된 값을 하나로 묶고 싶다 — 그게 (구조체)다.

BEGIN OF ~ END OF

로 구조체를 선언한다. 묶을 필드를 BEGIN OF와 END OF 사이에 둔다.

ABAP
1
2
3
4
5
DATA: BEGIN OF ls_line,
        dan    TYPE i,
        mul    TYPE i,
        result TYPE i,
      END OF ls_line.

이렇게 만든 변수 하나를 (작업 영역)라 부른다 — 한 건(레코드)을 담는 그릇이다. ls_ 접두어가 관례(local structure).

컴포넌트 접근 — 하이픈(-)

구조체 안의 필드(컴포넌트)는 하이픈 - 로 접근한다.

ABAP
1
2
3
4
5
ls_line-dan    = 2.
ls_line-mul    = 3.
ls_line-result = ls_line-dan * ls_line-mul.   " 6

WRITE: / ls_line-dan, 'x', ls_line-mul, '=', ls_line-result.

이제 "구구단 한 줄"이 변수 하나(ls_line)로 다뤄진다 — 셋이 한 묶음임이 코드에도 보인다.

타입으로 빼두기 (재사용)

구조체 모양을 로 정의하면 여러 변수가 같은 구조를 공유한다. 사람 한 명(이름·나이·금액)처럼 다른 묶음도 똑같이 만든다.

ABAP
1
2
3

5

8
9

11
TYPES: BEGIN OF ty_person,
         name   TYPE string,
         age    TYPE i,
         amount TYPE p LENGTH 8 DECIMALS 2,
       END OF ty_person.

DATA ls_person TYPE ty_person.

ls_person-name   = '정훈영'.
ls_person-age    = 30.
ls_person-amount = '1500.50'.

흔한 실수와 점검

  • 점(.)으로 접근 → ABAP의 컴포넌트 접근은 -다. ls_line-dan(O), .dan(X).
  • END OF 이름 → END OF ls_line. 처럼 같은 이름으로 닫는다.

구구단 한 줄을 깔끔히 묶었다. 그런데 이 구조도 프로그램마다 또 정의하긴 마찬가지 — 전역으로 공유하고 싶다 → Structure(다음 레슨).
