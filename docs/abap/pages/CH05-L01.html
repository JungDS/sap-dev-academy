CH05 · Internal Table

Internal Table 기초

같은 모양의 행을 여러 개 — 내부 테이블 선언과 행 추가.

한 건으로는 부족하다

CH04의 구조체는 사람 한 명을 담는다. 그런데 명단은 보통 여러 명이다. 같은 모양(구조)의 행을 여러 개 쌓아 두고 싶다 — 그게 (내부 테이블)다.

선언 — TYPE TABLE OF

행의 모양(구조 타입) 위에 TABLE OF를 붙인다.

TYPES: BEGIN OF ty_person,
         name TYPE string,
         age  TYPE i,
       END OF ty_person.

DATA: ls_person TYPE ty_person,          " 한 건(Work Area)
      lt_person TYPE TABLE OF ty_person. " 여러 건(Internal Table)
  • lt_ 접두어가 관례(local table). 은 메모리에만 존재한다.
  • 한 건을 담는 (ls_)와 짝지어 쓴다.

행 추가 — APPEND

에 값을 채우고 APPEND로 테이블에 쌓는다.

ls_person-name = 'Kim'. ls_person-age = 30.
APPEND ls_person TO lt_person.

CLEAR ls_person.
ls_person-name = 'Lee'. ls_person-age = 25.
APPEND ls_person TO lt_person.

이제 lt_person에는 행이 둘이다. lines( lt_person )으로 개수를 센다.

비우기 — CLEAR / REFRESH

CLEAR   lt_person.   " 모든 행 제거
REFRESH lt_person.   " 동일하게 행 제거

구조체(한 건)는 CLEAR로 값만 비웠다. 내부 테이블은 CLEAR/REFRESH로 행 전체를 비운다.

흔한 실수와 점검

  • APPEND 전에 CLEAR 안 함 → 이전 값이 남아 섞일 수 있다. 새 행 전에 CLEAR ls_person 습관.
  • 헤더라인 옛 방식 → 오래된 코드의 "header line" 대신, 지금은 INTO ls_ 작업영역 방식을 쓴다(명확·안전).

행을 쌓았다. 그런데 이 테이블 타입도 로컬·전역으로 나뉜다 → 다음 레슨(Table Type).
