잠금은 언제 풀리나

  • DEQUEUE 명시 호출 시.
  • COMMIT WORK / ROLLBACK WORK 시 — 보통 종료와 함께 자동 해제된다.
  • 종료 시.

DEQUEUE_ALL로 현재 프로그램이 건 모든 잠금을 한 번에 풀 수도 있다.

foreign_lock 예외 — 정중히 거절

이미 남이 잠갔으면 foreign_lock이 난다. 무시하지 말고 사용자에게 안내한다.

CALL FUNCTION 'ENQUEUE_EZ_BOOKING'
  EXPORTING  booking_id = lv_id
  EXCEPTIONS foreign_lock = 1 OTHERS = 2.

IF sy-subrc = 1.
  MESSAGE '다른 사용자가 편집 중입니다' TYPE 'E'.
ENDIF.

⚠️ 누가 잠갔는지(잠금 소유자)는 sy-msgv1 같은 시스템 변수에 항상 채워진다고 가정하지 말 것. 필요하면 SM12(잠금 목록)나 잠금 조회 함수로 별도로 얻는다.

흔한 실수와 점검

  • 장시간 잠금 → 사용자가 화면을 켜 둔 채 점심. 가능하면 짧게 잠그고 푼다.
  • 안 잡음 → foreign_lock을 무시하면 사용자 경험이 나쁘다. 안내한다.
