SALV/Grid ALV 표시 제어 심화

SALV Layout / Variant 심화

표시 설정·컬럼 텍스트·레이아웃 저장 — 개발자 기본값 + 사용자 보기.

Track 1Chapter 22Lesson 2고급

왜 — 정렬·필터만으론 부족 🎨

정렬·필터를 줘도 긴 컬럼이 잘리고, 제목이 DDIC 기술명 그대로 나오고, 매번 같은 컬럼 순서를 다시 맞춰야 하면 보고서가 불편하다. SALV layout·variant는 이걸 줄인다 — **개발자는 "처음 보기 좋은 기본값"**을 주고, **사용자는 "내 업무 방식 보기"**를 저장한다.

세 층의 표시 설정 🧩

층 역할
표 전체 get_display_settings( ) 제목(list header)·줄무늬
컬럼 묶음 get_columns( ) 폭 최적화·특정 컬럼 접근
variant 저장 get_layout( ) report별 layout key·저장 허용

DATA(lo_disp) = lo_salv->get_display_settings( ).
lo_disp->set_striped_pattern( abap_true ).
lo_disp->set_list_header( '회차별 예매 현황' ).

DATA(lo_cols) = lo_salv->get_columns( ).
lo_cols->set_optimize( abap_true ).               " 컬럼 폭 최적화

TRY.
    DATA(lo_cap) = lo_cols->get_column( 'CAPACITY' ).
    lo_cap->set_short_text(  '정원' ).            " ★ 셋 다 맞춘다
    lo_cap->set_medium_text( '정원' ).
    lo_cap->set_long_text(   '정원' ).
  CATCH cx_salv_not_found.
    MESSAGE 'CAPACITY 컬럼을 찾을 수 없습니다.' TYPE 'E'.
ENDTRY.

⚠️ set_long_text만 바꾸면 헤더가 그대로일 수 있다. SALV는 컬럼 폭에 따라 short/medium/long 중 하나를 골라 쓰므로, 셋을 함께 맞춰야 초보자 혼란이 없다.

Variant 저장 💾

ABAP
1
2
3
DATA(lo_layout) = lo_salv->get_layout( ).
lo_layout->set_key( VALUE salv_s_layout_key( report = sy-repid ) ).   " 어느 리포트의 layout인지
lo_layout->set_save_restriction( if_salv_c_layout=>restrict_none ).   " 저장 허용

set_key로 "이 variant가 어느 리포트의 layout인지" 식별한다(sy-repid = 현재 프로그램). 그래야 사용자가 컬럼 순서·숨김·폭을 자기 보기로 저장할 수 있다.

흔한 실수와 점검 🚧

  • long_text만 바꿈 → 헤더가 안 바뀔 수 있다. short/medium/long을 함께.
  • 컬럼명 오타 → get_column( )은 화면 제목이 아니라 내부 필드명(대문자)으로 접근. 틀리면 cx_salv_not_found.
  • 레이아웃 키 누락 → variant를 저장하려면 set_key로 식별부터.
  • variant ≠ 데이터 변경 → 컬럼 순서·숨김·폭 같은 표시 설정만 저장된다. 또 ALV 숨김은 권한 제어가 아니다 — 보안상 숨길 값은 애초에 조회하지 않는다.

SALV를 다듬었다. 이제 Grid ALV의 컬럼을 field catalog로 더 세밀히 → Grid ALV Column 제어 심화.
