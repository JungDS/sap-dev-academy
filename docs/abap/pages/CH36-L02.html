기반 모델 — ZI_

CH22처럼 기반(ZI_) 뷰를 짠다. 예매(ZI_Booking)를 root로, 공연·회차를 association으로 잇는다.

define root view entity ZI_Booking
  as select from zbooking
  association [1..1] to ZI_Concert as _Concert
    on $projection.concert_id = _Concert.concert_id
  association [1..1] to ZI_Perf as _Perf
    on  $projection.concert_id = _Perf.concert_id
    and $projection.perf_no    = _Perf.perf_no
{
  key booking_id,
      concert_id,
      perf_no,
      customer,
      seats,
      status,
      @Semantics.user.createdBy: true
      created_by,
      _Concert,                 // 노출
      _Perf
}
  • 예매를 root로( 대상).
  • 공연·회차 association을 노출해 화면에서 따라가게.
  • 감사필드(created_by)에 시맨틱 주석.

흔한 실수와 점검

  • 카디널리티 → 예매:공연 = [1..1]처럼 관계를 정확히.
  • 노출 누락 → association을 { }에 넣어야 소비 가능.

기반을 놓았다. 화면용 Projection → 다음 레슨.
