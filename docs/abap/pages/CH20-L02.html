Advanced ABAP SQL

WITH와 CTE — 중간 결과에 이름 붙이기

복잡한 SELECT를 "중간 표를 만든다 → 다시 읽는다"로 나누기.

Track 1Chapter 20Lesson 2고급

왜 — 두 단계를 한 문장에 넣기 벅차다 🪜

공연별 잔여석을 구하려면 보통 두 단계가 필요하다.

  1. 예약 테이블에서 공연별 예약 좌석을 합산한다.
  2. 공연 마스터의 정원에서 그 합계를 뺀다.

이걸 한 SELECT에 다 밀어 넣을 수도 있지만, 입문자에겐 읽기 어렵다. 는 이 문제를 푼다. 먼저 +booked라는 중간 결과를 만들고, 그다음 main query에서 +booked를 일반 테이블처럼 읽는다.

무엇 — WITH · + 이름 📛

(Common Table Expression, 중간 결과에 이름을 붙인 임시 표)는 ABAP SQL에서 WITH 문으로 정의하고, 이름은 +로 시작한다.

WITH
  +booked AS (
    SELECT concert_id,
           SUM( seats ) AS booked_seats
      FROM zbooking
      WHERE status <> 'C'
      GROUP BY concert_id )
  SELECT c~concert_id,
         c~title,
         c~capacity,
         COALESCE( b~booked_seats, 0 )              AS booked,
         c~capacity - COALESCE( b~booked_seats, 0 ) AS remaining
    FROM zconcert AS c
    LEFT OUTER JOIN +booked AS b
      ON b~concert_id = c~concert_id
    INTO TABLE @DATA(lt_capacity).

읽는 순서는 이렇다.

부분 의미
WITH +booked AS ( … ) +booked라는 임시 결과를 만든다
SUM( seats ) 예약 좌석을 합산
WHERE status <> 'C' 취소 예약은 합산에서 제외
GROUP BY concert_id 공연별로 합계를 만든다
LEFT OUTER JOIN +booked 공연 마스터는 유지하고 예약 합계를 붙인다
COALESCE( b~booked_seats, 0 ) 예약 없는 공연의 null 합계를 0으로(지난 챕터)

+booked는 실제 DB 테이블을 새로 만드는 게 아니다. 지금 이 SELECT 문장 안에서만 살아 있는 이름 붙은 결과다.

어떻게 확인 — +booked를 먼저 그려라 🖊️

를 처음 배울 땐 main query 결과만 보면 안 된다. +booked가 어떤 표라고 가정되는지 먼저 손으로 그린다.

입력 예약(zbooking):

concert_id seats status
C001 2 N
C001 3 N
C001 1 C
C002 4 N

+booked 예상 결과:

concert_id booked_seats
C001 5
C002 4

취소(C) 1석이 빠졌는지 확인한다. 이후 공연 마스터와 붙였을 때 예약 없는 공연이 사라지지 않는지(LEFT OUTER JOIN 덕분)도 확인한다.

흔한 실수와 점검 🚧

  • 이름을 실제 테이블로 착각 → +booked는 이 문장 안에서만 사는 임시 이름이다. 다른 프로그램에서 못 읽는다.
  • 계산 컬럼에 별칭 누락 → SUM( seats ) AS booked_seats처럼 AS로 읽기 쉬운 이름을 준다.
  • CTE를 너무 많이 → 1~2개까지가 읽기 쉽다. 그 이상이면 오히려 숨겨진 절차가 된다.
  • COALESCE 빠뜨림 → 예약 없는 공연의 합계가 null이 되어 잔여석 계산이 어긋난다.

중간 결과에 이름을 붙였다. 이제 조건 안에서 존재 여부를 묻는 subquery를 보자 → Subquery와 EXISTS.
