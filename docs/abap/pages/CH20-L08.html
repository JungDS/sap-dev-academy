부모 타입 변수에 자식 객체

· 덕에 부모(또는 인터페이스) 타입 변수에 여러 자식 를 담아 똑같이 다룰 수 있다(다형성). 그런데 가끔 실제 타입으로 되돌려 다뤄야 한다.

내려보내기 — CAST ( ?= )

ABAP
1
2
3

5
6
DATA lo_base TYPE REF TO zcl_booking_manager.
lo_base = NEW zcl_vip_booking( ... ).        " 부모 타입에 자식 담기

" 실제 타입(VIP)으로 변환해 VIP 전용 메서드 호출
DATA(lo_vip) = CAST zcl_vip_booking( lo_base ).
lo_vip->vip_only( ).
  • CAST 타입( ref )(모던) = ref ?= ...(classic) — down-cast(부모→자식).
  • 실제 타입이 안 맞으면 CX_SY_MOVE_CAST_ERROR → TRY/CATCH로 대비.

타입 분기 — CASE TYPE OF

의 실제 타입에 따라 분기한다.

ABAP
1
2
3
4
5
6
CASE TYPE OF lo_base.
  WHEN TYPE zcl_vip_booking.
    " VIP 처리
  WHEN OTHERS.
    " 일반 처리
ENDCASE.

흔한 실수와 점검

  • 무분별한 CAST → 타입 불일치 . 가능하면 다형성(공통 )으로 풀고, CAST는 최소화.
  • CASE TYPE OF 남용 → 타입별 분기가 많으면 설계(다형성)를 재고.

타입을 자유롭게 다룬다. 마지막으로 가 사건을 알리는 → 다음 레슨.
