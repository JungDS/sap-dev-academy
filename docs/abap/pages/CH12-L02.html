SELECT-OPTIONS와 Range Table

SELECT-OPTIONS 기본 문법

화면 입력칸과 Range Table을 한 번에 만드는 SELECT-OPTIONS.

Track 1Chapter 12Lesson 2중급

매번 손으로 적을 순 없다

Range Table 구조에서 의 구조(SIGN·OPTION·LOW·HIGH)를 배웠다. 하지만 사용자에게 "SIGN은 I, OPTION은 BT로 적으세요"라고 시킬 순 없다. 사용자는 그저 SAP 화면에서 From / To 칸에 값을 넣고, 필요하면 다중 선택 버튼을 누르는 데 익숙하다.

는 바로 이 연결을 만들어 준다. 화면에는 범위 입력칸을 만들고, 프로그램 안에는 같은 이름의 Range Table을 만든다. 즉 SELECT-OPTIONS 하나가 "화면 요소"이면서 동시에 "조건을 담는 "이다.

TABLES zbooking.

SELECT-OPTIONS: so_conc FOR zbooking-concert_id,
                so_stat FOR zbooking-status.

DATA ls_conc LIKE LINE OF so_conc.

START-OF-SELECTION.
  LOOP AT so_conc INTO ls_conc.
    WRITE: / ls_conc-sign, ls_conc-option,
             ls_conc-low,  ls_conc-high.
  ENDLOOP.

줄별로 읽으면 이렇다.

코드 의미
TABLES zbooking. zbooking-concert_id처럼 Dictionary 필드를 참조할 수 있게 한다.
SELECT-OPTIONS so_conc FOR zbooking-concert_id. 화면에 공연ID 범위 입력을 만들고, 프로그램 안에 so_conc 을 만든다.
so_stat FOR zbooking-status. 상태값용 Range Table을 만든다.
DATA ls_conc LIKE LINE OF so_conc. so_conc의 한 행을 담을 work area를 만든다.
LOOP AT so_conc INTO ls_conc. 사용자가 입력한 조건 행을 한 줄씩 꺼내 본다.

🔎 [선행 사용] TABLES zbooking은 zbooking과 같은 모양의 작업 영역을 만들어 zbooking-concert_id 같은 필드 참조를 가능하게 한다(DB 조회가 아니다). 은 선택화면 입력이 끝나고 실제 처리가 시작되는 지점이다. 둘 다 지금은 "이 자리에 필요해서 미리 쓴다" 정도로만 — 정식 설명은 Chapter 15 · 이벤트와 선택화면에서 한다.

이름 관례도 알아 두자. so_는 select-options를 뜻하는 표준 관례로, so_conc, so_stat처럼 쓰면 코드만 봐도 "사용자 선택 조건"임이 보인다. 실무에선 짧게 s_로 줄여 쓰기도 한다(선택화면 pa_를 p_로 줄이는 습관과 같다). 단, 이름은 최대 8자까지만 가능하다(so_+최대 5자).

화면에서는 어떻게 보이나

실행하면 선택화면에 so_conc·so_stat이 나타난다. 각 항목은 단일 입력칸 하나가 아니라 From / To 두 칸 + 우측 다중 선택(화살표) 버튼을 가진다. 사용자가 단일 값, 범위, 목록, 제외를 넣으면 그대로 행이 된다.

아래에서 직접 From/To를 입력하고 다중 선택을 더해 so_conc가 어떻게 채워지는지, LOOP AT so_conc가 무엇을 출력하는지 확인해 보자.

흔한 실수와 점검

  • 를 단일 값으로 본다 → so_conc-low 첫 행만 보는 습관이 들면 다중 선택 행을 놓친다. 조건은 table 전체로 다룬다.
  • TABLES를 조회 문장으로 오해한다 → TABLES zbooking.은 DB를 읽지 않는다. 필드 참조용 work area 선언일 뿐이다.
  • 심화 옵션을 지금 다 배우려 한다 → OBLIGATORY·NO-EXTENSION·NO INTERVALS 같은 선택화면 옵션은 Chapter 15 · 이벤트와 선택화면에서 정리한다.

정리

  • 는 선택화면 입력 영역과 프로그램 내부 을 동시에 만든다.
  • 그 Range Table은 L01의 SIGN/OPTION/LOW/HIGH 구조 그대로다.
  • FOR <필드>가 입력칸의 타입과 도움 속성 기준을 정한다.
  • 이름은 s_ 관례 + 8자 이내.

화면 입력이 Range Table에 담겼다. 이제 이 조건을 실제 조회에 꽂아 예매 목록을 줄여 보자 → 다음 레슨.
