변수와 표준 타입 (+ Local Type)

변수 선언(DATA)

값을 담을 그릇 — DATA로 변수를 선언한다.

Track 1Chapter 2Lesson 1입문

리터럴의 한계

CH01에서 WRITE 'Hello'.처럼 값을 따옴표에 직접 적었다. 그런데 같은 값을 여러 번 쓰거나, 계산 결과를 보관하려면? 따옴표 리터럴로는 안 된다 — 값을 담아 둘 그릇, 곧 가 필요하다.

DATA로 변수 선언

문으로 변수를 선언한다. 이름 TYPE 타입 형태다.

DATA lv_name TYPE string.
DATA lv_count TYPE i.
  • lv_ 접두어는 관례(local variable). 강제는 아니지만 팀 표준을 따른다.
  • TYPE 뒤에 어떤 타입인지 적는다(다음 레슨들의 주제).

값 넣고 출력하기

선언한 변수에 값을 대입(=)하고 출력해 본다.

DATA lv_msg TYPE string.
lv_msg = 'Hello, ABAP!'.
WRITE lv_msg.

이제 메시지를 한 곳(lv_msg)에서 관리한다 — 바꿀 때 한 줄만 고치면 된다.

여러 변수 한 번에 (체인)

콜론(:)으로 를 체인해 여러 변수를 한 번에 선언한다.

DATA: lv_first TYPE string,
      lv_last  TYPE string,
      lv_age   TYPE i.

흔한 실수와 점검

  • 선언 안 한 변수 사용 → lv_xxx is unknown 오류. 먼저 로 선언.
  • 타입 누락 → DATA lv_x. 만으론 부족. TYPE을 붙인다.
  • 대입과 비교 → 둘 다 =를 쓴다(ABAP). 문맥으로 구분된다.

값은 담았다. 그런데 TYPE 뒤에 뭘 적나? 타입에는 길이를 안 적어도 되는 것과 적어야 하는 것이 있다. 먼저 전자(Complete) → 다음 레슨.
