변수·표준 타입·상수·Text Symbol

변수 선언(DATA)

값을 담을 그릇 — DATA로 변수를 선언한다.

Track 1Chapter 2Lesson 1입문

리터럴의 한계

CH01에서 WRITE 'Hello'.처럼 값을 따옴표에 직접 적었다. 그런데 같은 값을 여러 번 쓰거나, 계산 결과를 보관하려면? 따옴표 로는 안 된다 — 값을 담아 둘 그릇, 곧 가 필요하다.

DATA로 변수 선언

문으로 변수를 선언한다. 이름 TYPE 타입 형태다.

ABAP
1
2
DATA gv_name TYPE string.
DATA gv_count TYPE i.
  • gv_ 접두어는 관례(global variable). 아직 Subroutine(프로그램을 잘게 나누는 단위)을 안 배웠으니 선언하는 변수는 모두 전역 — 그래서 g로 시작한다(로컬 변수 lv_는 뒤에서). 강제는 아니지만 팀 표준을 따른다.
  • TYPE 뒤에 어떤 타입인지 적는다(다음 레슨들의 주제).

값 넣고 출력하기

선언한 변수에 값을 대입(=)하고 출력해 본다.

ABAP
1
2
3
DATA gv_msg TYPE string.
gv_msg = 'Hello, ABAP!'.
WRITE gv_msg.

이제 메시지를 한 곳(gv_msg)에서 관리한다 — 바꿀 때 한 줄만 고치면 된다.

선언과 동시에 초기값 — VALUE

=로 나중에 넣는 대신, 선언하면서 바로 초기값을 주려면 VALUE를 붙인다.

ABAP
1
2
DATA gv_qty TYPE i VALUE 10.            " 선언과 동시에 10으로 시작
DATA gv_tag TYPE c LENGTH 1 VALUE 'X'.

VALUE를 안 적으면 변수는 타입의 기본값으로 시작한다 — 숫자는 0, 문자는 공백.

여러 변수 한 번에 (체인)

콜론(:)으로 를 체인해 여러 변수를 한 번에 선언한다.

ABAP
1
2
3
DATA: gv_first TYPE string,
      gv_last  TYPE string,
      gv_age   TYPE i.

TYPE 말고 LIKE — 기존 변수를 본떠

TYPE 뒤에는 타입 이름을 적지만, LIKE 뒤에는 이미 선언한 변수를 적는다. "저 변수와 똑같은 타입으로 하나 더"라는 뜻이다.

ABAP
1
2
DATA gv_price TYPE p LENGTH 8 DECIMALS 2.
DATA gv_total LIKE gv_price.     " gv_price와 같은 타입

원본(gv_price)의 타입이 바뀌면 gv_total도 자동으로 따라 바뀐다. "이 변수와 짝이 되는 변수"를 만들 때 편하다.

흔한 실수와 점검

  • 선언 안 한 변수 사용 → gv_xxx is unknown 오류. 먼저 로 선언.
  • 타입 누락 → DATA gv_x. 만으론 부족. TYPE을 붙인다.
  • 대입과 비교 → 둘 다 =를 쓴다(ABAP). 문맥으로 구분된다.

값은 담았다. 그런데 TYPE 뒤에 뭘 적나? 타입에는 길이를 안 적어도 되는 것과 적어야 하는 것이 있다. 먼저 전자(Complete) → 다음 레슨.
