종합 실습: 구구단

배운 연산·분기·반복을 모아 구구단을 만든다.

Track 1Chapter 4Lesson 7입문

다 모아서, 구구단

산술·판단·반복·디버깅을 한 번에 쓸 차례다. 목표는 구구단. 이 구구단은 앞으로 여러 챕터를 따라다니며 "같은 결과물을 한 단계씩 키우는" 의 출발점이 된다.

1단계 — 한 단 출력

곱하는 수(1~9)만큼 반복하면 한 단이 나온다. 로 몇 단인지 입력받는다. 정답을 보기 전에, 반복(DO)·회차(sy-index)·곱셈(*)을 떠올리며 빈칸을 직접 채워 보자.

2단계 — 전체(2~9단) 출력: 반복 속의 반복

단도 1~9로 바꾸면 반복 안에 반복(중첩 )이 된다. 바깥은 단, 안쪽은 곱하는 수다. 바깥의 회차는 안쪽 루프가 를 덮어쓰기 전에 변수에 담아 둔다 — 이 점을 직접 채우며 확인해 보자.

3단계 — 범위만 (예: 3~5단)

시작·끝 단을 입력받아 그 범위만 출력한다. 로 "현재 단이 범위 안인가"를 가린다 — 범위 밖을 어떻게 건너뛸지 직접 채우며 떠올려 보자.

디버거로 들여다보기

2단계 코드에 BREAK-POINT를 걸고(CH04-L06) F5로 진행하며 gv_dan·gv_mul· gv_res가 어떻게 변하는지 본다. 바깥 루프가 한 번 돌 때 안쪽 루프가 9번 도는 구조가 눈에 들어온다.

흔한 실수와 점검

  • 중첩 루프에서 단을 잃어버림 → 바깥 sy-index를 변수(gv_dan)에 담은 뒤 안쪽 루프로 들어간다. 안 그러면 안쪽 sy-index에 덮어써진다.
  • 줄바꿈 → 각 줄은 WRITE: /로 시작(/가 새 줄).
  • 범위 입력 역전 → p_from이 p_to보다 크면 아무것도 안 나온다. 입력 점검.

정리 — 그리고 다음 불편

연산·분기·반복으로 구구단을 완성했다. 그런데 한 줄을 표현하려고 gv_dan· gv_mul·gv_res 변수 세 개를 따로 들고 다녔다. 이 셋은 사실 "구구단 한 줄"이라는 한 묶음인데 코드에는 그렇게 안 보인다. 관련된 값을 하나로 묶고 싶다 → CH05(Structure).
