Open SQL 2차: JOIN·집계

JOIN / FAE / ABAP 처리 선택 기준

합치기 방법 셋 — JOIN·FAE·ABAP 루프 중 무엇을.

Track 1Chapter 13Lesson 7중급

셋 중 무엇을

여러 데이터를 합치는 길은 셋이다 — DB의 , (FAE), 그리고 ABAP에서 읽어와 직접 맞추기. 상황별로 정리한다.

한눈에

방법 일을 누가 적합
DB 두 테이블을 키로 합쳐 한 번에 — 보통 1순위
FAE DB(여러 번) 이미 에 있는 목록 기준으로 조회
ABAP 루프 앱 서버 단순·소량, 또는 JOIN/FAE가 부적절할 때

원칙(직관)

  • 가능하면 — DB가 합치게 둔다(데이터 이동 최소).
  • 합칠 대상이 이미 내부 테이블로 있으면 FAE(빈 테이블·중복 주의).
  • LOOP 안에서 SELECT(중첩 조회) 는 피한다 — 성능의 대표 안티패턴.

직접 골라 보기

상황마다 데이터가 어디 있고 질문이 무엇인지 먼저 묻고, 알맞은 도구로 분류해 보자.

흔한 실수와 점검

  • LOOP + SELECT → 행마다 DB 왕복. /FAE로 바꾼다.
  • 무조건 JOIN → 너무 많은 테이블 JOIN도 무겁다. 균형.

이제 classic Open SQL로 합치고·하고·정렬·선택하는 큰 줄기를 다 봤다. 마지막으로 이 판단을 공연별 예매현황 리포트(LEFT OUTER + GROUP BY + SUM)로 묶어 실습하자 → 다음 레슨. (그리고 Chapter 19 · Modern Open SQL에서 이 모든 SQL을 modern @·인라인으로 다시 만난다.)
