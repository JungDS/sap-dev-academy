물려받아 확장

비슷한 를 처음부터 다시 만들 필요 없이, 기존 클래스를 (INHERITING)해 차이만 더한다.

CLASS zcl_vip_booking DEFINITION PUBLIC
  INHERITING FROM zcl_booking_manager.
  PUBLIC SECTION.
    METHODS book REDEFINITION.      " 부모 메서드를 다시 정의
ENDCLASS.

CLASS zcl_vip_booking IMPLEMENTATION.
  METHOD book.
    " VIP 추가 처리 후 부모 동작 호출
    super->book( iv_seats = iv_seats ).
  ENDMETHOD.
ENDCLASS.
  • INHERITING FROM — 부모를 물려받는다.
  • REDEFINITION — 부모 를 새로 구현(오버라이드).
  • super-> — 부모의 원래 구현을 부른다.

ABSTRACT · FINAL

  • ABSTRACT — 직접 못 만듦, 상속 전용(틀만 제공).
  • FINAL — 더 이상 상속/재정의 금지.

흔한 실수와 점검

  • REDEFINITION 없이 오버라이드 → 부모 를 바꾸려면 REDEFINITION 선언.
  • super-> 누락 → 부모 동작이 필요하면 명시적으로 호출.

상속으로 확장했다. 그런데 부모 타입 변수에 담긴 자식을 원래 타입으로 다루려면? → 다음 레슨.
