사람 눈만으론 부족

운영에 올리기 전, 성능·보안·문법·표준 위반을 자동으로 점검한다. (SCI)와 그 상위 (ABAP Test Cockpit)가 그 도구다.

무엇을 잡나

  • 성능 — SELECT in LOOP·인덱스 누락(CH32).
  • 보안 — SQL injection·인증 누락.
  • 표준 준수 — 명명·금지 구문·ABAP Cloud(Released API) 위반(CH23-L08).
  • 문법·미사용 변수 등.

흐름: Variant(검사 항목) 지정 → 실행 → Finding 목록을 우선순위(에러/경고)로 수정.

흔한 실수와 점검

  • 무시하고 이송 → 운영에서 사고. 이송 전 통과를 게이트로.
  • 모든 경고 끄기 → 진짜 문제를 놓친다. 예외는 근거와 함께.

품질을 정적으로 점검했다. 동작 회귀를 막는 ABAP Unit → 다음 레슨.
