Text Symbol — 화면 글자의 다국어

코드에 박은 텍스트를 번호로 빼서, 언어마다 다르게 보이게 한다.

Track 1Chapter 2Lesson 6입문

화면 글자를 코드에 박으면 생기는 문제

WRITE '금액'.처럼 화면에 보일 텍스트를 코드에 직접 적으면, 영어 사용자가 로그인해도 여전히 '금액'이라고 나온다. SAP는 전 세계가 같은 프로그램을 각자 언어로 쓰는 시스템이라, 화면 텍스트는 언어마다 바뀔 수 있어야 한다. 그 장치가 (텍스트 심볼)이다.

텍스트를 번호로 빼기 — TEXT-nnn

화면 텍스트를 코드에 직접 두지 않고, 세 자리 번호로 관리한다. 코드에서는 TEXT-001처럼 부르고, 실제 글자("금액")는 별도의 텍스트 목록(SE38의 Text Elements)에 언어별로 등록한다.

ABAP
1
WRITE: / TEXT-001, gv_amt.     " TEXT-001 = '금액'(한국어) / 'Amount'(영어)

로그인 언어가 한국어면 '금액', 영어면 'Amount'가 자동으로 나온다. 코드는 그대로다.

TEXT-001과 '금액'(001)의 차이 (중요)

두 가지로 적을 수 있는데 동작이 다르다.

적는 법 번역이 없을 때 효과
TEXT-001 공백이 나온다 번역 누락이 화면에서 바로 들통남
'금액'(001) '금액'(괄호 앞 기본값)이 나온다 번역이 없어도 일단 보이지만, 누락이 숨겨짐

'금액'(001)은 기본값('금액')과 번호(001)를 함께 적는 방식이다. 번역을 빠뜨려도 한국어가 보여 편하지만, 누락을 놓치기 쉽다. 번역 관리를 엄격히 하려면 TEXT-001 쪽이 안전하다.

표로만 보면 아리송하다. 직접 눌러 보자. 등록/미등록과 로그인 언어를 바꿔 가며 두 문법의 출력이 어떻게 갈리는지 확인하면 한 번에 잡힌다.

흔한 실수와 점검

  • 번호만 적고 등록 안 함 → TEXT-001만 쓰고 Text Elements에 안 넣으면 공백.
  • 메시지와 혼동 → 사용자에게 띄우는 알림(MESSAGE)은 또 다른 장치다(뒤 챕터). 은 화면에 박히는 라벨·제목 용도다.
  • 세 자리 번호 → TEXT-1이 아니라 TEXT-001.
