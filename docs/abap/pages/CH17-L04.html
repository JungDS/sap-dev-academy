Field Catalog 기초

데이터 필드를 사용자가 보는 컬럼으로 — 제목·너비·순서 제어.

Track 1Chapter 17Lesson 4중급

컬럼을 내 맘대로 — 표의 "컬럼 설명서"

데이터만 있으면 ALV가 표를 그릴 수는 있다. 하지만 실무 화면은 기술 필드명 SEATS를 그대로 두기보다 "좌석수" 라는 제목이 좋고, 어떤 컬럼은 좁게·어떤 컬럼은 넓게·어떤 컬럼은 숨기거나 강조해야 한다. 이 역할이 — 표의 컬럼 설명서다.

  • 타입 LVC_T_FCAT은 컬럼 설정들의 내부 테이블, 각 행 LVC_S_FCAT은 한 컬럼의 설정이다.
  • 주요 칸: fieldname(어떤 데이터 필드와 연결), coltext(컬럼 제목), outputlen(출력 너비).

자주 만지는 속성만 추려 두자. 세부는 필요할 때 찾아 쓴다.

속성 뜻
fieldname 어떤 데이터 필드의 컬럼인지
coltext 컬럼 제목(헤더)
outputlen 출력 너비
no_out 처음엔 숨김(사용자가 펼치면 보임)
key 키 컬럼으로 고정 표시
do_sum 숫자 컬럼 합계(맛보기 — Grid ALV Column 제어 심화)

자동으로 만들기 — LVC_FIELDCATALOG_MERGE

행 Structure가 DDIC에 있으면 함수로 기본 를 자동 생성한다.

DATA gt_fcat TYPE lvc_t_fcat.

CALL FUNCTION 'LVC_FIELDCATALOG_MERGE'
  EXPORTING
    i_structure_name = 'ZBOOKING'      " ← DDIC Structure 이름(변수명 아님)
  CHANGING
    ct_fieldcat      = gt_fcat
  EXCEPTIONS
    inconsistent_interface = 1
    program_error          = 2
    OTHERS                 = 3.

IF sy-subrc <> 0.
  MESSAGE 'Field Catalog를 생성하지 못했습니다.' TYPE 'E'.
ENDIF.

손으로 다듬기

자동 생성 후, 필요한 컬럼만 코드로 다듬는다.

DATA ls_fcat TYPE lvc_s_fcat.

LOOP AT gt_fcat INTO ls_fcat.
  IF ls_fcat-fieldname = 'SEATS'.
    ls_fcat-coltext   = '좌석수'.            " 컬럼 제목
    ls_fcat-outputlen = 8.                  " 출력 너비
    MODIFY gt_fcat FROM ls_fcat INDEX sy-tabix.   " 현재 행에 반영
  ENDIF.
ENDLOOP.

직접 편집해 보기 — 컬럼 설명서 편집기

왼쪽 gt_booking 미리보기를 보면서, 오른쪽 gt_fcat에서 컬럼 제목(coltext)·너비를 바꿔 보자. 헤더가 즉시 바뀐다. 단, fieldname을 소문자로 잡으면 DDIC 필드명과 맞지 않아 수정이 적용되지 않는 것도 확인하자.

흔한 실수와 점검

  • fieldname 오타·소문자 → 출력 데이터 필드명('SEATS')과 정확히 같아야 적용된다. 대소문자도 맞춰야 한다.
  • i_structure_name에 변수명 → DDIC Structure 이름('ZBOOKING')을 넣어야 한다. 내부 테이블 변수명을 넣으면 의도대로 안 된다.
  • fcat 비움 → "데이터가 있는데 표가 이상하다"면, 먼저 gt_fcat이 채워졌는지 확인한다.

정리

  • 이 레슨은 "데이터 필드"를 "사용자가 보는 컬럼"으로 번역하는 단계다.
  • LVC_FIELDCATALOG_MERGE로 기본 카탈로그를 만들고, coltext·outputlen 등을 손으로 다듬는다.

컬럼을 정의했다. 이제 컬럼 하나하나가 아니라 표 전체의 보기 방식(줄무늬·선택 모드 등)을 정하는 Layout으로 넘어가자 → Layout 기본 설정.
