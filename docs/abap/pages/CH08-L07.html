조회 실패와 MESSAGE (기초)

결과가 없을 때 — sy-subrc 분기와 MESSAGE 맛보기.

결과가 없을 수도 있다

조회는 늘 성공하지 않는다. 우리 ZGUGUDAN에는 2·3단만 넣었으니, 5단을 조회하면 한 건도 안 나온다. 이때 빈 화면만 두면 사용자는 영문을 모른다 — 알려 줘야 한다.

SELECT * FROM zgugudan INTO TABLE gt_gugu WHERE dan = 5.

IF sy-subrc <> 0.
  WRITE: / '5단 데이터가 없습니다.'.
ELSE.
  LOOP AT gt_gugu INTO gs_line.
    WRITE: / gs_line-dan, 'x', gs_line-mul, '=', gs_line-result.
  ENDLOOP.
ENDIF.

MESSAGE — 사용자에게 알리기 (맛보기)

WRITE로 화면에 찍는 것보다, MESSAGE로 알림을 띄우는 게 더 적절할 때가 많다.

ABAP
1
2
3
IF sy-subrc <> 0.
  MESSAGE '해당 단의 데이터가 없습니다.' TYPE 'S'.   " S = 상태바 알림
ENDIF.
  • TYPE 'S'(Status, 상태바)·TYPE 'I'(Information, 팝업)가 가벼운 알림이다.
  • 여기선 맛보기만 — 메시지 타입 전체(I/S/W/E/A/X)와 메시지 클래스는 CH15에서 정식으로 배운다.

dan을 바꿔 조회하고(2·3=있음, 5=없음), 실패했을 때 세 가지 피드백이 화면에서 어떻게 다른지 직접 확인해 보자.

흔한 실수와 점검

  • 결과 미확인 진행 → 빈 테이블로 다음 로직을 돌리면 사고. 항상 sy-subrc 분기.
  • 모든 걸 E로 → 단순 "없음"은 S/I가 적절. E(오류)는 흐름을 막는다(CH15).

여기까지가 읽기 기초의 완성이다 — 값(CH02)→전역(CH03)→묶음(CH05)→여러 건(CH06) →영속(CH07)→다시 읽기(CH08). 그런데 carrid만 입력해도 항공사명이 F4로 뜨면 좋겠다. 입력을 똑똑하게 만드는 DDIC 관계와 F4 → CH09.
