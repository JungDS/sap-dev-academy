콘서트앱 6단계 — 공연 마스터를 표준 화면으로 등록·조회.

Track 1Chapter 14Lesson 9중급

🎫 6단계 — 공연 마스터 유지보수

지금까지 공연 데이터를 Create Entries로 한 건씩 넣었다(CH07). 이제 표준 유지보수 화면으로 다건 등록하고, 공연+회차를 한 뷰로 조회한다.

1) Database View — 공연 + 회차

SE11 → View → ZV_CONCERT → ZCONCERT와 ZPERF를 조인(CONCERT_ID). 공연명·회차·일시를 한 화면에서 본다(CH14-L01).

2) Maintenance View / SM30 — 등록 화면

ZCONCERT에 Table Maintenance Generator(CH14-L05)로 유지보수 화면을 만들고 SM30으로 연다. 입력 검증·F4(CH09)가 그대로 동작한다.

직접 해 볼 과제

  1. SM30으로 공연 2개를 더 등록한다(아티스트=이름 풀에서).
  2. ZV_CONCERT 뷰로 공연+회차가 함께 보이는지 확인.
  3. (도전) 회차(ZPERF)도 SM30 화면을 만들어 회차를 추가한다.

흔한 실수와 점검

  • TMG 미생성 → SM30이 "테이블 없음" → Table Maintenance Generator 먼저(CH14-L05).
  • 뷰 조인 조건 → CONCERT_ID로 정확히 이어야 중복/누락이 없다.

공연 마스터를 화면으로 관리한다. 이제 "공연별 예매현황"을 이벤트·검증을 갖춘 리포트로 만들자 → CH15.
