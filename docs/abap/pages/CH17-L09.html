행 색상 기초

중요한 행을 색으로 강조 — 색 코드 컬럼 + layout info_fname.

Track 1Chapter 17Lesson 9중급

매진 회차를 눈에 띄게

업무 표에서 모든 행이 같은 무게로 보이면 사용자는 중요한 상태를 놓친다. 예매 목록이라면 매진된 회차가 먼저 눈에 들어와야 한다. 색은 데이터를 바꾸는 게 아니라 주의를 올바른 곳으로 보내는 표시 기능이다. 행 전체 색은 각 행에 색 코드 컬럼을 두고, Layout에 그 컬럼 이름을 알려 주는 방식이다.

TYPES: BEGIN OF ts_perf_alv,
         concert_id TYPE zperf-concert_id,
         perf_date  TYPE zperf-perf_date,
         seatsocc   TYPE zperf-seatsocc,
         capacity   TYPE zperf-capacity,
         rowcolor   TYPE c LENGTH 4,        " ← 색 코드 컬럼
       END OF ts_perf_alv.

LOOP AT gt_perf INTO ls_perf.
  IF ls_perf-seatsocc >= ls_perf-capacity.   " 매진이면
    ls_perf-rowcolor = 'C610'.               " 빨강 계열 코드
    MODIFY gt_perf FROM ls_perf INDEX sy-tabix.
  ENDIF.
ENDLOOP.

gs_layout-info_fname = 'ROWCOLOR'.            " 색 코드가 든 '필드명'을 layout에

색이 화면에 뜨려면 네 가지가 맞아야 한다. 하나라도 빠지면 색이 보이지 않는다.

  1. 데이터 행에 색 코드 컬럼(예 rowcolor)이 있다.
  2. 각 행의 색 코드 컬럼에 코드 값(예 'C610')을 넣었다.
  3. gs_layout-info_fname에 그 컬럼 이름('ROWCOLOR')을 지정했다.
  4. set_table_for_first_display(또는 refresh_table_display)로 화면에 반영했다.

⚠️ info_fname은 "색 코드 값"이 아니라 "색 코드가 들어 있는 필드명" 이다. gs_layout-info_fname = 'C610'처럼 쓰면 안 된다. 'ROWCOLOR'(필드명)를 넣는다. 색 코드는 임의 RGB가 아니라 C + 색번호 + 강조 + 반전 형식(C610)이다.

직접 색칠해 보기 — 매진 판정과 색 연결

좌석 점유를 바꿔 회차 상태(여유/임박/매진)를 만들고, 색 코드 쓰기 → info_fname 연결 → 표시 순서로 매진 행에 색이 입혀지는지 보자. info_fname을 틀린 필드명으로 두면 왜 색이 안 보이는지도 확인한다.

🔶 셀 하나만 색·스타일을 다르게 하는 건 SALV/Grid ALV 표시 제어 심화, 더블클릭 같은 는 ALV 고급 Event 응용에서 다룬다. 여기선 행 색상까지.

흔한 실수와 점검

  • info_fname 미지정 → 색 컬럼을 둬도 layout에 안 알리면 색이 안 먹는다.
  • info_fname에 색 코드 값 → 필드명('ROWCOLOR')을 넣는다. 'C610'이 아니다.
  • 색 코드 형식 → C로 시작하는 4자리. 임의 RGB가 아니다.

정리

  • 중요한 행을 사용자가 놓치지 않게 색으로 표시한다.
  • 데이터 행의 색 코드 필드와 layout의 info_fname(필드명) 이 함께 있어야 색이 보인다.

화면에 박는 Grid ALV의 조각을 모두 배웠다(컨테이너→그리드→fieldcat→layout→display→ refresh→색). 이제 그 조각들을 하나의 실행 프로그램으로 모아 보자 → 종합 실습 — 예매 목록 Grid ALV 완성.
