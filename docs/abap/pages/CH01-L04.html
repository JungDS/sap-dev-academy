WRITE로 문자열 출력

WRITE로 화면에 첫 글자를 찍고, 줄바꿈·체인·리터럴을 직접 실행해 구분한다.

🖥️이번 Lesson에서 다루는 트랜잭션 코드

화면이 아직 비어 있다

환경도 갖췄고, 한 줄로 프로그램 뼈대도 세웠다. 그런데 실행하면 화면은 여전히 백지다. 지금 가장 하고 싶은 건 딱 하나. "뭐라도 눈으로 보고 싶다." 그 한 줄을 책임지는 게 다. 이번 레슨에서 처음으로 개발 루프(작성 → 활성화 → 실행)를 끝까지 한 바퀴 돌려 본다.

🎯 환경 세팅과 프로그램 뼈대를 지나, 드디어 결과가 눈에 보이는 첫 순간이다.

WRITE — 리스트 화면에 찍는다

는 값을 화면에 찍는 가장 기본 출력문이다. 실행하면 ABAP은 리스트(List) 화면 이라는 출력 전용 화면을 자동으로 열고, WRITE가 그 위에 한 줄씩 글자를 써 나간다.

ABAP
1
2
3
REPORT zhello.

WRITE 'Hello, ABAP!'.

작성 → 저장 → 활성화(Ctrl+F3) → 실행(F8). 그러면 리스트 화면에 Hello, ABAP!이 찍힌다.

📋 리스트 화면이란, 프로그램을 실행하면 자동으로 열리는 스크롤되는 출력 결과창이다. 우리가 다룰 첫 출력 무대다.

직접 출력해 보자

말로 듣는 것보다 직접 고쳐 실행하는 게 빠르다. 아래 에디터의 코드를 바꾸고 ▶ 실행(F8) 을 누르면 SAP 리스트 화면에 결과가 그려진다. 위쪽 예제 칩으로 네 가지 출력 패턴을 갈아 끼우며, /를 넣고 빼면 줄이 어떻게 달라지는지 눈으로 확인해 보자.

🖱️ 예제 칩 "이어 붙기 (/ 없음)" 와 "줄바꿈 (/)" 을 번갈아 눌러 보면, 같은 세 줄의 코드가 한 줄로 붙는지 / 세 줄로 나뉘는지가 한눈에 비교된다.

같은 줄? 새 줄? — /의 규칙

는 기본적으로 앞 출력에 이어 같은 줄에 찍는다. 줄을 바꾸려면 값 앞에 /를 붙인다.

코드 결과
WRITE 'A'.
WRITE 'B'.		 A B (같은 줄에 이어 붙음)
WRITE 'A'.
WRITE / 'B'.		 A
B (/에서 새 줄)
ABAP
1
2
WRITE 'first line'.
WRITE / 'second line'.

💡 /는 "여기서 줄을 바꿔라"는 신호다. 빠뜨리면 다음 출력이 옆에 붙어 버린다. 위 시뮬레이터에서 /를 지웠다 살렸다 해 보면 바로 체감된다.

여러 값 한 번에 — 콜론(:) 체인

같은 키워드를 반복하기 귀찮을 때, 콜론(:) 뒤에 콤마(,)로 나열하면 한 문장처럼 묶어 쓸 수 있다.

ABAP
1
WRITE: / 'A', 'B', 'C'.

이 한 줄은 아래 세 문장과 완전히 같다. (/는 그게 붙은 값 하나에만 적용된다.)

ABAP
1
2
3
WRITE / 'A'.
WRITE 'B'.
WRITE 'C'.

콜론은 "콜론 왼쪽 전체"를 각 항목마다 반복한다. 그래서 /의 위치가 중요하다. /를 콜론 앞에 두면(/:) 반복되는 부분에 /가 포함돼 모든 항목이 새 줄에서 시작한다.

ABAP
1
WRITE /: 'A', 'B'.    " WRITE / 'A'. 그리고 WRITE / 'B'. → A, B가 각각 새 줄

💡 줄을 바꾸는 또 다른 방법으로 NEW-LINE. 문도 있다. 단독으로 쓰면 그다음 출력이 새 줄에서 시작한다.

콜론 체인은 WRITE 전용이 아니다 (더 보기)

콜론(:) + 콤마 체인은 ABAP 공통 문법이다. 앞으로 배울 DATA:, PARAMETERS: 등에서도 똑같이 "키워드 한 번 + 콤마로 나열"이 가능하다. 지금은 WRITE로 그 감만 잡아 두면 된다.

문자열 리터럴

작은따옴표로 감싼 '...'가 문자(텍스트) 이다. 코드에 값을 직접 적어 둔 것이다.

ABAP
1
WRITE '값을 직접 적은 것이 리터럴'.

지금은 출력할 값을 코드에 직접 박아 넣고 있다. 따옴표 안 글자가 그대로 화면에 나온다. 시뮬레이터에서 따옴표 없이 gv_name 같은 걸 적으면 "변수"로 취급되는데, 그건 다음 챕터의 이야기다.

흔한 실수와 점검

세 가지 모두 위 시뮬레이터에서 직접 재현해 볼 수 있다(에러 메시지가 그대로 뜬다).

⚠️ 따옴표 짝 안 맞음 — '를 열고 안 닫으면 문자열이 끝나지 않아 오류. (WRITE 'oops. 실행해 보기)

⚠️ 줄바꿈이 안 됨 — /를 빠뜨렸다. 새 줄로 내리려면 WRITE / ....

⚠️ 마침표 누락 — 앞 레슨의 그 규칙. 여전히 1번 단골 에러다. 모든 문장은 .으로 끝난다.

정리

  • 는 ABAP의 기본 출력문이다. 값을 리스트 화면에 한 줄씩 찍는다.
  • 같은 줄에 이어 붙는 게 기본, 줄을 바꾸려면 값 앞에 /.
  • 콜론(:) + 콤마로 여러 WRITE를 한 문장처럼 체인할 수 있다.
  • 따옴표로 감싼 '...'는 코드에 직접 적은 .

첫 출력이 화면에 떴다. 그런데 곧 아쉬워진다. 숫자가 칸이 안 맞고, 글자 정렬도 투박하고, 중요한 값을 강조할 수도 없다. "출력을 보기 좋게 다듬고 싶다." 그게 다음 레슨 WRITE 심화(정렬·폭·색·구분선) 다.
