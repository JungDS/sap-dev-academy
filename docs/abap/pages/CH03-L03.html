CH03 · DDIC Domain·Data Element + PARAMETERS

PARAMETERS로 보상(F4·라벨)

DDIC의 수고가 화면에서 보상받는다 — PARAMETERS의 라벨·F4.

수고의 보상을 받을 시간

·를 만드느라 손이 갔다. 그 투자가 화면에서 한 번에 회수되는 걸 로 체감한다.

PARAMETERS — 가장 단순한 입력 화면

는 프로그램 실행 시 입력 필드 한 칸을 선택화면에 만든다.

REPORT zdemo_param.

PARAMETERS p_amt TYPE zde_amount.

START-OF-SELECTION.
  WRITE: / '입력값:', p_amt.

실행(F8)하면 입력 화면이 먼저 뜨고, 값을 넣으면 p_amt로 들어온다.

공짜로 따라오는 것들

p_amt의 타입을 zde_amount로 줬더니:

  • 라벨 — Data Element의 Field Label("금액")이 입력칸 옆에 자동으로 뜬다.
  • 도움말 — Domain에 이나 검색도움말이 있으면 입력칸에서 F4를 눌러 선택 목록이 나온다.
  • 입력검증 — 고정값을 벗어난 값은 자동으로 거부된다.

이게 "보상"이다. 타입을 TYPE c LENGTH 8처럼 줬다면 라벨도 F4도 없다. Data Element로 줬기에 화면이 똑똑해졌다.

흔한 실수와 점검

  • 라벨이 안 떠요 → Data Element의 Field Label이 비었거나, 타입을 표준 타입으로 직접 줬다. Data Element 참조로 바꾼다.
  • F4가 없어요 → Domain에 고정값/검색도움말이 없다. 입력검증이 필요하면 추가.
  • PARAMETERS 이름 길이 → 8자 이내(예: p_amt).

전역 정의(Domain·Data Element)의 가치를 화면에서 확인했다. 그런데 지금은 값을 낱개로만 다룬다. 관련된 값 여러 개(이름+나이+금액)를 하나로 묶고 싶다 → CH04(Structure).
