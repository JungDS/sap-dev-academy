DDIC 관계와 입력도움말(F4)

PARAMETERS와 DDIC F4 Help 연결

DDIC의 검증·F4가 PARAMETERS 화면으로 자동 연결되는 원리.

Track 1Chapter 9Lesson 6초급

수고가 화면으로 연결되는 지점

지금까지 만든 ··가 실제 입력 화면 ()에서 자동으로 F4·검증으로 나타나는 원리를 정리한다. CH03-L03의 "보상"을 더 깊이 보는 셈.

자동 연결 — Data Element 경유

PARAMETERS p_id TYPE zde_persid. 처럼 로 타입을 주면:

  • Data Element/Domain에 붙은 , , 필드의 ****가 그 의 F4·검증으로 그대로 따라온다.
  • 별도 코드 없이 화면이 똑똑해진다.

명시적으로 붙이기 — MATCHCODE OBJECT

특정 검색 도움말을 직접 지정하고 싶으면:

ABAP
1
2
PARAMETERS p_id TYPE zde_persid
  MATCHCODE OBJECT zsh_person_coll.
  • MATCHCODE OBJECT로 이 입력칸에 쓸 를 명시한다.

F4가 오는 세 경로

  1. (Domain) — 짧은 코드 목록.
  2. /Check Table — 마스터 테이블 값.
  3. — 설계된 검색 화면(가장 유연).

Search Help는 어디에 붙이나 — 부착 지점

같은 라도 어디에 붙이느냐에 따라 영향 범위가 다르다.

붙이는 곳 효과
그 DE를 쓰는 모든 필드에 한꺼번에(가장 광범위)
테이블/구조의 필드 그 필드 하나에
Structure 그 구조를 쓰는 화면 필드에
프로그램·화면 필드(MATCHCODE OBJECT) 그 입력칸 하나에 콕

💡 "널리 공유" → Data Element에, "이 칸만" → MATCHCODE OBJECT로. 넓게 붙일수록 재사용이 크고, 좁게 붙일수록 통제가 정확하다.

흔한 실수와 점검

  • F4가 안 떠요 → 표준 타입으로 직접 줬다. 경유 또는 MATCHCODE로.
  • 여러 도움말 충돌 → 우선순위를 이해하고 하나로 명확히(MATCHCODE 권장).

도움말 붙이는 법까지 봤다. 그런데 한 칸에 여럿이 걸리면 무엇이 뜰까? F4 호출 우선순위를 보자 → 다음 레슨.
