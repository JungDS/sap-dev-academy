CH09 · 모듈화 기초

Subroutine / Function / Class 선택 기준

셋 중 무엇을 — 상황별 모듈화 선택 기준.

셋 중 무엇을 쓰나

(FORM) · · (메서드) — 모듈화 도구가 셋이다. 상황별로 정리한다.

한눈에

도구 범위 권장도 언제
Subroutine(FORM) 프로그램 내부 ▲ 레거시 기존 코드 유지보수
Function Module 전역 재사용 ○ 절차적 재사용, RFC(원격), 표준 BAPI
Class / Method 전역 재사용 ◎ 권장 신규 개발 전반(현대 표준)

원칙

  • 신규 코드는 클래스(메서드) 로 — 현대 ABAP의 기본.
  • 함수 모듈은 여전히 유효: 원격 호출(RFC), 다수의 표준 인터페이스가 FM.
  • 서브루틴(FORM) 은 새로 만들지 말 것. 읽고 유지보수하는 용도로만 안다.

흔한 실수와 점검

  • 신규에 FORM 사용 → 지양. 메서드로.
  • 무조건 클래스 → RFC가 필요하면 함수 모듈(또는 RFC 가능 FM)을 쓴다.

이제 코드를 재사용 단위로 묶을 수 있다. 그런데 결과를 WRITE 리스트로 보자니 투박하다 — 표로 깔끔하게 보고 싶다 → CH10(SALV / 간단 ALV).
