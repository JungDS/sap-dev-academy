Enhancement / BAdI / User Exit

Implicit / Explicit Enhancement 판단

어떤 확장 수단을 쓸지 고른다.

Track 2Chapter 29Lesson 4고급

확장점이 있나 없나

구분 뜻
Explicit 개발자가 미리 심어 둔 확장점(BAdI·Enhancement Point/Section)
Implicit 모든 ·함수·FORM의 시작/끝에 자동으로 존재

선택 기준

  1. BAdI가 있으면 BAdI(가장 권장·OO·필터).
  2. 없고 적당한 Explicit Point가 있으면 그걸.
  3. 그것도 없으면 Implicit( 시작/끝)에 최소한으로.
  4. 그 무엇도 아니면 표준 직접 수정 — 최후의 수단(Modification, 등록·이유 필요).

즉 확장 강도: BAdI > Explicit > Implicit > Modification. 위쪽일수록 안전·권장.

흔한 실수와 점검

  • Implicit 남용 → 코드 중간이 아니라 시작/끝만 가능. 흐름 중간 개입엔 부적합.
  • Modification 먼저 → 거의 마지막 선택. 가능한 다른 수단을 먼저.

수단을 골랐다. 마지막으로 클라우드 시대의 확장 원칙 → 다음 레슨.
