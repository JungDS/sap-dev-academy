버튼을 누른 뒤 — PAI

(Process After Input)는 사용자가 버튼·Enter를 누른 뒤 실행된다. 어떤 버튼이 눌렸는지는 명령 필드 OK_CODE(또는 SY-UCOMM)로 안다.

MODULE user_command_0100 INPUT.
  DATA lv_code TYPE sy-ucomm.
  lv_code = ok_code.
  CLEAR ok_code.                       " 다음을 위해 비움(관례)

  CASE lv_code.
    WHEN 'SAVE'.
      PERFORM save_booking.            " 예매 저장(모듈화는 CH10)
    WHEN 'BACK' OR 'EXIT' OR 'CANCEL'.
      LEAVE TO SCREEN 0.               " 화면 종료
  ENDCASE.
ENDMODULE.

💡 OK_CODE를 읽은 뒤 CLEAR 하는 건 관례다(이전 명령이 남아 오작동하는 걸 막는다).

화면을 떠나는 세 가지

문 동작
LEAVE TO SCREEN 0 현재 화면 종료 → 이전/호출자로 돌아간다
LEAVE PROGRAM 프로그램 전체 종료(EXIT 맥락)
LEAVE SCREEN 현재 화면 처리 끝내고 다음 화면(SET SCREEN) 으로

BACK · EXIT · CANCEL — 관례

화면 위 표준 버튼 셋은 역할이 정해져 있다.

  • BACK(초록 화살표) — 한 단계 뒤로.
  • EXIT — 트랜잭션 종료(보통 LEAVE PROGRAM).
  • CANCEL — 입력 취소하고 뒤로.

보통 세 버튼을 CASE에서 비슷하게 처리하되, 저장 안 한 변경이 있으면 확인을 띄운다.

🔶 화면에 목록을 보여줄 땐 Table Control 대신 ALV(CH17) 를 쓴다(이 과정은 Table Control을 다루지 않는다).

흔한 실수와 점검

  • OK_CODE 안 비움 → 이전 명령이 남아 엉뚱하게 분기. 읽은 뒤 CLEAR.
  • LEAVE 혼동 → 화면 종료=LEAVE TO SCREEN 0, 프로그램 종료=LEAVE PROGRAM.

입력 처리까지 했다. 이제 화면 위 메뉴·툴바·제목을 다는 법 → 다음 레슨.
