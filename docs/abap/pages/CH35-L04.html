대량·정기 작업은 배치로

야간 정산·대량 (CH24·CH30)는 화면 없이 정해진 시각에 자동 실행한다. 그게 이다.

  • SM36 — 잡 정의·스케줄(주기·시작 조건).
  • SM37 — 잡 모니터(상태·로그·스풀).
  • 프로그램은 보통 SUBMIT ... VIA JOB ... AND RETURN으로 잡에 넣거나, 잡 API로 등록.
  • VIA JOB은 반드시 AND RETURN과 함께 써야 한다 — 잡에 등록만 하고 호출한 쪽은 계속 진행(공식 규칙).

운영 원칙

  • 재시작 가능하게 설계(CH24-L05의 재시작점·로그).
  • 실패 시 알림(메일·로그)으로 즉시 인지.
  • 자원·시간대를 고려해 스케줄(야간·부하 분산).

흔한 실수와 점검

  • 무한·중복 실행 → 스케줄·잠금으로 중복 방지.
  • 실패 방치 → SM37 모니터·알림으로 추적.

배치 운영을 봤다. 마지막으로 운영 오류를 남기고 추적하는 Application Log → 다음 레슨.
