화면 위 메뉴·툴바 — GUI Status

화면에는 입력칸만 있는 게 아니다. 위쪽에는 메뉴바·표준 툴바·application toolbar· function key가 있고, 사용자는 그 버튼으로 저장하거나 뒤로 가거나 도움말을 연다. 이 상단 버튼들의 function code를 묶은 객체가 (PF-STATUS)다. SE41(Menu Painter)에서 만들고, 어떤 버튼을 둘지·각 버튼의 FctCode를 정한다.

PBO에서 켜기 — SET PF-STATUS · SET TITLEBAR

와 제목은 PBO 처리 흐름의 MODULE ... OUTPUT에서 켠다.

ABAP
1
2
3
4
MODULE status_0100 OUTPUT.
  SET PF-STATUS 'ST100'.     " 이 화면에 쓸 GUI Status
  SET TITLEBAR  'TB100'.     " 제목 표시줄
ENDMODULE.
  • SET PF-STATUS로 상태를 켜면, 그 버튼이 눌릴 때 FctCode가 OK_CODE로 온다 (PAI 처리 흐름 — OK_CODE와 화면 떠나기).
  • 설정 후 sy-pfkey에 현재 status 이름이, sy-title에 제목이 들어온다.

버튼을 동적으로 숨기기 — EXCLUDING

특정 상황에서 일부 버튼만 비활성화하려면 EXCLUDING에 제외할 function code 목록을 넘긴다. (공식문서: EXCLUDING된 기능은 화면에서 선택할 수 없게 된다.)

DATA gt_excl  TYPE STANDARD TABLE OF sy-ucomm.
DATA gv_fcode TYPE sy-ucomm.

MODULE status_0100 OUTPUT.
  CLEAR gt_excl.
  IF gv_locked = 'X'.
    gv_fcode = 'SAVE'.
    APPEND gv_fcode TO gt_excl.          " 잠금 상태면 SAVE 비활성
  ENDIF.

  SET PF-STATUS 'ST100' EXCLUDING gt_excl.
  SET TITLEBAR  'TB100' WITH gv_conc.     " 아래 참고
ENDMODULE.

제목에 값 넣기 — TITLEBAR ... WITH

제목 텍스트에 &1~`&9자리표시자를 두고WITH로 값을 채운다. 제목 TB100이 **공연 &1 예매** 라면, SET TITLEBAR 'TB100' WITH gv_conc는 공연 C001 예매`가 된다.

제목은 장식이 아니다 — 사용자가 지금 어느 공연·어느 업무 화면에 있는지 알려 주는 안전 장치다.

직접 매핑해 보기 — Toolbar와 Function Code 매핑판

툴바 버튼을 눌러 그 function code가 OK_CODE로 전달되고 PAI CASE에서 처리되는지 보자. 잠금으로 SAVE를 EXCLUDING하면 어떻게 빠지는지, 그리고 CASE에 없는 버튼(HELP) 을 누르면 어떤 경고가 뜨는지 확인하자.

흔한 실수와 점검

  • PF-STATUS 미설정 → 표준 버튼만 떠 내 기능키가 없다. PBO에서 늦지 않게 SET PF-STATUS.
  • FctCode 불일치 → Menu Painter의 버튼 코드와 CASE save_ok 값이 같아야 한다. 다르면 버튼은 눌렸지만 처리할 branch가 없다.
  • EXCLUDING 오타 → 없는 function code를 넣으면 조용히 무시된다. 코드를 점검한다.
  • 제목 placeholder 누락 → &1이 있는데 WITH를 안 주면 자리표시자가 그대로 보인다.

정리

  • PF-STATUS는 화면 위 버튼과 function code의 설계도, TITLEBAR는 현재 화면의 업무 맥락이다.
  • 둘 다 PBO에서 설정하고(EXCLUDING으로 동적 비활성화, WITH로 제목 값 채움), PAI의 OK_CODE 처리와 같은 function code 체계로 맞춘다.

화면·입력·메뉴까지 갖췄다. 이제 화면 안에 ALV·트리 같은 컨트롤을 박는 자리, Custom Control을 보자 → Custom Control과 Container · Tabstrip · Subscreen.
