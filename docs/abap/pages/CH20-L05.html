Window Expression — 행을 유지하며 그룹 계산

GROUP BY는 행을 접지만, window는 상세 행을 두고 합계·순번을 붙인다.

Track 1Chapter 20Lesson 5고급

왜 — GROUP BY는 상세 행이 사라진다 🪟

GROUP BY는 강력하지만 대가가 있다. 상세 행이 사라진다.

예약 상세를 보면서 같은 공연의 총 예약 좌석도 함께 보고 싶다고 하자. GROUP BY concert_id를 쓰면 공연별 합계만 남고 개별 예약 행은 없어진다. 개별 행을 유지하면서 그룹 합계를 붙이고 싶을 때 을 쓴다.

무엇 — OVER · PARTITION BY 🎛️

(상세 행을 유지한 채 그룹 계산을 붙이는 SQL 식)은 함수 뒤에 OVER( … )를 붙인다.

SELECT concert_id,
       booking_id,
       seats,
       SUM( seats ) OVER( PARTITION BY concert_id )                       AS booked_total,
       ROW_NUMBER( ) OVER( PARTITION BY concert_id ORDER BY booking_id )  AS row_in_concert
  FROM zbooking
  WHERE status <> 'C'
  INTO TABLE @DATA(lt_window).

읽는 법은 이렇다.

부분 의미
SUM( seats ) 좌석 수를 합산
OVER( … ) 일반 가 아니라 window로 계산
PARTITION BY concert_id 같은 공연을 하나의 계산 묶음으로
ROW_NUMBER( ) 묶음 안에서 행 번호를 붙임
ORDER BY booking_id 행 번호를 어떤 순서로 붙일지

GROUP BY와의 차이는 표로 확인해야 한다.

입력(zbooking):

booking_id concert_id seats
B001 C001 2
B002 C001 3
B003 C002 4

GROUP BY 결과(행이 접힘):

concert_id sum_seats
C001 5
C002 4

Window 결과(상세 행 유지):

booking_id concert_id seats booked_total row_in_concert
B001 C001 2 5 1
B002 C001 3 5 2
B003 C002 4 4 1

순위 함수 — RANK · DENSE_RANK 🏅

말고 순위도 붙일 수 있다.

ABAP

SELECT concert_id,
       seats,
       RANK( )       OVER( PARTITION BY concert_id ORDER BY seats DESCENDING ) AS seat_rank,
       DENSE_RANK( ) OVER( PARTITION BY concert_id ORDER BY seats DESCENDING ) AS dense_rank
  FROM zbooking
  WHERE status <> 'C'
  INTO TABLE @DATA(lt_ranked).

RANK는 같은 값이 있으면 다음 순위를 건너뛴다(1, 1, 3). DENSE_RANK는 촘촘히 이어 간다(1, 1, 2). 둘 다 OVER( … ORDER BY … )의 순서 정의가 매우 중요하다.

어떻게 확인 ✅

  1. PARTITION BY가 어떤 묶음을 만드는지 표시한다.
  2. 묶음 안에서 ORDER BY가 어떤 순서를 만드는지 표시한다.
  3. 함수 결과가 각 상세 행에 반복되어 붙는지 확인한다.
  4. OVER( … ORDER BY … )와 문장 끝 ORDER BY를 혼동하지 않는다. 앞은 계산 순서, 뒤는 표시 순서로 서로 독립이다.

흔한 실수와 점검 🚧

  • WHERE에서 window로 필터 → window는 SELECT list에서만 쓴다. WHERE·GROUP BY에 직접 쓰지 못한다.
  • PARTITION BY 빠뜨림 → 빼면 전체가 한 묶음이 된다. 전체 합계가 모든 행에 붙는 게 의도인지 확인한다.
  • 순위인데 ORDER BY 없음 → RANK·DENSE_RANK엔 OVER 안 ORDER BY가 필수다. 없으면 순위가 무의미하다.
  • DB 지원 가정 → window는 DB·릴리스별 지원 차이가 있을 수 있다. 내 시스템에서 syntax check로 확인한다.
