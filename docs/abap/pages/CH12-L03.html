SELECT-OPTIONS와 Range Table

WHERE … IN (classic range)

Range Table을 조회 조건으로 — classic WHERE … IN과 sy-subrc·sy-dbcnt.

Track 1Chapter 12Lesson 3중급

담기만 해선 줄지 않는다

SELECT-OPTIONS 기본 문법에서 사용자의 조건이 so_conc·so_stat 같은 에 담겼다. 하지만 담기만 해서는 예매 목록이 줄어들지 않는다. 그 조건을 DB 조회 문장에 꽂아야 실제로 걸러진다.

핵심은 한 줄, WHERE field IN range_table이다. 이 한 줄이 사용자가 넣은 단일 값, 여러 값, 범위, 제외 조건을 SELECT의 필터로 바꿔 준다.

이 챕터는 classic이다. WHERE concert_id IN so_conc처럼 @ 없이 쓴다. 호스트 변수 표시(@)와 콤마 필드 목록 같은 현대식 Open SQL은 Chapter 19 · Modern Open SQL에서 배운다.

DATA lt_book TYPE TABLE OF zbooking.

SELECT * FROM zbooking INTO TABLE lt_book
  WHERE concert_id IN so_conc
    AND status     IN so_stat.

사람 말로 읽으면 이렇다.

코드 사람 말
concert_id IN so_conc 공연ID가 사용자가 고른 공연 조건에 들면
status IN so_stat 상태가 사용자가 고른 상태 조건에 들면
AND 두 조건을 모두 만족하는 행만

📌 빈 range는 true. so_conc가 비어 있으면 concert_id IN so_conc는 조건을 걸지 않은 것처럼 동작한다(공식 문서: initial ranges table → 조건이 DB에 전달되지 않아 결과는 항상 참). 그래서 둘 다 비우면 전체 예매가 조회된다. 편리하지만, 실수로 비우면 테이블 전체를 읽어 버릴 수 있다는 뜻이기도 하다.

결과 확인 — sy-subrc · sy-dbcnt

예매 6건을 기준으로 시나리오별 결과를 예측해 보자.

so_conc so_stat 예상 결과
비어 있음 비어 있음 6건 전체
I EQ C001 비어 있음 C001 공연 4건
비어 있음 I EQ N 예약 N 5건
I EQ C001 I EQ N C001이면서 N인 4건
I EQ C999 비어 있음 0건 (없는 공연)

실행 뒤에는 시스템 필드로 확인한다.

  • 행이 하나라도 담기면 sy-subrc = 0, sy-dbcnt = 담긴 행 수.
  • 결과가 비면 sy-subrc = 4, sy-dbcnt = 0.
  • 단, **빈 range로 전체가 나온 것도 sy-subrc = 0**이다. "성공"과 "의도한 필터"는 따로 확인해야 한다.

흔한 실수와 점검

  • 빈 range의 의미를 거꾸로 기억한다 → 빈 so_conc는 "아무것도 선택 안 했으니 0건"이 아니라 "공연 조건은 제한 안 함"이다.
  • 전체 조회를 정상 필터로 착각한다 → 조건이 비어 전체가 나온 걸 결과로 믿으면 위험하다(성능·권한). 입력 필수 검증은 Chapter 15 · 이벤트와 선택화면에서 다룬다.
  • classic과 modern을 섞는다 → 이 예제는 classic range 조건이다. @ 호스트 표시·콤마 필드 목록은 쓰지 않는다(modern Open SQL은 다음 구간에서).

정리

  • WHERE field IN range_table은 을 SELECT 조건으로 평가한다.
  • 가 만든 selection table도 같은 구조라 그대로 IN에 쓴다.
  • 빈 range는 true → 전체 조회가 될 수 있다.
  • 결과는 sy-subrc·sy-dbcnt·담긴 행 수로 확인한다.

조건은 SELECT에 꽂았다. 그런데 화면의 다중 선택 팝업에서 포함(I)과 제외(E)가 섞이면 결과가 어떻게 정해질까 → 다음 레슨.
