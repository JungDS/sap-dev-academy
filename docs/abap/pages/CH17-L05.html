표 전체의 보기 설정

Field Catalog가 컬럼별 설명서라면, (타입 LVC_S_LAYO)은 표 전체의 보기 설정이다. 같은 데이터라도 줄무늬가 있으면 읽기 쉽고, 컬럼 너비가 자동으로 맞으면 가로 스크롤이 줄고, 제목이 있으면 지금 보는 목록이 무엇인지 바로 안다. 업무 화면에서 표는 반복해서 읽고 비교하는 도구다. 읽기 편한 표가 사용자 실수를 줄인다.

DATA gs_layout TYPE lvc_s_layo.

gs_layout-zebra      = abap_true.    " 줄무늬
gs_layout-sel_mode   = 'A'.          " 행 선택 모드
gs_layout-grid_title = '예매 목록'.   " 표 제목
gs_layout-cwidth_opt = abap_true.    " 컬럼 너비 자동 최적화
필드 의미 초보자 설명
zebra 행 배경을 번갈아 긴 목록에서 줄을 놓치지 않게
sel_mode 행 선택 방식 한 행만/여러 행 선택 기준
grid_title 표 제목 이 표가 무엇을 보여 주는지
cwidth_opt 컬럼 너비 자동 조정 데이터 길이에 맞춰 너비
info_fname 행 색상 컬럼 이름 행 색상용 — 행 색상 기초에서

직접 켜고 꺼 보기 — Layout 토글 패널

각 옵션을 켜고 끄며 미리보기 표가 어떻게 달라지는지 보자. 특히 cwidth_opt를 끄면 긴 고객명이 잘리고, 켜면 컬럼이 넓어진다. 여기서 바뀌는 건 데이터가 아니라 표 전체의 보기 설정이라는 점에 주목하자.

흔한 실수와 점검

  • 만들고 안 넘김 → gs_layout을 채워도 SET_TABLE_FOR_FIRST_DISPLAY의 is_layout에 넘겨야 적용된다. 만들었는지와 전달했는지를 따로 확인한다.
  • 옵션 과다 → 처음엔 zebra·cwidth_opt·grid_title·sel_mode 정도로 충분하다. 셀 스타일·세부 표시 제어는 후속 챕터로.

정리

  • 이 레슨은 표 전체를 읽기 좋게 만드는 설정을 다룬다.
  • LVC_S_LAYO를 채워 is_layout으로 넘긴다. 채우기와 전달은 별개의 단계다.

표 모양을 정했다. 이제 사용자가 컬럼 배치·필터 상태를 저장하고 다시 불러오는 Variant를 연결하자 → Variant 기본 설정.
