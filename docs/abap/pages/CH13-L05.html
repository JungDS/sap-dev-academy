Classic DDIC View·유지보수 객체

Table Maintenance Generator / SM30

테이블/뷰에 표준 유지보수 화면을 생성 — SM30로 운영.

Track 1Chapter 13Lesson 5중급

입력 화면을 자동 생성

테이블이나 에 표준 입력 화면을 자동으로 만들어 주는 게 다. 만든 뒤엔 SM30으로 데이터를 입력·수정한다.

생성 (SE11 → Utilities)

SE11 → 테이블/뷰 → Utilities → Table Maintenance Generator.

  • Authorization Group(권한 그룹), 화면 번호, Maintenance Type(1단계/2단계) 지정.
  • 생성하면 표준 유지보수 화면(추가·수정·삭제)이 만들어진다.

운영 (SM30)

SM30 → 테이블/뷰 입력 → Maintain → 행 추가·수정·삭제 → 저장
  • SM30 — 단일 뷰/테이블 유지보수.
  • 저장 시 (이송 대상이면) 에 기록된다(CH01-L05의 그 흐름).

흔한 실수와 점검

  • Generator 없이 SM30 → "유지보수 불가" 오류. 먼저 Generator로 화면 생성.
  • 권한 그룹 미지정 → 운영에서 권한 통제가 안 된다.

이제 데이터를 표준 화면으로 유지보수한다. 그런데 이 클래식 뷰들은 현대의 CDS View로 진화한다 — 그 비교를 준비하자 → 다음 레슨.
