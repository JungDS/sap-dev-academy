ABAP을 OData로 (classic Gateway)

CH23의 RAP 이전, ABAP을 웹·Fiori에 노출하는 전통적 방법이 SAP Gateway(SEGW)다. 지금도 많은 시스템이 쓴다.

SEGW 프로젝트 구성

  • Data Model — EntityType(필드)·EntitySet(컬렉션)·Association.
  • Service Implementation — 각 작업(GetEntitySet·GetEntity·Create 등)을 로 구현.
  • Runtime/Model Provider — 생성된 *_DPC_EXT(데이터)·*_MPC_EXT(모델) .
  • 등록(Register)·활성화하면 /sap/opu/odata/... URL로 호출된다.

흔한 실수와 점검

  • DPC_EXT vs MPC_EXT → 데이터 로직은 DPC, 모델은 MPC.
  • 서비스 미등록 → /IWFND/MAINT_SERVICE에 등록해야 호출 가능.

게이트웨이 구조를 봤다. 이제 실제 조회 를 구현하자 → 다음 레슨.
