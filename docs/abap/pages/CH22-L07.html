🎫 11단계 — 콘서트 모델을 CDS로

테이블 위에 재사용 가능한 CDS 뷰(CH22)를 얹는다 — 기반 ZI_와 소비 ZC_.

define view entity ZI_Concert
  as select from zconcert
  association [0..*] to ZI_Perf as _Perf
    on $projection.concert_id = _Perf.concert_id
{
  key concert_id,
      artist,
      venue,
      capacity,
      _Perf
}

@EndUserText.label: '공연'
define view entity ZC_Concert
  as projection on ZI_Concert
{
  @UI.lineItem: [{ position: 10 }]
  key concert_id,
  @UI.lineItem: [{ position: 20 }]
      artist,
      venue,
      capacity,
      _Perf
}

직접 해 볼 과제

  1. ZI_Perf(회차)도 만들고 association으로 잇는다.
  2. ZC_Concert를 Data Preview로 조회한다.
  3. (도전) 잔여석을 association+로 계산해 노출(심화).

흔한 실수와 점검

  • association 미노출 → { }에 _Perf를 넣어야 소비 측에서 따라간다(CH22-L03).
  • 활성화 → CDS도 활성화해야 뷰가 생긴다.

모델을 CDS로 끌어올렸다. 이제 그 위에 트랜잭션(RAP) 을 얹자 → CH23.
