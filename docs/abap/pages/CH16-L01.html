정형 화면을 넘어서

CH15까지의 선택화면(PARAMETERS·SELECT-OPTIONS)은 SAP이 정해 준 틀이다. 그런데 "공연을 고르고, 좌석 수를 넣고, 예매 버튼을 누르는" 내가 설계한 화면이 필요하다. 그게 (Dynamic Program, 화면)이고, 화면 중심 프로그램이 이다.

리포트 vs Module Pool

  • 은 실행할 T-code가 반드시 있어야 한다(CH01-L07의 T-code, 이번엔 필수).
  • SE80(또는 SE51 Screen Painter)에서 화면을 그린다.

화면의 두 박자 — PBO / PAI

화면 하나가 뜨고 처리되는 흐름은 두 박자다.

  • PBO(Process Before Output) — 화면을 그리기 직전(초기값·상태 준비).
  • PAI(Process After Input) — 사용자가 버튼·Enter를 누른 뒤(입력 처리).

흔한 실수와 점검

  • T-code 없이 실행 → 은 T-code로만 실행된다(SE93로 생성).
  • 리포트 이벤트 기대 → 은 START-OF-SELECTION이 아니라 PBO/PAI 흐름.

화면 프로그램의 뼈대를 잡았다. 이제 실제 화면 번호와 Screen Painter로 화면을 그려 보자 → 다음 레슨.
