managed implementation in class zbp_i_booking unique;

define behavior for ZI_Booking alias Booking
persistent table zbooking
draft table zbooking_d              // 초안 보관(L07)
lock master
{
  create; update; delete;
  field ( readonly ) booking_id, created_by;

  determination setDefaults on modify { create; }
  validation validateCapacity on save { field seats, concert_id, perf_no; }
  action ( features : instance ) cancelBooking result [1] $self;

  mapping for zbooking { booking_id = booking_id; }
}

CLASS lhc_booking DEFINITION INHERITING FROM cl_abap_behavior_handler.
  PRIVATE SECTION.
    METHODS setDefaults     FOR DETERMINE ON MODIFY IMPORTING keys FOR Booking~setDefaults.
    METHODS validateCapacity FOR VALIDATE ON SAVE   IMPORTING keys FOR Booking~validateCapacity.
    METHODS cancelBooking   FOR MODIFY            IMPORTING keys FOR ACTION Booking~cancelBooking RESULT result.
ENDCLASS.
  • draft table — 미저장 변경을 초안으로(L07).
  • determination/validation/action을 핸들러에 구현(다음 레슨).

흔한 실수와 점검

  • mapping → CDS 필드↔DB 컬럼 이름이 다르면 매핑.
  • features instance → 상황별로 액션 활성/비활성 제어 가능.

동작을 선언했다. 그 알맹이 — Action·Validation·Determination → 다음 레슨.
