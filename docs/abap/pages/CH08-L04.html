INTO 대상 형태

결과를 어디에 담나 — Work Area·개별 변수·CORRESPONDING·APPENDING.

Track 1Chapter 8Lesson 4초급

같은 SELECT, 다른 담는 그릇

읽은 결과를 담는 방법은 여러 가지다. 상황에 맞게 고른다.

INTO wa / INTO TABLE

ABAP
1
2
SELECT SINGLE * FROM sflight INTO ls_flight WHERE carrid = 'KE' AND connid = '0701' AND fldate = '20260623'.
SELECT * FROM sflight INTO TABLE lt_flight WHERE carrid = 'KE'.

한 건은 구조체(INTO ls_), 여러 건은 내부 테이블(INTO TABLE).

INTO (필드들) — 개별 변수로

몇 개 값만 변수로 직접 받는다. SELECT 필드 순서와 변수 순서가 1:1로 맞아야 한다.

ABAP
1
2
3
4
5
6
DATA: lv_occ TYPE sflight-seatsocc,
      lv_max TYPE sflight-seatsmax.

SELECT SINGLE seatsocc seatsmax FROM sflight
  INTO (lv_occ, lv_max)
  WHERE carrid = 'KE' AND connid = '0701' AND fldate = '20260623'.

CORRESPONDING FIELDS OF — 이름 맞춰 담기

SELECT 필드와 받는 구조의 같은 이름 필드끼리 자동으로 맞춰 담는다(순서 무관).

ABAP
1
2
3
SELECT carrid connid fldate FROM sflight
  INTO CORRESPONDING FIELDS OF TABLE lt_flight
  WHERE carrid = 'KE'.

APPENDING — 기존 테이블에 덧붙이기

INTO TABLE은 기존 내용을 지우고 채우지만, APPENDING TABLE은 이어 붙인다.

ABAP
1
SELECT * FROM sflight APPENDING TABLE lt_flight WHERE carrid = 'LH'.

흔한 실수와 점검

  • INTO (v1,v2) 순서 어긋남 → SELECT 필드 순서와 변수 순서가 1:1로 맞아야 한다.
  • CORRESPONDING 이름 불일치 → 이름이 다르면 안 담긴다. 이름을 맞추거나 별칭(AS, CH13).
  • INTO vs APPENDING → INTO는 덮어쓰기, APPENDING은 누적.

담는 법까지 익혔다. 이제 어느 행만 읽을지를 정교하게 — WHERE 상세 → 다음 레슨.
