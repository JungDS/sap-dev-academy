화면에 ALV를 박으려면

다음 챕터(CH17)에서 화면에 ALV 그리드(좌석표·예매 목록)를 박는다. 그러려면 화면에 Custom Control 영역을 두고, 거기에 객체를 연결해야 한다.

🔎 [선행 사용] 아래의 TYPE REF TO·CREATE OBJECT는 본격적으로 CH20(OO) 에서 배운다. 지금은 "컨테이너 객체를 만들어 화면 영역에 연결한다" 정도로만 받아들이면 된다.

ABAP
1
2
3
4
DATA go_cont TYPE REF TO cl_gui_custom_container.

CREATE OBJECT go_cont
  EXPORTING container_name = 'CONT100'.   " 화면의 Custom Control 이름

이 go_cont에 CH17에서 ALV 그리드를 얹는다.

화면을 나누는 요소들

요소 뜻
Tabstrip 한 화면을 여러 탭으로 나눔
Subscreen Area 화면 안에 다른 화면을 끼워 넣는 영역
Status Icon 상태를 아이콘으로 표시(예: 매진 🔴)
  • Tabstrip은 "공연 정보 / 예매 내역"처럼 한 화면에 여러 탭을 둘 때.
  • Subscreen은 공통 머리말 화면을 여러 화면에서 재사용할 때.

🔶 Table Control은 다루지 않는다 — 화면 안의 표는 거의 항상 ALV로 대체한다(CH17).

흔한 실수와 점검

  • Container 이름 불일치 → 화면의 Custom Control 이름과 container_name이 같아야 한다.
  • 컨테이너만 만들고 비움 → 컨테이너는 자리일 뿐. 내용(ALV 등)은 CH17에서 얹는다.

화면에 컨트롤을 박을 자리를 마련했다. 배운 걸 콘서트앱에 적용해 예매 입력 화면을 만들어 보자 → CH16-L08(실습).
