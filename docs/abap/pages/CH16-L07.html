화면에 무엇을 얹을 자리

입력칸만으로 끝나는 업무는 많지 않다. 예매 화면에는 좌석표나 예매 목록을 ALV로 보여 주고 싶고(Grid ALV 기초), 한 화면을 공연 정보·예매자 정보·상태로 나누고 싶을 수도 있다. 이 레슨은 화면 안에 "무엇을 얹을 자리" 를 만드는 요소들을 다룬다. 완성 ALV는 여기서 구현하지 않는다. 자리와 구조만 잡는다.

요소 역할 이 챕터에서의 범위
Custom Control 화면 안의 빈 사각 영역 ALV Grid를 얹을 자리
Custom Control 영역과 control 객체를 연결 CL_GUI_CUSTOM_CONTAINER 선행 사용
Tabstrip 한 화면을 여러 탭 페이지로 분리 탭 title·function code·subscreen 연결
Subscreen Area 화면 안에 다른 화면을 끼워 넣는 영역 CALL SUBSCREEN 흐름
Status Icon 상태를 아이콘·텍스트로 표시 매진·가능·오류 표시

Custom Control과 Container

화면에 Custom Control 영역(CONT100)을 두고, 거기에 객체를 연결한다.

ABAP

8
DATA go_cont TYPE REF TO cl_gui_custom_container.

MODULE create_container_0100 OUTPUT.
  IF go_cont IS INITIAL.                   " 한 번만 생성
    CREATE OBJECT go_cont
      EXPORTING container_name = 'CONT100'.  " 화면 영역 이름과 같아야
  ENDIF.
ENDMODULE.

🔎 [선행 사용] TYPE REF TO·CREATE OBJECT는 Global Class 생성과 객체에서 정식으로 배운다. 여기선 "Custom Control 이름 CONT100에 container 객체를 연결한다"는 블랙박스로만 받아들이면 된다.

  • ⚠️ Container를 PBO마다 새로 만들면 lifecycle 문제가 생긴다 → IF go_cont IS INITIAL로 한 번만 생성한다.
  • Custom Control은 내용물이 아니라 빈 사각 영역이다. container와 실제 control을 만들어야 내용이 보인다(ALV는 Grid ALV 기초에서 이 go_cont에 얹는다).

Subscreen — 화면 안에 다른 화면

Subscreen Area는 화면 안에 다른 dynpro를 끼워 넣는 영역이다. Flow Logic에서 CALL SUBSCREEN으로 포함한다.

ABAP
1
2
3
4
5
6
PROCESS BEFORE OUTPUT.
  CALL SUBSCREEN sub_area INCLUDING sy-repid '0110'.

PROCESS AFTER INPUT.
  CALL SUBSCREEN sub_area.
  MODULE user_command_0100.
  • ⚠️ Subscreen은 자체 OK field를 가질 수 없다(공식문서). 사용자 명령의 function code는 main 화면의 OK_CODE로 모인다 — 처리 책임은 main의 PAI 흐름에 있다.

Tabstrip · Status Icon

  • Tabstrip — 탭마다 title(function code) 과 subscreen area가 짝이다. ABAP에서 CONTROLS tab_main TYPE TABSTRIP.으로 선언하고, tab_main-activetab에 활성 탭의 function code를 보관한다.
  • Status Icon — 상태를 아이콘·텍스트·툴팁으로 보여 준다(예: 매진 🔴 / 가능 🟢 / 오류 ⚠).

🔶 Table Control은 다루지 않는다. 공식문서엔 있지만, 표 형태 데이터 표시는 이 과정에서 Grid ALV 기초로 대체한다.

직접 설계해 보기 — 화면 분할과 컨테이너 설계판

요소를 골라 Screen Painter 영역 이름과 ABAP 연결을 맞춰 보자. container_name이 화면 영역과 다르면 어떤 경고가 나는지, 탭·subscreen·status icon이 어떻게 연결되는지 본다.

흔한 실수와 점검

  • Container 이름 불일치 → 화면 Custom Control 이름과 container_name이 같아야 붙는다.
  • 컨테이너를 PBO마다 생성 → IF go_cont IS INITIAL로 한 번만.
  • Subscreen에 OK field 기대 → 없다. 명령은 main 화면 OK_CODE로 처리.
  • Tabstrip을 탭만 그리고 끝 → 탭 title function code·activetab·subscreen area·CALL SUBSCREEN이 모두 맞아야 동작한다.

정리

  • Custom Control은 ALV를 담을 자리, Subscreen·Tabstrip은 화면을 나누는 구조다.
  • OO 문법과 Grid 구현은 뒤로 미루고, 지금은 "화면 영역과 프로그램 연결" 까지만 정확히 잡는다.

화면에 컨트롤을 박을 자리를 마련했다. 배운 걸 콘서트앱에 적용해 예매 입력 화면을 만들어 보자 → 실습 — 예매 입력 화면 (Dynpro).
