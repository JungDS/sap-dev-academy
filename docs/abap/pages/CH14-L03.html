Help View와 Search Help 연결

F4 도움말을 풍부하게 — 여러 테이블을 묶는 Help View.

Track 1Chapter 14Lesson 3중급

사용자는 코드를 외우지 않는다

공연 등록 화면에서 "P001을 입력하라"고 하면 개발자는 당연하지만, 업무 담당자는 "클래식 갈라가 어느 코드인지"를 찾아야 한다. F4 입력도움말이 이 간격을 줄인다. 사용자는 코드나 설명 일부로 검색하고, 고른 값이 입력 필드로 돌아온다.

Chapter 9 · 입력도움말(F4)에서 의 기본을 배웠다. 단일 테이블로 충분하면 selection method가 테이블이면 된다. 하지만 코드 + 설명 + 보조 정보를 함께 보여 주려면 가 후보가 된다.

Help View = Search Help의 소스

는 에서 쓰는 DDIC table view다. 공식문서 기준 두 가지가 핵심이다.

  • F4 전용 — ABAP SQL로 직접 읽는 대상이 아니다(프로그램 조회는 Database/Projection View).
  • primary table 보존(outer join 성격) — foreign key로 보조 테이블을 묶되, 주 테이블 행은 남고 보조 설명이 없으면 그 칸만 비어 보일 수 있다.

Database View와 목적이 다르다.

질문 Database View Help View
ABAP SQL에서 읽나 예 아니오
주 사용처 프로그램 조회 F4 Search Help
여러 테이블 결합 inner join primary table 보존(outer)

아래에서 perf_id 필드에 F4를 눌러, 사용자가 설명을 보고 고르면 key가 화면에 돌아오는 흐름을 확인하자. 설명 테이블이 누락되거나 Export가 꺼지면 어떻게 되는지도 보자.

흔한 실수와 점검

  • 를 프로그램 조회용으로 → ABAP SQL data source가 아니다. 조회는 Database/Projection View나 코드 JOIN.
  • parameter를 대충 둔다 → 표시 필드가 많아도 Export가 안 맞으면 선택값이 화면에 안 돌아오고, Import가 틀리면 입력 검색어가 반영되지 않는다.
  • F4를 너무 복잡하게 → 처음엔 코드·대표명·상태면 충분하다. exit·동적 제어는 심화.

정리

  • 는 F4 를 풍부하게 만드는 View다(프로그램 조회용 아님).
  • primary table을 기준으로 보조 정보를 붙여 사용자의 선택을 돕는다.
  • F4는 단순 팝업이 아니라 DDIC와 화면 필드가 연결된 데이터 흐름이다.

조회용·F4용 View까지 봤다. 이제 데이터를 입력·수정하는 유지보수 관점의 View다 → 다음 레슨(Maintenance View).
