개발 환경과 첫 프로그램

T-code와 SE38 첫 실행

명령창과 SE38(ABAP Editor)로 첫 프로그램을 만들고 $TMP(Local Object)에 저장한다.

🖥️이번 Lesson에서 다루는 트랜잭션 코드

"메뉴 뒤지기"가 답답하다

메뉴 트리로 ABAP 에디터까지 가려면 한참을 펼쳐야 한다. SAP은 이 답답함을 (트랜잭션 코드) 로 해결한다 — 화면마다 붙은 짧은 주소다.

T-code: 어디든 바로 가는 주소

화면 왼쪽 위 명령창에 코드를 입력하고 Enter를 치면, 메뉴를 거치지 않고 그 화면으로 직행한다. 웹의 URL 직접 입력과 같다.

자주 쓰는 접두어:

  • SE38 — 그냥 입력하면 현재 창에서 이동
  • /nSE38 — 현재 작업을 닫고 로 (/n = new task)
  • /oSE38 — 새 창에서 열기 (/o = open)
  • /nex — 확인 없이 즉시 로그오프

처음 익힐 개발 : (ABAP 에디터), SE11(Dictionary), SE16N(데이터 조회), SE80(오브젝트 내비게이터). 이 레슨에선 만.

SE38 — ABAP Editor 열기

명령창에 SE38을 입력해 ABAP Editor로 간다. 여기서 실행형 프로그램(Report)을 작성·활성화·실행한다.

첫 프로그램 생성하기

  1. Program 칸에 이름을 입력한다. 이름은 반드시 Z 또는 Y로 시작 — 이 영역(네임스페이스)이 고객(우리)에게 열려 있다. 예: ZHELLO.
  2. Create 클릭.
  3. 속성 화면에서 Type = Executable Program(실행형)을 고르고 저장.

왜 Z/Y로 시작하나? 그 외 이름은 SAP 표준 객체용으로 예약돼 있다. 고객 개발물은 충돌을 피하려 Z/Y 네임스페이스를 쓴다.

$TMP에 저장하기 (Local Object)

저장하면 시스템이 "이 객체를 어느 패키지에 넣을까?" 하고 묻는다. 지금은 가장 간단한 길 — Local Object 버튼을 누른다. 그러면 객체가 라는 로컬 전용 패키지에 들어가고, 이송요청 없이 바로 저장된다.

  • 장점: 복잡한 절차 없이 바로 개발·실험 가능.
  • 한계: 에 둔 객체는 다른 시스템으로 이송할 수 없고, 팀과 공유되지 않는다. "내 PC 바탕화면에만 둔 파일"인 셈.

이 한계가 불편으로 다가오면 그때 개발 패키지로 옮긴다 → CH01-L06.

활성화와 실행 — 개발 루프

ABAP은 작성한 코드를 "활성화(Activate)" 해야 실제로 동작한다.

  1. 작성 → 2. 저장(Ctrl+S) → 3. 활성화(Ctrl+F3) → 4. 실행(F8)

이 네 박자가 ABAP 개발의 기본 루프다. 앞으로 수백 번 반복하게 된다.

흔한 실수와 점검

  • "Name range" 오류 → 이름이 Z/Y로 시작하지 않는다.
  • 수정했는데 결과가 그대로 → 활성화(Ctrl+F3) 를 안 했다. 비활성 상태의 변경은 실행에 반영되지 않는다.
  • 저장 시 패키지 창에서 막힘 → 지금은 망설이지 말고 Local Object.
