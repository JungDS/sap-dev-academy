개발 환경과 첫 프로그램

T-code와 SE38 첫 실행

명령창과 SE38(ABAP Editor)로 첫 프로그램을 만들고 $TMP(Local Object)에 저장한다.

Track 1Chapter 1Lesson 2입문

🖥️이번 Lesson에서 다루는 트랜잭션 코드

필요한 프로그램을 찾아서 실행하기가 어렵다

메뉴 트리로 ABAP 에디터까지 가려면 한참을 펼쳐야 한다. SAP은 이 불편을 (트랜잭션 코드) 로 해결한다 — 화면마다 붙은 짧은 주소다.

T-code: 어디든 바로 가는 주소

화면 왼쪽 위 명령창에 코드를 입력하고 Enter를 치면, 메뉴를 거치지 않고 그 화면으로 직행한다. 웹의 URL 직접 입력과 같다.

⚠️ 핵심 함정 — 접두어 없는 SE38은 로그인 직후 Easy Access에서만 통한다. SE11 등 다른 트랜잭션 안에서 이동하려면 앞에 /n을 붙여 /nSE38 으로. 단 /n은 현재 화면을 닫고 이동하므로, 저장 안 한 입력이 있다면 먼저 저장부터.

💡 현재 화면은 그대로 두고 새 창(세션) 으로 열려면 /oSE38. 반대로 /nex 는 묻지도 않고 즉시 로그오프하는 코드라 실수로 치면 곤란하다 — 위 플레이그라운드에서 안전하게 겪어 보자.

SE38 — ABAP Editor 열기

명령창에 SE38을 입력해 ABAP Editor로 간다. 실행형 프로그램(Report)을 작성·활성화·실행하는 이 과정의 주 작업장이다 — 이번 레슨의 첫 프로그램도 여기서 만든다.

아래가 그 첫 화면(Initial Screen)이다. Program 칸에 이름을 넣고 Create를 누르면 시작한다.

SE11 — ABAP Dictionary

SE11은 ABAP Dictionary — 테이블·데이터 타입 같은 "데이터의 설계도" 를 만들고 보는 곳이다. 프로그램이 다룰 데이터의 구조를 여기서 정의한다.

아래 첫 화면에서 보듯, Database table·View·Data type·Domain·Search help 같은 사전 오브젝트를 골라 만든다. 지금은 위치만 기억해 두고, 정식으로는 Domain·Data Element 만들기부터 본격적으로 쓴다.

SE80 — Object Navigator

SE80은 Object Navigator — 프로그램·클래스·화면 등 여러 개발 오브젝트를 한 화면의 트리로 묶어 오가는 통합 작업장이다.

아래처럼 왼쪽 Repository Browser 트리에서 오브젝트를 찾아 오간다(예: $TMP 로컬 오브젝트). 입문 단계엔 SE38 하나로 충분하고, 다룰 오브젝트가 늘면 그때 편해진다.

첫 프로그램 생성하기

이제 SE38로 돌아가 첫 프로그램을 만들어 보자.

  1. Program 칸에 이름을 입력한다. 이름은 반드시 Z 또는 Y로 시작 — 이 영역(네임스페이스)이 고객(우리)에게 열려 있다. 예: ZHELLO.
  2. Create 클릭.
  3. 속성 화면에서 Type = Executable Program(실행형)을 고르고 저장.

왜 Z/Y로 시작하나? 그 외 이름은 SAP 표준 객체용으로 예약돼 있다. 고객 개발물은 충돌을 피하려 Z/Y 네임스페이스를 쓴다.

$TMP에 저장하기 (Local Object)

저장하면 시스템이 "이 객체를 어느 패키지에 넣을까?" 하고 묻는다. 지금은 가장 간단한 길 — Local Object 버튼을 누른다. 그러면 객체가 라는 로컬 전용 패키지에 들어가고, 이송요청 없이 바로 저장된다.

  • 장점: 복잡한 절차 없이 바로 개발·실험 가능.
  • 한계: $TMP에 둔 객체는 다른 시스템으로 이송할 수 없고, 팀과 공유되지 않는다. "내 PC 바탕화면에만 둔 파일"인 셈.

이 한계가 불편으로 다가오면 그때 개발 패키지로 옮기기를 배운다.

활성화와 실행 — 개발 루프

ABAP은 작성한 코드를 "활성화(Activate)" 해야 실제로 동작한다.

  1. 작성 → 2. 저장(Ctrl+S) → 3. 활성화(Ctrl+F3) → 4. 실행(F8)

이 네 박자가 ABAP 개발의 기본 루프다. 앞으로 수백 번 반복하게 된다.

흔한 실수와 점검

  • 명령창에 SE38을 쳤는데 이동이 안 됨 → 지금 Easy Access가 아니라 다른 트랜잭션(SE11 등) 안이다. /nSE38로 이동한다(접두어 없는 코드는 Easy Access 전용).
  • "Name range" 오류 → 이름이 Z/Y로 시작하지 않는다.
  • 수정했는데 결과가 그대로 → 활성화(Ctrl+F3) 를 안 했다. 비활성 상태의 변경은 실행에 반영되지 않는다.
  • 저장 시 패키지 창에서 막힘 → 지금은 망설이지 말고 Local Object.

자리도 잡고 빈 프로그램도 만들었다. 이제 그 안을 채울 차례 — 프로그램은 어떤 뼈대로 생겼고, 설명(주석)은 어떻게 다는가? → 다음 레슨.
