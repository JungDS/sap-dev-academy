SALV 1차 (간단 ALV)

SALV의 목적과 CL_SALV_TABLE 개요

WRITE 리스트를 넘어 — 표 형태 출력 SALV(CL_SALV_TABLE).

Track 1Chapter 11Lesson 1초급

WRITE 리스트는 투박하다

CH05~CH08에서 데이터를 다뤘지만, WRITE로 찍은 리스트는 정렬·필터·열 너비 조정도 안 되고 모양도 단조롭다. 사용자가 쓰기 좋은 표(그리드) 로 보여주고 싶다 — 그게 (ABAP List Viewer)이고, 그 가장 단순한 길이 다.

SALV = 단순·읽기전용 ALV

는 CL_SALV_TABLE 클래스로 표를 그리는 현대적·간결한 다.

  • 장점 — 코드 몇 줄로 정렬·필터·합계·엑셀 내보내기 등 표준 기능이 공짜.
  • 범위 — 기본은 읽기 전용 표시. 셀 편집·복잡한 이벤트는 Grid ALV(CH17)· 심화(CH21)에서.
방식 성격
WRITE 리스트 투박, 수동
SALV (CL_SALV_TABLE) 간결, 표준 기능 풍부, 읽기전용
Grid ALV (CL_GUI_ALV_GRID) 강력·편집 가능, 코드 많음 (CH17)

왜 SALV부터인가

내부 테이블 하나만 있으면 몇 줄로 그럴듯한 리포트가 나온다. 입문 단계의 "결과를 보기 좋게"에 가장 잘 맞는다.

직접 해 보기

같은 예매 6행을 WRITE와 로 번갈아 보며, 정렬·합계가 한쪽에서만 되는 걸 확인해 보자.

흔한 오해와 점검

  • 로 셀 편집 → 안 된다(읽기전용). 편집은 Grid .
  • CL_SALV_TABLE는 클래스 → CH10의 클래스 호출 지식이 여기서 쓰인다.

개념은 잡았다. 이제 실제로 내부 테이블을 SALV로 띄우는 factory 호출을 보자 → 다음 레슨.
