Transparent Table (SE11 · SE16N)

데이터 입력(SE16N)

만든 테이블에 데이터를 넣고 확인한다(SE16N).

Track 1Chapter 6Lesson 2초급

빈 테이블 채우기

을 만들었으니 값을 넣어야 조회도 의미가 있다. 학습 단계에선 SE16N으로 손쉽게 입력·확인한다.

SE16N로 조회·입력

SE16N → 테이블명 ZTPERSON → 실행(F8)하면 현재 행이 보인다.

  • 조회 — 조건을 주고 실행하면 해당 행만 나온다.
  • 입력 — 신규 행 입력은 권한·설정에 따라 다르다. 교육 시스템의 안내(또는 SE11 → Contents, 전용 입력 프로그램)를 따른다.

실무에서는 보통 입력용 화면/프로그램(또는 유지보수 뷰, CH13)으로 데이터를 넣는다. 지금은 "테이블에 행이 들어간다"를 확인하는 게 목적.

키 중복은 안 된다

키가 같은 행을 두 번 넣을 수 없다. MANDT + PERSID가 키라면 같은 PERSID는 하나뿐.

흔한 실수와 점검

  • 입력 권한 없음 → 교육 시스템 안내를 따른다. SE16N 직접 편집은 제한될 수 있다.
  • 클라이언트 혼동 → 다른 Client에서 넣은 데이터는 안 보인다( 분리).

이제 데이터가 디스크에 영구히 남았다. 그런데 프로그램에서 이 저장된 값을 다시 읽어오려면? → CH07(Open SQL 기본 조회).
