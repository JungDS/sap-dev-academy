Modern ABAP Syntax

VALUE Constructor Expression

구조체·내부 테이블을 한 식으로 채운다 — VALUE

Track 1Chapter 18Lesson 2중급

한 줄 한 줄 채우기 — 번거롭다

classic에서는 구조체 필드를 하나씩 대입하고, 테이블은 APPEND를 여러 번 했다. 모던 VALUE #( )는 한 식으로 채운다.

구조체 채우기

ABAP
1
2
3
4
5
6
" classic
DATA ls_line TYPE ty_line.
ls_line-dan = 2. ls_line-mul = 3. ls_line-result = 6.

" modern
DATA(ls_line) = VALUE ty_line( dan = 2 mul = 3 result = 6 ).

내부 테이블 채우기

각 ( ... )가 한 행이다.

ABAP
1
2
3
4
DATA(lt_gugu) = VALUE ty_line_tab(
  ( dan = 2 mul = 1 result = 2 )
  ( dan = 2 mul = 2 result = 4 )
  ( dan = 2 mul = 3 result = 6 ) ).

기존에 더하기 — BASE

기존 테이블을 유지하면서 행을 더하려면 BASE를 쓴다.

ABAP
1
lt_gugu = VALUE #( BASE lt_gugu ( dan = 3 mul = 1 result = 3 ) ).

FOR로 반복 생성 — 테이블을 한 식으로

행을 일일이 적는 대신, 반복으로 한꺼번에 만들 수 있다. APPEND를 도는 LOOP( CH06)을 VALUE 안의 FOR 한 줄로 줄이는 셈이다.

숫자 범위로 생성 — 앞에서 3줄만 손으로 적었지만, 2단 전체(1～9)를 식 하나로:

ABAP
1
2
3
DATA(lt_gugu) = VALUE ty_line_tab(
  FOR i = 1 WHILE i <= 9
  ( dan = 2 mul = i result = 2 * i ) ).
  • FOR i = 1 WHILE i <= 9 — i를 1부터 조건이 참인 동안 1씩 늘리며 반복(LOOP과 같은 흐름).
  • 뒤의 ( ... ) — 그 회차에 만들 새 행 하나. 결과는 9행짜리 테이블.

이미 있는 테이블을 돌며 변환할 수도 있다:

ABAP
1
2
3
4
" lt_gugu의 각 행(line)을 훑어 결과를 2배로 바꾼 새 테이블
DATA(lt_double) = VALUE ty_line_tab(
  FOR line IN lt_gugu
  ( dan = line-dan mul = line-mul result = line-result * 2 ) ).
  • FOR line IN lt_gugu — lt_gugu를 한 행씩(line) 훑는다.
  • 원본은 그대로 두고 새 테이블을 만든다(원본 행 수만큼).

💡 이 VALUE #( FOR … )는 실무에서 매우 자주 쓰인다 — 특히 한 테이블을 다른 형태로 옮길 때. CH36의 RAP 코드도 이 방식으로 수정할 행 묶음을 만든다.

흔한 실수와 점검

  • 타입 생략 VALUE #( ) → 좌변 타입이 분명할 때만 #. 아니면 타입명을 적는다.
  • BASE 빠뜨림 → VALUE만 쓰면 기존 내용이 사라진다. 누적은 BASE.
  • FOR 변수의 수명 → FOR line IN …의 line은 그 식 안에서만 산다(밖에선 못 쓴다).

값을 한 식으로 채웠다. 이제 구조가 다른 둘 사이를 이름으로 매핑하는 CORRESPONDING → 다음 레슨.
