Modern ABAP Syntax

VALUE Constructor Expression

구조체·내부 테이블을 결과 모양 그대로 — VALUE

Track 1Chapter 18Lesson 2중급

한 줄 한 줄 채우기 — 절차가 의도를 가린다

classic에서는 구조체 필드를 하나씩 대입하고, 테이블엔 APPEND를 반복했다. 절차는 분명하지만 값의 모양이 안 읽힌다 — "2단 1행을 만들고 싶다"는 의도보다 "변수 만들기· 필드 채우기·append"라는 작업이 더 커 보인다. 행이 늘면 CLEAR 누락 같은 실수도 는다.

VALUE constructor expression은 절차를 나열하는 대신 결과가 어떤 값인지를 표현한다.

ABAP
1
2
3
4
TYPES: BEGIN OF ty_line,
         dan TYPE i, mul TYPE i, result TYPE i,
       END OF ty_line.
TYPES ty_line_tab TYPE STANDARD TABLE OF ty_line WITH EMPTY KEY.

구조체 채우기

ABAP
1
2
3
4
5
6
" classic
DATA ls_line TYPE ty_line.
ls_line-dan = 2. ls_line-mul = 3. ls_line-result = 6.

" modern — "이 값은 dan 2, mul 3, result 6이다"로 한 덩어리로 읽힘
DATA(ls_line) = VALUE ty_line( dan = 2 mul = 3 result = 6 ).

내부 테이블 채우기 — 각 ( )가 한 행

ABAP
1
2
3
4
DATA(lt_gugu) = VALUE ty_line_tab(
  ( dan = 2 mul = 1 result = 2 )
  ( dan = 2 mul = 2 result = 4 )
  ( dan = 2 mul = 3 result = 6 ) ).

기존에 더하기 — BASE

기존 테이블을 유지하면서 행을 더하려면 BASE를 쓴다. #는 좌변 lt_gugu가 타입을 알려 줘 가능하다.

ABAP
1
lt_gugu = VALUE #( BASE lt_gugu ( dan = 3 mul = 1 result = 3 ) ).

⚠️ BASE를 빼고 lt_gugu = VALUE #( ( … ) )를 쓰면 기존 내용이 유지되는 게 아니라 새 값으로 대체된다. VALUE는 "새 값을 만든다"는 성격이 강하다.

FOR로 반복 생성

행을 일일이 적는 대신 반복으로 한꺼번에 만든다. 숫자 범위로:

ABAP
1
2
3
DATA(lt_gugu_2) = VALUE ty_line_tab(
  FOR i = 1 WHILE i <= 9
  ( dan = 2 mul = i result = 2 * i ) ).   " 9행

이미 있는 테이블을 돌며 변환할 수도 있다:

ABAP
1
2
3
DATA(lt_double) = VALUE ty_line_tab(
  FOR line IN lt_gugu_2
  ( dan = line-dan mul = line-mul result = line-result * 2 ) ).
  • FOR i = 1 WHILE i <= 9·FOR line IN lt_gugu_2 — 그 회차에 만들 새 행 하나를 뒤의 ( )로.
  • 원본은 그대로 두고 새 테이블을 만든다. i·line은 그 표현식 안에서만 산다.

직접 만들어 보기 — VALUE Builder

테이블을 VALUE로 만들고, BASE로 더하거나 BASE 없이 대입했을 때 기존 행이 어떻게 되는지, FOR로 9행을 한 번에 만드는지 확인하자. unique key 모드에서 중복 행을 넣으면 어떤 오류가 나는지도 본다.

흔한 실수와 점검

  • 타입 생략 VALUE #( ) → 좌변 타입이 분명할 때만 #. 처음엔 VALUE ty_line( … )처럼 타입을 명시.
  • BASE 빠뜨림 → VALUE만 쓰면 기존 내용이 사라진다. 누적은 BASE 기존테이블.
  • FOR 변수의 수명 → FOR line IN …의 line은 그 식 안에서만 산다(밖에선 못 쓴다).
  • 중복 key → sorted/hashed·unique key 테이블에 VALUE로 중복 행을 넣으면 런타임 오류다. 테이블 타입의 key 규칙은 VALUE에서도 그대로 적용된다.

💡 VALUE #( FOR … )가 읽기 어려울 만큼 길어지면 다시 LOOP가 낫다. Modern ABAP은 한 줄로 압축하는 대회가 아니다 — 조건이 많고 중간 검증이 필요하면 절차형이 더 안전하고 교육적이다.

정리

  • VALUE는 구조체·Internal Table을 결과 모양 그대로 만든다 — 한 건은 VALUE ty_line( … ), 여러 건은 VALUE ty_line_tab( ( … ) ( … ) ).
  • 기존을 유지하려면 BASE, 반복 생성엔 FOR. #의 타입 추론과 BASE 누락이 가장 자주 틀리는 지점이다.

값을 한 식으로 채웠다. 이제 구조가 다른 둘 사이를 이름으로 매핑하는 CORRESPONDING → CORRESPONDING과 구조 매핑.
