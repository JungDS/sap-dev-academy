DDIC 관계와 입력도움말(F4)

Elementary Search Help

F4 목록을 설계한다 — 단일 소스 Elementary Search Help.

Track 1Chapter 9Lesson 4초급

F4를 제대로 설계하기

Foreign Key의 F4는 "Check Table 그대로"다. 그런데 검색 화면(어떤 컬럼을 보여줄지, 무엇으로 검색할지)을 직접 설계하고 싶다. 그게 (검색 도움말)다. 가장 기본은 Elementary(단일 소스) 검색 도움말.

Elementary Search Help 구성

SE11 → → ZSH_PERSON → Create (Elementary).

핵심 요소:

  • Selection Method — 값을 가져올 테이블/뷰.
  • Parameters — F4 팝업에 보일 필드들. 각 파라미터의 역할:
    • IMP(Import) — 검색 조건으로 입력받는 값.
    • EXP(Export) — 선택 시 화면 필드로 돌려줄 값.
    • LPos / SPos — 목록(List)·검색(Selection) 화면에서의 위치.

어디에 붙이나

검색 도움말은 ****나 테이블 필드에 붙인다(Attach). 그러면 그 타입을 쓰는 모든 화면·PARAMETERS에서 F4가 이 도움말로 동작한다.

CH03에서 "Data Element에 라벨·도움말이 붙는다"고 했다. 그 '도움말'의 정체가 바로 이 다 — 나선형으로 다시 만났다.

흔한 실수와 점검

  • EXP 미지정 → 골랐는데 화면에 값이 안 돌아온다. 돌려줄 파라미터를 EXP로.
  • Selection Method 범위 → 큰 테이블이면 적절한 뷰·검색조건으로 제한.

소스가 하나면 Elementary로 충분하다. 그런데 "사번으로도, 이름으로도, 부서로도" 여러 방식으로 찾고 싶다면? → Collective (다음 레슨).
