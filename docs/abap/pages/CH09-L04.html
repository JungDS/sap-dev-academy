DDIC 관계와 입력도움말(F4)

Collective Search Help 기초

여러 Elementary를 묶어 탭으로 — Collective Search Help.

Track 1Chapter 9Lesson 4초급

여러 방식으로 찾고 싶다

사람을 "사번으로", "이름으로", "부서로" 각각 찾고 싶다. 검색 방식마다 Elementary 검색 도움말을 만들고, 이들을 하나로 묶는 게 다.

구성

SE11 → Search Help → ZSH_PERSON_COLL → Create (Collective).

  • 여러 **Elementary **를 포함시킨다.
  • F4 팝업에 각 Elementary가 탭으로 나타나, 사용자가 검색 방식을 고른다.
  • 포함된 Elementary들의 파라미터를 Collective의 파라미터와 대응시킨다.

ZSH_PERSON_COLL (Collective)
 ├─ ZSH_BY_ID    (사번으로)
 ├─ ZSH_BY_NAME  (이름으로)
 └─ ZSH_BY_DEPT  (부서로)

언제 무엇을

  • 검색 소스/방식이 하나 → Elementary.
  • 여러 방식을 한 F4에서 → Collective(탭으로 묶음).

흔한 실수와 점검

  • 파라미터 대응 누락 → 탭은 떠도 값이 안 돌아온다. Collective↔Elementary 파라미터 매핑 확인.
  • 과설계 → 방식이 하나뿐이면 Collective는 과하다. Elementary로.

도움말을 설계했다. 이제 이게 PARAMETERS 화면에서 어떻게 연결되는지 정리하자 → 다음 레슨.
