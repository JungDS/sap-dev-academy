SELECT-OPTIONS와 Range Table

EQ / BT / CP 옵션 이해

비교 방식 OPTION — EQ(같음)·BT(범위)·CP(패턴)를 SIGN과 함께 읽기.

Track 1Chapter 12Lesson 5중급

OPTION — 어떻게 비교할까

Multiple Selection과 Include/Exclude에서 [[SIGN]]이 포함/제외를 정한다고 배웠다. [[OPTION]]은 그 옆에서 어떤 방식으로 비교할지를 정한다. 을 잘못 고르면 결과가 완전히 달라진다.

"C001만"은 EQ, "C001부터 C003까지"는 BT, "C로 시작하는 모든 공연"은 CP다. 이 차이를 모르고 전부 EQ로만 쓰면 다중 조건의 장점을 거의 못 쓴다.

자주 쓰는 OPTION은 이렇다.

OPTION 뜻 LOW HIGH 예
EQ 같음 사용 비움 I EQ C001
NE 다름 사용 비움 I NE C001
BT 사이(between) 사용 사용 I BT C001 C003
NB 사이 밖 사용 사용 I NB C001 C003
GT / GE 초과 / 이상 사용 비움 I GE 10
LT / LE 미만 / 이하 사용 비움 I LT 10
CP 패턴에 맞음 사용 비움 I CP C*
NP 패턴에 안 맞음 사용 비움 I NP C*

이 챕터에서는 특히 EQ·BT·CP 를 확실히 잡는다.

SIGN과 OPTION은 함께 읽는다

조건 하나를 읽을 때는 SIGN과 OPTION을 같이 본다.

  • I EQ C001 — C001을 포함.
  • E EQ C001 — C001을 제외.
  • I CP C* — C로 시작하는 값을 포함.
  • E CP C* — C로 시작하는 값을 제외.

CP — 패턴 비교 (*와 +)

CP는 Conforms to Pattern(패턴에 부합)의 약자다. 선택화면 도움말에는 "Contains Pattern"으로 표시되기도 해 "포함 문자열 검색"처럼 느끼기 쉽지만, 실제로는 패턴 전체에 부합하는지를 판단한다. 와일드카드로 원하는 모양을 그린다.

패턴 의미
C* C로 시작
*01 01로 끝
C+01 C 다음 한 글자, 그 뒤 01 — 길이 4 고정 (예: C001·CA01은 맞고, C0010은 길이가 달라 안 맞음)

*는 여러 글자, +는 정확히 한 글자에 대응한다. SQL LIKE의 %·_와는 기호가 다르다. Range Table의 CP에는 *·+를 쓴다고 기억하자.

직접 을 바꿔 가며 같은 후보가 어떻게 다르게 걸리는지 실험해 보자.

흔한 실수와 점검

  • CP를 일반 포함 검색처럼 쓴다 → 와일드카드(*·+) 없이 CP C001을 쓰면 정확 매칭과 헷갈린다. 정확히 같은 값이면 EQ를 쓴다.
  • BT에 HIGH를 비운다 → 사이값 비교라 끝 값이 꼭 필요하다. (LOW/HIGH 순서도 주의)
  • 문자 코드에 BT를 숫자처럼 생각한다 → 문자 필드의 BT는 문자 정렬 순서를 따른다. C001~`C999`가 숫자 범위처럼 보여도, 코드 체계에 따라 예상과 다르게 걸릴 수 있다.
  • SQL LIKE의 %·_를 섞는다 → selection option의 CP에는 *·+를 쓴다.

정리

  • [[OPTION]]은 비교 방식을 정한다.
  • EQ는 정확히 같음, BT는 범위, CP는 와일드카드 패턴 비교다.
  • [[SIGN]]과 OPTION은 항상 함께 읽는다.
  • CP의 와일드카드는 *(여러 글자)와 +(한 글자)다.
