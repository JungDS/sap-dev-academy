DDIC 관계와 입력도움말(F4)

Foreign Key와 Check Table

아무 값이나 막는다 — Foreign Key로 입력을 다른 테이블 값으로 제한.

Track 1Chapter 8Lesson 1초급

아무 값이나 들어온다

CH06의 테이블에 country = 'XX'처럼 존재하지 않는 값도 그냥 저장된다. 입력을 올바른 값으로 제한하고 싶다. DDIC가 선언적으로 해주는 첫 장치가 (외래 키)와 이다.

Foreign Key = 다른 테이블을 참조

어떤 필드의 값이 다른 테이블에 실제로 존재하는 값이어야 한다고 DDIC에 선언한다. 그 "다른 테이블"이 이다.

예: ZTPERSON-COUNTRY가 국가 마스터(ZTCOUNTRY)에 있는 코드만 받도록.

SE11에서 설정

SE11 → 테이블 → 필드 선택 → Foreign Keys 버튼.

  • Check Table 지정(참조할 테이블).
  • 키 필드 대응(이 필드 ↔ Check Table의 키)을 매핑.
  • 활성화.

그러면 두 가지가 공짜로 생긴다:

  • 입력검증 — Check Table에 없는 값은 입력 화면/유지보수에서 거부.
  • 도움말 — 그 필드에서 F4 시 Check Table의 값 목록.

흔한 실수와 점검

  • 검증이 안 먹어요 → Foreign Key는 화면/유지보수 레벨 검증이다. INSERT 같은 직접 DB 쓰기는 자동 검증되지 않는다(프로그램이 책임 — CH08-L06).
  • 키 대응 누락 → 매핑이 정확해야 검증이 동작.

Foreign Key는 "이 테이블을 보라"고 가리킨다. 그런데 에도 비슷한 "Value Table"이 있다. 둘이 뭐가 다를까? → 다음 레슨.
