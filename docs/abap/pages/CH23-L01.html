RAP 아키텍처 개요

현대 SAP 트랜잭션 앱의 표준 — RAP의 큰 그림.

읽기를 넘어 — 쓰기·검증·서비스까지

CH22의 CDS는 데이터를 읽기/모델링했다. 그런데 실무 앱은 생성·수정·삭제·검증과 Fiori 화면 연결까지 필요하다. 이 전부를 현대적으로 묶는 표준이 (RESTful Application Programming Model)다.

managed vs unmanaged

시나리오 누가 저장하나
managed 런타임이 자동으로 DB 저장 처리
unmanaged 기존 로직을 직접 연결(레거시 통합)

신규 개발은 보통 managed로 시작한다(코드가 적다).

흔한 실수와 점검

  • =CDS로 오해 → CDS는 데이터 모델, RAP은 그 위 트랜잭션·서비스까지.
  • 계층 건너뛰기 → 보통 ZI_(root)→ZC_→BDEF→Service 순으로 쌓는다.
