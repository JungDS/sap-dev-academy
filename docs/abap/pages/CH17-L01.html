화면 안에 표를 박고 싶다

Chapter 11 · SALV 1차 (간단 ALV)의 는 전체 화면을 차지하는 간단한 표였다. 그런데 Custom Control과 Container · Tabstrip · Subscreen에서 만든 내 화면 안의 한 영역에 표를 박고, 버튼·입력칸과 함께 두고 싶다. 그러려면 더 세밀한 Grid ALV()와, 그것이 살 컨테이너가 필요하다.

🔎 [선행 사용] 이 챕터는 객체(TYPE REF TO·CREATE OBJECT)와 메서드 호출을 많이 쓴다. 두 낱말만 감으로 잡아 두자. TYPE REF TO 는 객체를 직접 담는 상자가 아니라 만들어진 객체를 가리키는 리모컨 같은 참조 변수를 만드는 것이고, CREATE OBJECT 는 표준 클래스(설계도)로 객체를 하나 찍어내 그 리모컨에 물리는 것이다. 지금은 "표준 클래스로 객체를 만들어(CREATE OBJECT) 메서드를 부른다"는 흐름만 따라가면 된다(분석·암기 불필요). 클래스가 무엇이고 어떻게 설계하는지는 Global Class 생성과 객체에서 배운다.

전체 그림 — 조각을 쌓는 순서

Grid ALV는 부품이 많다. 레슨마다 조각 하나씩이니, 먼저 전체가 어떤 순서로 쌓이는지 한눈에 보자. 비유하면 방을 잡고(컨테이너) → 방에 표를 놓고(그리드) → 자료·컬럼 설명서· 보기 설정을 얹은 뒤 → "그려라"라고 말하는 흐름이다.

순서 조각 역할(비유)
1 CL_GUI_CUSTOM_CONTAINER 표가 살 방 (지금)
2 CL_GUI_ALV_GRID 방에 놓는 표
3 출력용 Internal Table 표에 채울 자료
4 Field Catalog 컬럼 설명서(제목·너비)
5 Layout 표 보기 설정(줄무늬 등)
6 Variant 내 보기 저장
7 set_table_for_first_display 첫 표시("이제 그려라")
8 refresh_table_display 자료 바뀌면 다시 그리기
9 행 색상 중요한 행 강조

지금 레슨은 맨 위 방을 잡는 단계다. 각 조각이 왜 필요한지는 그 레슨에서 불편과 함께 만난다.

세 가지 이름을 구분하자

이 레슨에서 헷갈리기 쉬운 이름이 세 개다. 미리 갈라 두자.

이름 어디에 있나 의미
CONT100 Screen Painter 화면에 그려 둔 Custom Control 영역의 이름
go_cont ABAP 전역 변수 CONT100 영역을 붙잡는 object reference
CL_GUI_CUSTOM_CONTAINER SAP 표준 클래스 화면 영역과 Control Framework를 잇는 클래스

💡 여기서 컨테이너는 데이터를 담는 내부 테이블이 아니라, 화면 컨트롤을 담는 프레젠테이션 영역이다. 이름이 비슷해 헷갈리기 쉬우니 주의.

Custom Container 만들기

Custom Control과 Container · Tabstrip · Subscreen에서 화면에 둔 Custom Control 영역(CONT100)에 객체를 연결한다.

ABAP
1
2
3
4
DATA go_cont TYPE REF TO cl_gui_custom_container.

CREATE OBJECT go_cont
  EXPORTING container_name = 'CONT100'.
  • container_name은 ABAP 변수명이 아니라 Screen Painter Custom Control의 이름이다. 실제 화면 요소 이름과 정확히 같아야 붙는다.
  • 이 go_cont 위에 다음 레슨에서 를 얹는다.

직접 연결해 보기 — 컨테이너 바인딩 스텝퍼

이름을 맞추거나 틀리게 두고, Screen Painter 확인 → PBO 실행 → CREATE OBJECT를 한 단계씩 진행해 보자. go_cont가 initial에서 bound로 바뀌는지, 이름이 틀리면 왜 못 붙는지 확인한다.

흔한 실수와 점검

  • 이름 불일치 → 화면에서 CC_ALV로 만들고 코드에 CONT100을 넘기면 연결되지 않는다. 화면 Custom Control 이름과 container_name이 같아야 한다.
  • PBO마다 생성 → 컨테이너·그리드는 화면 컨트롤이라, 매 PBO마다 새로 만들면 중복 생성· 깜박임·Control Framework 오류가 난다. IF go_cont IS INITIAL로 한 번만.

정리

  • 이 레슨은 "화면에 빈 영역을 만들었다"에서 "ABAP 코드가 그 영역을 붙잡았다" 로 넘어가는 단계다.
  • 아직 표는 없지만, 이 단계가 없으면 다음 단계의 Grid ALV가 들어갈 자리가 없다.

표가 살 자리(컨테이너)를 만들었다. 이제 그 위에 올릴 ALV 그리드 객체를 만들자 → CL_GUI_ALV_GRID 생성.
