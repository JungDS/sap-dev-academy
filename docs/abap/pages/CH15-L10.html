PARAMETERS · SELECT-OPTIONS 옵션 총정리

옵션을 한자리에 모아 보기

OBLIGATORY·DEFAULT·AS CHECKBOX 같은 옵션을 그동안 Chapter 3 · Data Element — 의미·라벨 입히기, Chapter 12 · SELECT-OPTIONS와 Range Table, Selection Screen UI 구성 곳곳에서 만났다. 이 레슨은 와 에 붙이는 옵션을 한눈에 정리한다. 필요할 때 찾아보는 카탈로그다.

핵심 마음가짐: 입력 필수·기본값·대소문자·복수 선택·화면 제어를 event 코드로 억지 구현하기 전에, 옵션으로 해결되는지 먼저 본다.

PARAMETERS 옵션

ABAP

PARAMETERS pa_conc TYPE zconcert-concert_id OBLIGATORY.  " 필수 입력
PARAMETERS pa_note TYPE c LENGTH 20 LOWER CASE MEMORY ID znt.
PARAMETERS pa_test AS CHECKBOX.
옵션 뜻 주의
TYPE / LIKE 타입 지정 / 기존 필드와 같게 DDIC 필드를 참조하면 라벨·F1 연결이 쉬움
OBLIGATORY 필수 입력(비우면 진행 불가) 필수 여부 ≠ 업무 유효성
DEFAULT <값> 시작값 제안 MEMORY ID보다 우선될 수 있음
LOWER CASE 소문자 보존(기본은 대문자 변환) character-like 필드에만 의미 있음
AS CHECKBOX / RADIOBUTTON GROUP g 체크박스 / 라디오 길이 1 character 성격
MEMORY ID <pid> Parameter ID와 연결 — 마지막 값 기억 이전 사용자 값이 들어올 수 있음
VALUE CHECK 고정값/check table 기반 검사 입력 직후 자동 검증
MODIF ID <grp> 화면 제어용 그룹 이름 SCREEN-GROUP1에 저장(아래)

⚠️ LOWER CASE는 문자형 필드 전용이다(공식문서: character-like 필드의 대문자 변환을 막음). TYPE p(숫자) 같은 필드에 붙이는 옵션이 아니다.

DATA gv_date TYPE d.

SELECT-OPTIONS s_date FOR gv_date
  DEFAULT sy-datum TO sy-datum   " 첫 조건 행의 시작/끝 기본값
  OBLIGATORY                     " 조건 최소 한 줄 필수
  NO-EXTENSION                   " 다중 선택(여러 줄) 버튼 숨김
  NO INTERVALS.                  " From–To 중 To 칸 숨김
옵션 뜻 주의
DEFAULT <low> [TO <high>] 첫 조건 행의 시작값/범위 내부값(internal) 기준
OBLIGATORY 조건 입력 필수 복수 선택 UX와 함께 확인
LOWER CASE 소문자 보존 문자형 한정
NO-EXTENSION 다중 선택(여러 줄) 버튼 숨김 — 복수 조건 막음
NO INTERVALS To 칸 숨김(단일값만) 단독 사용 시 다중선택 dialog에선 interval이 여전히 가능할 수 있음
MEMORY ID / MODIF ID 값 기억 / 화면 제어 그룹
(SIGN·OPTION) 부호·비교연산자 — Chapter 12 · SELECT-OPTIONS와 Range Table

⚠️ NO-EXTENSION(복수 선택 막기)과 NO INTERVALS(To 칸 숨기기)는 다른 옵션이다. 헷갈리지 말 것 — high 필드를 숨기는 것과 복수 조건을 막는 것은 별개다.

MODIF ID — 한 그룹을 한꺼번에 제어

같은 MODIF ID를 준 항목들을, 화면이 뜨기 직전 묶어서 보이기/숨기기·잠그기 할 수 있다.

PARAMETERS: pa_basic AS CHECKBOX,
            pa_adv1  TYPE c LENGTH 10 MODIF ID adv,
            pa_adv2  TYPE c LENGTH 10 MODIF ID adv.

DATA gs_screen TYPE screen.

AT SELECTION-SCREEN OUTPUT.                  " 화면 뜨기 직전
  LOOP AT SCREEN INTO gs_screen.
    IF gs_screen-group1 = 'ADV' AND pa_basic = 'X'.  " MODIF ID adv → GROUP1='ADV'
      gs_screen-active = '0'.                " 0=숨김 (1=표시)
      MODIFY SCREEN FROM gs_screen.          " 반드시 반영
    ENDIF.
  ENDLOOP.
  • MODIF ID adv는 화면 runtime에서 SCREEN-GROUP1에 ADV(대문자) 로 들어간다.
  • 의 LOOP AT SCREEN INTO gs_screen에서 group1로 묶어 active/input을 바꾸고 MODIFY SCREEN FROM gs_screen으로 반영한다.

직접 켜고 꺼 보기 — 옵션 스위치보드

같은 입력 항목에 옵션을 하나씩 켜고 끄며, 화면 모양·내부 값·복수 선택 가능 여부가 어떻게 달라지는지 보자. 특히 NO INTERVALS만 켰을 때 다중 선택 창에서의 동작도 확인한다.

흔한 실수와 점검

  • OBLIGATORY로 검증 끝이라 착각 → 필수 입력과 값의 업무적 유효성은 다르다.
  • LOWER CASE를 아무 타입에나 → 문자형 필드에만 의미가 있다.
  • MEMORY ID 충돌 → 같은 Parameter ID를 엉뚱한 필드에 쓰면 값이 섞인다. 고유하게.
  • MODIF ID 제어가 안 먹음 → AT SELECTION-SCREEN OUTPUT에서 MODIFY SCREEN FROM gs_screen까지 해야 반영된다.

정리

  • 선택화면 옵션은 작은 문법처럼 보이지만 사용자 경험을 크게 바꾼다.
  • 입력 필수·기본값·대소문자·복수 선택·화면 제어 group을 옵션으로 먼저 해결하고, 그래도 안 되면 event 코드로 간다.

옵션까지 다 갖췄다. 이제 화면을 여러 개 두고 골라 부르는 법으로 → 여러 선택화면 — 화면번호·CALL·Variant.
