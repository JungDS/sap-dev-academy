SALV/Grid ALV 표시 제어 심화

실습 — 매진 회차 색 강조

콘서트앱 — 잔여석 계산을 셀 색으로(매진 빨강·임박 노랑).

Track 1Chapter 21Lesson 8고급

🎫 상태를 색으로 — 세 단계가 이어지기

회차 현황을 ALV로 띄우되, 매진은 빨강·잔여 적으면 노랑으로 강조한다. 완성 기준은 "색이 예쁘다"가 아니라 세 단계가 이어지는 것이다: 업무 계산(remaining) → 표시용 필드(seats_left) → deep component(cellcolors) → layout(ctab_fname) 연결.

TYPES: BEGIN OF ty_row,
         concert_id TYPE zconcert-concert_id,
         perf_no    TYPE zbooking-perf_no,
         capacity   TYPE i,
         seats_left TYPE i,
         cellcolors TYPE lvc_t_scol,        " ALV 표시용 deep(DB 저장 아님)
       END OF ty_row.

LOOP AT lt_row ASSIGNING FIELD-SYMBOL(<r>).
  DATA(lo_mgr) = NEW zcl_booking_manager(
    iv_concert = <r>-concert_id iv_perf = <r>-perf_no ).
  <r>-seats_left = lo_mgr->remaining( ).             " 업무 계산(이전 챕터 클래스)
  CLEAR <r>-cellcolors.
  IF <r>-seats_left <= 0.
    APPEND VALUE lvc_s_scol( fname = 'SEATS_LEFT'
      color-col = col_negative color-int = abap_true ) TO <r>-cellcolors.   " 빨강
  ELSEIF <r>-seats_left <= 5.
    APPEND VALUE lvc_s_scol( fname = 'SEATS_LEFT'
      color-col = col_total color-int = abap_true ) TO <r>-cellcolors.      " 노랑
  ENDIF.
ENDLOOP.

DATA ls_layout TYPE lvc_s_layo.
ls_layout-ctab_fname = 'CELLCOLORS'.                 " 셀색 컬럼 연결(이전 레슨)

표시할 때 layout과 field catalog를 함께 넘기고, CELLCOLORS는 화면에 보일 컬럼이 아니라 기술용으로 숨긴다.

LOOP AT lt_fcat ASSIGNING FIELD-SYMBOL(<fc>).
  IF <fc>-fieldname = 'CELLCOLORS'. <fc>-tech = abap_true. ENDIF.   " 화면에서 숨김

go_grid->set_table_for_first_display(
  EXPORTING  is_layout       = ls_layout
  CHANGING   it_outtab       = lt_row
             it_fieldcatalog = lt_fcat ).

직접 해 볼 과제

  1. 회차별 잔여석을 ZCL_BOOKING_MANAGER의 remaining( )으로 계산해 seats_left에 채운다.
  2. 잔여석 0·3·12인 테스트 회차를 두고, 매진(0)→빨강·임박(≤5)→노랑이 입혀지는지 확인.
  3. 예매 1건을 더해 잔여석 6 → 5가 되면 그 셀이 노랑으로 바뀌는지(refresh) 본다.
  4. (도전) 더블클릭 시 그 회차 예매 목록 열기 — 본격 는 이후 ALV 이벤트 챕터.

흔한 실수와 점검 🚧

  • fname ≠ 필드명 → 색 줄의 fname은 output table component 이름(SEATS_LEFT)과 정확히 같아야. 틀리면 그 셀 색이 안 먹는다.
  • ctab_fname 미지정 → 색 컬럼을 둬도 layout에 안 알리면 무시된다(셀 색 연결).
  • 루프마다 매니저 생성 → 학습용으론 명확하지만 대량 데이터엔 비효율(이후 성능 장에서 batch/로).
  • 색 = 검증 착각 → 색은 표시 결과다. 매진 여부의 진짜 검증은 ZCL_BOOKING_MANAGER 서버 로직에서.

상태를 색으로 보여 줬다. 이 챕터는 "내부 테이블을 ALV로 잘 보여 주는 기술"이었다 — 이제 데이터 모델 자체를 재사용 가능한 뷰로 정의하는 CDS로 → CDS View Entity 기본 구조.
