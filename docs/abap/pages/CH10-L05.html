전역에 있는 도구 불러 쓰기

은 그 프로그램 안에서만 살았다. 처럼 전역에서 공유되는 OO 버전이 (전역 클래스, SE24의 ZCL_...)다. SAP 표준에도, 동료가 만든 것에도 유용한 전역 클래스가 많다.

정적 메서드를 =>로 호출 — [선행 사용]

🔎 [선행 사용] 클래스를 직접 설계하는 법은 Chapter 20 · OO 기초에서 배운다. 여기서는 이미 만들어진 클래스의 메서드를 블랙박스처럼 불러 쓰기만 한다(분석·암기 불필요).

객체를 만들지 않아도 되는 정적 메서드는 "클래스=>메서드"로 부른다.

DATA lv_text TYPE string.

lv_text = zcl_util=>format_amount( 1000 ).   " 전역 클래스의 정적 메서드
WRITE lv_text.

DATA lv_r TYPE i.
zcl_util=>calc( EXPORTING iv_a = 1 iv_b = 2 IMPORTING ev_r = lv_r ).

표준 클래스도 많다

CL_로 시작하는 SAP 표준 가 풍부하다(예: ALV의 CL_SALV_TABLE, Chapter 11). 곧 ALV에서 이런 표준 클래스를 정적 팩토리 메서드로 불러 쓰게 된다.

인스턴스가 필요한 경우 (개념)

상태를 가진 객체가 필요한 메서드는 객체를 만들어 ->로 부른다. 객체 생성·참조 변수는 Chapter 20 · OO 기초에서 정식으로 배운다. 지금은 "정적이면 =>로 바로 부른다"만 기억한다.

직접 해 보기

이미 만들어진 전역 클래스를 내부 구현을 모른 채 계약(IMPORTING/RETURNING)만 보고 호출해 보자. 필수 를 빼면 어떻게 되는지도 확인해 보자.

흔한 실수와 점검

  • 정적인데 객체부터 → 정적 메서드는 =>로 바로. 객체 불필요.
  • => vs -> → 정적은 =>(클래스로 직접), 인스턴스는 ->(객체로, Chapter 20).
