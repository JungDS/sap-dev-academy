Incomplete 타입(C·N·P)

길이(와 소수 자릿수)를 함께 지정해야 하는 타입.

"길이를 정해야" 완성되는 타입

어떤 타입은 길이를 함께 지정해야 비로소 완전해진다. 이를 이라 한다. 대표는 C·N·P다.

C · N · P

타입 뜻 지정할 것
C 고정 길이 문자(character) 길이
N 숫자형 문자(numeric text) 길이
P 압축 소수(packed) 길이 + 소수 자릿수 
DATA: lv_code TYPE c LENGTH 4,            " 4글자 코드
      lv_zip  TYPE n LENGTH 5,            " 우편번호(숫자 문자)
      lv_amt  TYPE p LENGTH 8 DECIMALS 2. " 금액, 소수 2자리

lv_code = 'ABCD'.
lv_amt  = 7 / 2.    " 3.50 (정확)
WRITE: / lv_code, / lv_amt.
  • C LENGTH n — 딱 n글자. 모자라면 공백으로 채워진다.
  • N LENGTH n — 숫자처럼 보이지만 문자다(앞의 0이 보존됨, 예: 우편번호).
  • P LENGTH n DECIMALS d — 금액·수량 등 정확한 소수 계산용.

왜 N을 쓰나? 우편번호 01234를 정수 I에 넣으면 앞의 0이 사라져 1234가 된다. "숫자처럼 보이지만 계산하지 않는 값"은 N(또는 C)으로 둔다.

흔한 실수와 점검

  • 길이 누락 → DATA lv_x TYPE c. 는 길이 1로 잘린다. LENGTH를 명시.
  • 금액에 F 사용 → 부동소수점은 오차가 난다. 돈은 반드시 P … DECIMALS.
  • N에 알파벳 대입 → N에는 숫자 문자만. 알파벳은 C.

이제 표준 타입은 다 봤다. 그런데 p length 8 decimals 2를 프로그램마다 또 적자니 번거롭다 — "한 번 정의해 재사용하고 싶다." 그게 다 → 다음 레슨.
