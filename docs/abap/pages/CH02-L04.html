변수와 표준 타입 (+ Local Type)

Local Type(TYPES) 재사용

같은 타입 정의를 매번 반복하지 말고, TYPES로 이름 붙여 재사용한다.

Track 1Chapter 2Lesson 4입문

같은 타입을 또 적기 싫다

금액을 다루는 변수가 여럿이면 TYPE p LENGTH 8 DECIMALS 2를 매번 반복하게 된다. 정의가 바뀌면(예: 소수 3자리로) 전부 고쳐야 한다. 한 번 정의해 이름을 붙이고 재사용하고 싶다 — 그게 다.

TYPES로 내 타입 만들기

ABAP
1
2
3
4
TYPES ty_amount TYPE p LENGTH 8 DECIMALS 2.

DATA: lv_price TYPE ty_amount,
      lv_total TYPE ty_amount.

이제 금액 타입은 ty_amount 한 곳에서 관리된다. 정의를 바꾸면 그걸 쓰는 모든 변수가 함께 바뀐다.

  • ty_ 접두어는 타입 관례.
  • 는 타입을 정의할 뿐 메모리를 잡지 않는다(변수는 가 만든다).

여러 타입 한 번에

ABAP
1
2
TYPES: ty_amount TYPE p LENGTH 8 DECIMALS 2,
       ty_code   TYPE c LENGTH 4.

아직은 "로컬"이다

여기서 만든 는 이 프로그램 안에서만 쓰는 로컬 타입이다. 그런데 금액·코드 타입을 다른 프로그램과도 공유하고 싶어진다 — 그러려면 전역(DDIC) 으로 올려야 한다. 그게 다음 챕터의 출발점이다.

흔한 실수와 점검

  • TYPES로 값 대입 시도 → 타입은 틀일 뿐. 값은 그 타입으로 만든 변수에.
  • 타입 이름 충돌 → ty_ 접두어로 변수(lv_)와 구분한다.

로컬 타입으로 반복은 줄였다. 하지만 프로그램마다 또 정의하는 건 여전하다 — "전역으로 공유하고 싶다." → CH03(DDIC Domain·Data Element).
