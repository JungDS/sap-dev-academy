New Open SQL / Modern ABAP SQL

Classic Open SQL과 Modern ABAP SQL 비교

같은 조회를 classic과 modern으로 — 무엇이, 왜 달라졌나.

Track 1Chapter 19Lesson 1중급

SELECT만 아직 옛날식이었다 🕰️

지난 챕터에서 ABAP 코드는 모던으로 바꿨지만, SELECT(Open SQL) 만은 일부러 classic으로 뒀다(공백 구분·@ 없음). 이유가 있다 — SQL의 모던화는 단순한 축약이 아니라, ABAP 프로그램과 데이터베이스 사이의 경계를 다시 긋는 일이기 때문이다. 이제 그 경계를 연다. 이것이 Modern ABAP SQL이다.

📌 ★ 여기부터 modern SQL(@·콤마·SQL 식)을 쓴다. 이전 챕터까지의 SELECT는 전부 classic이었다.

" classic — 지금까지 쓰던 형태
SELECT carrid connid seatsmax
  FROM sflight
  INTO TABLE lt_flight
  WHERE carrid = lv_carr.

" modern — 이 챕터부터
SELECT carrid, connid, seatsmax       " 콤마로 필드 구분
  FROM sflight
  WHERE carrid = @lv_carr             " @ 로 ABAP 변수 표시
  INTO TABLE @lt_flight.              " 대상도 @ · INTO는 맨 뒤

달라진 점은 셋이다.

  • 콤마로 필드를 구분한다(공백 ✕).
  • ABAP 변수·대상 앞에 @(host variable, "이건 ABAP 값"이라는 표시)를 붙인다.
  • INTO를 문장 맨 뒤에 둔다 → SELECT list → FROM → WHERE → INTO 순으로 읽힌다.

어느 이름이 DB 것이고, 어느 게 ABAP 것인가 🧭

modern SQL의 핵심은 멋이 아니라 경계 표시다. DB 테이블의 컬럼은 그냥 쓰고, ABAP 프로그램의 값은 @로 표시(escape)한다.

이름 정체 예
carrid DB 테이블 sflight의 컬럼 SELECT carrid
@lv_carr ABAP 프로그램의 변수 값 WHERE carrid = @lv_carr
@lt_flight ABAP 프로그램의 결과 내부 테이블 INTO TABLE @lt_flight
carrid, connid 콤마로 구분한 SELECT list SELECT carrid, connid

공식 문서 기준으로 공백으로 필드를 나열하는 옛 방식은 obsolete(낡은 것) 쪽으로 밀려 있고, 콤마 리스트와 @는 strict 모드(더 엄격한 문법 검사)를 켠다. strict 모드는 애매한 SQL을 컴파일 단계에서 더 빨리 잡아 준다.

결과는 그대로 — 확인 습관도 그대로 ✅

표기가 모던으로 바뀌어도 읽어 오는 데이터는 똑같다. 실행 뒤 보는 곳도 classic 때와 같다.

  1. 결과 내부 테이블에 몇 행이 들어왔나.
  2. sy-subrc가 0인가 4인가(찾았나 / 못 찾았나).
  3. sy-dbcnt가 실제 전달된 행 수와 같은가.

예를 들어 lv_carr = 'LH'이고 데이터가 있으면 sy-subrc = 0, sy-dbcnt > 0. 없는 코드('ZZ')라면 테이블은 비고 sy-subrc = 4, sy-dbcnt = 0이다. 이 관찰 지점은 이 챕터 내내 그대로 유지된다.

흔한 실수와 점검 🚧

  • 콤마와 공백을 섞음 → SELECT carrid connid, seatsmax처럼 섞으면 오류. 모던으로 갈 거면 일관되게 콤마.
  • DB 컬럼에 @를 붙임 → @는 ABAP 값/대상에만. carrid는 DB 컬럼이라 그냥 쓴다.
  • @ 빠뜨림 → modern SQL에서 ABAP 변수엔 @가 필수. 없으면 오류.
  • INTO를 중간에 둠 → 마지막 clause로 두는 습관을 처음부터. strict 모드에선 강제되는 경우가 있다.
  • 무심한 SELECT * → 필요한 컬럼만 읽는 원칙은 모던에서도 그대로다.

큰 그림을 봤다. 이제 그 @의 정체 — host variable과 한 걸음 더 나아간 host expression을 보자 → @ Host Variable과 Host Expression.
