Deep Structure 개념

구조 안에 내부 테이블/문자열이 든 'Deep' 구조 — 개념 소개.

행 안에 또 목록이 있다면

주문 한 건에 "주문 항목 여러 줄"이 딸린다. 즉 한 행 안에 또 다른 내부 테이블 이 들어가야 한다. 이렇게 구조 안에 테이블이나 문자열 같은 가변 요소가 든 것을 Deep Structure(깊은 구조)라 한다. 여기선 개념만 잡는다(구현은 뒤에서).

Flat vs Deep

  • Flat 구조 — 모든 컴포넌트가 고정 길이 기본형(지금까지의 구조).
  • Deep 구조 — 컴포넌트에 ·STRING·참조 등 가변 요소가 포함.
TYPES: BEGIN OF ty_order,
         id    TYPE i,
         items TYPE ty_person_tab,  " 행 안에 테이블! → Deep
       END OF ty_order.

왜 미리 알아두나

  • ALV 표시나 DB 저장에서 Deep 구조는 Flat과 다르게 취급된다.
  • 지금은 "구조 안에 테이블이 들어갈 수 있다"는 가능성만 기억하면 된다. 실제 활용·제약은 ALV·구조 심화에서 다시 만난다.

흔한 오해와 점검

  • Deep을 투명 테이블 컬럼으로 → 안 된다. 투명 테이블 행은 Flat이어야 한다(→ CH06).
  • 지금 깊이 파지 말 것 → 개념만. 나선형으로 뒤에서 깊어진다.

이제 여러 건을 메모리에서 자유롭게 다룬다. 그런데 프로그램이 끝나면 이 테이블은 사라진다. 값을 영구히 저장하고 싶다 → CH06(Transparent Table).
