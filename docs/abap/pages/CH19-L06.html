메모리 테이블도 SQL로

modern SQL은 DB 테이블뿐 아니라 내부 테이블을 소스로 조회할 수 있다. 이미 메모리에 있는 데이터를 SQL 문법(집계·정렬·필터)으로 가공할 때 편하다.

SELECT carrid, SUM( seatsocc ) AS total_occ
  FROM @lt_flight AS f                 " 내부 테이블이 소스
  GROUP BY carrid
  INTO TABLE @DATA(lt_sum).
  • FROM @lt_flight — 메모리의 내부 테이블을 소스로.
  • DB 왕복 없이, 가져온 데이터를 SQL식으로 재가공.

언제 쓰나

  • 이미 SELECT로 가져온 데이터를 다시 집계/정렬할 때.
  • 외부에서 받은 데이터(파일·인터페이스)를 SQL식으로 다룰 때.

흔한 실수와 점검

  • @ 누락 → 소스 내부 테이블도 @lt로 escape.
  • DB 대용으로 남용 → 대량은 DB에서 직접. itab SELECT는 메모리 데이터 재가공용.
