SQL Monitor / SQLM

운영 전체에서 무거운 SQL을 찾는다 — SQLM.

Track 2Chapter 32Lesson 3고급

운영 전체를 보는 눈

ST05는 한 실행을 본다. SQLM(SQL Monitor)은 운영 시스템 전체에서 실제로 무거운 SQL을 누적 해, "어디를 먼저 고칠지" 우선순위를 준다.

  • SQLM 활성화 → 일정 기간 운영 데이터 수집 → 분석.
  • 총 실행시간·횟수· 수가 큰 SQL부터 개선.
  • SWLT(SQL Performance Tuning Worklist)로 코드 점검과 결합.

Secondary Index 판단

CH08-L06에서 본 보조 를, 여기서 데이터로 판단한다.

  • 자주·느리게 도는 비키 조건 → 인덱스 후보.
  • 단, 인덱스는 쓰기 비용·공간이 들고 S/4HANA에선 권장 제한(엔진이 빠름).

흔한 실수와 점검

  • 로컬 측정만 → 운영의 실제 부하는 SQLM으로.
  • 남발 → 측정 근거 없이 추가 금지.

병목을 찾았다. 가장 흔한 원인 — LOOP 안 SELECT를 없애자 → 다음 레슨.
