입력 조건을 업무 단위로 정리

선택화면도 UI다. 입력칸이 많아질수록 사용자는 "어떤 값이 필수인지, 어떤 조건이 같은 묶음인지"를 알기 어려워진다. 문으로 블록으로 묶고, 체크박스·라디오·버튼을 더하면 조건을 빠르게 이해하고 실수 없이 실행할 수 있다.

이 레슨은 "화면을 꾸미는 문법 목록"이 아니라 입력 조건을 업무 단위로 정리하는 방법이다. 멋지게 만드는 것보다 실행 실수를 줄이는 것이 먼저다.

SELECTION-SCREEN BEGIN OF BLOCK b_main WITH FRAME TITLE TEXT-001.
  PARAMETERS p_conc TYPE zconcert-concert_id.
  SELECTION-SCREEN COMMENT /1(50) TEXT-002 FOR FIELD p_conc.  " 설명 줄
  SELECTION-SCREEN ULINE.                       " 구분선
  SELECTION-SCREEN SKIP.                         " 빈 줄
SELECTION-SCREEN END OF BLOCK b_main.
  • BEGIN/END OF BLOCK ... WITH FRAME TITLE — 테두리+제목으로 한 묶음.
  • COMMENT 설명 텍스트 · ULINE 구분선 · SKIP 빈 줄.
  • ⚠️ COMMENT에는 가능하면 FOR FIELD <필드> 를 붙인다. 그래야 그 설명이 특정 입력칸과 연결되어 F1 도움말 사용성이 살아난다.

여러 요소를 한 줄에 나란히 두려면 BEGIN OF LINE~`END OF LINE으로 묶고 POSITION`으로 칸 위치를 지정한다.

버튼 — 본문 PUSHBUTTON · 툴바 FUNCTION KEY

선택화면에는 두 종류의 버튼을 둘 수 있다. 본문 버튼(화면 안)과 툴바 버튼(위쪽 application toolbar)이다. 둘 다 누르면 그 function code가 의 ucomm으로 전달된다 — AT SELECTION-SCREEN에서 받는다.

TABLES sscrfields.

SELECTION-SCREEN PUSHBUTTON /1(20) but_ref USER-COMMAND ref.  " 본문 버튼
SELECTION-SCREEN FUNCTION KEY 1.                              " 툴바 버튼 FC01

INITIALIZATION.
  but_ref = '조건 새로고침'.            " 본문 버튼 텍스트
  sscrfields-functxt_01 = '도움말'.     " 툴바 버튼 텍스트

AT SELECTION-SCREEN.
  CASE sscrfields-ucomm.                " ← sy-ucomm 아님!
    WHEN 'REF'.
      MESSAGE '조건을 다시 계산했습니다' TYPE 'S'.
    WHEN 'FC01'.
      MESSAGE '이 리포트는 공연 예매 조회용입니다' TYPE 'I'.
  ENDCASE.
  • FUNCTION KEY 1~5 → function code FC01~FC05. 텍스트는 FUNCTXT_0n.
  • ⚠️ 선택화면 버튼은 SY-UCOMM이 아니라 SSCRFIELDS-UCOMM 으로 처리한다(공식 규칙).
  • ⚠️ PUSHBUTTON은 조회 실행을 대체하는 버튼이 아니다. 실제 조회는 Execute 흐름과 분리해 설계한다(버튼은 화면 갱신·도움말 같은 보조 동작용).

탭으로 나누기 — TABBED BLOCK

조건 영역이 여러 개면 탭으로 나눌 수 있다.

  • SELECTION-SCREEN BEGIN OF TABBED BLOCK ... — 여러 화면을 탭으로 묶기.
  • 각 탭의 내용은 Subscreen(선택화면의 일부 영역)으로 채운다. 탭마다 다른 입력 묶음을 끼워 넣는 구조다.
  • ⚠️ subscreen 연결이 필수다 — 연결을 빠뜨리면 빈 탭이 되어 사용자가 혼란스럽고, 잘못 연결하면 runtime 문제가 날 수 있다. (복잡한 tab framework는 실습 심화로 둔다.)

직접 꾸며 보기 — Selection Screen 레이아웃 빌더

아래 미리보기에서 버튼을 눌러 SSCRFIELDS-UCOMM에 어떤 코드(REF·FC01)가 들어오는지 보고, 버튼 처리 방식과 COMMENT 연결을 바꿔 가며 어떤 경고가 뜨는지 확인하자.

흔한 실수와 점검

  • BLOCK 짝 안 맞음 → BEGIN OF BLOCK은 END OF BLOCK으로 닫는다.
  • 라디오 그룹 미지정 → 같은 그룹이라야 배타 선택이 된다.
  • 버튼 처리에 SY-UCOMM 사용 → 선택화면에서는 SSCRFIELDS-UCOMM으로 받는다.
  • 장식 과다 → 업무 단위 grouping이 먼저, 꾸미기는 그 다음. 한꺼번에 다 쓰면 화면만 복잡해진다.

정리

  • 선택화면 UI 구성은 조건을 업무 언어로 정리하는 일이다.
  • 블록으로 묶고, COMMENT(가능하면 FOR FIELD)로 보충하고, checkbox/radio로 선택 방식을 명확히 한다.
  • PUSHBUTTON·FUNCTION KEY는 SSCRFIELDS-UCOMM 으로 처리한다.

선택화면을 업무 단위로 구성했다. 이제 입력 항목에 붙는 옵션을 한자리에 정리하자 → PARAMETERS · SELECT-OPTIONS 옵션 총정리.
