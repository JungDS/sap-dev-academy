SALV 객체 만들기 — factory

는 정적 메서드 factory( )로 내부 테이블을 받아 ALV 객체를 만든다. (CH09의 정적 메서드 => 호출이다.)

DATA lo_alv TYPE REF TO cl_salv_table.

TRY.
    cl_salv_table=>factory(
      IMPORTING r_salv_table = lo_alv      " 만들어진 ALV 객체
      CHANGING  t_table      = lt_person ). " 표시할 내부 테이블
  CATCH cx_salv_msg.
    " 생성 실패 처리
ENDTRY.
  • IMPORTING r_salv_table — 결과 ALV 객체를 받는다.
  • CHANGING t_table — 표시할 (참조로 연결).
  • factory는 예외 cx_salv_msg를 던질 수 있어 TRY ... CATCH로 감싼다.

컬럼은 자동

테이블 행이 로 타입팅돼 있으면 컬럼 제목(라벨)이 자동으로 붙는다 — CH03에서 만든 라벨이 또 보상받는 지점.

흔한 실수와 점검

  • TRY/CATCH 없이 factory → 예외 미처리로 덤프 위험.
  • 빈 테이블 → 오류는 아니나 빈 표가 뜬다. 데이터 유무는 미리 확인.

객체는 만들었다. 그런데 아직 화면에 안 떴다 — 툴바 기능을 켜고 표시하자 → 다음 레슨.
