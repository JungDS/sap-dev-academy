Domain 생성 시뮬레이터 (SE11 — 저장 → 검사 → 활성화)

SE11에서 Domain을 만드는 흐름을 그대로 체험합니다. 값을 채우고 💾 저장 → ✓ 검사 → 🔥 활성화 순서로 눌러 보세요. 활성화해야 비로소 런타임에서 쓸 수 있는 객체가 됩니다.

예제:
📚Dictionary: Domain 유지보수 신규 (미저장)
전역 객체는 Z 또는 Y로 시작합니다.
소수점 이하 자릿수. 길이보다 작아야 합니다(DEC 전용).

정해진 값만 받게 하려면 고정값(Fixed Values)을 넣습니다. (예: O=Open, C=Closed) — 이 목록은 나중에 화면 F4·입력 검증으로 되돌아옵니다.

고정값설명
ℹ️ 준비됨
값을 채운 뒤 💾 저장 → ✓ 검사 → 🔥 활성화 순으로 눌러 보세요.
💡 DDIC 객체의 철칙 — 저장만 하면 비활성(inactive) 상태라 런타임에서 쓸 수 없습니다. 활성화해야 런타임 객체가 생성됩니다. 활성 후 필드를 고치면 다시 비활성이 되어 재활성화가 필요합니다.
