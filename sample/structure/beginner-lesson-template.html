내부 테이블(Internal Table)이 뭐예요?

SAP/ABAP을 오늘 처음 보셔도 괜찮아요. 어려운 말은 그때그때 풀어드릴게요. 이 페이지를 끝까지 보면, "여러 줄의 데이터를 다루는 가장 기본 도구"를 손으로 직접 만들어 보게 됩니다.

📖 읽는 법: 점선 밑줄이 그어진 단어는 마우스를 올리면 뜻이 다시 나와요. "▸ 더 자세히"는 눌러서 펼쳐 보세요.

① 왜

먼저, 왜 이게 필요할까요?

회사에 사원이 5명 있어요: 정훈영, 손흥민, 유재석, 김연아, 아이유. 이 다섯 명의 급여를 화면에 한 번에 보여주고 싶어요.

그런데 변수(값 하나만 담는 상자) 하나에는 한 명분의 급여밖에 못 담아요. 그럼 다섯 명이면 상자 5개, 100명이면 상자 100개를 일일이 만들어야 할까요? 😵

✗ 변수만 쓰면

lv_sal1 = 5000 lv_sal2 = 3200 lv_sal3 = 4100 … 100명이면 100개 😱

✓ 표(내부 테이블)를 쓰면

이름급여
정훈영5000
손흥민3200
……

한 개의 "표"에 줄줄이 담으면 끝!

이 불편을 해결하는 도구가 바로 내부 테이블이에요. "왜 배우는지" 먼저 느꼈으니, 이제 정체를 알아볼까요?

② 무엇

내부 테이블은 무엇인가요?

내부 테이블(Internal Table)(프로그램이 실행되는 동안 메모리에 잠깐 두는 "표")은, 같은 모양의 행(가로 한 줄)을 여러 개 담아두는 그릇이에요.

엑셀 표나 출석부를 떠올리면 쉬워요. 열(칼럼)은 "이름, 급여"처럼 항목이고, 행은 사원 한 명 한 명이에요. 다만 엑셀 파일처럼 저장되는 게 아니라, 프로그램이 도는 동안에만 메모리에 잠깐 존재했다 사라진다는 점이 달라요.

🧺
한 문장 비유

내부 테이블 = "사원 명단을 적는 빈 표 한 장". 줄을 추가하면 명단이 늘고, 한 줄씩 읽으면 한 명씩 호명하는 셈이에요.

더 자세히 — "같은 모양"이라는 게 무슨 뜻이죠?

표의 모든 행이 똑같은 칸 구성을 가진다는 뜻이에요. 1번 행이 (이름, 급여)면 2번 행도 (이름, 급여)예요. 이 "칸 구성"을 ABAP에서는 구조라고 부릅니다.

그래서 내부 테이블을 만들 땐 "어떤 모양의 행을 담을 표인지"를 먼저 정해줘요. (아래 ③에서 직접 해봅니다.)

③ 어떻게

그럼 어떻게 쓰나요? (3단계)

딱 세 동작만 기억하면 돼요: ① 표를 선언하고 → ② 줄을 추가하고(APPEND) → ③ 한 줄씩 읽는다(LOOP). 아래 코드를 보고, 바로 밑에서 직접 눌러볼게요.

ZDEMO_TABLE.abap
* ① 사원 명단을 담을 빈 표를 선언
DATA lt_emp TYPE TABLE OF ty_emp.   " ty_emp = (이름, 급여) 모양

* ② 표에 한 줄씩 추가(APPEND = 맨 끝에 붙이기)
APPEND VALUE #( name = '정훈영' sal = 5000 ) TO lt_emp.

* ③ 표를 한 줄씩 읽기(LOOP = 반복해서 한 행씩)
LOOP AT lt_emp INTO DATA(ls_emp).
  WRITE: / ls_emp-name, ls_emp-sal.
ENDLOOP.

글로만 보면 와닿지 않죠. 아래 버튼으로 표를 직접 만들어 보세요. 줄이 어떻게 쌓이고, LOOP가 어떻게 한 줄씩 훑는지 눈으로 확인할 수 있어요.

#이름 (name)급여 (sal)
아직 빈 표예요. "+ 줄 추가"를 눌러 사원을 한 명씩 넣어 보세요.
여기에 실행 로그가 나와요.
더 자세히 — APPEND 말고 다른 추가 방법도 있나요?

APPEND는 "맨 끝에 붙이기"예요. 가장 많이 씁니다. 이 외에 INSERT(원하는 자리에 끼워넣기), 정렬된 표에서의 자동 정렬 추가 등이 있어요. 입문 단계에선 APPEND 하나만 확실히 익히면 충분합니다.

④ 주의

입문자가 자주 하는 실수

미리 알아두면 피할 수 있는 함정들이에요. 지금 다 외울 필요는 없고, "이런 게 있구나" 정도면 충분해요. 궁금한 것만 펼쳐 보세요.

실수 1 — 빈 표인데 첫 줄을 읽으려 한다

표가 비어 있으면 LOOP는 한 번도 돌지 않아요(에러가 아니라 그냥 아무 일도 안 함). "결과가 안 나와요!" 할 때 십중팔구 표가 비어 있는 경우예요. 추가(APPEND)를 먼저 했는지 확인하세요.

실수 2 — LOOP 도는 중에 그 표를 건드린다

한 줄씩 읽는 도중에 같은 표에서 줄을 지우거나 추가하면 순서가 꼬여 예상치 못한 결과가 나와요. 읽는 중에는 그 표를 바꾸지 않기 — 이것만 기억하세요.

실수 3 — "메모리에만 있다"는 걸 잊는다

내부 테이블은 프로그램이 끝나면 사라져요. 영구 저장이 필요하면 데이터베이스에 저장해야 합니다. 내부 테이블은 어디까지나 "작업용 임시 표"예요.

⚠️
가장 중요한 한 가지

읽기 전에 항상 "이 표에 줄이 들어있나?"를 떠올리세요. 빈 표는 입문 단계 오류의 1순위예요.

오늘 챙긴 것
