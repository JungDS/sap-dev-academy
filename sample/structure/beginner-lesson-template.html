Internal Table이 뭐예요?

Track 1Chapter 6Lesson 1입문

🎯학습 목표

SAP/ABAP을 오늘 처음 보셔도 괜찮아요. 어려운 말은 그때그때 풀어드릴게요. 이 페이지를 끝까지 보면 "여러 줄의 데이터를 다루는 가장 기본 도구"인 Internal Table을 직접 만들어 보게 됩니다. 변수 하나로는 왜 부족한지부터, 표를 선언하고 APPEND로 줄을 넣고 LOOP AT으로 한 줄씩 읽는 3단계까지요. 🙂

1. 먼저, 왜 이게 필요할까요?

회사에 사원이 5명 있어요: 정훈영, 손흥민, 유재석, 김연아, 아이유. 이 다섯 명의 급여를 화면에 한 번에 보여주고 싶어요. 그런데 변수 하나에는 한 명분의 급여밖에 못 담아요. 다섯 명이면 상자 5개, 100명이면 100개를 일일이 만들어야 할까요? 😵

✗ 변수만 쓰면

DATA lv_sal1 TYPE i. "정훈영 DATA lv_sal2 TYPE i. "손흥민 DATA lv_sal3 TYPE i. "유재석 … 100명이면 100줄 😱

✓ 표(Internal Table)를 쓰면

이름급여
정훈영5000
손흥민3200
… (한 표에 줄줄이)…
그래서: 이 불편을 해결하는 도구가 바로 Internal Table이에요. "왜 배우는지"를 먼저 느꼈으니, 이제 정체를 알아볼까요? 👀

2. Internal Table은 무엇인가요?

Internal Table(프로그램이 도는 동안 메모리에 잠깐 두는 "표")은, 같은 모양의 행을 여러 개 담아두는 그릇이에요. 엑셀 표나 출석부를 떠올리면 쉬워요.

열(칼럼)은 "이름·급여" 같은 항목이고, 행은 사원 한 명 한 명이에요. 다만 엑셀 파일처럼 저장되는 게 아니라, 프로그램이 도는 동안에만 메모리에 잠깐 있다 사라집니다.

한 문장 비유: Internal Table = "사원 명단을 적는 빈 표 한 장". 줄을 추가하면 명단이 늘고, 한 줄씩 읽으면 한 명씩 호명하는 셈이에요. 📋
더 자세히 — "같은 모양"이라는 게 무슨 뜻이죠?
표의 모든 행이 똑같은 칸 구성을 가진다는 뜻이에요. 1번 행이 (이름, 급여)면 2번 행도 (이름, 급여)예요. 이 "칸 구성"을 ABAP에서는 Structure라고 부릅니다.

3. 그럼 어떻게 쓰나요? (3단계 + 직접 해보기)

딱 세 동작만 기억하면 돼요: ① 표 선언 → ② 줄 추가(APPEND) → ③ 한 줄씩 읽기(LOOP). 아래 코드를 먼저 눈으로 보고, 바로 밑에서 직접 눌러볼게요.

* ① 표 선언: 사원 명단을 담을 빈 표
DATA lt_emp TYPE TABLE OF ty_emp.

* ② APPEND: 표 맨 끝에 한 줄 추가
APPEND VALUE #( name = '정훈영'
                sal  = 5000 ) TO lt_emp.

* ③ LOOP: 한 줄씩 꺼내 읽기
LOOP AT lt_emp INTO DATA(ls_emp).
  WRITE: / ls_emp-name, ls_emp-sal.
ENDLOOP.

글로만 보면 잘 와닿지 않죠. 아래 버튼으로 표를 직접 만들어 보세요. 줄이 어떻게 쌓이고(APPEND), LOOP가 어떻게 한 줄씩 훑는지 눈으로 확인할 수 있어요. 👇

#이름 (name)급여 (sal)
아직 빈 표예요. "➕ 줄 추가"를 눌러 사원을 한 명씩 넣어 보세요.
여기에 실행 로그(APPEND·LOOP 결과)가 표시돼요.

4. 입문자가 자주 하는 실수

미리 알아두면 피할 수 있는 함정들이에요. 궁금한 것만 펼쳐 보세요.

실수 1 — 빈 표인데 바로 읽으려 한다
표가 비어 있으면 LOOP는 한 번도 돌지 않아요(에러가 아니라 그냥 아무 일도 안 함). "결과가 안 나와요!" 할 때 십중팔구 표가 비어 있는 경우예요. (위 빌더에서 빈 채로 🔁를 눌러 직접 확인해 보세요!)
실수 2 — LOOP 도는 중에 그 표를 건드린다
한 줄씩 읽는 도중에 같은 표에서 줄을 지우거나 추가하면 순서가 꼬여 예상치 못한 결과가 나와요. 읽는 중에는 그 표를 바꾸지 않기.
실수 3 — "메모리에만 있다"는 걸 잊는다
Internal Table은 프로그램이 끝나면 사라져요. 영구 보관이 필요하면 Database에 저장해야 합니다.
가장 중요한 한 가지: 읽기 전에 항상 "이 표에 줄이 들어있나?"를 떠올리세요. 빈 표는 입문 단계 오류 1순위예요. ⚠️

🧾한눈에 정리

왜? 🤔변수 하나는 값 1개뿐. 여러 줄을 다루려면 "표"가 필요해서.
무엇? 📋같은 모양의 행을 메모리에 줄줄이 담는 임시 표 — Internal Table.
어떻게? 🛠️① 선언 → ② APPEND(줄 추가) → ③ LOOP AT(한 줄씩 읽기).
주의! ⚠️빈 표면 LOOP는 0번 돈다. 읽기 전에 "줄 있나?" 확인.
