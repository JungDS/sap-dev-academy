🖥️ 이번 Lesson 트랜잭션 코드

복습SE11

SAP GUI에서 DDIC 객체를 생성·조회하는 전통적 트랜잭션 코드.

🎯 학습 목표

Domain을 "값의 기술 규격"으로 설계합니다. Data Type·Length·Fixed Value의 역할을 나눠 설명하고, SE11에서 저장·검사·활성화 흐름을 직접 시뮬레이션할 수 있어야 합니다.

읽고 이해했다면 눌러 단계를 채우세요.

1. Domain은 값의 재질을 정한다

Domain은 DDIC에서 가장 아래 부품입니다. "주문 상태"라는 업무 의미보다 먼저 "한 글자 코드인가, 숫자인가, 허용값이 고정인가"라는 값의 재질을 정합니다.

비유: Domain = 그릇의 규격, 의미를 붙이는 건 다음 단계(Data Element)에서 "이름표".

2. 네 가지 속성을 나눠서 본다

지금은 "이런 게 있구나" 정도면 충분합니다. 필요할 때 펼쳐 보세요.

Data Type & Length
계산할 값(숫자)/코드(문자)를 정하고 저장 길이를 정함. 예: 상태 코드 CHAR 1.
Fixed Value
값을 정해진 목록으로만 받게 제한. 작은 코드 목록(A/B/C)에 적합, 입력 도움·검증에 쓰임.

3. SE11에서 만드는 흐름

SE11에서 Domain을 만들면 저장 → 검사 → 활성화 3단계. 활성화까지 끝나야 다른 객체가 이 Domain을 씁니다.

주의: 저장만 하고 활성화를 안 하면 "왜 안 보이지?". 활성화 = "이제 진짜 쓸게요" 선언.
